# <a name="dir"></a>dir



디렉터리의 파일과 하위 디렉터리의 목록을 표시합니다. 매개 변수 없이 사용 하는 경우 **dir** 디스크의 볼륨 레이블 및 일련 번호, 이름별 날짜 및 마지막으로 수정 된 각 시간 등 디스크에 파일과 디렉터리의 목록을 표시 합니다. 파일에 대 한 **dir** 확장명과 크기를 바이트 단위로 표시 합니다. **Dir** 또한 파일 및 디렉터리 나열, 합친 크기 및 디스크에 남아 있는 바이트 단위로 사용 가능한 공간의 총 수를 표시 합니다.

이 명령을 사용하는 방법의 예는 [예](#examples)를 참조하세요.

## <a name="syntax"></a>구문

```
dir [<Drive>:][<Path>][<FileName>] [...] [/p] [/q] [/w] [/d] [/a[[:]<Attributes>]][/o[[:]<SortOrder>]] [/t[[:]<TimeField>]] [/s] [/b] [/l] [/n] [/x] [/c] [/4]
```

## <a name="remarks"></a>설명

-   사용 하 여 *FileName* 매개 변수, 공백, 쉼표 또는 세미콜론을 사용 하 여 각 파일 이름을 구분 합니다.
-   와일드 카드 문자를 사용할 수 있습니다 (**&#42;** 하거나 **?** ), 파일 이름에 하나 이상의 문자를 나타내려면 및 파일 또는 하위 디렉터리의 하위 집합을 표시 합니다.

    **별표 (\*):** 예를 들어, 모든 문자열의 대 안으로 별표를 사용 합니다.  
    -   **dir \*.txt** .txt1,.txt_old.txt,.txt으로 시작 하는 확장을 사용 하 여 현재 디렉터리의 모든 파일을 나열 합니다.
    -   **dir 읽을\*.txt** "읽기" 및.txt.txt_old.txt,.txt1, 등으로 시작 하는 확장을 사용 하 여 시작 하는 현재 디렉터리의 모든 파일을 나열 합니다.
    -   **dir 읽을\*.\***  모든 파일이 현재 디렉터리에 "읽기"로 시작 하는 모든 확장을 사용 하 여 나열 합니다.

    별표 와일드 카드 항상 짧은 파일 이름 매핑이 사용 되므로 예기치 않은 결과가 발생할 수 있습니다. 예를 들어 다음 디렉터리에 두 개의 파일 (t.txt2 및 t97.txt 파일이 있습니다).  
    ```
    C:\test>dir /x
    Volume in drive C has no label.
    Volume Serial Number is B86A-EF32
    
    Directory of C:\test
    
    11/30/2004  01:40 PM <DIR>  .
    11/30/2004  01:40 PM <DIR> ..
    11/30/2004  11:05 AM 0 T97B4~1.TXT t.txt2
    11/30/2004  01:16 PM 0 t97.txt
    ```  
    해당 입력 예상한 **dir t97\***  t97.txt 파일 파일이 있습니다를 반환 합니다. 그러나 입력 **dir t97\***  별표 와일드 카드의 짧은 이름 매핑 T97B4~1.TXT를 사용 하 여 파일 t.txt2 t97.txt 파일이 있습니다에 일치 하기 때문에 파일을 모두 반환 합니다. 마찬가지로, 입력 **del t97\***  파일을 모두 삭제 됩니다.

    **물음표 (?).** 이름에 단일 문자 대신 물음표를 사용 합니다. 예를 들어 입력 **읽기 dir??? 합니다. txt** "읽기"로 시작 되는 최대 3 개의 문자 뒤에 확장명이.txt 인 현재 디렉터리의 모든 파일을 나열 합니다. 이 포함까지, Read1.txt, Read12.txt, Read123.txt, 및 Readme1.txt, Readme12.txt 되지 됩니다.
-   파일 표시 속성 지정

    사용 하는 경우 **/a** 에서 둘 이상의 값을 사용 하 여 *특성*합니다 **dir** 만 지정된 된 모든 특성을 사용 하 여 해당 파일의 이름을 표시 합니다. 예를 들어, 사용 하는 경우 **/a** 사용 하 여 **r** 하 고 **-h** 특성으로 (중 하나를 사용 하 여 **/ a: r-h** 나 **/ar-h** )를 **dir** 숨겨지지 않은 읽기 전용 파일의 이름을 표시 됩니다.
-   파일 이름 정렬 지정

    둘 이상의 지정 하는 경우 *SortOrder* 값을 **dir** 순서에 따라, 다음 두 번째 조건에서 파일 이름으로 정렬 합니다. 예를 들어, 사용 하는 경우 **/o** 사용 하 여 합니다 **e** 하 고 **-s** 에 대 한 값 *SortOrder* (중 하나를 사용 하 여 **으로-s**나 **/oe-s**), **dir** 가장 큰 첫 번째를 사용 하 여 디렉터리 및 파일 이름 확장명, 정렬 및 최종 결과 표시 합니다. 확장 하 여 알파벳 정렬 파일 이름 확장명이 없는 나타나고 다음 디렉터리 이름 및 파일 이름 확장명을 가진 하면 됩니다.
-   리디렉션 기호 및 파이프를 사용 하 여

    리디렉션 기호를 사용 하는 경우 ( **>** )를 보내려고 **dir** 출력을 파일 또는 파이프 ( **|** ) 보낼 **dir**다른 명령에 사용 하 여 출력 **/a:-d** 하 고 **/b** 파일 이름만 나열 합니다. 사용할 수 있습니다 *FileName* 사용 하 여 **/b** 하 고 **/s** 지정 하는 **dir** 현재 디렉터리와 모든 파일에 대 한 하위 디렉터리를 검색 하는 것 일치 하는 이름을 *FileName*합니다. **Dir** 드라이브 문자, 디렉터리 이름, 파일 이름 및 파일 이름 확장명 (줄당 하나의 경로)에서 각 파일 이름을 찾습니다만 나열 합니다. 보낼 파이프를 사용 하기 전에 **dir** 다른 명령 출력을 설정 해야 TEMP 환경 변수 Autoexec.nt 파일에 있습니다.
-   합니다 **dir** 다른 매개 변수와 함께 명령을 복구 콘솔에서 사용할 수 있습니다.

## <a name="examples"></a>예

다른 알파벳 순서로, 넓은 형식 및 각 화면 후 일시 중지 후에 모든 디렉터리 하나를 표시 하도록 루트 디렉터리를 현재 디렉터리 인지 확인 한 다음 입력:
```
dir /s/w/o/p
```
**Dir** 루트 디렉터리, 하위 디렉터리를 및 확장을 비롯 하 여 루트 디렉터리의 파일을 나열 합니다. 차례로 **dir** 하위 디렉터리 이름 및 각 하위 트리에서 파일 이름을 나열 합니다.

앞의 예제를 변경할 수 있도록 **dir** 파일 이름과 확장명을 표시 하지만 생략 디렉터리 이름, 유형:
```
dir /s/w/o/p/a:-d
```
디렉터리 목록에 인쇄 하려면 다음을 입력 합니다.
```
dir > prn
```
지정 하는 경우 **prn**, 디렉터리 목록 LPT1 포트에 연결 되어 있는 프린터로 보내집니다. 프린터는 다른 포트에 연결 되 면 바꿔야 **prn** 올바른 포트의 이름입니다.

출력을 리디렉션할 수도 있습니다는 **dir** 대체 하 여 파일에 명령을 **prn** 파일 이름입니다. 경로 입력할 수도 있습니다. 예를 들어, 직접 하 **dir** 형식 레코드 디렉터리에 파일 dir.doc 출력:
```
dir > \records\dir.doc
```
Dir.doc 없으면 **dir** 레코드 디렉터리가 존재 하지 않는 경우가 아니면 만듭니다. 이 경우 다음과 같은 메시지가 표시 됩니다.

`File creation error`

C 드라이브에 모든 디렉터리에서 확장명이.txt 인 모든 파일 이름 목록을 표시 하려면 다음을 입력 합니다.
```
dir c:\*.txt /w/o/s/p
```
**Dir** 넓은 형식으로 표시 디렉터리 마다 사전순된 목록이 일치 하는 파일 이름을 지정 하 고 화면 가득 계속 하려면 아무 키나를 누를 때까지 일시 중지 합니다.
