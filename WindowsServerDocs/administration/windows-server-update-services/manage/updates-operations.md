# <a name="updates-operations"></a>업데이트 작업

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

업데이트 WSUS 서버, 동기화 된 후 해당 검색 됩니다 자동으로 서버의 클라이언트 컴퓨터에는 관련성에 대 한 합니다. 그러나 네트워크의 컴퓨터에 배포 되기 전에 업데이트를 승인 해야 합니다. 업데이트를 승인 하면은 본질적 WSUS와 수행할 작업 (선택 사항이 **설치** 또는 **거부** 새 업데이트에 대 한). 에 대 한 업데이트를 승인할 수 있습니다 합니다 **모든 컴퓨터** 그룹 또는 하위 그룹에 대 한 합니다. 업데이트를 승인 하면 승인 상태가 유지 됩니다 **승인 되지 않은**, WSUS 서버 클라이언트에서는 업데이트가 필요한 지 여부를 평가할 수 있습니다.

WSUS 서버를 복제 모드에서 실행 하는 경우 WSUS 서버의 업데이트를 승인할 수 없습니다. 복제 모드에 대 한 자세한 내용은 참조 하세요. [실행 중인 WSUS 복제 모드](running-wsus-replica-mode.md)합니다.

## <a name="approving-updates"></a>업데이트 승인
WSUS 네트워크에 있는 모든 컴퓨터 또는 다른 컴퓨터 그룹에 대 한 업데이트 설치를 승인할 수 있습니다. 업데이트가 승인 되 면 다음 중 하나 (또는 이상)를 수행할 수 있습니다.

-   있는 경우이 승인 자식 그룹에 적용 됩니다.

-   자동 설치 최종 기한을 설정 합니다. 이 옵션을 선택 하면 설정한 특정 시간 및 날짜 업데이트를 설치 하려면 클라이언트 컴퓨터에서 모든 설정을 재정의 합니다. 또한 대 한 업데이트를 즉시 (다음에 클라이언트 컴퓨터가 WSUS 서버 설치)를 승인 하려면 최종 기한에 대 한 과거 날짜를 지정할 수 있습니다.

-   해당 업데이트가 제거를 지 원하는 경우에 설치 된 업데이트를 제거 합니다.

유념 해야 하는 두 가지 중요 한 고려 사항이 있습니다.

-   먼저 사용자 입력 (예: 업데이트와 관련 된 설정을 지정) 필요한 경우 업데이트의 자동 설치에 대 한 마감일을 설정할 수 없습니다. 살펴볼 업데이트 사용자 입력 해야 하는지 여부를 결정할 합니다 **사용자 입력을 요청할 수 있습니다** 필드에 표시 되는 업데이트에 대 한 업데이트 속성에는 **업데이트** 페이지입니다. 메시지를 확인 합니다 **업데이트 승인** 상자는 "**선택한 업데이트를 사용자 입력이 필요 하며 설치 마감일을 지원 하지 않습니다**."

-   WSUS 서버 구성 요소에 대 한 업데이트의 경우 WSUS 업데이트 승인 될 때까지 클라이언트 시스템에 대 한 다른 업데이트를 승인할 수 없습니다. 업데이트 승인 대화 상자에서이 경고 메시지가 표시 됩니다. "업데이트가 WSUS 승인 되지 않은 것입니다. WSUS 업데이트 승인이 업데이트를 승인 하기 전에. " 이 경우 WSUS 업데이트 노드를 클릭 하 고 해당 보기에서 업데이트를 모두 일반 업데이트를 반환 하기 전에 승인 되어 있는지 확인 해야 합니다.

#### <a name="to-approve-updates"></a>업데이트를 승인하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트** 을 클릭 한 다음 **업데이트를 모두**입니다.

2.  업데이트 목록에서 업데이트를 승인 하 고 마우스 오른쪽 단추로 클릭 (또는 작업 창으로 이동)를 선택 및 업데이트 승인 대화 상자에서 업데이트를 승인할 컴퓨터 그룹을 선택 하 고 옆에 있는 화살표를 클릭 합니다.

3.  선택 **설치 승인**를 클릭 하 고 **승인**합니다.

4.  합니다 **승인 진행률** 승인을 완료 진행률 창에 표시 됩니다. 프로세스가 완료 되 면 합니다 **닫기** 단추가 나타납니다. **닫기**를 클릭합니다.

5.  업데이트를 마우스 오른쪽 단추로 클릭, 적절 한 컴퓨터 그룹을 선택, 옆에 화살표를 클릭 하 고 클릭 한 다음 여 마감일을 선택할 수 있습니다 **최종 기한**합니다.

    -   표준 마감 기한은 예 (1 주, 2 주, 1 개월), 중 하나를 선택할 수 있습니다 또는 클릭할 수 **사용자 지정** 날짜 및 시간을 지정 합니다.

    -   클라이언트 컴퓨터 연락처 서버 되 자 마자 설치에 대 한 업데이트를 클릭 **사용자 지정**, 한 다음 현재 날짜 및 시간 또는 지난에서 하나에 날짜 및 시간을 설정 합니다.

#### <a name="to-approve-multiple-updates"></a>여러 업데이트를 승인 하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트** 을 클릭 한 다음 **업데이트를 모두**입니다.

2.  여러 연속 업데이트를 선택 하려면 키를 누릅니다 **shift** 상태 업데이트를 선택 합니다. 여러 개의 업데이트를 선택 하려면 키를 누른 **CTRL** 상태 업데이트를 선택 합니다.

3.  선택 영역을 마우스 오른쪽 단추로 클릭 하 고 클릭 **승인**합니다. **업데이트 승인** 대화 상자가 열리고 합니다 **승인 상태** 로 설정 **기존 승인 유지** 하며 **확인** 단추 사용 안 함.

4.  개별 그룹에 대 한 승인을 변경할 수 있지만 이렇게 하면 되므로 영향을 주지 않습니다 자식 승인 합니다. 승인, 변경 하려는 그룹을 선택 하 고 왼쪽 화살표를 클릭 합니다. 바로 가기 메뉴를 클릭 **설치 승인**합니다.

5.  선택한 그룹에 대 한 승인 변경 **설치**합니다. 모든 자식 그룹의 경우 승인을 남아 **기존 승인 유지**합니다. 자식 그룹에 대 한 승인을 변경 하려면 그룹을 클릭 하 고 왼쪽에 있는 화살표를 클릭 합니다. 바로 가기 메뉴를 클릭 **자식에 적용**합니다.

6.  모든 승인 부모 로부터 상속 하도록 특정 자식을 설정 하려면 자식을 클릭 하 고 왼쪽 화살표를 클릭 합니다. 바로 가기 메뉴를 클릭 **부모와 같습니다.** 합니다. 자식의 승인을 상속 하도록 설정 해도 부모 승인을 변경 하지 않는 경우 자식 부모의 기존 승인을 상속 합니다.

7.  모든 자식에 대 한 변경 승인 동작을 원한다 면 승인 **모든 컴퓨터**를 선택한 후 **자식에 적용**합니다.

8.  클릭 **확인** 모든 승인을 설정한 후 합니다. 합니다 **승인 진행률** 승인을 완료 진행률 창에 표시 됩니다. 프로세스가 완료 되 면 합니다 **닫기** 단추를 사용할 수 있습니다. **닫기**를 클릭합니다.

## <a name="declining-updates"></a>업데이트 거부
이 옵션을 선택 하면 사용 가능한 업데이트의 기본 목록에서 업데이트를 제거 하 고 WSUS 서버를 클라이언트에 평가 또는 설치 업데이트를 제공 하지 않습니다. 이 옵션은 업데이트 또는 업데이트 하 고 마우스 오른쪽 단추로 클릭 또는 작업 창에 그룹을 선택 하 여 연결할 수 있습니다. 선택 하는 경우에 거부 업데이트가 업데이트 목록에 나타납니다 **거부 됨** 아래에 있는 업데이트 목록에 대 한 필터를 지정 하는 경우 승인 목록의 **보기**합니다.

#### <a name="to-decline-updates"></a>업데이트를 거부 하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트**를 클릭 하 고 **업데이트를 모두**입니다.

2.  업데이트 목록에서 하나 이상의 업데이트를 거부 하려면 선택 합니다.

3.  선택 **거절**를 클릭 하 고 **예** 확인 메시지에서.

## <a name="cleaning-up-declined-updates"></a>거부 된 업데이트 정리
거부 된 업데이트 계속 일부 WSUS 서버 리소스를 사용 하는 것입니다. The 서버 정리 WSUS 데이터베이스에서 거부 된 업데이트를 제거 하는 마법사를 실행 해야 합니다. 참조 [서버 정리 마법사](the-server-cleanup-wizard.md)에 대 한 자세한 내용은 합니다.

## <a name="reinstating-declined-updates"></a>거부 된 업데이트를 제거 하는 복원
업데이트 거부 후 계속 복구할 수 있습니다.

#### <a name="to-reinstate-declined-updates"></a>거부 된 업데이트를 다시 시작 하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트** 을 클릭 한 다음 **업데이트를 모두**입니다.

2.  변경할 **승인** 하 **거부 됨** 클릭 **새로 고침**합니다. 거부 된 업데이트 목록을 로드합니다.

3.  업데이트 목록에서 복구 하려는 하나 이상의 거부 된 업데이트를 선택 합니다.

4.  선택 및 업데이트에는 특정 업데이트를 다시 시작 하려면 마우스 오른쪽 단추로 클릭 **승인**합니다. 에 **업데이트 승인** 대화 상자에서 클릭 **확인** 기본 "승인 되지 않음" 승인 상태를 다시 적용 합니다. 업데이트 목록에 표시 됩니다 **승인 되지 않은** 거부 하는 대신 합니다.

거부 된 업데이트는 WSUS 서버 정리 마법사를 사용 하 여 정리 되지 않은 후 WSUS 서버에서 삭제 됩니다 하 고 모든 업데이트 보기에서 더 이상 나타나지 않습니다. 거부 됨, Microsoft Update 카탈로그에서 업데이트 정리를 다시 가져올 수 있습니다. 자세한 내용은 참조 하세요. [WSUS 및 카탈로그 사이트](wsus-and-the-catalog-site.md)합니다.

## <a name="change-an-approved-update-to-not-approved"></a>승인 된 업데이트 승인 되지 않은 변경
업데이트를 승인 하지 않기로 결정이 시간에 설치 하는 대신 나중에 대 한 저장 하는 경우에 승인 되지 않은 상태로 업데이트를 변경할 수 있습니다. 즉 업데이트가 사용 가능한 업데이트의 기본 목록에 남아 및 클라이언트 규정 준수를 보고 합니다 있지만 클라이언트에 설치 되지 것입니다.

#### <a name="to-change-an-update-from-approved-to-not-approved"></a>승인 되지 승인 된 업데이트에서 변경 하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트**를 클릭 하 고 **업데이트를 모두**입니다.

2.  업데이트 목록에서 승인 되지 않은 변경 하려는 하나 이상의 승인 된 업데이트를 선택 합니다.

3.  바로 가기 메뉴에서 또는 **동작** 창 **승인 되지**를 클릭 하 고 **예** 확인 메시지에서.

## <a name="approving-updates-for-removal"></a>제거에 대 한 업데이트를 승인
제거에 대 한 업데이트를 승인할 수 있습니다 (즉, 이미 설치 된 업데이트를 제거 하려면). 이 옵션은 업데이트가 이미 설치 되어 지원 제거 하는 경우에 사용할 수 있습니다. 제거할 업데이트에 대 한 최종 기한을 지정 하거나 업데이트를 즉시 (다음에 클라이언트 컴퓨터가 WSUS 서버에 연결)를 제거 하려는 경우 최종 기한에 대 한 과거 날짜를 지정할 수 있습니다.

모든 업데이트 제거 지 말할 것도 중요 한 것입니다. 업데이트는 개별 업데이트를 선택 하 고 확인 하 여 제거를 지원 하는지 여부를 표시 합니다 **세부 정보** 창. 아래 **추가 세부 정보**에 표시 됩니다는 **이동식** 범주입니다. WSUS를 통해 업데이트를 제거할 수 없습니다, 경우에 따라서는 제거할 수 있습니다 사용 하 여 **프로그램 추가 / 제거** 에서 **제어판**합니다.

#### <a name="to-approve-updates-for-removal"></a>제거에 대 한 업데이트를 승인 하려면

1.  WSUS 관리 콘솔에서 클릭 **업데이트** 을 클릭 한 다음 **업데이트를 모두**입니다.

2.  업데이트 목록에서 제거를 위한 승인 마우스 오른쪽 단추로 클릭 하려는 하나 이상의 업데이트를 선택 (또는 이동 합니다 **작업** 창).

3.  에 **업데이트 승인** 대화 상자에서 해당 업데이트를 제거 하려는 컴퓨터 그룹을 선택 하 고 옆에 있는 화살표를 클릭 합니다.

4.  선택 **제거 승인**를 클릭 하 고는 **제거** 단추입니다.

5.  제거 승인 완료 된 후에 업데이트를 한 번 더 마우스 오른쪽 단추로 클릭 하 고 적절 한 컴퓨터 그룹을 선택 하면 옆에 있는 화살표를 클릭 하 여 마감일을 선택할 수 있습니다. 선택한 **최종 기한**합니다. 표준 마감 기한은 예 (1 주, 2 주, 1 개월), 중 하나를 선택할 수 있습니다 또는 클릭할 수 **사용자 지정** 특정 날짜 및 시간을 선택 합니다.

6.  클라이언트 컴퓨터 연락처 서버 되 자 마자 제거에 대 한 업데이트를 클릭 **사용자 지정**, 과거의 날짜를 설정 합니다.

## <a name="approving-updates-automatically"></a>업데이트를 자동으로 승인
특정 업데이트의 자동 승인에 대 한 WSUS 서버를 구성할 수 있습니다. 도 사용 가능 해지면 자동 승인의 기존 업데이트 수정 버전을 지정할 수 있습니다. 이 옵션은 기본적으로 선택되어 있습니다. 수정 버전 변경에 있던 업데이트의 버전입니다 (예: 만료 된 것 같습니다 또는 해당 적용 가능성 규칙 변경 했을 수)입니다. 수정 된 버전의 업데이트를 자동으로 승인 하도록 선택 하지 않으면 하는 경우 WSUS에서 이전 버전을 사용할 및 업데이트 수정 버전을 수동으로 승인 해야 합니다.

WSUS 서버 동기화 중 자동으로 적용 되는 규칙을 만들 수 있습니다. 설치, 업데이트 분류, 제품 및 컴퓨터 그룹에 의해 자동으로 승인 하려는 업데이트를 지정 합니다. 이 수정 된 업데이트와 달리 새 업데이트에만 적용 됩니다. 일 수 및 승인 된 업데이트 설치 최종 기한 전에 제품에 대 한 특정 시간을 설정 하는 업데이트 승인 마감 날짜를 지정할 수도 있습니다. 이러한 설정을 사용할 수는 **옵션** 창 아래에 있는 **자동 승인**합니다.

#### <a name="to-automatically-approve-updates"></a>자동으로 업데이트를 승인 하려면

1.  WSUS 관리 콘솔에서 클릭 **옵션**를 클릭 하 고 **자동 승인**합니다.

2.  **업데이트 규칙**에서 **새 규칙**을 클릭합니다.

3.  에 **규칙 추가** 대화 상자 아래에 있는 **1 단계: 속성을 선택**, 사용할 것인지 선택 **특정 분류의 업데이트 인 경우** 또는 **업데이트 인 경우 특정 제품에** (또는 둘 다) 기준으로 합니다. 필요에 따라 선택할 것인지 **최종 기한 설정** 승인에 대 한 합니다.

4.  **2 단계: 속성 편집** 분류, 제품 및 해당 하는 경우 자동 승인 하려는 컴퓨터 그룹을 선택 하는 밑줄이 그어진된 속성을 클릭 합니다. 업데이트 승인 필요에 따라 선택 날짜와 시간을 최종 기한입니다.

5.  **3 단계: 이름 상자를 지정**, 규칙에 대 한 고유한 이름을 입력 합니다.

6.  **확인**을 클릭합니다.

자동 승인 규칙 업데이트 요구 하는 서버에서 아직 수락 하지 않았습니다는 최종 사용자 사용권 계약 (EULA)에 적용 되지 않습니다. 자동 승인 규칙을 적용 해도 모든 관련 업데이트를 승인 한다면 이러한 업데이트를 수동으로 승인 해야 합니다.

## <a name="automatically-approving-revisions-to-updates-and-declining-expired-updates"></a>만료 된 업데이트를 자동으로 업데이트의 수정 버전을 승인 및 거부
옵션 창의 자동 승인 섹션을 자동으로 승인 된 업데이트의 수정 버전을 승인 하는 기본 옵션을 포함 합니다. 또한 자동으로 만료 된 업데이트를 거부 하려면 WSUS 서버를 설정할 수 있습니다. 수정 된 버전의 업데이트를 자동으로 승인 하지 않으려는 경우 WSUS 서버를 이전 수정 버전을 사용 하 고 업데이트 수정 버전을 수동으로 승인 해야 합니다.

> [!NOTE]
> 수정 버전 변경 된 업데이트의 버전입니다 (예를 들어, 만료 또는 적용 가능성 규칙 업데이트) 합니다.

#### <a name="to-automatically-approve-revisions-to-updates-and-decline-expired-updates"></a>만료 된 업데이트를 자동으로 업데이트 수정 버전을 승인 하 고 거부

1.  WSUS 관리 콘솔에서 클릭 **옵션**를 클릭 하 고 **자동 승인**합니다.

2.  에 **고급** 탭에서 확인 둘 다 **승인 된 업데이트의 새 수정 버전을 자동으로 승인** 및 **새 수정 버전이만료되도록하는경우업데이트를자동으로거부** 선택 됩니다.

3.  확인을 클릭합니다.

    > [!NOTE]
    > 이러한 옵션에 대 한 기본값을 유지 하 WSUS 네트워크에서 좋은 성능을 유지할 수 있습니다. 만료 된 업데이트를 자동으로 거부 하지 않으려면 주기적으로 이러한 작업을 수동으로 거부 하도록 해야 합니다.

## <a name="automatically-declining-superseded-updates"></a>교체 된 업데이트를 자동으로 거부
자동으로 승인 하는 기존 업데이트를 대체 하는 새 업데이트를 승인 하면 대체 되는 업데이트 "해당 없음"에 있게 컴퓨터 또는 장치를 최신 업데이트를 설치 합니다. 모든 컴퓨터에 대 한 업데이트를 적용할 수 없음 WSUS 콘솔에서 확인할 수 있습니다. 업데이트 하는 경우 안전 하 게 거부 될 수 있습니다. 또한 업데이트 WSUS 서버 정리 마법사를 실행할 때 자동으로 거부할 수 있습니다.

대체 된 업데이트를 검색 하려면 모든 업데이트 보기 및 해당 열에 정렬 "대체" 플래그 열을 선택할 수 있습니다. 4 개의 그룹이 됩니다.

-   적 하는 업데이트 (빈 아이콘)를 대체 합니다.

-   업데이트 되지 않았지만, 대체는 다른 업데이트 (맨 아래에 파란색 사각형을 사용 하 여 아이콘)를 대체 합니다.

-   대체 하 고 다른 업데이트 (중간에 있는 파란색 사각형을 사용 하 여 아이콘)를 대체 하는 업데이트입니다.

-   업데이트가 다른 업데이트 (맨 위에 있는 파란색 사각형을 사용 하 여 아이콘)를 대체 했습니다.

Windows Server Update Services는 자동으로 거부 교체 된 업데이트의 최신 업데이트 승인 시에 기능이 없습니다. 먼저 승인에 승인 되지 않음 ","를 설정 하 고 서버 정리 마법사를 사용 하 여 모든 관련 된 조건이 충족 되 면 자동으로 업데이트를 거부 하는 것이 좋습니다. 참조 항목: [서버 정리 마법사](the-server-cleanup-wizard.md)합니다.

## <a name="approving-superseding-or-superseded-updates"></a>대체 하는 업데이트나 교체 된 업데이트 승인
일반적으로 다른 업데이트를 대체 하는 업데이트에는 다음 중 하나 이상을 수행 합니다.

-   개선, 향상, 또는 이전에 릴리스된 업데이트 하나 이상에서 제공 하는 수정에 추가 합니다.

-   업데이트 설치에 대 한 승인 된 경우 클라이언트 컴퓨터에 설치 되는 해당 업데이트 파일 패키지의 효율성을 개선 합니다. 예를 들어 대체 되는 업데이트 관련 문제를 해결 하거나 해당 파일을 대체 하는 업데이트 파일 패키지에 포함 되지 않습니다 있도록 이제 새 업데이트에서 지 원하는 운영 체제를 더 이상 없는 파일을 포함할 수 없습니다.

-   최신 버전의 운영 체제를 업데이트합니다. 것도 중요를 대체 하는 업데이트가 이전 버전의 운영 체제를 지원 하지 않을 수 있습니다.

반대로 다른 업데이트에 의해 대체 된 업데이트는 다음을 수행 합니다.

-   대체 하는 업데이트의 유사한 문제를 해결 합니다. 그러나 대체 하는 업데이트 대체 되는 업데이트를 제공 하는 수정 프로그램을 향상 시킬 수 있습니다.

-   이전 버전의 운영 체제를 업데이트합니다. 경우에 따라 이러한 버전의 운영 체제는 더 이상 대체 하는 업데이트에 의해 업데이트 됩니다.

개별 업데이트의 세부 정보 창에서 정보 아이콘 및 맨 위에 있는 메시지에는 대체 하거나 다른 업데이트에 의해 대체 나타냅니다. 또한 업데이트를 교체 또는 업데이트로 교체 되는 확인 하 여을 확인할 수 있습니다 합니다 **이 업데이트를 대체 하는 업데이트** 하 고 **이 업데이트로 대체 된 업데이트** 항목에는 **추가 정보** 부분을 **속성**합니다. 업데이트의 세부 정보 창 업데이트 목록 아래에 표시 됩니다.

WSUS는 대체 거부 업데이트를 자동으로 하 고는 대체 된 업데이트를 반드시 거부 해야는 대체 업데이트를 가정 하지 않는 것이 좋습니다. 대체 되는 업데이트를 거부 하기 전에 클라이언트 컴퓨터에서 더 이상 필요 없는 있는지를 확인 합니다. 다음은 대체 된 업데이트를 설치 해야 할 수 있는 시나리오의 예입니다.

-   업데이트를 대체 하는 새로운 버전의 운영 체제를 지원 하 고 클라이언트 컴퓨터의 일부 이전 버전의 운영 체제를 실행 합니다.

-   대체 하는 업데이트의 업데이트 보다 적응성이 제한 되어 있으면 그에 의해 대체는 부적합 하 게 됩니다 일부 클라이언트 컴퓨터에 대 한 합니다.

-   경우는 업데이트는 더 이상 새 변경 내용으로 인해 이전에 릴리스된 업데이트를 대체 합니다. 각 릴리스마다 변경 내용을 통해를 대체할 수 없는 이전 버전에서 대체 되는 업데이트 가능 합니다. 이 시나리오에서는 여전히 표시 됩니다 대체 된 업데이트에 대 한 메시지를 하지만 대체 하는 업데이트 하지 않는 업데이트에 의해 대체 되었습니다.
