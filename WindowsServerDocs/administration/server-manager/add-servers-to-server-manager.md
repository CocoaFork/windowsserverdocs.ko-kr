# <a name="add-servers-to-server-manager"></a>서버 관리자에 서버 추가

>적용 대상: Windows Server(반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Windows Server에서 단일 서버 관리자 콘솔을 사용하여 여러 원격 서버를 관리할 수 있습니다. 서버 관리자를 사용하여 관리하려는 서버는 Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 또는 Windows Server 2008을 실행할 수 있습니다. 이전 버전의 서버 관리자에서 Windows Server의 최신 릴리스를 관리할 수 없습니다.

이 항목에서는 서버 관리자 서버 풀에 서버를 추가하는 방법을 설명합니다.

> [!NOTE]
> 테스트에서 Windows Server 2012 및 Windows Server 이후 버전에서 서버 관리자를 사용하여 일반적인 작업을 통해 구성된 최대 100개의 서버를 관리할 수 있습니다. 단일 서버 관리자 콘솔을 사용하여 관리할 수 있는 서버 수는 관리되는 서버에서 요청한 데이터의 양과 서버 관리자를 실행하는 컴퓨터에서 사용할 수 있는 하드웨어 및 네트워크 리소스에 따라 달라질 수 있습니다. 표시하려는 데이터의 양은 해당 컴퓨터의 리소스 용량에 접근하므로 서버 관리자의 응답 시간이 느려지고 새로 고침 시 완료가 지연될 수 있습니다. 서버 관리자를 사용하여 관리할 수 있는 서버 수를 늘리기 위해 **이벤트 데이터 구성** 대화 상자의 설정을 사용하여 서버 관리자가 관리되는 서버에서 가져오는 이벤트 데이터를 제한하는 것을 권장합니다. 이벤트 데이터 구성은 **이벤트** 타일의 **작업** 메뉴에서 열 수 있습니다. 조직에서 엔터프라이즈 수준 숫자의 서버를 관리해야 하는 경우 [Microsoft System Center 제품군](https://go.microsoft.com/fwlink/p/?LinkId=239437)에서 제품을 평가하는 것이 좋습니다.
>
> 서버 관리자는 Windows Server 2003을 실행하는 서버에서 온라인 또는 오프라인 상태만 받을 수 있습니다. 서버 관리자를 사용하여 Windows Server 2008 R2 또는 Windows Server 2008을 실행하는 서버에서 관리 작업을 수행할 수 있지만 Windows Server 2008 R2, Windows Server 2008 또는 Windows Server 2003을 실행하는 서버에서 역할 및 기능을 추가할 수는 없습니다.

> 서버 관리자는 Windows Server 운영 체제의 최신 버전을 관리하는데 사용할 수 없습니다. Windows Server 2012 R2, Windows Server 2012, Windows 8.1 또는 Windows 8에서 실행되는 서버 관리자는 Windows Server 2016을 실행하는 서버를 관리하는 데 사용할 수 없습니다.

이 항목에는 다음 섹션이 포함됩니다.

-   [관리할 서버 추가](#BKMK_add)

-   [다음으로 관리 명령을 사용하여 자격 증명 제공](#BKMK_creds)

## <a name="BKMK_creds"></a>다음으로 관리 명령을 사용하여 자격 증명 제공
서버 관리자에 원격 서버를 추가하면 추가하는 일부 서버에 액세스하거나 관리하기 위해 다른 사용자 계정 자격 증명이 필요할 수 있습니다. 서버 관리자를 실행하는 컴퓨터에 로그인하기 위해 사용하는 서버가 아닌 관리되는 서버에 대해 자격 증명을 지정하려면, 역할 또는 그룹 홈 페이지의 **서버** 타일에서 관리되는 서버에 대한 항목을 마우스 오른쪽 단추로 클릭하여 액세스할 수 있는 서버 관리자에 서버를 추가한 뒤 **다음으로 관리** 명령을 사용합니다. **다음으로 관리**를 클릭하면 다음 형식 중 하나로 관리되는 서버에서 액세스 권한이 있는 사용자 이름을 제공할 수 있는 **Windows 보안** 대화 상자가 열립니다.

-   *사용자 이름*

-   *사용자 이름*@example.domain.com

-   *도메인*\\*사용자 이름*

**다음으로 관리** 명령에 의해 열린 **Windows 보안** 대화 상자는 스마트 카드 자격 증명을 수락할 수 없습니다. 서버 관리자를 통해 스마트 카드 자격 증명을 제공하는 것은 지원되지 않습니다. **다음으로 관리** 명령을 사용하여 관리되는 서버에 제공한 자격 증명은 캐시되며 사용자가 현재 서버 관리자를 실행하는 컴퓨터를 사용하여 서버를 관리하는 한 또는 동일한 서버에 대해 비어 있거나 다른 자격 증명을 지정하여 이를 덮어쓰지 않는 한 지속됩니다. 서버 관리자 설정을 다른 컴퓨터에서 사용할 수 있도록 서버 관리자 설정을 다른 컴퓨터로 내보내거나 로밍할 도메인 프로필을 구성하면 사용자 서버 풀의 서버에 대한 **다음으로 관리** 자격 증명은 로밍 프로필에 저장되지 않습니다. 서버 관리자 사용자는 관리하려는 각 컴퓨터에서 이를 추가해야 합니다.

이 항목의 절차에 따라 관리할 서버를 추가한 후, **다음으로 관리** 명령을 사용하여 추가한 서버를 관리하는 데 필요할 수 있는 대체 자격 증명을 지정하기 전에 서버에 대해 다음과 같은 관리 효율성 상태 오류가 표시될 수 있습니다.

-   Kerberos 대상 확인 오류

-   Kerberos 인증 오류

-   온라인 - 액세스 거부됨

> [!NOTE]
> **다음으로 관리** 명령을 지원하지 않는 역할 및 기능에는 원격 데스크톱 서비스(RDS) 및 IP 관리(IPAM) 서버가 있습니다. 서버 관리자를 실행하는 컴퓨터에서 사용하는 것과 동일한 자격 증명을 사용하여 원격 RDS 또는 IPAM 서버를 관리할 수 없는 경우 이러한 원격 서버를 관리하기 위해 일반적으로 사용하는 계정을 서버 관리자를 실행하는 컴퓨터의 관리자 그룹에 추가해 보세요. 그런 다음 사용 중인 계정으로 서버 관리자를 실행하는 컴퓨터에 로그인하여 rdS 또는 IPAM을 실행하는 원격 서버를 관리합니다.

## <a name="BKMK_add"></a>관리할 서버 추가
**서버 추가** 대화 상자에서 다음과 같은 세가지 방법 중 하나를 사용하여 서버 관리자에 관리할 서버를 추가할 수 있습니다.

-   **Active Directory Domain Services** Active Directory가 로컬 컴퓨터와 동일한 도메인에서 찾은 관리할 서버를 추가합니다.

-   **DNS(Domain Name System) 항목** 관리할 서버를 컴퓨터 이름이나 IP 주소를 기준으로 검색합니다.

-   **여러 서버 가져오기** 컴퓨터 이름 또는 IP 주소에서 나열된 서버를 포함하는 파일에서 가져올 여러 서버를 지정합니다.

#### <a name="to-add-servers-to-the-server-pool"></a>서버 풀에 서버를 추가하려면

1.  서버 관리자가 이미 열려 있으면 다음 단계로 이동합니다. 서버 관리자가 이미 열려 있지 않으면 다음 중 하나를 수행하여 엽니다.

    -   Windows 바탕 화면에서 Windows 작업 표시줄의 **서버 관리자**를 클릭하여 서버 관리자를 시작합니다.

    -   Windows **시작** 화면에서 서버 관리자 타일을 클릭합니다.

2.  **관리** 메뉴에서 **서버 추가**를 클릭합니다.

3.  다음 중 하나를 수행합니다.

    -   **Active directory** 탭에서 현재 도메인에 있는 서버를 선택합니다. 여러 서버를 선택하려면 **Ctrl** 키를 누른 채 선택합니다. 오른쪽 화살표 단추를 클릭하여 선택한 서버를 **선택된** 목록으로 이동합니다.

    -   **DNS** 탭에서 컴퓨터 이름이나 IP 주소의 처음 몇 자를 입력한 다음 **Enter** 키를 누르거나 **검색**을 클릭합니다. 추가하려는 서버를 선택하고 오른쪽 화살표 단추를 클릭합니다.

    -   **가져오기** 탭에서 추가할 컴퓨터의 DNS 이름 또는 IP 주소를 포함하는 텍스트 파일을 한 줄당 하나의 이름 또는 IP 주소로 찾습니다.

4.  서버를 모두 추가했으면 **확인**을 클릭합니다.

### <a name="add-and-manage-servers-in-workgroups"></a>작업 그룹에서 서버 추가 및 관리
서버 관리자에 작업 그룹에 속한 서버를 추가하는 것에 성공할 수 있지만, 서버를 추가한 후, 작업 그룹 서버를 포함하는 역할 또는 그룹 페이지에서 **서버** 타일의 **관리 효율성** 열이 원격 작업 그룹 서버에 연결하거나 여기에서 데이터를 수집하려고 시도할 때 발생한 **자격 증명이 잘못됨** 오류를 표시할 수 있습니다.

이 오류 또는 유사한 오류는 다음 조건에서 발생할 수 있습니다.

-   관리되는 서버가 서버 관리자를 실행하는 컴퓨터와 동일한 작업 그룹에 있습니다.

-   관리되는 서버가 서버 관리자를 실행하는 컴퓨터와 다른 작업 그룹에 있습니다.

-   컴퓨터 중 하나는 작업 그룹에, 다른 하나는 도메인에 있습니다.

-   서버 관리자를 실행하는 컴퓨터는 작업 그룹에 있으며 관리되는 원격 서버는 다른 서브넷에 있습니다.

-   두 컴퓨터가 도메인에 있지만 두 도메인 간에 트러스트 관계가 없습니다.

-   두 컴퓨터가 도메인에 있지만 두 도메인 간에 단방향 트러스트 관계만 있습니다.

-   관리하려는 서버가 IP 주소를 사용하여 추가되었습니다.

##### <a name="to-add-remote-workgroup-servers-to-server-manager"></a>서버 관리자에 원격 작업 그룹 서버를 추가하려면

1.  서버 관리자를 실행하는 컴퓨터에서 **TrustedHosts** 목록에 작업 그룹 서버 이름을 추가합니다. 이는 NTLM 인증의 요구 사항입니다. 신뢰할 수 있는 호스트의 기존 목록에 컴퓨터 이름을 추가하려면 명령에 `Concatenate` 매개 변수를 추가합니다. 예를 들어 `Server01` 컴퓨터를 신뢰할 수 있는 호스트의 기존 목록에 추가하려면 다음 명령을 사용합니다.

    ```
    Set-Item wsman:\localhost\Client\TrustedHosts Server01 -Concatenate -force
    ```

2.  관리하려는 작업 그룹 서버가 서버 관리자를 실행하는 컴퓨터와 동일한 서브넷에 있는지 확인합니다.

    두 컴퓨터가 동일한 서브넷에 있는 경우 또는 작업 그룹 서버의 네트워크 프로필이 **네트워크 및 공유 센터**에서 **비공개**로 설정된 경우 다음 단계로 이동합니다.

    동일한 서브넷에 있지 않은 경우, 또는 작업 그룹 서버의 네트워크 프로필이 **비공개**로 설정되지 않은 경우, 작업 그룹 서버에서 설정의 **속성** 대화 상자에 있는 **컴퓨터** 탭에 컴퓨터 이름을 추가하여 Windows 방화벽의 인바운드 **Windows 원격 관리(HTTP-In)** 설정을 원격 컴퓨터의 연결을 명시적으로 허용하도록 변경합니다.

3.  > [!IMPORTANT]
    > 이 단계에서 cmdlet을 실행하면 기본 제공 관리자 또는 시스템 계정이 프로세스에서 실행되고 있지 않는 한 관리자 권한 프로세스가 작업 그룹 컴퓨터에서 실행되지 못하게 하는 사용자 계정 컨트롤(UAC) 측정값이 재정의됩니다. cmdlet을 통해 관리자 그룹의 구성원이 기본 제공 관리자로 로그인하지 않고도 작업 그룹 서버를 관리할 수 있습니다. 추가 사용자가 작업 그룹 서버를 관리할 수 있도록 하면 보안이 약화될 수 있습니다. 그러나 이는 기본 제공 관리자 계정 자격 증명을 작업 그룹 서버를 관리하는 여러 사람에게 제공하는 것보다 안전합니다.

    작업 그룹 컴퓨터에서 관리자 권한으로 실행 중인 프로세스에 대해 UAC 제한을 무시하려면 다음 cmdlet을 실행하여 작업 그룹 서버에 **LocalAccountTokenFilterPolicy**라는 레지스트리 항목을 만듭니다.

    ```
    New-ItemProperty -Name LocalAccountTokenFilterPolicy -path HKLM:\SOFTWARE\Microsoft\Windows\Currentversion\Policies\System -propertytype DWord -value 1
    ```

4.  서버 관리자를 실행하는 컴퓨터에서 **모든 서버** 페이지를 엽니다.

5.  서버 관리자를 실행하는 컴퓨터와 대상 작업 그룹 서버가 동일한 작업 그룹에 있는 경우 마지막 단계를 건너뜁니다. 두 컴퓨터가 같은 작업 그룹에 있지 않다면 **서버** 타일에서 대상 작업 그룹 서버를 마우스 오른쪽 단추로 클릭한 다음 **다음으로 관리**를 클릭합니다.

6.  작업 그룹 서버에 대한 기본 제공 관리자 계정을 사용하여 작업 그룹 서버에 로그인합니다.

7.  **모든 서버** 페이지를 새로 고쳐 서버 관리자가 작업 그룹 서버에 연결하고 이 데이터를 수집할 수 있는지 확인한 다음 작업 그룹 서버에 대한 관리 효율성 상태를 검토합니다.

##### <a name="to-add-remote-servers-when-server-manager-is-running-on-a-workgroup-computer"></a>서버 관리자가 작업 그룹 컴퓨터에서 실행될 때 원격 서버를 추가하려면

1.  서버 관리자를 실행하는 컴퓨터에서 Windows PowerShell 세션에서 원격 서버를 로컬 컴퓨터의 **TrustedHosts** 목록에 추가합니다. 신뢰할 수 있는 호스트의 기존 목록에 컴퓨터 이름을 추가하려면 `Concatenate` 매개 변수를 명령에 추가합니다. 예를 들어 `Server01` 컴퓨터를 신뢰할 수 있는 호스트의 기존 목록에 추가하려면 다음 명령을 사용합니다.

    ```
    Set-Item wsman:\localhost\Client\TrustedHosts Server01 -Concatenate -force
    ```

2.  관리하려는 서버가 서버 관리자를 실행하는 작업 그룹 컴퓨터와 동일한 서브넷에 있는지 확인합니다.

    두 컴퓨터가 동일한 서브넷에 있는 경우 또는 작업 그룹 컴퓨터의 네트워크 프로필이 **네트워크 및 공유 센터**에서 **비공개**로 설정된 경우 다음 단계로 이동합니다.

    동일한 서브넷에 있지 않은 경우, 또는 작업 그룹 컴퓨터의 네트워크 프로필이 **비공개**로 설정되지 않은 경우, 서버 관리자를 실행하는 작업 그룹 컴퓨터에서 설정의 **속성** 대화 상자에 있는 **컴퓨터** 탭에 컴퓨터 이름을 추가하여 Windows 방화벽의 인바운드 **Windows 원격 관리(HTTP-In)** 설정을 원격 컴퓨터의 연결을 명시적으로 허용하도록 변경합니다.

3.  서버 관리자를 실행하는 컴퓨터에서 **모든 서버** 페이지를 엽니다.

4.  **모든 서버** 페이지를 새로 고쳐 서버 관리자가 원격 서버에 연결하고 이 데이터를 수집할 수 있는지 확인한 다음 원격 서버에 대한 관리 효율성 상태를 검토합니다. **서버** 타일에 원격 서버에 대한 관리 효율성 오류가 계속 표시되면 다음 단계로 이동합니다.

5.  서버 관리자를 실행하는 컴퓨터에서 로그아웃한 다음 기본 제공 관리자 계정을 사용하여 다시 로그인합니다. 위의 단계를 반복하여 서버 관리자가 원격 서버에 연결하고 이 서버의 데이터를 수집할 수 있는지 확인합니다.

이 섹션의 절차를 수행하였지만 작업 그룹 컴퓨터 또는 작업 그룹 컴퓨터가 아닌 컴퓨터를 관리하는 데 문제가 계속 발생하는 경우 Microsoft 웹 사이트에서 [about_remote_Troubleshooting](https://technet.microsoft.com/library/dd347642.aspx)을 참조하십시오.

### <a name="add-and-manage-servers-in-clusters"></a>클러스터에서 서버 추가 및 관리
서버 관리자를 사용하여 장애 조치 클러스터(서버 클러스터 또는 MSCS라고도 함)에 있는 서버를 관리할 수 있습니다. (클러스터 노드가 실제 또는 가상인지 여부와 상관없이)장애 조치 클러스터에 있는 서버는 서버 관리자에서 일부 고유한 동작 및 관리 제한이 있습니다.

-   클러스터에서 실제 및 가상 서버는 클러스터의 한 서버가 서버 관리자에 추가되는 경우 자동으로 서버 관리자에 추가됩니다. 마찬가지로, 서버 관리자에서 클러스터형 서버를 제거하면 클러스터에서 다른 서버를 제거하라는 메시지가 표시됩니다.

-   데이터는 동적이고 가상 클러스터형 노드가 호스팅되는 서버에 대한 데이터와 동일하기 때문에 서버 관리자는 클러스터형 가상 서버에 대한 데이터를 표시하지 않습니다. 가상 서버를 호스팅하는 서버를 선택하여 해당 데이터를 볼 수 있습니다.

-   서버의 가상 클러스터 개체 이름을 사용하여 서버를 서버 관리자에 추가하면 (예상된) 실제 서버 이름 대신 가상 개체 이름이 서버 관리자에 표시됩니다.

-   클러스터형 가상 서버에 역할 및 기능을 설치할 수 없습니다.
