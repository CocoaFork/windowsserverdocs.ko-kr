# <a name="cluster-aware-updating-advanced-options-and-updating-run-profiles"></a>클러스터 인식 업데이트 고급 옵션 및 업데이트 실행된 프로필

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

이 항목에 대해 구성할 수 있는 업데이트 실행 옵션에 설명 합니다는 [Cluster-aware Updating](cluster-aware-updating.md) (CAU) 업데이트 실행 합니다. 업데이트를 적용 하거나 자체 업데이트 옵션을 구성 하려면 CAU UI 또는 CAU Windows PowerShell cmdlet을 사용 하는 경우에 이러한 고급 옵션을 구성할 수 있습니다.

대부분의 구성 설정은 업데이트 실행 프로필이라는 XML 파일로 저장되며 이후에 업데이트 실행 시 다시 사용할 수 있습니다. CAU에서 제공되는 업데이트 실행 옵션의 기본값은 많은 클러스터 환경에서도 사용할 수 있습니다.

각 업데이트 실행 및 업데이트 실행 프로필에 대해 지정할 수 있는 추가 옵션에 대한 자세한 내용을 보려면 이 항목 후반부에 나오는 다음 섹션을 참조하십시오.

업데이트 실행 사용 하 여 업데이트 실행 프로필 옵션을 업데이트 실행 프로필에서 설정할 수 있는 요청 하는 경우를 지정 하는 옵션

다음 표에는 CAU 업데이트 실행 프로필에서 설정할 수 있는 옵션이 나열되어 있습니다. 

> [!NOTE] 
> PreUpdateScript 또는 PostUpdateScript 옵션을 설정 하려면 Windows PowerShell 및.NET Framework 4.6 또는 4.5를 설치 하 고 PowerShell 원격 클러스터의 각 노드에서 활성화 되어 있는지 확인 합니다. 자세한 내용은의 원격 관리를 위한 노드 구성 [요구 사항 및 클러스터 인식 업데이트에 대 한 모범 사례](cluster-aware-updating-requirements.md)합니다.

##  <a name="BKMK_profile"></a> 업데이트 실행된 프로필 사용  
 각 업데이트 실행을 특정 업데이트 실행 프로필과 연결할 수 있습니다. 업데이트 실행 프로필에 저장 된 기본값을 *%windir%\cluster* 폴더입니다. 원격 업데이트에서 CAU UI를 사용 하는 경우 모드에서 업데이트를 적용 하는 시 업데이트 실행 프로필을 지정 하거나 기본 업데이트 실행 프로필을 사용할 수 있습니다. CAU 자동 업데이트 모드를 사용 하는 경우 자동 업데이트 옵션을 구성할 때 지정한 업데이트 실행 프로필에서 설정을 가져올 수 있습니다. 두 경우 모두 필요에 따라 업데이트 실행 옵션에 대해 표시된 값을 재정의할 수 있습니다. 원할 경우 업데이트 실행 프로필 옵션을 같은 파일 이름이나 다른 파일 이름을 사용하여 업데이트 실행 프로필로 저장할 수 있습니다. 다음 번에 업데이트를 적용하거나 자동 업데이트 옵션을 구성할 경우 CAU에서 이전에 선택했던 업데이트 실행 프로필을 자동으로 선택합니다.  
  
 기존 업데이트 실행 프로필을 수정 하거나 선택 하 여 새로 만들 수 있습니다 **만들기 또는 업데이트 실행 프로필 수정** CAU ui에서입니다.

업데이트 실행 프로필을 사용 하는 방법에 대 한 몇 가지 주의 사항은 다음과 같습니다.

* 업데이트 실행 프로필을 관리자 자격 증명 같은 클러스터 특정 정보를 저장 하지 않습니다. CAU 자동 업데이트 모드를 사용 하는 경우 업데이트 실행 프로필도 저장 하지 일정 정보를 자동으로 업데이트 합니다. 이를 통해 지정된 클래스의 모든 장애 조치(failover) 클러스터에서 업데이트 실행 프로필을 공유할 수 있게 됩니다.
* 업데이트 실행 프로필을 사용 하 여 자동 업데이트 옵션을 구성 하 고 나중에 업데이트 실행 옵션에 대 한 값이 서로 다른 프로필을 수정 하는 경우 자동 업데이트 구성은 자동으로 변경 되지 않습니다. 새 업데이트를 실행 설정을 적용하려면 자동 업데이트 옵션을 다시 구성해야 합니다.
* 실행 프로필 편집기와 같은 공간을 포함 하는 파일 경로 지원 하지 않습니다 *C:\Program Files*합니다. 대 안으로 이전 버전을 저장 하 고 공백을 하거나 PowerShell을 실행 하는 경우 경로 주위에 따옴표를 배치 하는 실행 프로필을 관리할을 단독으로 사용 하지 않는 경로에 업데이트 스크립트를 게시 **Invoke-caurun**합니다.

### <a name="windows-powershell-equivalent-commands"></a>Windows PowerShell 해당 명령
  
 실행 하면 업데이트 실행 프로필에서 설정을 가져올 수 있습니다 합니다 **Invoke-caurun**를 **Add-cauclusterrole**, 또는 **Set-cauclusterrole** cmdlet.  
  
 다음 예제에서는 *C:\Windows\Cluster\DefaultParameters.xml*에 지정된 업데이트 실행 옵션을 사용하여 *CONTOSO-FC1*이라는 이름의 클러스터에서 스캔 및 전체 업데이트 실행을 수행하는 과정을 보여줍니다. 기본값은 나머지 cmdlet 매개변수에 대해 사용됩니다.  
  
```powershell  
$MyRunProfile = Import-Clixml C:\Windows\Cluster\DefaultParameters.xml  
Invoke-CauRun –ClusterName CONTOSO-FC1 @MyRunProfile   
```  
  
 업데이트 실행 프로필을 사용하여 예외 관리, 시간 범위 및 기타 조작 매개변수에 대한 일관된 설정으로 장애 조치(failover) 클러스터를 반복 가능한 방식으로 업데이트할 수 있습니다. 이러한 설정은 일반적으로 "모든 Microsoft SQL Server 서버" 또는 "내 업무상 중요한 클러스터"와 같은 장애 조치(failover) 클러스터의 클래스에만 적용되기 때문에 사용되는 장애 조치(failover) 클러스터의 클래스에 따라 업데이트 실행 프로필 이름을 각자 지정하려 할 수도 있습니다. 또한, IT 조직에서 특정 클래스의 모든 장애 조치(failover) 클러스터에 액세스할 수 있는 파일 공유에서 업데이트 실행 프로필을 관리하려 할 수도 있습니다.
