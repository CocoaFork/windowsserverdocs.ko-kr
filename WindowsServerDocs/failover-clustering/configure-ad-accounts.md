# <a name="configuring-cluster-accounts-in-active-directory"></a>Active Directory에서 클러스터 계정 구성하는 중


적용 대상: Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 및 Windows Server 2008

Windows Server 장애 조치 클러스터를 만들고 클러스터 된 서비스 또는 응용 프로그램을 구성 하는 경우 장애 조치 클러스터 마법사 (컴퓨터 개체 라고도 함) 수 있는 Active Directory 컴퓨터 계정에 필요한 만들고 특정 권한을 부여 합니다. 마법사 제외 하 고 Hyper-v 가상 컴퓨터 (이 계정은 라고도 함은 클러스터 이름 개체 또는 CNO) 클러스터 자체에 대 한 컴퓨터에 대 한 계정 및 대부분의 클러스터 된 서비스 및 응용 프로그램에 대 한 컴퓨터 계정을 만듭니다. 장애 조치 클러스터 마법사에서 이러한 계정에 대 한 사용 권한은 자동으로 설정 됩니다. 사용 권한을 변경 하는 경우 클러스터 요구 사항에 맞게 다시 변경 해야 합니다. 이 가이드 이러한 Active Directory 계정 및 권한에 대해 설명 하 고, 중요 한 이유에 대 한 배경 정보를 제공, 구성 하 고 계정을 관리 하는 단계를 설명 합니다.
      

## <a name="overview-of-active-directory-accounts-needed-by-a-failover-cluster"></a>장애 조치 클러스터에 필요한 Active Directory 계정 개요

이 섹션에는 Active Directory 컴퓨터 계정을 (Active Directory 컴퓨터 개체 라고도 함)는 장애 조치 클러스터에 대 한 중요 하다 고 설명 합니다. 이러한 계정은 아래와 같습니다.

  - **클러스터를 만드는 데 사용 되는 사용자 계정입니다.** 클러스터 만들기 마법사를 시작 하는 데 사용 하는 사용자 계정입니다. 계정을 클러스터 자체에 대 한 컴퓨터 계정이 생성 되는 기초를 제공 하기 때문에 중요 합니다.  
      
  - **클러스터 이름 계정입니다.** (클러스터 자체의 컴퓨터 계정 또는 클러스터 이름 개체 또는 CNO). 이 계정은 클러스터 만들기 마법사에서 자동으로 만들어진 및 클러스터와 동일한 이름을 가진 합니다. 클러스터 이름 계정이 만들어지기 때문에이 계정을 통해 다른 계정을 자동으로 클러스터에서 새 서비스 및 응용 프로그램을 구성할 때 매우 중요 합니다. 클러스터 이름 계정 복원 되거나 권한이 복원 될 때까지 클러스터에서 클러스터 이름 계정을 삭제 하거나 권한을 빼앗길, 다른 계정으로 만들 수 없습니다 하는 경우 필요 합니다.  
      
    예를 들어 Cluster1 이라는 클러스터를 만들고 클러스터에서 PrintServer1 이라는 클러스터 된 인쇄 서버를 구성 하려고 하는 경우 Active Directory에서 Cluster1 계정 보관 해야 합니다 올바른 권한을 컴퓨터 만들기를 사용할 수 있도록 PrintServer1를 호출 하는 계정입니다.  
      
    클러스터 이름 계정에 Active Directory 컴퓨터 계정에 대 한 기본 컨테이너에 생성 됩니다. 기본적으로 "컴퓨터" 컨테이너 이지만 도메인 관리자는 다른 컨테이너를 리디렉션할 선택할 수 있습니다 또는 조직 구성 단위 (OU).  
      
  - **클러스터형된 서비스 또는 응용 프로그램의 컴퓨터 계정 (컴퓨터 개체)입니다.** 이러한 계정은 대부분의 클러스터 된 서비스 또는 응용 프로그램을 제외 하 고 Hyper-v 가상 컴퓨터를 만드는 과정의 일부로 고가용성 마법사에서 자동으로 만들어집니다. 클러스터 이름 계정에는 이러한 계정을 제어 하는 데 필요한 권한이 부여 됩니다.  
      
    예를 들어 Cluster1 이라는 클러스터 있고 FileServer1 라는 클러스터형된 파일 서버를 만든 경우 고가용성 마법사 FileServer1 이라는 Active Directory 컴퓨터 계정을 만듭니다. 고가용성 마법사는 또한 Cluster1 계정 FileServer1 계정을 제어 하는 데 필요한 권한을 제공 합니다.

## <a name="requirements-related-to-failover-clusters-active-directory-domains-and-accounts"></a>장애 조치 클러스터, Active Directory 도메인 및 계정 관련 된 요구 사항

클러스터 된 서비스 특정 요구 사항을 충족 해야 앞의 세 단원에서 설명한 대로, 및 장애 조치 클러스터에서 응용 프로그램을 성공적으로 구성할 수 있습니다. 가장 기본적인 요구 사항 (내에서 단일 도메인) 클러스터 노드의 위치 및 클러스터를 설치 하는 사용자의 계정 권한 수준을 다룹니다. 이러한 요구 사항이 충족 하는 경우 클러스터에 필요한 다른 계정은 장애 조치 클러스터 마법사에서 자동으로 만들 수 있습니다. 다음은 이러한 기본 요구 사항에 대 한 정보를 제공합니다.

  - **노드:** 모든 노드가 동일한 Active Directory 도메인에 있어야 합니다. (Active Directory를 포함 하지 않는 Windows NT 4.0 도메인을 만들 수 없습니다.)  
      
  - **클러스터를 설치 하는 사용자의 계정:** 클러스터를 설치 하는 사람은 다음 특징을 가진 계정을 사용 해야 합니다.  
      
      - 계정은 도메인 계정 이어야 합니다. 도메인 관리자 계정 이어야 하는 것이 없습니다. 이 목록의 다른 요구 사항을 충족 하는 경우 도메인 사용자 계정 수 있습니다.  
          
      - 계정에는 클러스터 노드로 지정할 서버에 관리 권한이 있어야 합니다. 이 제공 하는 가장 간단한 방법은 도메인 사용자 계정을 만들고 다음 각 클러스터 노드로 지정할 서버에 로컬 관리자 그룹에 해당 계정을 추가 하는 경우 자세한 내용은 [클러스터를 설치 하는 사람의 계정을 구성 하는 단계](#steps-for-configuring-the-account-for-the-person-who-installs-the-cluster)이 가이드의 뒷부분에 나오는.  
          
      - 계정 (또는 그룹의 구성원 인 계정)를 지정 해야 합니다는 **컴퓨터 개체 만들기** 하 고 **모든 속성 읽기** 도메인의 컴퓨터 계정에 사용 되는 컨테이너의 사용 권한. 또 다른 방법은 계정을 도메인 관리자 계정을 만드는 것입니다. 자세한 내용은 [클러스터를 설치 하는 사람의 계정을 구성 하는 단계](#steps-for-configuring-the-account-for-the-person-who-installs-the-cluster)이 가이드의 뒷부분에 나오는.  
          
      - 조직 선택 (클러스터와 동일한 이름 가진 컴퓨터 계정)의 클러스터 이름 계정을 미리 준비 하면 사전 준비 된 클러스터 이름 계정에는 클러스터를 설치 하는 사용자의 계정에 "모든 권한" 권한을 부여 해야 합니다. 클러스터 이름 계정을 미리 준비 하는 방법에 대 한 기타 중요 한 세부 정보를 참조 하세요 [클러스터를 사전 준비 단계에는 계정 이름을](#steps-for-prestaging-the-cluster-name-account)이 가이드의 뒷부분에 나오는.  
          

### <a name="planning-ahead-for-password-resets-and-other-account-maintenance"></a>암호 재설정 및 기타 계정 유지 관리에 대 한 미리 계획

9.  클릭 **개체 유형** 했는지 **컴퓨터** 를 선택한 다음 클릭 **확인**합니다. 그런 다음 **선택할 개체 이름 입력**클러스터 이름 계정을 입력 하 고 클릭 **확인**합니다. 메시지가 나타나면 추가 사용할 수 없는 개체를 클릭 하려고 한다고 말하는 **확인**합니다.

10. 클러스터 이름 계정 선택 되어 있는지 확인 및 그런 다음 다음을 **모든 권한**를 선택 합니다 **허용** 확인란 합니다.
    
    ![](media/configure-ad-accounts/Cc731002.2e614376-87a6-453a-81ba-90ff7ebc3fa2(WS.10).gif)

## <a name="steps-for-troubleshooting-problems-related-to-accounts-used-by-the-cluster"></a>클러스터에서 사용 하는 계정과 관련 된 문제 해결 단계

이 가이드의 앞부분에 설명 된 대로, 장애 조치 클러스터를 만들고 클러스터 된 서비스 또는 응용 프로그램을 구성 하는 경우, 장애 조치 클러스터 마법사 필요한 Active Directory 계정을 만들고 올바른 권한을 부여 합니다. 필요한 계정이 삭제 되, 필요한 사용 권한을 변경 하는 경우 문제가 발생할 수 있습니다. 다음 하위 섹션에서는 이러한 문제 해결 단계가 나와 있습니다.

  - [클러스터 이름 계정 사용 하 여 암호 관련 문제 해결 단계](#steps-for-troubleshooting-password-problems-with-the-cluster-name-account)
      
  - [클러스터와 관련 된 Active Directory 계정에 대 한 변경으로 인 한 문제 해결 단계](#steps-for-troubleshooting-problems-caused-by-changes-in-cluster-related-active-directory-accounts)
      

### <a name="steps-for-troubleshooting-password-problems-with-the-cluster-name-account"></a>클러스터 이름 계정 사용 하 여 암호 관련 문제 해결 단계

다음 텍스트를 포함 하는 클러스터 id 또는 컴퓨터 개체에 대 한 이벤트 메시지는 없는 경우이 절차를 따르십시오. 이 텍스트는 이벤트 메시지의 시작 부분에 없습니다. 이벤트 메시지 내의 될 참고:

`Logon failure: unknown user name or bad password.`

이전 설명을 맞는 이벤트 메시지는 클러스터 이름 계정에 대 한 암호와 클러스터링 소프트웨어에 의해 저장 된 해당 암호가 더 이상 일치 하는지 나타냅니다.

클러스터 관리자가 필요에 따라 다음 절차를 수행 하려면 올바른 권한이 있는지 확인 하는 것에 대 한 자세한 계획 암호 재설정 및이 가이드의 앞부분에 나오는 다른 계정 유지 관리에 대 한 계속 해 서 참조 합니다.

로컬의 멤버 자격이 **관리자** 그룹 또는 그에 해당 하는이 절차를 완료 하는 데 필요한 최소입니다. 또한 계정에 부여 해야 합니다 **암호 재설정** 클러스터 이름 계정에 대 한 권한을 (계정의 아닌를 **Domain Admins** 클러스터 이름 계정 작성자 소유자 또는 계정). 이 프로시저에 대 한 클러스터를 설치한 사용자에 게 사용 된 계정을 사용할 수 있습니다. 적절 한 계정을 사용 하는 방법에 대 한 세부 정보를 검토 하 고 그룹 구성원 자격 [ http://go.microsoft.com/fwlink/?LinkId=83477 ](http://go.microsoft.com/fwlink/?linkid=83477)합니다.

#### <a name="to-troubleshoot-password-problems-with-the-cluster-name-account"></a>클러스터 이름 계정 암호 문제를 해결 하려면

1.  장애 조치 클러스터 스냅인을 열려면 **시작**, **관리 도구**를 차례로 클릭한 다음 **장애 조치 클러스터 관리**를 클릭합니다. (경우 합니다 **사용자 계정 컨트롤** 는 표시 되는 작업은를 클릭 확인 대화 상자가 나타나면 **계속**.)

2.  장애 조치 클러스터 관리 스냅인에서, 구성하려는 클러스터가 콘솔 트리에 표시되지 않는 경우 **장애 조치 클러스터 관리**를 마우스 오른쪽 단추로 클릭한 다음 **클러스터 관리**를 클릭하고 원하는 클러스터를 선택하거나 지정합니다.

3.  가운데 창에서 확장 **클러스터 코어 리소스**합니다.

4.  아래 **클러스터 이름**를 마우스 오른쪽 단추로 클릭 합니다 **이름** 항목을 가리킵니다 **기타 작업**, 클릭 하 고 **복구 Active Directory 개체**합니다.

### <a name="steps-for-troubleshooting-problems-caused-by-changes-in-cluster-related-active-directory-accounts"></a>클러스터와 관련 된 Active Directory 계정에 대 한 변경으로 인 한 문제 해결 단계

클러스터 이름 계정을 삭제 하거나 권한을 벗어나면 수행 된 경우 새 클러스터형된 서비스 또는 응용 프로그램을 구성 하려고 할 때 문제가 발생 합니다. 여기서이 원인일 수, Active Directory 사용자를 사용할 수 있습니다 및 컴퓨터 스냅인 확인 하거나 클러스터를 변경 하는 문제를 해결 하려면 계정 이름 및 기타 관련 계정. 이러한 유형의 문제 발생 (이벤트 1193, 1194, 1206, 또는 1207)를 참조 하는 경우 기록 된 이벤트에 대 한 자세한 [ http://go.microsoft.com/fwlink/?LinkId=118271 ](http://go.microsoft.com/fwlink/?linkid=118271)합니다.

이 절차를 완료하려면 최소한 **Domain Admins** 그룹의 구성원이거나 이와 동등한 자격을 갖추고 있어야 합니다. 적절 한 계정을 사용 하는 방법에 대 한 세부 정보를 검토 하 고 그룹 구성원 자격 [ http://go.microsoft.com/fwlink/?LinkId=83477 ](http://go.microsoft.com/fwlink/?linkid=83477)합니다.

#### <a name="to-troubleshoot-problems-caused-by-changes-in-cluster-related-active-directory-accounts"></a>클러스터와 관련 된 Active Directory 계정에 대 한 변경으로 인 한 문제를 해결 하려면

1.  도메인 컨트롤러에서 클릭 **시작**, 클릭 **관리 도구**를 클릭 하 고 **Active Directory Users and Computers**합니다. **사용자 계정 컨트롤** 대화 상자가 나타나면 원하는 작업이 표시되었는지 확인한 다음 **계속**을 클릭합니다.

2.  기본 확장 **컴퓨터** 컨테이너나 폴더는 클러스터 이름 계정 (클러스터에 대 한 컴퓨터 계정)입니다. **컴퓨터** 에 위치한 <b>Active Directory 사용자 및 컴퓨터 /</b><i>도메인 노드</i><b>/Computers</b>합니다.

3.  클러스터 이름 계정에 대 한 아이콘을 검사 합니다. 아래쪽 화살표를 가질 수 없습니다, 즉, 계정 활성화 되어야 합니다. 사용 하지 않도록 설정할 나올 경우 마우스 오른쪽 단추로 클릭 하 고 명령을 검색할 **계정 사용**합니다. 명령에 표시 되 면 클릭 합니다.

4.  에 **보기** 메뉴에서 했는지 **고급 기능** 을 선택 합니다.
    
    때 **고급 기능** 을 선택 하면 볼 수 있습니다 합니다 **보안** 계정 (개체)의 속성 탭에서 **Active Directory Users and Computers**합니다.

5.  기본값을 마우스 오른쪽 단추로 클릭 **컴퓨터** 컨테이너나 폴더는 클러스터 이름 계정입니다.

6.  **속성**을 클릭합니다.

7.  **보안** 탭에서 **고급**을 클릭합니다.

8.  권한이 있는 계정 목록에서 클러스터 이름 계정을 클릭 하 고 클릭 **편집**합니다.
    

> [!NOTE]
> 클러스터 이름 계정 나열 되지 않으면, 클릭 <STRONG>추가</STRONG> 목록에 추가 합니다. 
<br>


9.  클러스터 이름 계정 (클러스터 이름 개체 또는 CNO 라고도 함)를 위해 **허용** 가 선택 합니다 **컴퓨터 개체 만들기** 하 고 **모든 속성 읽기** 사용 권한입니다.
    
    ![](media/configure-ad-accounts/Cc731002.f5977c4d-a62e-4b17-81e3-8c19ddca2078(WS.10).gif)

10. 클릭 **확인** 돌아갈 때까지 합니다 **Active Directory Users and Computers** 스냅인입니다.

11. 검토 도메인 정책 (해당 하는 경우 도메인 관리자를 사용 하 여 컨설팅) 컴퓨터 계정 (개체) 만들기와 관련이 있습니다. 클러스터 이름 계정 클러스터형된 서비스 또는 응용 프로그램 구성 될 때마다 컴퓨터 계정을 만들 수 있는지 확인 합니다. 예를 들어 도메인 관리자가 새 컴퓨터 계정을 모든 기본값 보다는 특수 한 컨테이너를 만들 수는 설정 구성 **컴퓨터** 컨테이너에 이러한 설정을 허용 하는지 확인 합니다 클러스터 이름 계정도 해당 컨테이너에 새 컴퓨터 계정을 만들 수 있습니다.

12. 기본 확장 **컴퓨터** 컨테이너 또는 컴퓨터 계정 클러스터링 된 서비스나 응용 프로그램 중 하나에 위치한 컨테이너입니다.

13. 클러스터링 된 서비스나 응용 프로그램 중 하나에 대 한 컴퓨터 계정을 마우스 오른쪽 단추로 누른 **속성**합니다.

14. 에 **보안** 탭에서 클러스터 이름 계정 권한만 제공 하 고 선택 하는 계정 간에 나열 되어 있는지 확인 합니다. 클러스터 이름 계정에 있는지 확인 **모든 권한** 권한 (합니다 **허용** 확인란을 선택). 표시 되지 않는 경우 클러스터 이름 계정 목록에 추가 하 고 부여 **전면적인** 권한.
    
    ![](media/configure-ad-accounts/Cc731002.2e614376-87a6-453a-81ba-90ff7ebc3fa2(WS.10).gif)

15. 각 클러스터 된 서비스 및 클러스터에서 구성 하는 응용 프로그램에 대 한 13-14 단계를 반복 합니다.

16. (기본적으로 10) 컴퓨터 개체를 만들기 위한 도메인 전체 할당량에 도달 하지 않은 (해당 하는 경우 도메인 관리자를 사용 하 여 컨설팅)를 확인 합니다. 이 절차에서는 이전 항목 모두 되었는지 검토 하 고 수정 하는 경우 및 할당량에 도달한 경우 할당량을 증가 하는 것이 좋습니다. 할당량을 변경 합니다.
    
    1.  실행 관리자 권한으로 명령 프롬프트를 열고 **ADSIEdit.msc**합니다.  
          
    2.  마우스 오른쪽 단추로 클릭 **ADSI 편집**, 클릭 **연결할**를 클릭 하 고 **확인**합니다. 합니다 **기본 명명 컨텍스트** 콘솔 트리에 추가 됩니다.  
          
    3.  두 번 클릭 **기본 명명 컨텍스트**, 아래에 있는 도메인 개체를 마우스 오른쪽 단추로 클릭 하 고 클릭 **속성**합니다.  
          
    4.  스크롤하여 **ms-DS-MachineAccountQuota**, 선택, 클릭 **편집**에서 값을 변경 하 고 클릭 **확인**합니다.
