# <a name="whats-new-in-failover-clustering"></a>장애 조치(failover) 클러스터링의 새로운 기능

> 적용 대상: Windows Server 2019, Windows Server 2016, Windows Server (반기 채널)

이 항목에서는 장애 조치 클러스터링에 대 한 Windows Server 2019, Windows Server 2016의에서 새로운 기능과 변경 된 기능에 설명 하 고 Windows 서버 반기 채널 해제 합니다.

## <a name="whats-new-in-windows-server-2019"></a>Windows Server 2019의 새로운 기능

- **클러스터 설정**

    클러스터 집합을 사용 하는 클러스터의 현재 제한 초과 하는 단일 소프트웨어 정의 데이터 센터 (SDDC) 솔루션에 있는 서버의 수를 늘릴 수 있습니다. 이렇게 하려면 클러스터 집합-여러 장애 조치 클러스터의 느슨하게 결합 된 그룹으로 여러 클러스터를 그룹화 하 여: 계산, 저장소 및 하이퍼 수렴 형입니다.
    클러스터 집합을 사용 하 여 온라인 가상 컴퓨터로 이동할 수 있습니다 (실시간 마이그레이션) 클러스터 내 클러스터를 설정 합니다.

    자세한 내용은 참조 하세요. [클러스터 집합](../storage/storage-spaces/cluster-sets.md)합니다.

- **Azure에서 인식 클러스터**

    장애 조치 클러스터 이제 자동으로 검색 하는 Azure IaaS virtual machines에서에서 실행 하 고 사전 장애 조치 및 최고 수준의 가용성을 달성 하기 위해 Azure 계획 된 유지 관리 이벤트의 로깅을 제공 하는 구성을 최적화할 때. 클러스터 이름에 대 한 동적 네트워크 이름으로 부하 분산 장치를 구성할 필요가 제거 하 여 배포도 간단 합니다.

- **도메인 간 클러스터 마이그레이션**

    장애 조치 클러스터 이제 동적으로 이동할 수 한 Active Directory 도메인에서 다른 도메인 통합을 단순화 하 고 하드웨어 파트너에 의해 만들어지고 나중에 고객의 도메인에 가입 된 클러스터를 허용 합니다.
- **USB 미러링 모니터 서버**

    이제 간단한 결정 클러스터용 쿼럼 감시로 네트워크 스위치에 연결 된 USB 드라이브를 사용할 수 있습니다. 이 모든 SMB2 규격 장치를 지원 하기 위해 파일 공유 감시를 확장 합니다.

- **클러스터 인프라 향상**

    이제 CSV 캐시는 가상 머신 성능을 향상 시키기 위해 기본적으로 사용 됩니다. MSDTC에서 클러스터 공유 볼륨을 지원하여 SQL Server와 같은 저장소 공간 다이렉트에 MSDTC 워크로드 배포를 허용합니다. 개선된 논리로 자동 복구를 통해 분할된 노드를 감지하고 노드를 클러스터 멤버십으로 반환합니다. 개선된 클러스터 네트워크 경로 감지 및 자동 복구.

- **저장소 공간 다이렉트 지원 클러스터 인식 업데이트**

    CAU(클러스터 인식 업데이트)가 저장소 공간 다이렉트에 통합 및 인식되어 각 노드에서 완료된 데이터 재동기화 유효성을 검사합니다. 클러스터 인식 업데이트는 업데이트를 지능적으로 필요한 경우에 다시 시작 검사 합니다. 이 통해 오케스트레이션 계획 된 유지 관리에 대 한 클러스터의 모든 서버를 다시 시작 합니다.

- **파일 공유 감시 향상 기능** 다음과 같은 시나리오에서 파일 공유 감시를 사용 하도록 설정한 것: 
    - 클라우드 감시를 사용 하지 못하게 하는 원격 위치 때문에 인터넷 액세스 없거나 극히 저하. 
    - 디스크 감시에 대 한 공유 드라이브의 부족 합니다. 이 SQL Server 항상에서 그룹 AG (가용성), 저장소 공간 다이렉트 하이퍼 수렴 형 구성 수 또는 * Exchange 데이터베이스 가용성 그룹 (DAG), 공유 디스크를 사용 하는 중입니다. 
    - DMZ 뒤 되는 클러스터로 인해 도메인 컨트롤러 연결을 부족 합니다. 
    - 작업 그룹 또는 도메인 간 클러스터를 가지는 없는 Active Directory 클러스터 이름 개체 (CNO)입니다. 관리 블로그 게시물에서는 이러한 개선 기능에 대해 자세히 알아봅니다. 장애 조치 클러스터 파일 공유 감시 및 DFS 합니다.
    
    에서는 이제 명시적으로 사용을 차단 DFS 네임 스페이스 공유 위치로 합니다. 파일 공유 감시를 추가 합니다. 공유를 DFS를 클러스터에 대 한 안정성 문제를 유발할 수 및이 구성은 지원 되지 않았습니다. 공유에는 DFS 네임 스페이스를 사용 하 고 DFS 네임 스페이스 감지 되 면 장애 조치 클러스터 관리자는 미러링 모니터 서버 생성을 차단 지원 되지 않습니다 하는 방법에 대 한 오류 메시지를 표시 하는 경우를 검색 하기 위한 논리가 추가 되었습니다.
- **클러스터 보안 강화**

    클러스터 공유 볼륨 및 저장소 공간 다이렉트에 대한 SMB(서버 메시지 블록)를 통한 클러스터 간 통신으로 인증서를 활용하여 더 안전한 플랫폼을 제공합니다. 이를 통해 장애 조치 클러스터는 NTLM에 대한 종속성 없이 작동하고 보안 기준이 활성화됩니다.
- **장애 조치 클러스터에는 더 이상 NTLM 인증 사용**

    장애 조치 클러스터는 더 이상 NTLM 인증을 사용합니다. 대신 인증서 기반 인증 및 Kerberos는 단독으로 합니다. 이 보안 향상 된 기능을 활용 하 고 사용자 또는 배포 도구에 필요한 변경이 있습니다. 또한 장애 조치 클러스터를 NTLM에 사용할 수 있는 환경에서 배포할 수 있습니다. 


## <a name="whats-new-in-windows-server-2016"></a>Windows Server 2016의 새로운 기능

### <a name="BKMK_RollingUpgrade"></a>클러스터 운영 체제 롤링 업그레이드

클러스터 운영 체제 롤링 업그레이드 관리자가 Hyper-v 또는 스케일 아웃 파일 서버 워크 로드를 중지 하지 않고 최신 버전으로 Windows Server 2012 R2에서 클러스터 노드의 운영 체제를 업그레이드 합니다. 이 기능을 사용하면 SLA(서비스 수준 계약)의 가동 중지 시간 위반을 피할 수 있습니다. 

**이와 같은 변경을 통해 더해지는 가치**  

Hyper-v 또는 스케일 아웃 파일 서버 클러스터에서 Windows Server 2012 R2로 업그레이드 Windows Server 2016 이상 가동 중지 시간이 필요 합니다. Windows Server 2016를 실행 하는 모든 클러스터의 노드가 될 때까지 Windows Server 2012 R2 수준에서 작동 하도록 클러스터를 계속 합니다. Windows PowerShell cmdlt을 사용 하 여 클러스터 기능 수준을 Windows Server 2016으로 업그레이드 됩니다 `Update-ClusterFunctionalLevel`합니다. 

> [!WARNING]  
> -   클러스터 기능 수준을 업데이트 한 후 Windows Server 2012 R2 클러스터 기능 수준을로 다시 이동할 수 없습니다. 
> -   될 때까지 `Update-ClusterFunctionalLevel` cmdlet이 실행, 프로세스는 되돌릴 수 있어야 하 고 Windows Server 2012 R2 노드를 추가할 수 있습니다 및 Windows Server 2016 노드를 제거할 수 있습니다. 

**달라진 기능**  

Hyper-v 또는 스케일 아웃 파일 서버 장애 조치 클러스터를 쉽게 가동 중지 시간 없이 업그레이드할 수 이제 또는 Windows Server 2016 운영 체제를 실행 하는 노드를 사용 하 여 새 클러스터를 작성 해야 합니다. Windows Server 2012 R2로 마이그레이션 클러스터는 기존 클러스터를 오프 라인 및 각 노드에 대 한 새 운영 체제를 다시 설치 하 고, 그런 다음 클러스터를 다시 온라인 상태로 전환에 관여 합니다. 이전 프로세스가 였던 번거롭고 필요한 가동 중지 시간이 발생 합니다. 그러나 Windows Server 2016에서는 클러스터 필요가 없습니다 언제 든 지 오프 라인으로 전환 합니다. 

클러스터 운영 체제 업그레이드에 대 한 클러스터의 각 노드에 대해은 다음과 같습니다.  
-   노드 일시 중지 하 고 실행 중인 모든 가상 머신의 종료 합니다. 
-   가상 컴퓨터 (또는 다른 클러스터 워크 로드)은 클러스터의 다른 노드로 마이그레이션됩니다. 
-   기존 운영 체제는 제거 하 고 노드에서 Windows Server 2016 운영 체제 새로 설치 수행 됩니다. 
-   Windows Server 2016 운영 체제를 실행 하는 노드를 클러스터에 다시 추가 됩니다. 
-   이 시점에서 클러스터 클러스터 노드에서 Windows Server 2012 R2 또는 Windows Server 2016 실행 되므로 혼합 모드로 실행 이라고 합니다. 
-   Windows Server 2012 R2에서 클러스터 기능 수준을 유지 됩니다. 이 기능 수준에서 운영 체제의 이전 버전과 호환성에 영향을 주는 새로운 기능 Windows Server 2016에서 사용할 수 없게 됩니다. 
-   결국 모든 노드가 Windows Server 2016으로 업그레이드 됩니다. 
-   Windows PowerShell cmdlet을 사용 하 여 Windows Server 2016 클러스터 기능 수준을 변경한 다음 `Update-ClusterFunctionalLevel`합니다. 이 시점에서 Windows Server 2016 기능 활용을 걸릴 수 있습니다. 

자세한 내용은 [클러스터 운영 체제 롤링 업그레이드](cluster-operating-system-rolling-upgrade.md)합니다. 

### <a name="BKMK_SR"></a>저장소 복제본  
저장소 복제본은 저장소 독립적인 수 있는 새로운 기능이 블록 수준의 동기 복제 서버 또는 사이트 간 장애 조치 클러스터를 확장할 뿐만 아니라 재해 복구에 대 한 클러스터 사이입니다. 동기 복제를 사용하면 파일 시스템 수준에서 데이터가 손실되지 않고 크래시 일관성이 있는 볼륨을 사용하여 실제 사이트의 데이터를 미러링할 수 있습니다. 비동기 복제는 대도시 범위를 넘어 사이트를 확장합니다(데이터가 손실될 수도 있음). 

**이와 같은 변경을 통해 더해지는 가치**  

저장소 복제본을 사용 하면 다음을 수행할 수 있습니다.  

-   중요 업무용 워크로드의 계획된 중단 및 계획되지 않은 중단을 위한 단일 공급업체 재해 복구 솔루션을 제공합니다. 

-   안정성, 확장성 및 성능이 검증된 SMB3 전송을 사용합니다. 

-   대도시 거리로 Windows 장애 조치(failover) 클러스터를 확장합니다. 

-   저장소 및 클러스터링, Hyper-v, 저장소 복제본, 저장소 공간, 클러스터, 스케일 아웃 파일 서버, SMB3, 데이터 중복 제거 및 ReFS/NTFS와 같은 종단 간 Microsoft 소프트웨어를 사용 합니다. 

-   다음과 같이 비용 및 복잡성을 줄입니다.  

    -   하드웨어 제약이 없으므로 DAS 또는 SAN과 같은 특정 저장소 구성에 대한 요구 사항이 없습니다. 

    -   상용 저장소 및 네트워킹 기술을 허용합니다. 

    -   장애 조치(Failover) 클러스터 관리자를 통해 개별 노드 및 클러스터에 대한 그래픽 관리가 용이합니다. 

    -   Windows PowerShell을 통한 포괄적인 대규모 스크립팅 옵션을 포함합니다. 

-   가동 중지 시간을 줄이고, Windows의 고유한 안정성 및 생산성을 높입니다. 

-   지원 가능성, 성능 메트릭 및 진단 기능을 제공합니다. 

자세한 내용은 [Windows Server 2016의 저장소 복제본](../storage/storage-replica/storage-replica-overview.md)을 참조하세요. 


### <a name="BKMK_CloudWitness"></a>클라우드 감시  
클라우드 감시는 중재 지점으로 Microsoft Azure를 활용하는 Windows Server 2016에서 새로운 유형의 장애 조치 클러스터 쿼럼 감시 기능입니다. 다른 쿼럼 감시와 마찬가지로 클라우드 감시는 응답을 가져오고 쿼럼 계산에 참여할 수 있습니다. 클러스터 쿼럼 구성 마법사를 사용하여 쿼럼 감시로 클라우드 감시를 구성할 수 있습니다. 

**이와 같은 변경을 통해 더해지는 가치**  

장애 조치 클러스터로 클라우드 감시를 사용 하 여 쿼럼 감시 다음과 같은 이점이 있습니다.  

-   Microsoft Azure를 활용 하 고 세 번째 각기 다른 데이터 센터에 대 한 필요를 제거 합니다. 

-   표준 공개적으로 사용할 수 있는 Microsoft Azure Blob Storage 공용 클라우드에서 호스팅되는 Vm의 추가 유지 관리 오버 헤드를 제거 하는 사용 합니다. 

-   여러 클러스터 (클러스터로 파일을 하나의 blob; blob 파일 이름으로 사용 하는 고유한 클러스터 id)에 대 한 동일한 Microsoft Azure Storage 계정은 사용할 수 있습니다. 

-   저장소 계정 (매우 작은 데이터 blob 파일에는 클러스터 노드의 상태가 변경 될 때 한 번만 업데이트 된 blob 파일 마다 기록)에 매우 저렴 한 지속적인 비용을 제공 합니다. 

자세한 내용은 [는 클라우드 미러링 모니터 서버에 대 한 장애 조치 클러스터를 배포](deploy-cloud-witness.md)합니다. 

**달라진 기능**  

이 기능은 Windows Server 2016의 새로운 기능입니다. 

### <a name="BKMK_VMs"></a>가상 머신 복원 력이  
**복원 력 계산** Windows Server 2016에는 다음과 같이 계산 클러스터의 클러스터 간 통신 문제를 줄이기 위해 향상 된 가상 컴퓨터 계산 복원 력 포함 됩니다. 

-   **가상 머신에 사용할 수 있는 복원 력 옵션:**  이제 일시적 오류 중 가상 컴퓨터의 동작을 정의 하는 가상 머신 복원 력 옵션을 구성할 수 있습니다.  

    -   **복원 력 수준:** 일시적인 오류를 처리 하는 방법을 정의 하는 데 도움이 됩니다. 

    -   **복원 력 기간:**  모든 가상 컴퓨터 격리를 실행할 수 있습니다 기간을 정의 하는 데 도움이 됩니다. 

-   **비정상 노드 격리:** 비정상 노드 격리는 및를 더 이상 클러스터에 가입 하도록 허용 합니다. 이렇게 하면 다른 노드 및 전체 클러스터에 부정적인 영향에서 날개를 퍼 덕 노드를 않습니다. 

자세한 내용은 가상 컴퓨터 계산 복원 력 워크플로 및 노드 격리 설정에 대 한 격리 또는 격리에서 노드는 배치 하는 방법을 제어 하는 참조 [Windows Server 2016에서 가상 컴퓨터 계산 탄력성](http://blogs.msdn.com/b/clustering/archive/2015/06/03/10619308.aspx)합니다. 

**저장소 복원 력** Windows Server 2016에서에서 virtual machines는 임시 저장소 오류 복원 력이 더욱 높아집니다. 향상 된 가상 머신 복원 력이 저장소 중단이 발생할 경우 테 넌 트 가상 컴퓨터 세션 상태를 유지 하는 데 도움이 됩니다. 저장소 인프라 문제에 대 한 가상 컴퓨터를 지능적이 고 빠른 응답 하 여 수행 됩니다. 

가상 컴퓨터를 내부 저장소에서 연결을 끊습니다, 일시 중지 하 고 저장소 복구 될 때까지 기다립니다. 일시 중지 하는 동안 가상 컴퓨터에서 실행 중인 응용 프로그램의 컨텍스트를 유지 합니다. 해당 저장소에 가상 컴퓨터의 연결이 복원 되 면 가상 머신이 실행 중인 상태로 돌아갑니다. 결과적으로, 테 넌 트 컴퓨터의 세션 상태는 복구에 유지 됩니다. 

Windows Server 2016에서 가상 머신 저장소 복원 력이 너무 인식 하 고 게스트 클러스터에 대 한 최적화 합니다. 

### <a name="BKMK_Diagnostics"></a>장애 조치 클러스터링의 진단 개선 사항  
Windows Server 2016를 장애 조치 클러스터를 사용 하 여 문제를 진단 하려면 다음을 포함 합니다.  

-   클러스터는 (예: 표준 시간대 정보와 DiagnosticVerbose 로그) 로그 파일에 몇 가지 향상 된 장애 조치 클러스터링 문제를 해결 하려면 쉽습니다. 자세한 내용은 [Windows Server 2016 장애 조치 클러스터 문제 해결 기능 향상-클러스터 로그](http://blogs.msdn.com/b/clustering/archive/2015/05/15/10614930.aspx)합니다. 

-   새 덤프 유형의 **활성 메모리 덤프**, 가상 컴퓨터에 할당 된 대부분의 메모리 페이지를 필터링 하 고 훨씬 쉽고 작은 저장 하거나 복사 하 여 memory.dmp 사용 합니다. 자세한 내용은 [Windows Server 2016 장애 조치 클러스터 문제 해결 기능 향상-현재 덤프](http://blogs.msdn.com/b/clustering/archive/2015/05/18/10615526.aspx)합니다. 

### <a name="BKMK_SiteAware"></a>사이트 인식 장애 조치 클러스터  
Windows Server 2016에는 사이트 인식 장애 조치 클러스터는 물리적 위치 (사이트)에 따라 확장된 클러스터의 그룹 노드를 사용 하도록 설정 하는 포함 되어 있습니다. 클러스터 사이트 인식은 장애 조치 동작, 배치 정책, 노드와 쿼럼 동작 간의 하트 비트와 같은 클러스터 수명 주기 동안 주요 작업을 개선 합니다. 자세한 내용은 [Windows Server 2016의 사이트 인식 장애 조치 클러스터](http://blogs.msdn.com/b/clustering/archive/2015/08/19/10636304.aspx)합니다. 

### <a name="BKMK_multidomainclusters"></a>작업 그룹 및 다중 도메인 클러스터  
클러스터를 Windows Server 2012 R2 및 이전 버전에서는 동일한 도메인에 가입 된 구성원 노드 간에만 만들 수만 있습니다. Windows Server 2016에는 이러한 장벽을 허물고 Active Directory 종속성 없이 장애 조치(failover) 클러스터를 만들 수 있는 기능이 도입되었습니다. 이제 다음 구성의 장애 조치 클러스터를 만들 수 있습니다.  

-   **단일 도메인 클러스터입니다.** 동일한 도메인에 가입 된 모든 노드를 사용 하 여 클러스터를 제공 합니다. 

-   **다중 도메인 클러스터입니다.** 서로 다른 도메인의 구성원 인 노드를 사용 하 여 클러스터를 제공 합니다. 

-   **작업 그룹 클러스터입니다.** 구성원 서버는 노드가 있는 클러스터 / 작업 그룹 (가입 된 도메인이 아님). 

자세한 내용은 참조 하세요. [Windows Server 2016에서 작업 그룹 및 다중 도메인 클러스터](http://blogs.msdn.com/b/clustering/archive/2015/08/17/10635825.aspx)  
### <a name="BKMK_VMLoadBalancing"></a>가상 머신 부하 분산  
가상 머신 부하 분산은 클러스터의 노드 간에 가상 머신의 원활 하 게 부하 분산을 원활 하 게 장애 조치 클러스터링의 새로운 기능입니다. 오버 커밋된 노드 가상 머신 메모리 및 노드의 CPU 사용률에 따라 식별 됩니다. 가상 컴퓨터 (실시간 마이그레이션)을 이동할 노드에 해당 하는 경우 사용 가능한 대역폭을 사용 하 여 오버 커밋된 노드에서 합니다. 최적의 클러스터 성능 및 사용률을 확인 하는 분산의 강도 조정할 수 있습니다. 부하 분산은 Windows Server 2016 Technical Preview의 기본으로 사용 됩니다. 그러나 부하 분산 되지 SCVMM 동적 최적화를 사용할 때. 

### <a name="BKMK_VMStartOrder"></a>가상 컴퓨터 시작 순서  
가상 컴퓨터 시작 순서를 클러스터에 가상 컴퓨터 시작 순서 오케스트레이션 (및 모든 그룹)를 소개 하는 장애 조치 클러스터링의 새로운 기능을 않습니다. 가상 컴퓨터 계층에 이제 그룹화 할 수 있습니다 하 고 다른 계층 간에 시작 순서 종속성을 만들 수 있습니다. 이렇게 하면 가장 중요 한 가상 컴퓨터 (예: 도메인 컨트롤러 또는 유틸리티 가상 컴퓨터)를 첫 번째 시작 됩니다. 가상 컴퓨터에서 종속성 수는 가상 컴퓨터도 시작 때까지 시작 되지 않습니다. 

### <a name="BKMK_SMBMultiChannel"></a> 간소화 된 SMB 다중 채널 및 다중 NIC 클러스터 네트워크  
장애 조치 클러스터 네트워크 서브넷 당 단일 NIC로 제한 됩니다 더 이상 네트워크입니다. 간체 SMB 다중 채널 및 다중 NIC 클러스터 네트워크를 사용 하 여 네트워크 구성은 자동 및 워크 로드 및 클러스터 트래픽에 대 한 서브넷의 모든 NIC를 사용할 수 있습니다. 이 향상 된이 기능에는 Hyper-v, SQL Server 장애 조치 클러스터 인스턴스 및 다른 SMB 워크 로드에 대 한 네트워크 처리량을 최대화 하기 위해 고객이 수 있습니다. 

자세한 내용은 [간체 SMB 다중 채널 및 다중 NIC 클러스터 네트워크](smb-multichannel.md)합니다.
