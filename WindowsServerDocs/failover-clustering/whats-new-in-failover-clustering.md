# <a name="whats-new-in-failover-clustering-in-windows-server-2016"></a>클러스터링 Windows Server 2016의 새로운 기능
> 에 해당: (세미콜론 연간 채널) Windows Server, Windows Server 2016

이 항목 클러스터링 Windows Server 2016의 새로운 기능과 변경 된 기능에 설명 합니다.  

## <a name="BKMK_RollingUpgrade"></a>롤링 업그레이드 클러스터 운영 체제  
운영 체제 배포 업그레이드 클러스터 장애 클러스터링의 새로운 기능으로 사용 클러스터 노드에서의 운영 체제의 Hyper-v 또는 확장 파일 서버 작업 중단 없이 Windows Server 2016에 Windows Server 2012 r 2에서 업그레이드 하려면 관리자 있습니다. 이 기능을 사용 하 여, 작동 중지 저하 문제에 대 한 서비스 계약 (SLA 수준) 방지할 수 있습니다.  

**이 변경에 추가 어떤 값 되나요?**  

확장 파일 서버 또는 Hyper-v 클러스터에서에서 업그레이드 Windows Server 2012 r 2 Windows Server 2016에 더 이상 작동을 중지를 해야 합니다. 클러스터는 Windows Server 2016을 실행 하는 모든 노드에서 클러스터에 될 때까지 Windows Server 2012 r 2 수준에서 작동 하는 데 계속 됩니다. 클러스터 기능 수준을 Windows PowerShell cmdlt를 사용 하 여 Windows Server 2016으로 업그레이드 `Update-ClusterFunctionalLevel`합니다.  

> [!WARNING]  
> -   클러스터 기능 수준 업데이트 한 후 Windows Server 2012 r 2 클러스터 기능 수준으로 다시 이동할 수 없습니다.  
> -   될 때까지 `Update-ClusterFunctionalLevel` cmdlet 실행, 하는 과정은 취소할 수 및 Windows Server 2012 r 2 노드를 추가할 수 및 Windows Server 2016 노드 제거할 수 있습니다.  

**다르게 작동 무엇 인가요?**  

확장 파일 서버 또는 Hyper-v 장애 조치 클러스터 쉽게 모든 작동 중단 없이 업그레이드할 수 이제 하거나 새 클러스터 노드가 Windows Server 2016 운영 체제를 실행 하는 빌드하는 데 필요한 합니다. Windows Server 2012 r 2에 마이그레이션 클러스터 관련 오프 라인 기존 클러스터 각 노드에 대 한 새로운 운영 체제를 다시 설치 하 고 다음 클러스터를 다시 온라인 상태로 전환 됩니다. 오래 된 프로세스를 번거롭고 필요한 작동 중지 합니다. 그러나 Windows Server 2016에 클러스터 하지 않아도 언제 든 지 오프 라인으로 이동 합니다.  

업그레이드 단계에 클러스터 운영 체제에 대 한 각 노드에서 클러스터에는 다음과 같습니다.  
-   노드에서 일시 중지 되며에서 실행 되는 모든 가상 컴퓨터의 소모 수도 있습니다.  
-   가상 컴퓨터 (또는 다른 클러스터 작업) 클러스터의 다른 노드로 마이그레이션될 합니다. 가상 컴퓨터 다른 노드에서 클러스터에으로 이동지 않습니다.  
-   기존 운영 체제를 제거 하 고 새로 설치에 노드 Windows Server 2016 운영 체제의 수행 됩니다.  
-   Windows Server 2016 운영 체제를 실행 노드에서 클러스터에 다시 추가 됩니다.  
-   이 시점 클러스터 클러스터 노드에서 Windows Server 2012 R2 또는 Windows Server 2016 실행 되 고 있으므로 혼합된 모드에서 실행 되 고 라고 합니다.  
-   Windows Server 2012 r 2에는 클러스터 기능 수준을 유지 됩니다. 이 기능 수준에서 운영 체제의 이전 버전과 호환성에 영향을 주는 새로운 기능 Windows Server 2016에 사용할 수 없게 됩니다.  
-   결국 Windows Server 2016에 모든 노드 업그레이드 됩니다.  
-   다음 Windows PowerShell cmdlet 사용 하 여 Windows Server 2016 클러스터 기능 수준을 변경 됩니다 `Update-ClusterFunctionalLevel`합니다. 이 시점 Windows Server 2016 기능 기능을 사용할 수 있습니다.  

자세한 내용은 참조 [클러스터 운영 체제로 업그레이드 배포](cluster-operating-system-rolling-upgrade.md)합니다.  

## <a name="BKMK_SR"></a>저장소 복제

**이 변경에 추가 어떤 값 되나요?**  

저장소 복제 다음을 수행할 수 있습니다.  

-   Single 제공 임무 중요 한 작업 계획 한 예상치 못한 중단에 대 한 공급 업체 장애 복구 솔루션 합니다.  

-   SMB3 전송 입증된 안정성, 확장성 및 성능 사용 합니다.  

-   Windows 대도시 거리 클러스터 장애 조치를 확대 합니다.  

-   저장소 및 클러스터링, Hyper-v, 저장소 복제, 저장소 공간을 클러스터, 확장 파일 서버, SMB3, 데이터 중복 제거 ReFS/NTFS 등 끝 Microsoft 소프트웨어를 사용 합니다.  

-   줄일 비용과 복잡성 다음과 같습니다.  

    -   요구 DAS 또는 산와 같은 특정 저장 장치 구성에 대 한와 하드웨어 알 수 없는 경우  

    -   저장소 및 네트워킹 기술 상품 수 있게합니다.  

    -   기능 접근성 개별 노드와 클러스터 장애 조치를 통해 클러스터 관리자에 대 한 그래픽 관리 합니다.  

    -   Windows PowerShell 통해 포괄적인, 대규모 스크립팅 옵션을 포함합니다.  

-   줄이고 작동 중지, 안정성 및 windows 내장 생산성 향상 합니다.  

-   지원과 메트릭 성능, 진단 기능을 제공 합니다.  

자세한 내용은 참조는 [Windows Server 2016에 저장소 복제본](../storage/storage-replica/storage-replica-overview.md)합니다.  


## <a name="BKMK_CloudWitness"></a>클라우드 미러링  
클라우드 감시 클러스터 장애 조치 쿼럼 감시 Microsoft Azure 중재 지점으로 활용 하는 Windows Server 2016의 새로운 유형의입니다. 클라우드 감시 다른 쿼럼 감시 같은 투표를 가져오고 쿼럼 계산에 참여할 수 있습니다. 클라우드 감시 구성 클러스터 쿼럼 마법사를 사용 하 여 쿼럼 감시도 구성할 수 있습니다.  

**이 변경에 추가 어떤 값 되나요?**  

장애 클러스터로 감시 클라우드를 사용 하 여 쿼럼 감시 다음과 같은 이점을 제공 합니다.  

-   Microsoft Azure을 활용 하 고 세 번째 센터 필요가 없습니다.  

-   표준 공개적으로 사용할 수 있는 Microsoft Azure 물방울 저장소 추가 유지 관리 오버 공개 클라우드에서 호스트 Vm의 제거를 사용 합니다.  

-   동일한 Microsoft Azure 저장소 계정 여러 클러스터 (클러스터로 파일을 하나의 물방울, 물방울 파일 이름으로 사용 되는 클러스터 고유 id)에 사용할 수 있습니다.  

-   매우 낮은 지속적인 비용 저장소 계정 (매우 적은 데이터 클러스터 노드에서 상태를 변경할 때 한 번만 물방울 파일이 업데이트 기록 물방울 파일당)에 제공 합니다.  

자세한 내용은 참조 [는 클라우드 감시 클러스터 하기 위한 장애 조치를 배포](deploy-cloud-witness.md)합니다.  

**다르게 작동 무엇 인가요?**  

이 기능은 Windows Server 2016의 새로운 기능입니다.  

## <a name="BKMK_VMs"></a>가상 컴퓨터 복구  
**복구 계산** Windows Server 2016 클러스터 간 통신 문제 계산 클러스터에 다음과 같은 줄이는 데 도움이 되도록 향상 된 가상 컴퓨터 컴퓨팅 복구 포함 됩니다. 

-   **가상 컴퓨터에 대 한 사용 가능한 복구 옵션:** 일시적인 오류 중 가상 컴퓨터의 동작에 정의 가상 컴퓨터 복구 옵션 구성할 수 있습니다.  

    -   **복구 수준:** 는 일시적인 오류 처리 하는 방법을 정의 하 고 있습니다.  

    -   **복구 기간:** 을 실행할 격리 된 모든 가상 컴퓨터 수 기간을 정의 하는 데 도움이 됩니다.  

-   **비정상적인 노드 격리:** 비정상적인 격리 된 노드와 클러스터에 가입 수 없습니다. 부정적인 다른 노드와 클러스터 전체 영향에서 날개를 퍼 덕 노드 수 없습니다.  

설정에 대 한 자세한 내용은 가상 컴퓨터 컴퓨팅 복구 워크플로 및 노드 격리 노드가 격리 또는 격리에 배치 됩니다 방식을 제어 하는, 참조 [Windows Server 2016에서 가상 컴퓨터 계산 복구](http://blogs.msdn.com/b/clustering/archive/2015/06/03/10619308.aspx)합니다.  

**저장소 복구** Windows Server 2016 가상 컴퓨터는 실패 임시 저장소를 더 복원 합니다. 향상 된 가상 컴퓨터 복구 저장소를 더 이상 발생 했을 때 테 가상 컴퓨터 세션 상태를 유지 하는 데 도움이 됩니다. 이 저장소 infrastructure 문제에 대 한 빠른 하 고 지능적인 가상 컴퓨터 응답 하 여 수행 됩니다.  

가상 컴퓨터의 내부 저장소에서 연결을 끊고, 일시 중지 하 고 저장소를 복구할 때까지 기다립니다. 일시 중지 하는 동안 가상 컴퓨터에서 실행 되는 응용 프로그램의 상황에 맞는 유지 됩니다. 가상 컴퓨터의 연결을 저장소 복원 됩니다 가상 컴퓨터 뛰 상태로 돌아갑니다. 결과적으로, 복구 테 기계 세션 상태가 유지 됩니다.  

Windows Server 2016에서 가상 컴퓨터 메모리 복구는 인식 하 고 게스트 클러스터에 대 한 최적화 된 하기도 합니다.

## <a name="BKMK_multidomainclusters"></a>다중 도메인 및 작업 그룹 클러스터  
Windows Server 2012 r 2와 이전 버전 클러스터 회원 노드 같은 도메인에 가입 간에 만들 수만 있습니다.   Windows Server 2016 이러한 장벽 분할 한 Active Directory 종속성 없는 클러스터 장애 조치를 만들 수 있는 기능을 소개 합니다. 이제 다음과 같은 구성에서 클러스터 장애 조치를 만들 수 있습니다.  

-   **클러스터 단일 도메인입니다.** 동일한 도메인에 가입 하는 모든 노드와 클러스터 합니다.  

-   **다중 도메인 클러스터 합니다.** 다른 도메인에 속한는 노드와 클러스터 합니다.  

-   **작업 그룹 클러스터 합니다.** 구성원 서버가 노드와 클러스터 / 작업 그룹 (하지 도메인에 가입).  

자세한 내용은 참조 [다중 도메인 및 작업 그룹 클러스터 Windows Server 2016에](http://blogs.msdn.com/b/clustering/archive/2015/08/17/10635825.aspx)  
## <a name="BKMK_VMLoadBalancing"></a>가상 컴퓨터 부하 분산  
가상 컴퓨터 부하 분산 클러스터링 클러스터에서 노드에서 가상 컴퓨터의 원활한 부하 분산 용이 하 게의 새로운 기능입니다. 너무 많이 위해 최선을 다하고 노드 가상 컴퓨터 메모리 및 노드에서 cpu에 따라 표시 됩니다. 가상 컴퓨터 (실시간 마이그레이션) 이동 다음 (해당 하는 경우) 대역폭이 노드를 위해 최선을 다하고 과도 하 게 노드에서 합니다. 균형 수준 최적의 클러스터 성능과 활용도 조정할 수 있습니다.  부하 분산 Windows 서버 2016 Technical Preview에서 기본적으로 활성화 됩니다. 그러나 부하 분산 불가능 SCVMM 동적 최적화를 사용 합니다.  

## <a name="BKMK_VMStartOrder"></a>가상 컴퓨터 시작 주문  
가상 컴퓨터 시작 순서가 클러스터링 가상 컴퓨터에 대 한 순서 오케스트레이션 시작 (및 모든 그룹) 기능이 도입 된 새로운 기능 클러스터에서 합니다. 가상 컴퓨터가 계층을으로 그룹화 할 수 하 고 다른 계층 간에 시작 종속성 만들 수 있습니다. 이렇게 하면 (도메인 컨트롤러 또는 유틸리티 가상 컴퓨터) 등 가장 중요 한 가상 컴퓨터 첫 번째 시작 됩니다. 가상 컴퓨터에 의존 있는도 시작 될 때까지 가상 컴퓨터 시작 되지 않습니다.  

## <a name="BKMK_SMBMultiChannel"></a>간소화 된 SMB 다중 채널 및 다중 NIC 클러스터 네트워크  
클러스터 장애 조치 네트워크는 더 이상 각 단일 NIC 제한 / 네트워크입니다. 간체 SMB 다중 채널와 다중 NIC 클러스터 네트워크 네트워크 구성 자동 이며 교통 클러스터 및 작업에 대 한 모든 NIC 서브넷에 사용할 수 있습니다. 이 enhancement 고객을 Hyper-v, SQL Server 장애 클러스터 인스턴스 및 기타 SMB 작업에 대 한 네트워크 처리량 극대화할 수 있습니다.  

자세한 내용은 참조 [간체 SMB 다중 채널 및 다중 NIC 클러스터 네트워크](smb-multichannel.md)합니다.
