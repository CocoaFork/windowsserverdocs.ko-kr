# <a name="how-user-account-control-works"></a>사용자 계정 컨트롤의 작동 방식

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

제어 UAC (사용자 계정) 방지 악성 프로그램 (맬웨어) 로부터 컴퓨터 손상을 하 조직 보다 잘 관리 데스크톱을 배포 합니다. Uac, 특별히 관리자에 관리자 수준 시스템에 대 한 액세스 권한을 부여 하지 않는 한 응용 프로그램 및 작업을 항상 관리자가 아닌 계정, 보안 컨텍스트에서 실행 됩니다. UAC 무단으로 응용 프로그램은 자동으로 설치를 차단 하 고 시스템 설정 실수로 변경할 수 있습니다.

## <a name="uac-process-and-interactions"></a>UAC 프로세스와 상호 작용
각 응용 프로그램 관리자 액세스 토큰 요구 하는 데 동의 관리자를 라는 해야 합니다. 한 가지 예외는 간의 관계를 부모 및 자녀 프로세스입니다. 자녀가 프로세스 부모 과정에서 사용자 액세스 토큰을 상속합니다. 그러나 부모 및 자녀 프로세스 같은 무결성 수준이 있어야 합니다. Windows Server 2012의 무결성 수준 표시 하 여 프로세스를 보호 합니다. 무결성 수준은 측정 신뢰 됩니다. "고급" 무결성 응용 프로그램은 디스크 "낮은" 무결성 응용 프로그램이 운영 체제, 웹 브라우저 등 손상 될 가능성이 작업을 수행 하는 동안 응용 프로그램을 파티션 등 시스템 데이터를 수정 하는 작업을 수행 합니다. 낮은 무결성 수준으로 응용 프로그램 높은 무결성 수준으로 응용 프로그램의 데이터를 수정할 수 없습니다. 표준 사용자를 관리자 액세스 토큰 요구 하는 응용 프로그램을 실행할 때, UAC 사용자 유효한 관리자 자격 증명을 제공 해야 합니다.

이 프로세스 역시 방법을 이해를 돕기 위해은 Windows Server 2012 로그온 프로세스의 세부 정보를 확인 해야 합니다.

### <a name="windows-server-2012-logon-process"></a>Windows Server 2012 로그온 프로세스
다음 그림 어떻게 표준 사용자에 대 한 로그온 과정에서 다른 지 관리자로 로그온 프로세스를 보여 줍니다.

![표준 사용자에 대 한 로그온 과정에서 관리자로 로그온 프로세스 차이점 시연 그림](../media/How-User-Account-Control-Works/UACWindowsLogonProcess.gif)

기본적으로 표준 사용자와 관리자 리소스에 액세스 하 고 보안 컨텍스트에서 표준 사용자의 응용 프로그램을 실행 합니다. 사용자가 컴퓨터에 로그온 할 때 해당 사용자에 대 한 액세스 토큰을 생성 됩니다. 액세스 토큰 사용자 포함 하 여 특정 보안 Sid (식별자) 및 Windows 권한을 부여 되 액세스 수준에 대 한 정보를 포함 되어 있습니다.

관리자로 로그온, 사용자에 대해 두 개의 개별 액세스 토큰 생성 됩니다: 사용자 표준 액세스 토큰과 관리자 액세스 토큰 합니다. 표준 사용자 액세스 토큰에 관리자 액세스 토큰 같은 사용자 관련 정보가 되지만 관리 Windows 권한 및 Sid 제거 됩니다. 표준 사용자 액세스 토큰을 시작 (표준 사용자 응용 프로그램) 관리 작업을 수행 하지 않는 응용 프로그램 사용 됩니다. 표준 사용자 액세스 토큰 (Explorer.exe) 바탕 화면을 표시 하려면 사용 됩니다. Explorer.exe는 다른 모든 사용자가 시작한 프로세스 액세스 토큰 상속 부모 프로세스입니다. 결과적으로, 응용 프로그램을 모두 표준 사용자로 실행 하지 않으면 사용자의 동의 제공 하거나 자격 증명 모든 관리 액세스 토큰을 사용 하 여 응용 프로그램을 합니다.

사용자의 관리자가 그룹 구성원에에 로그인 하 고 웹을 검색 하 고 표준 사용자 액세스 토큰을 사용 하 여 전자 메일을 읽을 수 있습니다. 관리자가 자동으로 Windows Server 2012 관리자 액세스 토큰 요구 하는 작업을 수행 해야 하는 경우 사용자에 게 승인 라는 메시지가 표시 됩니다. 이 메시지는 상승 라고 및 보안 정책이 로컬 스냅인 (Secpol.msc) 또는 그룹 정책을 사용 하 여 해당 동작 구성할 수 있습니다.

> [!NOTE]
> 라는 용어 "따를" 사용자 동의 라는 메시지가 표시 하거나 전체 관리자 액세스 토큰을 사용 하 여 자격 증명 하는 Windows Server 2012에서 프로세스를 참조 하는 데 사용 됩니다.

### <a name="the-uac-user-experience"></a>UAC 사용자 환경
UAC를 사용 하는 경우에 표준 사용자에 대 한 사용자 환경을 승인 모드 관리자의에서 관리자는 다릅니다. Windows Server 2012를 실행 중 권장 되 고 더 안전 하 게 방법은 기본 사용자는 표준 사용자 계정의 계정을 확인 하는 것 합니다. 표준 사용자로 실행 관리 환경에 대 한 보안을 극대화 하는 데 도움이 됩니다. 기본 UAC 상승할 구성 요소와 표준 사용자가 로컬 관리자 계정에 대 한 올바른 자격 증명을 입력 하 여 관리 작업 쉽게 수행할 수 있습니다. 기본적으로 제공 UAC 상승할 구성 요소 표준 사용자를 위한 자격 증명 확인입니다.

표준 사용자 승인 모드 관리자의에서 관리자 권한으로 실행 하는 것으로 실행 하는 대신 합니다. 기본 UAC 상승할 구성 요소와 쉽게 로컬 관리자가 그룹의 회원 승인 제공 하 여 관리 작업을 수행 수 있습니다. 기본적으로 제공 UAC 상승할 구성 요소 승인 모드 관리자의에서 관리자 계정에 대 한 동의 확인을 라고 합니다. UAC 상승할 묻는 동작 로컬 보안 정책이 스냅인 (Secpol.msc) 또는 그룹 정책을 사용 하 여 구성할 수 있습니다.

**동의 하 고 자격 증명 표시 되는 메시지**

Uac 설정, Windows Server 2012 동의를 묻는 메시지가 표시 하거나 유효한 로컬 관리자 계정 자격 증명 프로그램이 나 전체 관리자 액세스 토큰 요구 하는 작업을 시작 하기 전에 확인 합니다. 이 메시지가 표시 되지 악성 소프트웨어를 자동으로 설치할 수 있는지 확인 합니다.

**동의 확인**

사용자는 사용자의 관리 액세스 토큰 요구 하는 작업을 수행 하려고 할 때 동의 확인 표시 됩니다. 다음은 화면 UAC 동의 확인입니다.

![UAC 동의 메시지의 스크린샷](../media/How-User-Account-Control-Works/UACConsentPrompt.gif)

**자격 증명 확인**

표준 사용자는 사용자의 관리 액세스 토큰 요구 하는 작업을 수행 하려고 할 때 자격 증명 확인 표시 됩니다. 로컬 보안 정책이 스냅인 (Secpol.msc) 또는 그룹 정책을 사용 하 여이 표준 사용자 기본 프롬프트 동작 구성할 수 있습니다. 관리자가 사용자 계정 컨트롤을 설정 하 여 해당 자격 증명을 제공 하기 위해 해야 할 수 있습니다: 관리자 자격 증명을 값 프롬프트를 설정 관리 승인 모드 정책에 대 한 상승 동작 합니다.

다음 스크린샷 자격 증명 UAC 프롬프트의 예입니다.

![UAC 자격 증명 확인 예로 보여 주는 화면](../media/How-User-Account-Control-Works/UACCredentialPrompt.gif)

**UAC 상승**

UAC 상승 색을 즉시 식별할 보안 위험이 있을 가능성이 응용 프로그램의 응용 프로그램에 고유으로 구분 됩니다. 응용 프로그램을 대 한 전체 액세스 토큰 프로그램 관리자 권한으로 실행 시도할 때 Windows Server 2012에서 실행 파일의 게시자를 확인 하려면 먼저 분석 합니다. 응용 프로그램 실행 파일이 게시자에 따라도 구분 먼저: (서명), Windows Server 2012 검증 된 게시자 및 게시자 확인 되지 않은 (부호가). 다음 그림은 Windows Server 2012를 사용자에 게 제공 하는 컬러 상승 결정 하는 방법을 보여 줍니다.

색 코드 지정 상승 프롬프트는 다음과 같습니다.

-   빨간색 보호 아이콘에 빨간색 배경에: 응용 프로그램 그룹 정책에서 차단 하거나 차단 된 게시자에서가 있습니다.

-   파란색과 골드 보호 아이콘 배경 블루: 응용 프로그램은 Windows Server 2012 관리 응용 프로그램, 제어판 항목을 등 합니다.

-   파란색 보호 아이콘 배경 블루: 응용 프로그램 Authenticode을 사용 하 여 로그인 하 고 로컬 컴퓨터에서 신뢰할 수 있습니다.

-   노란색 보호 아이콘 배경에 노란색: 응용 프로그램 서명 되지 않은 또는 서명 있지만 아직 로컬 컴퓨터에서 신뢰 하지 않습니다.

**보호 아이콘**

와 같은 일부 제어판 항목 **날짜 및 시간 속성**, 표준 사용자 작업 관리자의 조합으로 나만의 포함 되어 있습니다. 표준 사용자가 시간을 확인 하 고 해당 표준 시간대 변경 수 있지만 전체 관리자 액세스 토큰 로컬 시스템 시간을 변경 하는 데 합니다. 다음은의 스크린샷은 **날짜 및 시간 속성** 제어판 항목 합니다.

![화면을 보여 주는 스크린샷는 * * 날짜 및 시간 속성 * * 제어판 항목](../media/How-User-Account-Control-Works/UACShieldIcon.gif)

보호 아이콘은 **날짜 및 시간 변경** 단추 나타냅니다 프로세스는 전체 관리자 액세스 토큰 요구 및 UAC 상승 표시 됩니다.

**상승 보안**

상승 프로세스 프롬프트 보안 데스크톱으로 전환 하 여 보안을 강화할 합니다. Windows Server 2012에서 기본적으로 보안 데스크톱에 동의 하 고 자격 증명 메시지가 표시 됩니다. Windows 프로세스만 보안 데스크톱에 액세스할 수 있습니다. 높은 보안 수준을 유지 권장는 **사용자 계정 컨트롤: 상승할 확인 시 보안 데스크톱으로 전환** 정책 설정을 사용 합니다.

실행 파일 상승을 요청 하는 경우 사용자 데스크톱 라고도 대화형 데스크톱 보안 데스크톱으로 전환 됩니다. 보안 데스크톱 사용자 데스크톱에 어둡게 고 진행 하기 전에 응답할 수 있는 상승 표시 됩니다. 예를 클릭 하거나 데스크톱 사용자 데스크톱으로 전환 되는 아니요 때입니다.

맬웨어 보안 데스크톱의 하지만 사용자 계정 제어 한 복사본을 제공할 수 있습니다: 관리자 관리 승인 모드 정책 설정에 대 한 상승으로 설정 되어 프롬프트에 동의 예의 유효성을 클릭 하면 맬웨어가 상승할 수 없습니다. 프롬프트에 자격에 대 한 정책 설정을 설정 하는 경우 자격 증명 확인 모방한 맬웨어 자격 증명 사용자 로부터 수집 수 있습니다. 그러나 맬웨어가 높은 권한 수 없습니다 및 시스템에 맬웨어 사용자 인터페이스도 추수 한 암호를 사용 하 여 촬영을 완화 하는 다른 보호 합니다.

맬웨어 보안 데스크톱의 유효성을 제공할 수도 하세요 하는 동안 사용자의 컴퓨터에서 맬웨어를 설치 하지 않은이 문제가 발생할 수 없습니다. UAC를 사용 하는 관리자 액세스 토큰 요구 되는 프로세스 자동으로 설치 되지을 하기 때문에 사용자 명시적으로 제공 해야 동의 클릭 하 여 **예** 관리자 자격 증명을 제공 하 여 합니다. UAC 상승 특정 동작 그룹 정책에 따라 결정 됩니다.

### <a name="virtualization"></a>Virtualization
엔터프라이즈 환경에서 시스템 관리자는 보안 시스템 하려고, 때문에 많은 기간 업무 (lob 기간 업무) 응용 프로그램이 표준 사용자 액세스 토큰만 사용 하도록 설계 되어 있습니다. 결과적으로, IT 관리자 uac를 사용 하도록 설정한 Windows Server 2012를 실행 중인 경우 대부분의 응용 프로그램을 대체 필요가 없습니다.

 Windows Server 2012 응용 프로그램 UAC와 호환 되지 않는 및 제대로 실행 하려면 관리자 인 액세스 토큰 요구 하는 파일 및 레지스트리 가상화 기술이 포함 되어 있습니다. Virtualization은 UAC와 호환 되지 않는 응용 프로그램이 Windows Server 2012와 호환 되는지 확인 합니다. UAC와 호환 되지 않은 관리 응용 프로그램 등 Program Files 보호 디렉터리에 쓰는 하려고 할 때 UAC 제공 응용 프로그램을 변경 하려고 리소스에는 자체 가상화 보기 사용자의 프로필에 가상화 복사본을 유지 됩니다. 이 방법을 호환 되지 않는 응용 프로그램을 실행 하는 각 사용자는 가상화 파일의 복사본을 만듭니다.

대부분의 응용 프로그램 작업 virtualization 기능을 사용 하 여 정상적으로 작동 합니다. Virtualization 대부분의 응용 프로그램을 실행할 수 있게 합니다가 임시로 및 장기 솔루션 되지 않습니다. 응용 프로그램 개발자 응용 프로그램을 파일, 폴더 및 레지스트리 virtualization 대신 최대한 빨리 Windows Server 2012 로고 프로그램을 준수 수정 해야 합니다.

가상화 옵션은 다음과 같은 경우에는 다음과 같습니다.

1.  Virtualization 상승 되 고 모든 관리 액세스 토큰을 사용 하 여 실행 하는 응용 프로그램에 적용 되지 않습니다.

2.  Virtualization만 32 비트 응용 프로그램을 지원합니다. 비 권한이 64 비트 응용 프로그램 간단 하 게 액세스 거부 Windows 개체 (고유 식별자) 핸들을 가져오려는 경우 메시지를 받습니다. 기본 Windows 64 비트 응용 프로그램은 정확한 위치 데이터를 쓰려고 하 고 UAC과 호환 됩니다.

3.  응용 프로그램이 포함 요청된 실행 수준 특성 응용 프로그램 매니페스트 virtualization 응용 프로그램에 대 한 불가능 합니다.

### <a name="request-execution-levels"></a>실행 수준 요청
응용 프로그램 매니페스트입니다 XML 파일을 설명 하 고는 공유 하 고 개인 세로 정렬으로 어셈블리를 식별 응용 프로그램 실행 시에 연결 해야 합니다. Windows Server 2012에서 매니페스트에서 호환성을 위해 UAC 응용 프로그램 항목이 포함 되어 있습니다. 사용자의 액세스 토큰 액세스할 수 있는 권한이 묻는 매니페스트에서 항목이 포함 하는 응용 프로그램을 관리 합니다. 응용 프로그램 매니페스트에서 항목이, 없는 있지만 대부분 관리 응용 프로그램 호환성 수정 응용 프로그램을 사용 하 여 수정 하지 않고 실행할 수 있습니다. 응용 프로그램 호환성 수정 UAC Windows Server 2012에서 제대로 작동 하도록와 호환 되지 않는 응용 프로그램을 사용 하는 데이터베이스 항목이 있습니다.

UAC 호환 모든 응용 프로그램 실행 수준을 매니페스트에서에 추가 해야 합니다. 응용 프로그램을 시스템 관리자 액세스를 필요한 경우 시스템 관리자 응용 프로그램으로이 프로그램을 식별 하 고 필요한 상승 단계를 수행 하면 다음 "관리자 필요" 수준 요청한 실행와 응용 프로그램을 표시 합니다. 수준 권한이 필요한 응용 프로그램에 대 한 지정 실행을 요청 했습니다.

### <a name="installer-detection-technology"></a>설치 관리자 감지 기술
설치 프로그램은 응용 프로그램 소프트웨어 배포 하도록 설계 되었습니다. 대부분의 설치 프로그램 디렉터리 시스템 및 레지스트리 키를 작성합니다. 시스템 보호 저장할 위치로 이러한 위치는 표준 사용자가 프로그램을 설치에 대 한 액세스 권한이 없는 즉 Installer 감지 기술의 관리자가만 일반적으로 쓸 수 있습니다. Windows Server 2012 스스로 액세스 권한으로 실행 하는 데 설치 프로그램 및 요청 관리자 자격 증명 또는 관리자가 사용자의 승인을 감지 합니다. Windows Server 2012 업데이트와 응용 프로그램을 제거 하는 프로그램 스스로 감지 합니다. 설치는 사용자가 알지 실행 되지 않도록 방지 하 고 설치 프로그램 파일 시스템 및 레지스트리 보호 되는 영역에 쓰는 때문에 동의 UAC 디자인 목표 중 하나입니다.

설치 관리자 검색에만 적용 됩니다.

-   실행 파일 32 비트입니다.

-   요청한 실행 수준 특성 없이 응용 프로그램입니다.

-   Uac를 사용 표준 사용자도 실행 되는 대화형 프로세스입니다.

32 비트 과정을 생성 하기 전에 다음과 같은 특성 설치 관리자 인지 확인 하려면 검사는 다음과 같습니다.

-   파일 이름에 "업데이트" 또는 "설치", ", 설정을" 같은 키워드 포함

-   버전 리소스 필드에 다음과 같은 키워드: 공급 업체, 회사 이름, 제품 이름, 파일 설명, 원래 파일 이름, 내부 이름 및 내보내기 이름입니다.

-   실행 파일에는 세로 정렬으로 매니페스트에 키워드 포함 됩니다.

-   실행 파일에서 특정 StringTable 항목에서 키워드 연결 됩니다.

-   주요 리소스 스크립트 데이터 특성 실행 파일에 연결 됩니다.

-   실행 파일 내에서 바이트 대상으로 시퀀스 가지가 있습니다.

> [!NOTE]
> 키워드와 바이트 시퀀스 일반 특성 다양 한 installer 기술을 관찰에서 파생 되었습니다.

> [!NOTE]
> 사용자 계정 컨트롤: 응용 프로그램 설치할 및 라는 설치 프로그램을 감지 하 installer 검색에 대 한 사용 하도록 설정한 상승 정책 설정 해야 합니다. 이 설정은 기본적으로 활성화 되어 보안 정책이 로컬 스냅인 (Secpol.msc)를 사용 하 여 로컬 구성 하거나 도메인, OU 또는 특정 그룹에 대 한 (Gpedit.msc) 그룹 정책에서 구성 된 될 수 있습니다.
