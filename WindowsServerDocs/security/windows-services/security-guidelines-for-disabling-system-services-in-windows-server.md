## <a name="guidance-on-disabling-system-services-on-windows-server-2016-with-desktop-experience"></a>Windows Server 2016 데스크톱 환경에서 시스템 서비스를 사용 하지 않도록 설정에 대 한 지침

적용: Windows Server 2016

Windows 운영 체제는 중요 한 기능을 제공 하는 많은 시스템 서비스가 포함 됩니다. 다른 서비스는 다른 기본 시작 정책을: 일부는 시작 (자동) 기본적으로 일부 필요할 때 (수동), 하 고 일부는 기본적으로 사용 하지 않도록 설정 하 고 사용할 수 있어야 명시적으로 실행 될 수 있습니다. 고객이 일반적인 성능, 기능과 보안 하 각 서비스에 대 한 이러한 기본값 신중 하 게 선택 했습니다.

그러나 일부 기업 고객 자녀가 Windows Pc와는 절대 최소한의 공격을 따라서 하고자 하는 특정 환경에서 필요 하지 않은 모든 서비스를 비활성화 완벽 하 게 하는 서버에 대 한 보안 중심 잔액을 선호 될 수 있습니다. 해당 고객에 대 한 Microsoft®는에 대 한 서비스 수 안전 하 게 사용할 수 없습니다이 함께 제공 된 지침을 제공 합니다.

지침은 않는 데스크톱 경험와 Windows Server 2016 용 (최종 사용자에 게 데스크톱 대용으로 사용)입니다. 각 시스템 서비스에는 다음과 같은 분류 됩니다.

-   **사용 하지 않도록:** 보안 중심 엔터프라이즈 가능성이 큽니다 선호 하는이 서비스를 비활성화 및 기능을 중단 합니다 (자세한 내용은 아래 참조).
- **사용 안 함 확인:** 이 서비스는 일부 기업 유용한 기능을 제공 하 고 사용 되지 않는 보안 중심 기업 안전 하 게 비활성화할 수 있습니다.
- **해제 되지 않는:** 필수적인 기능에 영향을 미칠 하거나 기능 또는 특정 역할 제대로 작동 하지 못하도록는이 서비스를 사용 하지 않도록 설정 합니다. 따라서 것 하지 않도록 합니다.
-  **(지침 없음):** 이러한 서비스를 사용 하지 않도록 설정의 영향을 완전히 평가 되지 않습니다. 따라서 이러한 서비스의 기본 구성을 변경할 수 없습니다.


자녀가 Windows Pc와 서버 PowerShell 자동화 또는 보안 템플릿을 사용 하 여 해당 그룹 정책에서 선택 된 서비스를 사용 하지 않도록 설정 하는 고객 구성할 수 있습니다. 경우에 따라 제공 된 지침을 직접 자체 서비스를 사용 하지 않도록 설정 하는 대신 서비스의 기능 사용 안 함 특정 그룹 정책 설정을 포함 되어 있습니다.

### <a name="disabling-services-not-installed-by-default"></a>기본적으로 설치 되지 서비스를 사용 하지 않도록 설정

기본적으로 설치 되지 않는 서비스를 사용 하지 않도록 설정 하려면 Microsoft 정책을 적용에 대 한 권장 합니다.
-  서비스는 기능이 설치 되어 있는 경우 일반적으로 필요 합니다. 기능 또는 서비스를 설치 하려면 관리자 권한이 필요 합니다. 하지 서비스를 시작 하는 기능이 설치를 허용 하지 않습니다.
-  Microsoft Windows 서비스 차단 설치 아마도 하나 보안 위험이 높은와 유사한 제 3 자 동작을 관리 (또는 관리자가 아닌 경우에 따라) 중지 하지 않습니다.
-  초기 또는 벤치 마크 기본이 아닌 Windows 서비스 (예: W3SVC)를 비활성화 하는 느낌을 일부 감사 잘못 기술 (예: IIS) 안전 하지 않습니다 하 고 사용할 수는 없습니다.
-  기능 (및 서비스) 설치 하지 않은 경우이 추가 불필요 한 대량 기준선 인증 작동 합니다.

<br />
이 문서에 나열 된 모든 시스템 서비스에 대 한에 따라 두 테이블 열 및 Microsoft 권장 하는 설정 및 Windows Server 2016 데스크톱 환경에서 시스템 서비스 해제에 대 한 설명이 제공 합니다.

WinRM 서비스는 웹 사이트가 /wsman URL 접두사를 보유합니다. 관리자 iis 충돌을 방지 하기 IIS에서 호스트 웹 사이트가 /wsman URL 접두사 사용 하지 마십시오 확인 해야 합니다.
