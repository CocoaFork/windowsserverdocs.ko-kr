# <a name="troubleshooting-activation-error-codes"></a>정품 인증 오류 코드 문제 해결

**홈 사용자** 이 문서는 지원 담당자 및 IT 전문가용으로 제작되었습니다. Windows 정품 인증 오류 메시지에 대한 자세한 정보를 찾고 있다면 다음 Microsoft 웹 사이트로 이동하세요.

[Windows 정품 인증 오류에 대한 도움말 보기](https://support.microsoft.com/help/10738/windows-10-get-help-with-activation-errors) 

복수 정품 인증 키(MAK) 또는 키 관리 서비스(KMS)를 사용하여 하나 이상의 Windows 기반 컴퓨터에서 볼륨 정품 인증을 수행하려고 하면 특정 오류 코드가 포함되어 있는 오류 메시지가 나타납니다. 이 문서에서는 이러한 오류를 해결하는 방법을 설명합니다.

**오류 코드와 설명**

|오류 코드 |오류 메시지 |정품 인증 유형 |가능한 원인 |문제 해결 단계 |

MAK가 유효한지 확인하려면 [Microsoft 라이선싱 정품 인증 센터](https://www.microsoft.com/en-us/Licensing/existing-customer/activation-centers)에 문의하세요.    |
|    0xC004C020     |    정품 인증 서버에서 복수 정품 인증 키가 잠금 해제 제한을 초과한 것이 확인되었습니다.    |    MAK    |    MAK가 정품 인증 제한을 초과했습니다.    |    MAK는 정품 인증 횟수가 제한되도록 설계되었습니다.   [Microsoft 라이선싱 정품 인증 센터](https://www.microsoft.com/en-us/Licensing/existing-customer/activation-centers)에 문의하세요.    |
|    0xC004C021     |    정품 인증 서버에서 MAK(복수 정품 인증 키) 연장 제한이 초과되었다고 보고했습니다.    |    MAK    |    MAK가 정품 인증 제한을 초과했습니다.    |    MAK는 정품 인증 횟수가 제한되도록 설계되었습니다.   [Microsoft 라이선싱 정품 인증 센터](https://www.microsoft.com/en-us/Licensing/existing-customer/activation-centers)에 문의하세요.    |
|    0xC004F009     |    소프트웨어 보호 서비스에서 유예 기간이 만료되었다고 보고했습니다.    |    MAK    |    소프트웨어가 정품 인증되기 전에 유예 기간이 만료되었습니다. 이제 시스템이 알림 상태입니다.    |    "사용자 환경" 섹션을 참조하세요.    |
|    0xC004F00F     |    소프트웨어 라이선스 서버에서 하드웨어 ID 바인딩이 오차 허용 수준을 벗어났다고 보고했습니다.    |    MAK/KMS 클라이언트/KMS 호스트    |    하드웨어가 변경되거나 드라이버가 시스템에서 업데이트되었습니다.    |    MAK: 온라인 또는 전화 정품 인증을 사용하여 OOT 유예 기간 동안 시스템을 다시 정품 인증하세요.      KMS: 다시 시작하거나 slmgr.vbs   /ato를 실행하세요.    |
|    0xC004F014     |    소프트웨어 보호 서비스에서 제품 키를 사용할 수 없다고 보고했습니다.    |    MAK/KMS 클라이언트    |    시스템에 설치된 제품 키가 없습니다.    |    MAK 제품 키를 설치하거나 설치 미디어의 \sources\pid.txt에 있는 KMS 설치 키를 설치하세요.    |

|    0xC004F035     |    이 오류 코드는 “소프트웨어 보호 서비스에서 볼륨 라이선스 제품 키를 사용하여 컴퓨터의 정품을 인증할 수 없다고 보고했습니다...”에 해당합니다. 오류 텍스트가 올바르지만 모호합니다. 이 오류는 컴퓨터의 BIOS에 Windows 마커가 없음을 나타냅니다. 이 마커는 컴퓨터에 적격 Windows 버전이 함께 제공됨을 나타내기 위해 OEM 시스템에 제공되는 것으로, KMS 클라이언트 정품 인증에 필요합니다. 오류:   유효하지 않은 볼륨 라이선스 키. 정품을 인증하려면 제품 키를 유효한 MAK(복수 정품 인증 키) 또는 정품 키로 변경해야 합니다. 적격 운영 체제 라이선스와 볼륨 라이선스 Windows 7 업그레이드 라이선스 또는 소매점의 Windows 7 정품 라이선스가 있어야 합니다. 이 소프트웨어의 다른 모든 설치는 계약 및 관련 저작권 법에 위배됩니다.    |    KMS 클라이언트/KMS 호스트    |    Windows 7 볼륨 버전은 업그레이드용으로만 라이선스가 부여됩니다. 적격 운영 체제가 설치되지 않은 컴퓨터에 볼륨 운영 체제를 설치할 수는 없습니다.    |    적격 버전의 Microsoft 운영 체제를 설치한 후 MAK를 사용하여 정품 인증하세요.    |
|    0xC004F038     |    소프트웨어 보호 서비스에서 해당 컴퓨터를 정품 인증할 수 없다고 보고했습니다. KMS(키 관리 서비스)에서 반환한 수가 부족합니다. 시스템 관리자에게 문의하세요.    |    KMS 클라이언트    |    KMS 호스트의 수가 적습니다. KMS 수는 Windows Server의 경우 5개 이상, Windows 클라이언트의 경우 25개 이상이어야 합니다.    |    KMS 클라이언트를 정품 인증하려면 KMS 풀에 더 많은 컴퓨터가 필요합니다. KMS 호스트의 현재 수를 확인하려면 Slmgr.vbs /dli를 실행하세요.    |
|    0xC004F039     |    소프트웨어 보호 서비스에서 해당 컴퓨터를 정품 인증할 수 없다고 보고했습니다. KMS(키 관리 서비스)를 사용하고 있지 않습니다.    |    KMS 클라이언트    |    이 오류는 KMS 요청에 대한 응답이 없는 경우에 발생합니다.    |    KMS 호스트와 클라이언트 간의 네트워크 연결 문제를 해결하세요. TCP 포트 1688(기본값)이 방화벽으로 차단되거나 다른 방식으로 필터링되지 않았는지 확인하세요.    |
|    0xC004F041     |    소프트웨어 보호 서비스에서 KMS(키 관리 서비스)가 활성화되지 않은 것이 확인되었습니다. KMS를 활성화해야 합니다.    |    KMS 클라이언트    |    KMS 호스트가 정품 인증되지 않았습니다.    |    온라인 또는 전화 정품 인증을 통해 KMS 호스트를 정품 인증하세요.    |
|    0xC004F042     |    소프트웨어 보호 서비스에서 지정된 KMS(키 관리 서비스)를 사용할 수 없음이 확인되었습니다.    |    KMS 클라이언트    |    KMS 클라이언트와 KMS 호스트가 일치하지 않습니다.    |    이 오류는 KMS 클라이언트가 클라이언트 소프트웨어를 정품 인증할 수 없는 KMS 호스트에 연결한 경우에 발생합니다. 예를 들어 애플리케이션 및 운영 체제별 KMS 호스트를 포함하는 혼합된 환경에서 일반적으로 발생할 수 있습니다.    |

|    0xC004F051     |    소프트웨어 보호 서비스에서 제품 키가 차단되었다고 보고했습니다.    |    MAK/KMS    |    Microsoft에서 정품 인증 서버의 제품 키를 차단했습니다.    |    새 MAK/KMS 키를 받아서 시스템에 설치하고 정품 인증하세요.    |
