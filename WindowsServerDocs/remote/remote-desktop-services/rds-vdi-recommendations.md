# <a name="recommended-settings-for-vdi-desktops"></a>VDI 데스크톱 권장 설정

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows10

Microsoft Desktop Virtualization 장치 구성 및 네트워크 상태를 가져오고 사용자가 최신 실행 더 빨리 회사 응용 프로그램 및 데스크톱을 즉시 설치를 사용 하 여 자동으로 검색 및 것은 레거시에 대 한 액세스를 제공 하는 IT Windows 10으로 마이그레이션하는 동안 응용 프로그램입니다.

기회를 구체화 하는 Windows 10 운영 체제는 기본적으로 매우 잘 조정 되어, 있지만 회사의 Microsoft 가상 데스크톱 인프라 (VDI) 환경에 추가 합니다. VDI 환경에서 많은 백그라운드 서비스 및 작업부터에서 비활성화 됩니다.

이 항목에서는 청사진이 하지만 대신 가이드 또는 아닙니다 시작 지점입니다. 몇 가지 권장 시나리오에서 특정 설정을 조정의 혜택 및 비용을 고려해 야 하므로 사용 하려면 선호 하는 기능을 비활성화할 수 있습니다.

다음 지침과 권장 되는 설정이 Windows 10 1607 (10.0.1393 버전)에 적용 됩니다.

> [!NOTE]  
> 이 항목에서 특별히 언급 한 모든 설정을 유지할 수 있습니다 VDI 기능에 별다른 영향을 주지 않고 해당 기본값 (또는 요구 사항 및 정책 설정).

VDI 배포를 기반으로 이미지를 만들 때 사용 해야 합니다 **현재 분기**합니다. 현재 분기에 대 한 자세한 내용은 참조 하세요. [Windows 10 릴리스 정보](https://technet.microsoft.com/windows/release-info.aspx)합니다.

## <a name="microsoft-store-apps"></a>Microsoft Store 앱
VDI 이미지에서 제거할 수 있는 Microsoft Store 앱의 여러 가지 제거할 때 CPU 사용량 감소를 디스크 공간을 절약 합니다. 제거에 대 한 좋은 후보는 다음과 같습니다.

- Office 다운로드
- Skype (미리 보기)
- (특히 경우 인터넷 연결이 없는) 시작
- 피드백 허브
- Microsoft 카드놀이 컬렉션
- 유료 Wi-fi 및 셀룰러

VDI 이미지를 만드는 데 기본 사용자 프로필을 사용자 지정 하려면 기본 제공 관리자 계정을 사용 합니다. 이미 사용 하지 않는 경우 관리 컴퓨터에서에서 로컬 사용자 및 그룹을 사용 하 여이 수행 합니다. 다음 단계를 완료 하려면 관리자 계정에 로그인 합니다.

> [!NOTE]  
> 시스템 앱 등의 스토어 앱을 제거 하지 마세요. 다시 설치 하기 어렵습니다. 다른 앱은 스토어에서 쉽게 reinstallable입니다.

### <a name="delete-unwanted-apps-from-the-administrator-user-profile"></a>관리자 사용자 프로필에서 원치 않는 앱 삭제
1. Windows PowerShell에서 실행 `Get-AppxPackage | ft PackageFamilyName` 설치 된 앱 목록을 확인 합니다.
2. 제거 하려는 각 앱 패키지 작성 도구에 대 한 형식이 예의 cmdlet을 실행 합니다.

    `Get-AppxPackage *messaging* | Remove-AppxPackage`

    `Get-AppxPackage *WindowsMaps* | Remove-AppxPackage`

    `Get-AppxPackage *ZuneMusic* | Remove-AppxPackage`

### <a name="delete-the-payload-of-unwanted-store-apps"></a>원치 않는 스토어 앱의 페이로드를 삭제 합니다.
이렇게 하면 앱에서 다시 설치 하 고 하지 것입니다.
1. 이 cmdlet 사용 하 여 저장소의에서 데이터를 프로 비전 하는 다른 항목과 목록 스토어 앱: `Get-AppxProvisionedPackage -Online`합니다.
2. 사용 하 여 지정 된 패키지를 제거 `Remove-AppxProvisionedPackage -Online -PackageName MyAppPackage`, 적절 한 MyAppPackage를 사용 하 여 1 단계에서에서 반환 됩니다. 예를 들어 Zune 관련 패키지를 제거 하려면 실행 `Remove-AppxProvisionedPackage -Online -PackageName Microsoft.ZuneMusic_2019.17012.10311.0_neutral_~_8wekyb3d8bbwe`합니다.
