# 원격 데스크톱 클라이언트에 대 한 질문과 대답

>적용 대상: Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

원격 데스크톱 클라이언트에서 (Android, Mac, iOS 또는 Windows) 장치에서 설정한 했으므로 질문을 할 수 있습니다. 다음은 원격 데스크톱 클라이언트에 대 한 가장 자주 묻는 질문에 대답 합니다.

- [연결, 게이트웨이, 및 네트워크](#connection-gateway-and-networks)
- [웹 클라이언트](#web-client)
- [모니터, 오디오 및 마우스](#monitors-audio-and-mouse)
- [Mac 하드웨어](#mac-client---hardware-questions)
- [특정 오류 메시지](#specific-errors)

대부분의 이러한 질문 클라이언트의 모든 적용 되지만 항목이 몇 가지 클라이언트 특정 합니다.

응답을 원하는 추가 질문이 있는 경우 두면 피드백으로이 문서에서.

## 설정

### 에 연결할 수 있는 Pc

[지원 되는 구성](remote-desktop-supported-config.md) 문서에 연결할 수 있는 Pc에 대 한 정보를 확인 합니다.

### 원격 데스크톱 PC를 설정 어떻게 해야 하려면?

설정, 장치 살펴야 PC의 준비 생각 합니다. 도움말?

원격 데스크톱 설치 마법사 먼저 본 있습니까? 안내 원격 액세스를 위한 PC를 준비 합니다. 모든 PC에서 도구를 다운로드 및 실행 설정 합니다. 

그렇지 않으면 수동으로 수행 하려는 경우에서 읽습니다.

Windows 10의 경우 다음을 수행 합니다.

1. 연결 하려는 디바이스에서 **설정**을 엽니다.
2. **시스템** 및 **원격 데스크톱**을 선택 합니다.
3. 슬라이더를 사용 하 여 원격 데스크톱을 사용 하도록 설정 합니다.
4. 일반적으로 항시 대기 하 고 연결을 검색 PC를 유지 하는 것이 좋습니다. 이 설정을 변경할 수 있는 PC에 대 한 전원 설정으로 이동 하 게 **설정 표시** 를 클릭 합니다.
   > [!NOTE]
   > 절전 모드로 전환 되는 PC에 연결 하거나 절전 및 최대 절전 모드에서 원격 PC에 대 한 설정 **사용 안 함**으로 설정 되어 있는지 확인 하므로, 절전 수 없습니다. (최대 절전 모드를 모든 Pc에서 사용할 수 없습니다.)


**이 PC에 연결 하는 방법**에서이 PC의 이름을 기록해 둡니다. 클라이언트를 구성 하려면이 호출 해야 합니다.

그렇게, 클릭 **이 PC에 원격으로 액세스할 수 있는 사용자를 선택**하려면이 PC에 액세스할 수 있는 특정 사용자에 게 권한을 부여할 수 있습니다.
관리자 그룹의 구성원에는 자동으로 액세스할을 수 있습니다.

Windows 8.1 대 한 [원격 데스크톱 연결을 사용 하 여 다른 데스크톱에 연결](https://support.microsoft.com/en-us/help/17463/windows-7-connect-to-another-computer-remote-desktop-connection#1TC=windows-8)에서 원격 연결을 허용 하도록 지시를 따릅니다.



## 연결, 게이트웨이, 및 네트워크

### 원격 데스크톱을 사용 하 여 연결할 수 없는 이유는?

다음은 원격 PC에 연결 하려고 할 때 발생할 수 있는 일반적인 문제에 몇 가지 가능한 해결 방법입니다. 이러한 솔루션 작동 하지 않는 경우에 [Microsoft 커뮤니티 웹 사이트](https://go.microsoft.com/fwlink/p/?LinkId=242079)에서 더 많은 도움말을 찾을 수 있습니다.

- **원격 PC는 찾을 수 없습니다.** 올바른 PC 이름을 있고 다음 해당 이름을 올바르게 입력 했는지 확인 해야 합니다. 그래도 여전히 연결할 수 없으면 원격 PC의 IP 주소를 사용 하 여 PC 이름 대신 합니다.
- **네트워크를 사용 하 여 문제가 있습니다.** 인터넷에 연결 되어 있는지 확인 합니다. 
- **원격 데스크톱 포트는 방화벽에 의해 차단 될 수 있습니다.** Windows 방화벽을 사용 중인 경우 다음이 단계를 따르세요.

   1. Windows 방화벽을 엽니다. 
   2. **Windows 방화벽을 통해 또는 기능 허용 앱을**클릭 합니다. 
   3. **설정 변경을**클릭 합니다. 한 관리자 암호 또는 선택을 확인 하 라는 메시지가 표시 될 수 있습니다.
   4. **허용 되는 앱 및 기능을** **원격 데스크톱**, 선택 하 고 탭 하거나 **확인**을 클릭 합니다.

   다른 방화벽을 사용 중인 경우 원격 데스크톱 (일반적으로 3389)에 대 한 포트 열려 있는지 확인 합니다.
- **원격 PC에 대 한 원격 연결 설정할 수 있습니다.** 이 문제를 해결 하려면 백업할 스크롤하여 [원격 데스크톱 설정 PC 설치는 어떻게?](#how-do-i-set-up-a-pc-for-remote-desktop) 이 항목에서 질문 합니다.
- **원격 PC 네트워크 수준 인증을 설정 하는 Pc에 연결 하도록 허용할 수 있습니다.** 
- **원격 PC 꺼져 있을 수 있습니다.** 꺼져, PC에 연결 하거나 최대 절전 모드에 있는지 확인 설정을 절전 및 최대 절전 원격 PC에는 사용 **안** 함으로 설정에 대 한 수 없습니다 (최대 절전 모드를 모든 Pc에서 사용할 수 없습니다.).

### 찾을 또는 내 PC에 연결할 수 없는 이유 있나요?
다음 사항을 확인 합니다.
- PC 및 항시 대기 인가요?
- 올바른 이름 또는 IP 주소를 입력 했습니까?

   > [!IMPORTANT]
   > PC 이름을 사용 하 여 DNS 통해 올바르게 이름을 확인 하기 위해 네트워크에 필요 합니다. 대부분의 홈 네트워크에 연결 하는 대신 호스트 이름을 IP 주소를 사용 해야 합니다.
- PC가 다른 네트워크? 연결을 통해 외부 수 있도록 PC를 구성 했습니까?  도움말에 대 한 [허용 네트워크 외부 로부터 PC에 대 한 액세스](remote-desktop-allow-outside-access.md) 를 확인 합니다.
- 지원 되는 Windows 버전에 연결 되어 있습니까? 

   > [!NOTE]
   > Windows XP Home, Windows Media Center 버전, Windows Vista 홈 및 Windows 7 Home 또는 Starter 타사 소프트웨어 없이 지원 되지 않습니다.

### 원격 PC에 로그인 할 수 없습니다는 이유
원격 PC의 로그인 화면에 표시 하는 경우 로그인 할 수 없는 있습니다 하지 추가 되었을 원격 데스크톱 사용자 그룹으로 또는 원격 PC에 대 한 관리자 권한 가진 그룹. 이를 시스템 관리자에 게 문의 하십시오.

### 회사 네트워크에 대 한 연결 메서드에 지원 되나요?
회사 네트워크 외부에서 office 데스크톱에 액세스 하려는 경우 회사는 수단 원격 액세스를 제공 해야 합니다. RD 클라이언트는 현재 다음을 지원합니다.

- 터미널 서버 게이트웨이 또는 원격 데스크톱 게이트웨이
- 원격 데스크톱 웹 액세스
- (IOS 기본 제공 VPN 옵션)를 통해 VPN

### VPN 작동 하지 않는 경우

VPN 문제는 여러 원인이 있을 수 있습니다. 첫 번째 단계는 PC 또는 Mac 컴퓨터와 동일한 네트워크에서 VPN 작동 하는지 확인 합니다. PC 또는 Mac을 사용 하 여 테스트할 수는 없습니다, 장치의 브라우저를 사용 하 여 회사 인트라넷 웹 페이지에 액세스 하려고 하면 수 있습니다.

확인 하려면 다른 작업:
- **3g 네트워크 VPN 손상 또는 차단 합니다.** 전 세계의 차단 또는 손상 된 3g 트래픽을 하는 몇 가지 3g 공급자 있습니다. VPN 연결 분에 대 한 제대로 작동을 확인 합니다.
- **L2TP 또는 PPTP Vpn 합니다.** PPTP 또는 L2TP VPN을 사용 하는 경우 설정 하십시오 모든 트래픽을 보낼 on VPN 구성.
- **VPN 잘못 구성 되었습니다.** 잘못 구성 된 VPN 서버 이유는 VPN 연결 되거나 되지 않는 작업 떠나면 잠시 후 작동을 중지 원인이 될 수 있습니다. 이 경우 동일한 네트워크에서 iOS 디바이스의 웹 브라우저 또는 PC 또는 Mac을 사용 하 여 테스트를 확인 합니다.

### VPN이 제대로 작동 하는 경우를 테스트할 수는 방법
VPN 장치에서 활성화 되어 있는지 확인 합니다. 내부 네트워크에서 웹 페이지를 이동 하거나 에서만 VPN을 통해 사용할 수 있는 웹 서비스를 사용 하 여 VPN 연결을 테스트할 수 있습니다.

### L2TP 또는 PPTP VPN 연결을 구성 하려면 어떻게 해야 합니까?
PPTP 또는 L2TP VPN을 사용 하는 경우에 VPN 구성에서 **ON** 으로 **모든 트래픽을 보낼** 을 설정 해야 합니다.

## 웹 클라이언트

### 브라우저를 사용할 수 있나요?

웹 클라이언트에서 Microsoft Edge, Internet Explorer 11 Mozilla Firefox 지원 (v55.0 이상), 사파리, 및 Google Chrome 합니다.

### 웹 클라이언트에 액세스 하는 Pc 사용할 수 있나요?

웹 클라이언트는 Windows, macOS, Linux 및 ChromeOS 지원합니다. 이 이번에는 모바일 장치 지원 되지 않습니다.

### 게이트웨이 없이 원격 데스크톱 배포에서 웹 클라이언트를 사용할 수 있나요?

아니요. 클라이언트에 연결 하는 원격 데스크톱 게이트웨이 필요 합니다. 모르는 즉? 귀하의 관리자에 대 한 요청 합니다.

### 원격 데스크톱 웹 클라이언트 원격 데스크톱 웹 액세스 페이지를 대체 됩니까?

아니요. 원격 데스크톱 웹 클라이언트 원격 데스크톱 웹 액세스 페이지 보다 다른 URL에 호스트 됩니다. 브라우저에서 원격 리소스를 보려면 웹 클라이언트 또는 웹 액세스 페이지를 사용할 수 있습니다.

### 다른 웹 페이지에서 웹 클라이언트를 포함할 수 있습니까?

이 기능은 현재 지원 되지 않습니다.

## 모니터, 오디오 및 마우스

### 모니터의 모든 사용 하는 방법
두 개 이상의 화면을 사용 하려면 다음을 수행 합니다.

1. 여러 화면을 사용 하도록 설정 하려는 원격 데스크톱을 마우스 오른쪽 단추로 클릭 하 고 **편집**을 클릭 합니다.
2. **모든 모니터를 사용 하 여** 및 **전체 화면**을 사용 하도록 설정 합니다.

### 양방향 소리 지원 됩니까?
소리 업스트림 (마이크에 대 한 서버 클라이언트)에서 원격 데스크톱 클라이언트에 의해 지원 되지 않습니다.

### 소리 재생 되지 않는 경우 어떻게 해야 합니까?
세션에서 로그 아웃 (하지 방금 분리, 로그 아웃 놓고) 했다가 다시 로그인 합니다.

## Mac 클라이언트-하드웨어 질문
### 망막 해상도 지원 됩니까?
예, 원격 데스크톱 클라이언트 망막 해상도 지원합니다.

### 보조 마우스 오른쪽 단추로 클릭을 설정 하려면 어떻게 하나요?
확인 하기 위해 세 가지 옵션이 열려 세션 내에서 마우스 오른쪽 단추로 사용:

- 표준 PC 두 단추 USB 마우스
- Apple 매직 마우스: 마우스를 사용 하려면 클릭 **시스템 기본 설정** , 도킹 스테이션에서 **마우스**를 클릭 하 고 **보조 클릭**한 다음 사용 합니다.
- Apple 매직 트랙 패드 또는 트랙 패드 MacBook: 마우스를 사용 하도록 설정, 도킹 스테이션에서 **시스템 기본 설정** 클릭 하 고, **마우스**를 클릭 하 고 다음 **보조 클릭**을 사용 하도록 설정 합니다.

### AirPrint 지원 됩니까?
아니요, 원격 데스크톱 클라이언트 AirPrint 지원 하지 않습니다. (이 모두 Mac 및 iOS 클라이언트에 대해 true 임)

### 잘못 된 문자가 세션에 표시지 않습니다.
국제에서 키보드를 사용 하는 경우 표시 될 수 있습니다 문제 세션에 표시 되는 문자 문자 일치 작업을 수행 하는 Mac 키보드에서 입력 합니다.

이 다음과 같은 시나리오에서 발생할 수 있습니다.

- 원격 세션에서 인식 하지 못하는 키보드를 사용 하는 합니다. 원격 데스크톱 키보드를 인식할 수 없는 경우 기본적으로 마지막 원격 PC와 함께 사용 되는 언어.
- 원격 PC에서 이전에 연결이 끊긴된 세션에 연결 하는 원격 PC의 언어와 다른 키보드 언어를 사용 및 사용 하려는 현재 합니다.

원격 세션에 대 한 키보드 언어를 수동으로 설정 하 여이 문제를 해결할 수 있습니다. 다음 섹션의 단계를 참조 하세요.

### 언어 설정을 미치는 영향은 무엇입니까 키보드 원격 세션에서?
다양 한 유형의 Mac 자판 있습니다. 이들 중 일부는 Mac 특정 레이아웃 또는 사용자 지정 레이아웃을 정확 하 게 일치 하지 못할에 원격 windows 버전에서. 원격 세션 최선을 다 해 원격 PC에 사용할 수 있는 키보드 언어와 일치 하는 키보드를 매핑합니다. 

Mac 키보드 레이아웃으로 설정 된 경우 PC 버전의 모든 키 올바르게 매핑되어야 언어 키보드 (예: 프랑스어-PC) 및 키보드 작동 해야 합니다.

Mac 키보드 레이아웃에는 키보드 (예를 들어 프랑스어)의 Mac 버전으로 설정 된 경우 원격 세션 프랑스어의 PC 버전에 하면 매핑됩니다. Mac 바로 가기 키 OSX를 사용 하 여 사용 했던 일부 원격 Windows 세션에서 작동 하지 않습니다.

키보드 레이아웃에는 언어 (예를 들어 캐나다 프랑스어)의 변형으로 설정 된 경우 및 원격 세션 수를 정확 하 게 변형에 매핑할 수 없는 경우 원격 세션에 가장 가까운 언어 (예를 들어 프랑스어) 하면 매핑됩니다. Mac 바로 가기 키 OSX를 사용 하 여 사용 했던 일부 원격 Windows 세션에서 작동 하지 않습니다.

키보드 레이아웃에는 원격 세션 전혀 일치 하지는 레이아웃으로 설정 되 면 기본적으로 마지막으로 해당 PC와 함께 사용 되는 언어를 사용 하면 원격 세션 됩니다. 이 경우 또는 사용자 Mac 키보드에 맞게 원격 세션의 언어를 변경 해야 하는 경우에서 다음과 같이 사용 하려는 것과 거의 비슷한 경우 되는 언어를 원격 세션에서 키보드 언어를 수동으로 설정할 수 있습니다.

원격 데스크톱 세션 내에서 키보드 레이아웃을 변경 하려면 다음 지침을 사용 합니다.

**Windows 10 또는 Windows 8:**

1. 원격 세션 내에서 지역 및 언어를 엽니다. **시작 gt_ 설정 gt_ 시간 및 언어를**클릭 합니다. **지역 및 언어를**엽니다.
2. 사용할 언어를 추가 합니다. 다음 국가 및 언어 창을 닫습니다.
3. 이제 원격 세션에서 언어 간 전환 하는 기능이 표시 됩니다. (오른쪽에서 원격 세션을 시계 근처의.) (예: **Eng**)로 전환 하는 데 사용할 언어를 클릭 합니다.

닫고 적용 되려면 바로 변경 내용에 대 한 현재 사용 중인 응용 프로그램을 다시 시작 해야 할 수 있습니다.


## 특정 오류

### "권한이 없음" 오류가 발생 합니까?
연결 하려는 세션에 액세스할 수 없습니다. 가장 일반적인 원인 관리자 세션에 연결 하려는입니다. 관리자만 콘솔에 연결할 수 있습니다. 콘솔 스위치는 원격 데스크톱의 고급 설정에서 꺼져 확인 합니다. 문제의 원인이 없는 경우 시스템 관리자에 게 문의 문의 하세요.

### 이유는 클라이언트 말 없는 CAL는?
원격 데스크톱 클라이언트에서 원격 데스크톱 서버에 연결할 때 서버는 원격 데스크톱 서비스 클라이언트 액세스 라이선스 (RDS CAL) 클라이언트에 의해 저장 된 발급 합니다. 클라이언트 연결 될 때마다 다시 사용 하 여 해당 RDS CAL 및 서버 라이선스를 발급 하지 않습니다. 서버는 장치에 RDS CAL 없거나 손상 된 경우 다른 라이선스를 발급 합니다. 사용이 허가 된 디바이스의 최대 수에 도달 하면 서버 새 RDS Cal 발급 하지 않습니다. 네트워크 관리자에 게 문의 합니다.

### "액세스 거부" 오류는 왜 얻을 했습니까?
"액세스 거부" 오류는 연결 시도 하는 동안 원격 데스크톱 게이트웨이 잘못 된 자격 증명의 결과를 생성 합니다. 사용자 이름 및 암호를 확인 합니다. 연결 하기 전에 작업 최근에 오류가 발생 한 경우 되었을 수 Windows 사용자 계정 암호를 변경 하 고 업데이트 하지 않은 것 아직 원격 데스크톱 설정에서 합니다.

### "RPC 오류 23014" 또는 "오류 0x59e6" 평균 합니까?
**RPC 오류 23014** 또는 **몇 분 정도 기다린 후 다시 시도 0x59E6 오류를**발생 시 RD 게이트웨이 서버 활성 연결의 최대 수에 도달 했습니다. 연결의 최대 수 RD 게이트웨이에서 실행 중인 Windows 버전에 따라 다른: The Windows Server 2008 R2 Standard 구현 연결 250의 수를 제한 합니다. Windows Server 2008 R2 Foundation 구현 50에 대 한 연결 수가 제한 됩니다. 다른 모든 Windows 구현 개수에 연결을 허용합니다.

### "를 NTLM 챌린지를 구문 분석 하지 못했습니다." 오류는 무엇을 의미 합니까?
이 오류는 원격 PC에는 잘못 된 구성은 인해 발생 합니다. 반드시는 RDP 보안 수준 원격 PC에 설정 된 "클라이언트 호환 가능 합니다." (통신할 시스템 관리자가 작업을 수행 하는 데 도움이 필요 하면.)

### "TS_RAP 수 없습니다 지정된 된 호스트에 연결" 하는 것이 무엇을 의미?
이 오류는 게이트웨이 서버에서 리소스 권한 부여 정책 사용자 이름을 원격 PC에 연결 하지 못하도록 중지 될 때 발생 합니다. 이 다음과 같은 경우에 발생할 수 있습니다.

- 원격 PC 이름은 게이트웨이의 이름과 동일 합니다. 그런 다음 원격 PC에 연결 하려고 할 때 연결 이동 게이트웨이를 대신 하는 것에 액세스할 수 있는 권한이 없는 합니다. 게이트웨이에 연결 해야 하는 경우 외부 게이트웨이 이름 PC 이름으로 사용 하지 마십시오. 대신 "localhost" 또는 IP 주소 (127.0.0.1) 또는 내부 서버 이름을 사용 합니다.
- 사용자 계정에 대 한 원격 액세스에 대 한 사용자 그룹의 구성원이 아닙니다.
