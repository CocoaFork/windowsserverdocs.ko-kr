# IOS에서 원격 데스크톱을 사용 하 여 시작

>적용 대상: Windows 10, Windows 8.1, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2

Windows 앱, 리소스 및 데스크톱 (Iphone 및 iPads) iOS 장치에서 작동 하도록 iOS에 대 한 원격 데스크톱 클라이언트를 사용할 수 있습니다.

시작 하려면 다음 정보를 사용 합니다. 질문이 있는 경우 [FAQ](remote-desktop-client-faq.md) 를 확인 해야 합니다.

> [!NOTE]
> - IOS 클라이언트에 대 한 새 릴리스 용어나? 체크 아웃 [iOS에서 원격 데스크톱에 대 한 새로운?](ios-whatsnew.md)
> - IOS 클라이언트 iOS를 실행 하는 장치를 지 원하는 6.x 및 최신 버전입니다.

## 원격 데스크톱 클라이언트를 가져오고 사용을 시작합니다

### 원격 데스크톱 클라이언트 iOS 스토어에서 다운로드
IOS 디바이스에서 원격 데스크톱을 사용 하 여 시작 하려면 다음이 단계를 따르세요.

1. [ITunes](https://itunes.apple.com/us/app/microsoft-remote-desktop/id714464092?mt=8)에서 Microsoft 원격 데스크톱 클라이언트를 다운로드 합니다.
2. [원격 연결을 허용 하도록 PC를 설정](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop)합니다.
3. [원격 데스크톱 연결](#add-a-remote-desktop-connection) 또는 [원격 리소스](#add-a-remote-resource)를 추가 합니다. 연결을 사용 하 여에 연결 하는 직접 RemoteApp 프로그램을 사용 하 여 Windows PC 및 원격 리소스, 세션 기반 데스크톱 또는 가상 데스크톱 게시 온-프레미스 RemoteApp 및 데스크톱 연결을 사용 하 여 합니다. 이 기능은 일반적으로 기업 환경에서 사용할 수 있습니다.

### 원격 데스크톱 iOS 베타 클라이언트 다운로드
IOS 장치에서 원격 데스크톱 iOS 베타 클라이언트를 다운로드 하려면 [다음이 지침](https://aka.ms/rdiosbeta) 따릅니다.

### 원격 데스크톱 연결을 추가 합니다.

원격 데스크톱 연결을 만들려면: 
1. 연결 센터 탭에서 **+** 를 누른 다음 **추가 PC 또는 서버**.
2. 원격 데스크톱 연결에 대 한 다음 정보를 입력 합니다.
  - **PC 이름** -컴퓨터의 이름입니다. 이 Windows 컴퓨터 이름, 인터넷 도메인 이름으로 또는 IP 주소일 수 있습니다. 또한 PC 이름 (예를 들어 **MyDesktop:3389** 또는 **10.0.0.1:3389**) 포트 정보를 추가할 수 있습니다.
  - **사용자 이름** -원격 PC에 액세스 하는 데 사용자 이름입니다. 다음 형식을 사용할 수 있습니다: *사용자 _*, *domain\user_name*또는 *user_name@domain.com*합니다. 사용자 이름 및 암호를 묻는 메시지를 표시 여부를 지정할 수 있습니다.
3. 또한 다음과 같은 추가 옵션을 설정할 수 있습니다.
  - **이름 (선택 사항)** –에 연결 하는 PC에 대 한 기억 하기 쉬운 이름을입니다. 모든 문자열을 사용할 수 있지만 이름을 지정 하지 않으면 PC 이름이 표시 됩니다.
  - **게이트웨이 (선택 사항)** – 가상 데스크톱, RemoteApp 프로그램 및 세션 기반 데스크톱에서 회사 내부 네트워크에 연결 하는 데 사용할 수 있는 원격 데스크톱 게이트웨이. 시스템 관리자에서 게이트웨이 대 한 정보를 가져옵니다.
  - **소리** – 원격 세션 동안 오디오를 사용 하 여 장치를 선택 합니다. 원격 장치를 로컬 장치에서 하거나 전혀 없는 소리를 재생 선택할 수 있습니다.
  - **마우스 단추 스왑** – 마우스 제스처를 마우스 왼쪽된 단추를 사용 하 여 명령을 전송 하는 경우, 동일한 명령을 보냅니다 마우스 오른쪽 단추를 사용 하 여 대신 합니다. 이 왼손 마우스 모드에 대 한 원격 PC 구성 된 경우 필요 합니다.
  - **관리자 모드** -Windows Server 2003 이상을 실행 하는 서버에서 관리 세션에 연결 합니다.
4. **저장**을 누릅니다.

이러한 설정을 편집할 필요가? 편집 하려는 데스크톱 누른 상태 다음 설정 아이콘을 탭 합니다. 

### 원격 리소스를 추가 합니다.
원격 리소스는 RemoteApp 프로그램, 세션 기반의 데스크톱 및 가상 데스크톱 RemoteApp 및 데스크톱 연결을 사용 하 여 게시 합니다.

- URL은 RemoteApp 및 데스크톱 연결에 액세스할 수 있는 RD 웹 액세스 서버에 있는 링크를 표시 합니다.
- 구성 된 RemoteApp 및 데스크톱 연결은 나열 됩니다.

에 원격 리소스를 추가 합니다.

1. 탭 연결 센터 화면에서 **+** 를 누른 다음 **원격 리소스를 추가**합니다. 
2. 원격 리소스에 대 한 정보를 입력 합니다.
   - **피드 URL** -RD 웹 액세스 서버의 URL입니다. 전자 메일 주소와 관련 된 RD 웹 액세스 서버에 대 한 검색 하려면 클라이언트를 인지를 –이 필드에 회사 메일 계정에 입력할 수 있습니다.
   - **사용자 이름** -에 연결 하는 RD 웹 액세스 서버에 사용할 사용자 이름입니다.
   - **암호** -RD 웹 액세스 서버에 연결 하는 데 암호입니다.
3. **저장**을 누릅니다.

원격 리소스 연결 가운데에 표시 됩니다.


## 내부 자산에 액세스할 수 있는 RD 게이트웨이에 연결

원격 데스크톱 게이트웨이 (RD 게이트웨이)를 사용 하면 어디에서 나 회사 네트워크에서 원격 컴퓨터에 연결에서 인터넷 합니다. 만들 수 있으며 원격 데스크톱 클라이언트를 사용 하 여 게이트웨이 관리.

새로운 게이트웨이를 설정 하려면:

1. 연결 중앙에서 **설정을 gt_ 게이트웨이**누릅니다. 
2. **추가 원격 데스크톱 게이트웨이**탭 합니다.
3. 다음 정보를 입력 합니다.
  - **서버 이름** -게이트웨이 사용 하려는 컴퓨터의 이름입니다. 이 Windows 컴퓨터 이름, 인터넷 도메인 이름으로 또는 IP 주소일 수 있습니다. 서버 이름으로 포트 정보를 추가할 수도 있습니다 (예: **RDGateway:443** 또는 **10.0.0.1:443**).
  - **사용자 이름** -사용자 이름 및 암호를 연결 하는 원격 데스크톱 게이트웨이 위한 사용 해야 합니다. 원격 데스크톱 연결에 사용 되는 것과 동일한 사용자 이름 및 암호를 사용 하 여 **연결 자격 증명 사용** 을 선택할 수 있습니다.


## 사용자 계정 관리 

데스크톱 또는 원격 리소스에 연결할 때 다시에서 선택 하도록 사용자 계정을 저장할 수 있습니다. 원격 데스크톱 클라이언트를 사용 하 여 사용자 계정을 관리할 수 있습니다.

새 사용자 계정을 만들려면:

1. 연결 센터에서 **설정**, 누르고 **사용자 이름**입니다.
2. **사용자 계정을 추가**누릅니다.
3. 다음 정보를 입력 합니다.
   - **사용자 이름** -원격 연결에 사용 하기 위해 저장 하는 사용자의 이름입니다. 다음 형식 중 하나로 사용자 이름을 입력할 수 있습니다: 사용자 _, domain\user_name, 또는 user_name@domain.com 합니다.
   - **암호** -지정한 사용자에 대 한 암호입니다. 원격 연결에 사용 하도록 저장할 하려는 모든 사용자 계정 암호 연결 되어 있어야 합니다.
4. **저장**을 누르고 **설정**합니다.
5. 새 구성을 저장 하려면 **완료** 를 탭 합니다.

사용자 계정을 삭제 합니다.

1. 연결 중앙에서 **설정을 gt_ 사용자 이름을**누릅니다.
2. 살짝 밀기 행 오른쪽에서 왼쪽으로 사용자를 선택 합니다.
3. **삭제**를 누릅니다.



## 원격 데스크톱 세션 이동
원격 데스크톱 세션을 시작할 때 사용할 수 있는 세션 이동 하는 데 사용할 수 있는 도구가 있습니다.

### 원격 데스크톱 연결을 시작 합니다.

1. 원격 데스크톱 세션을 시작 하려면 원격 데스크톱 연결을 누릅니다. 
2. 원격 데스크톱에 대 한 인증서를 확인 하는 메시지가 **동의**누릅니다. 항상 **표시 안 함이 컴퓨터에 연결에 대 한 다시** 토글 **ON**으로 이동 하 여 수락 하도록 선택할 수 있습니다. 

### 연결 표시줄

연결 표시줄은 추가 탐색 컨트롤에 대 한 액세스 제공 합니다. 

- **이동 컨트롤**: 팬 컨트롤을 사용 하면 화면을 확대 하 고 이동 합니다. 참고 이동 컨트롤은 직접 터치를 사용 하 여 사용할 수만 있습니다.
   - 활성화 / 이동 컨트롤을 사용 하지 않도록 설정: 화면을 확대/축소 및 이동 컨트롤을 표시 하려면 연결 표시줄에 이동 아이콘을 탭 합니다. 연결 표시줄 숨기기를 기존 픽셀 화면 돌아갑니다에 팬 아이콘을 탭 합니다.
   - 이동 컨트롤을 사용 하 여: 팬 누르고 제어 하 고 끌어서 화면을 이동 방향으로 합니다.
   - 이동 컨트롤을 이동: 화면에 컨트롤을 이동 컨트롤을 길게 두 번 탭 합니다.
- **연결 이름**: 현재 연결 이름이 표시 됩니다. 세션 선택 모음을 표시 하려면 연결 이름을 누릅니다.
- **키보드**: 키보드 아이콘을 표시 하거나 숨기려면 키보드를 탭 합니다. 이동 컨트롤이 키보드 표시 될 때 자동으로 표시 됩니다.
- **연결 표시줄 이동**: 탭 및 보류 연결 표시줄 및 다음 끌어서 놓기를 새 위치로 화면 맨 위에 있는 합니다.

### 세션 선택
이와 동시에 여러 Pc에 여러 연결을 사용할 수 있습니다. 화면의 왼쪽에 있는 세션 선택 모음을 표시 하려면 연결 막대를 탭 합니다. 세션 선택 모음에 열려 있는 연결을 살펴보고 간에 전환할 수 있습니다. 

- 열린 원격 리소스 세션에서 앱 간에 전환 합니다.

    원격 리소스에 연결할 때 확장 메뉴를 탭 하 고 사용 가능한 항목의 목록에서 선택 하 여 해당 세션 내에서 열려 있는 응용 프로그램 간에 전환할 수 있습니다.
- 새 세션을 시작

  현재 연결 내에서 새 응용 프로그램 또는에서 데스크톱 세션을 시작할 수 있습니다: **새 시작**, 탭 하 고 사용 가능한 항목의 목록에서 선택 합니다.

- 세션 연결 끊기

  연결을 끊으려면 세션 타일의 왼쪽에 X 세션 탭 합니다.

### 명령 모음

명령 모음 8.0.1 버전 부터는 바 유틸리티를 대체 합니다. 마우스 모드 간에 전환 하 고 명령 모음에서 연결 중심으로 돌아갈 수 있습니다.

| 직접 터치  | 마우스 오른쪽 단추로 클릭          | 1 손가락으로 탭 하기 및 누르고 있기                                      |

| 마우스 포인터 | 오른쪽 클릭 후 끌기 | 2 손가락으로 두 번 탭 하기 및 고정, 다음 끕니다.                    |

## 원격 세션에서 키보드를 사용 합니다.

하나를 사용할 수 있는 화상 키보드 또는 원격 세션에서 실제 키보드 합니다.

화상 키보드를 사용 하 여 단추 키보드 위쪽 표시줄의 오른쪽 가장자리에을 표준 및 추가 키보드 간에 전환 합니다.

IOS 디바이스에서 Bluetooth를 사용 하는 경우 클라이언트에 Bluetooth 키보드를 자동으로 검색 합니다.

OS에 제한으로 인해 Ctrl, 옵션 및 함수와 같은 특수 키가 예상 대로 작동 하지 Bluetooth 키보드를 사용 하 여 명심 하세요. 다음 키가 작동 합니다.

- 영숫자 키
- 커서 키
- 탭: 탭 작동 하지만 Shift + Tab 작동 하지 않는
- Home / Pos1: Alt + 왼쪽 홈 =
- Alt + 오른쪽 종료: 종료 =
- Page Up: Alt + 위쪽 Page Up =
- Page Down: Alt + 아래쪽 Page Down =
- 모두 선택: 명령 + A = Ctrl + A (대부분의 프로그램에서 모두 선택)
- 잘라내기: 명령 + X = Ctrl + X (대부분의 프로그램에 컷)
- 복사: 명령 + C = Ctrl + C (대부분의 프로그램에서 복사)
- 붙여넣기: 명령 + V = Ctrl + V (대부분의 프로그램에 붙여넣기)
- 기호: Alt + 영숫자 키 생성 언어 구성에 따라 서로 다른 기호
