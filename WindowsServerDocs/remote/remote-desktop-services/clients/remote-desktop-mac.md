# Mac에서 원격 데스크톱을 사용 하 여 시작

>적용 대상: Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

Mac 컴퓨터에서 Windows 앱, 리소스 및 데스크톱 작업할 Mac에 대 한 원격 데스크톱 클라이언트를 사용할 수 있습니다. 다음 정보를 사용 하 여 시작 및 질문이 있는 경우 [FAQ](remote-desktop-client-faq.md) 를 확인 합니다.

>[!Note]
> - MacOS 클라이언트에 대 한 새 릴리스 용어나? 체크 아웃 [Mac에서 원격 데스크톱에 대 한 새로운 기능?](mac-whatsnew.md)
> - Mac 클라이언트 10.10 이상 macOS를 실행 하는 컴퓨터에서 실행 됩니다.
> - 이 문서의 정보 주로 전체 버전의 Mac 클라이언트-Mac AppStore에서 사용할 수 있는 버전에 적용 됩니다. 여기에 미리 보기 앱을 다운로드 하 여 새 기능 테스트: [베타 클라이언트 릴리스 정보](https://go.microsoft.com/fwlink/?LinkID=619698&clcid=0x409).

## 원격 데스크톱 클라이언트
Mac에서 원격 데스크톱을 사용 하 여 시작 하려면 다음이 단계를 따르세요.

1. Microsoft 원격 데스크톱 클라이언트 [Mac 앱 스토어](https://itunes.apple.com/us/app/microsoft-remote-desktop/id1295203466?mt=12)에서 다운로드 합니다.
2. [원격 연결을 허용 하도록 PC를 설정](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop)합니다. (이 단계를 건너뛰면 연결할 수 없습니다 PC에.)
3. 원격 데스크톱 연결 또는 원격 리소스를 추가 합니다. 연결을 사용 하 여 RemoteApp 프로그램, 세션 기반 데스크톱을 사용 하 여 Windows PC 및 원격 리소스에 직접 연결할지 또는 가상 데스크톱 게시 온-프레미스 RemoteApp 및 데스크톱 연결을 사용 합니다. 이 기능은 일반적으로 기업 환경에서 사용할 수 있습니다.

## Mac 베타 클라이언트 어떤가요?
에서는 새로운 기능 HockeyApp에는 미리 보기 채널에서 테스트할 수 있습니다. 체크 아웃 하려면? [Microsoft 원격](https://go.microsoft.com/fwlink/?LinkID=619698) 데스크톱 Mac에 이동한 다음 **다운로드**를 클릭 합니다. HockeyApp 베타 클라이언트 다운로드에 계정 또는 사이 니 지를 만들 필요가 없습니다.

클라이언트를 이미 있는 경우 최신 버전 보장 하기 위해 업데이트를 확인할 수 있습니다. 베타 클라이언트에서 맨 위에 있는 **Microsoft 원격 데스크톱 베타** 를 클릭 하 고 **업데이트 확인**을 클릭 합니다. 

## 원격 데스크톱 연결을 추가 합니다.
원격 데스크톱 연결을 만들려면:

1. 연결 센터에서 **+**, 다음 **바탕 화면**을 클릭 합니다.
2. 다음 정보를 입력 합니다.
   - **PC 이름** -컴퓨터의 이름입니다.
      - 이 ( **시스템** 설정에서 찾을 수)는 Windows 컴퓨터 이름, 도메인 이름 또는 IP 주소일 수 있습니다.
      - 이 이름은 *MyDesktop:3389*같은 끝에 포트 정보를 추가할 수도 있습니다.
   - **사용자 계정** 원격 PC를 액세스 하는 데 사용 하 여 사용자 계정을 추가 합니다.
      - Active Directory (AD) 컴퓨터 또는 로컬 계정에 연결을 사용 하 여 다음이 형식 중 하나: *사용자 _*, *domain\user_name*또는 *user_name@domain.com*합니다.
      - Azure Active Directory (AAD) 가입 컴퓨터를 사용 하 여 다음이 형식 중 하나: *AzureAD\user_name* 또는 *AzureAD\ user_name@domain.com*합니다.
      - 암호 필요 여부를 선택할 수 있습니다.
      - 동일한 사용자 이름 사용 하 여 여러 사용자 계정을 관리 하는 경우 계정을 식별 하도록 이름을 설정 합니다.
      - 앱의 기본 설정에 저장 된 사용자 계정 관리 합니다. 

3. 또한 연결에 대 한 이러한 선택적 설정을 설정할 수 있습니다.
   - 이름을 설정 합니다. 
   - 게이트웨이 추가 합니다.
   - 사운드 출력 설정
   - 마우스 단추 바꾸기
   - 관리자 모드를 사용 하도록 설정
   - 원격 세션에 로컬 폴더 리디렉션
   - 앞으로 로컬 프린터
   - 앞으로 스마트 카드
4. **저장**을 클릭합니다.

연결을 시작 하려면 두 번 것입니다. 원격 리소스에 대 한도 마찬가지입니다.

### 연결 가져오기 및 내보내기
원격 데스크톱 연결 정의 내보내고 다른 장치에서 사용할 수 있습니다. 원격 데스크톱은 별도 저장 됩니다. RDP 파일입니다.

1. 연결 센터에서 원격 데스크톱을 마우스 오른쪽 단추로 클릭 합니다.
2. **내보내기**를 클릭 합니다.
3. 원격 데스크톱을 저장 하려는 위치로 이동 합니다. RDP 파일입니다.
4. **확인**을 클릭합니다.

다음 단계를 사용 하 여 원격 데스크톱을 가져옵니다. RDP 파일입니다.

1. 메뉴 모음에서 **파일 > 가져오기**합니다.
2. 이동 합니다. RDP 파일입니다.
3. **열기**를 클릭합니다.

## 원격 리소스를 추가 합니다.
원격 리소스는 RemoteApp 프로그램, 세션 기반의 데스크톱 및 가상 데스크톱 RemoteApp 및 데스크톱 연결을 사용 하 여 게시 합니다.

- URL은 RemoteApp 및 데스크톱 연결에 액세스할 수 있는 RD 웹 액세스 서버에 링크를 표시 합니다.
- 구성 된 RemoteApp 및 데스크톱 연결은 나열 됩니다.

원격 리소스를 추가 합니다.

1. 연결 센터에서 **+**, 다음 **원격 리소스 추가**클릭 합니다. 
2. 원격 리소스에 대 한 정보를 입력 합니다.
   - **피드 URL** -RD 웹 액세스 서버의 URL입니다. 전자 메일 주소와 관련 된 RD 웹 액세스 서버에 대 한 검색 하려면 클라이언트를 인지를-이 필드에 회사 메일 계정에 입력할 수 있습니다.
   - **사용자 이름** -RD 웹 액세스 서버에 연결 하는에 사용할 사용자 이름입니다.
   - **암호** -RD 웹 액세스 서버에 연결 하는 데 암호입니다.
3. **저장**을 클릭합니다.


원격 리소스 연결 중앙에 표시 됩니다.


## 내부 자산에 액세스할 수 RD 게이트웨이에 연결

원격 데스크톱 게이트웨이 (RD 게이트웨이)를 사용 하면 어디에서 나 회사 네트워크에서 원격 컴퓨터에 연결에서 인터넷 합니다. 만들 하 고 새 데스크톱 연결을 설정 하는 동안 또는 앱의 기본 설정에서 사용자 게이트웨이 관리할 수 있습니다.

기본 설정에서 새 게이트웨이를 설정 하려면:

1. 연결 센터 클릭 **기본 설정 > 게이트웨이**합니다. 
2. 클릭 하 고 **+** 다음 정보를 입력 테이블의 맨 아래에 단추:
  - **서버 이름** -게이트웨이로 사용 하려는 컴퓨터의 이름입니다. 이 Windows 컴퓨터 이름, 인터넷 도메인 이름 또는 IP 주소일 수 있습니다. 서버 이름으로 포트 정보를 추가할 수도 있습니다 (예: **RDGateway:443** 또는 **10.0.0.1:443**).
  - **사용자 이름** -사용자 이름 및 암호에 연결 하는 원격 데스크톱 게이트웨이 사용할 수 있습니다. 원격 데스크톱 연결에 사용 되는 것과 동일한 사용자 이름 및 암호를 사용 하 여 **연결 자격 증명 사용** 을 선택할 수 있습니다.


## 사용자 계정 관리

데스크톱 또는 원격 리소스에 연결 하는 경우 다시에서 선택 하도록 사용자 계정을 저장할 수 있습니다. 원격 데스크톱 클라이언트를 사용 하 여 사용자 계정을 관리할 수 있습니다.

새 사용자 계정을 만들려면:

1. 연결 중앙에서 **설정**을 클릭 > **계정**입니다.
2. **사용자 계정을 추가**클릭 합니다.
3. 다음 정보를 입력 합니다.
   - **사용자 이름** -원격 연결에 사용 하기 위해 저장 하는 사용자의 이름입니다. 다음 형식 중 하나로 사용자 이름을 입력할 수 있습니다: 사용자 _, domain\user_name, 또는 user_name@domain.com 합니다.
   - **암호** -지정한 사용자에 대 한 암호입니다. 원격 연결에 사용 하 여 저장 하려는 모든 사용자 계정 연결 된 암호 필요 합니다.
   - **이름** -다른 암호를 사용 하 여 동일한 사용자 계정을 사용 하는 경우 해당 사용자 계정을 구분 하기 위해 이름을 설정 합니다.
4. **저장**을 누르고 **설정**합니다.

## 디스플레이 해상도 사용자 지정
원격 데스크톱 세션에 대 한 디스플레이 해상도 지정할 수 있습니다.

1. 연결 센터에서 **기본 설정**을 클릭 합니다.
2. **확인**을 클릭 합니다. 
3. 클릭 **+**.
4. 해상도 높이 너비를 입력 한 다음 누르고 **확인 합니다.**

해상도 삭제 하려면 선택 하 고, 클릭 한 다음 **-**.

**표시는 별도 공간** Mac OS X 10.9를 실행 하 고 Mavericks에 **디스플레이 별도 공간을** 사용 하지 않도록 설정 하는 경우 (**시스템 기본 설정 > 필수 제어**), 같은 옵션을 사용 하 여 원격 데스크톱 클라이언트에서이 설정을 구성 해야 합니다.

### 원격 리소스에 대 한 드라이브 리디렉션
드라이브 리디렉션은 로컬에으로에 원격 응용 프로그램을 사용 하 여 생성 된 파일을 저장할 수 있도록 원격 리소스에 대 한 지원 리디렉션된 폴더는 항상 원격 세션에서 네트워크 드라이브로 표시 홈 디렉터리입니다.

> [!NOTE]
> 이 기능을 사용 하려면 관리자가 서버에 적절 한 설정 해야 합니다.


## 원격 세션에서 키보드를 사용 합니다.

Windows 키보드 레이아웃에서 Mac 키보드 레이아웃 달라 집니다. 

- 명령 키를 Mac 키보드에서 Windows 키를 같습니다.
- Mac에서 명령 단추를 사용 하는 작업을 수행 하려면 Windows 컨트롤 버튼을 사용 해야 합니다 (예:: 복사 = Ctrl + C).
- 또한 FN 키를 누르면 세션에서 기능 키를 활성화할 수 있습니다 (예:: FN + F1).
- Alt 키를를 Mac 키보드에 스페이스바의 오른쪽 창에서 Alt Gr/오른쪽 Alt 키를 같습니다.

기본적으로 원격 세션에 클라이언트를 실행 중인 운영 체제와 동일한 키보드 로캘을 사용 합니다. (Mac는 en 실행 하는 경우-우리도 원격 세션에 사용 되는 운영 체제입니다. OS 키보드 로캘 사용 하지 않는 경우 원격 PC에 설정 하 고 설정을 변경 하면 키보드 확인 수동으로 합니다. 키보드 및 로캘에 대 한 자세한 내용은 대 한 [원격 데스크톱 클라이언트 FAQ](remote-desktop-client-faq.md) 를 참조 하세요.


## 원격 데스크톱 게이트웨이 플러그 가능한 인증 및 권한 부여에 대 한 지원

Windows Server 2012 R2에는 새로운 인증 방법, 원격 데스크톱 게이트웨이 플러그 가능한 인증 및 사용자 지정 인증 루틴에 대 한 더 많은 유연성을 제공 하는 인증에 대 한 지원을 도입 되었습니다. 하면 이제이 인증 모델 Mac 클라이언트를 사용 하 여 됩니다.

> 위의 문서에 설명 하는 있지만 Windows 8.1 하기 전에 사용자 지정 인증 및 권한 부여 모델 지원 되지 않습니다.

확인이 기능에 대 한 자세한 내용은 [http://aka.ms/paa-sample](http://aka.ms/paa-sample).
