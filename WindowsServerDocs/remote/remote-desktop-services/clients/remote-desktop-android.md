# Android에서 원격 데스크톱을 사용 하 여 시작

>적용 대상: Windows 10, Windows 8.1, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2

Android 장치에서 직접 Windows 앱 및 데스크톱 작업할 Android에 대 한 원격 데스크톱 클라이언트를 사용할 수 있습니다.

시작 하려면 다음 정보를 사용 합니다. 질문이 있는 경우 [FAQ](remote-desktop-client-faq.md) 를 확인 해야 합니다.

> [!NOTE]
> - Android 클라이언트에 대 한 새 릴리스 용어나? 체크 아웃 [Android에서 원격 데스크톱에 대 한 새로운 기능?](android-whatsnew.md)
> Android 4.1 및 최신 장치 뿐만 아니라 Chromebooks ChromeOS 53 설치를 사용 하 여 클라이언트를 실행할 수 있습니다. Chrome에서 Android 응용 프로그램에 자세히 알아보기 [다음과 같습니다](https://sites.google.com/a/chromium.org/dev/chromium-os/chrome-os-systems-supporting-android-apps).

## RD 클라이언트를 가져오고 사용을 시작합니다

Android 디바이스에 원격 데스크톱을 사용 하 여 시작 하려면 다음이 단계를 따르세요.

1. [Google Play](https://play.google.com/store/apps/details?id=com.microsoft.rdc.android)에서 원격 데스크톱 클라이언트를 다운로드 합니다. 
2. [원격 연결을 허용 하도록 PC를 설정](remote-desktop-allow-access.md)합니다.
3. 원격 데스크톱 연결 또는 원격 리소스를 추가 합니다. RemoteApp 프로그램, 세션 기반 데스크톱을 사용 하 여 Windows PC 및 원격 리소스에 직접 연결 하려면 연결을 사용 하거나 가상 데스크톱 온-프레미스 게시 합니다. 
4. 원격 데스크톱을 신속 하 게 가져올 수 있도록 위젯을 만듭니다.

> [!NOTE]
> 새로운 기능을 이전 플라이트 하려는 경우에 Google Play 스토어에서 [Microsoft 원격 데스크톱 베타](https://play.google.com/store/apps/details?id=com.microsoft.rdc.android.beta) 응용 프로그램을 다운로드 하는 것이 좋습니다. 

### 원격 데스크톱 연결을 추가 합니다.

원격 데스크톱 연결을 만들려면:

1. 연결 센터 탭에서 **+** 를 이동한 다음 **바탕 화면**을 탭 합니다.
2. 연결 하려는 컴퓨터에 대 한 다음 정보를 입력 합니다.
  - **PC 이름** -컴퓨터의 이름입니다. 이 Windows 컴퓨터 이름, 인터넷 도메인 이름으로 또는 IP 주소일 수 있습니다. 또한 PC 이름 (예를 들어 **MyDesktop:3389** 또는 **10.0.0.1:3389**) 포트 정보를 추가할 수 있습니다.
  - **사용자 이름** -원격 PC에 액세스 하는 데 사용자 이름입니다. 다음 형식을 사용할 수 있습니다: *사용자 _*, *domain\user_name*또는 *user_name@domain.com*합니다. 사용자 이름 및 암호를 묻는 메시지를 표시 여부를 지정할 수 있습니다.
3. 또한 다음과 같은 추가 옵션을 설정할 수 있습니다.
  - **식별 이름** -에 연결 하는 PC에 대 한 기억 하기 쉬운 이름을입니다. 모든 문자열을 사용할 수 있지만 이름을 지정 하지 않으면 PC 이름이 표시 됩니다.
  - **게이트웨이** -가상 데스크톱, RemoteApp 프로그램 및 세션 기반 데스크톱에서 회사 내부 네트워크에 연결 하는 데 사용할 수 있는 원격 데스크톱 게이트웨이. 시스템 관리자에서 게이트웨이 대 한 정보를 가져옵니다.
    원격 데스크톱 게이트웨이 구성 해야 하나요?
  - **소리** – 원격 세션 동안 오디오를 사용 하 여 장치를 선택 합니다. 원격 장치를 로컬 장치에서 하거나 전혀 없는 소리를 재생 선택할 수 있습니다.
  - 이 설정을 사용 하면 연결에 대 한 사용자 지정 해상도 설정 **사용자 지정 디스플레이 해상도** -합니다. 해상도 오프 적용 될 때 앱의 전역 설정에서 정의 합니다.
  - **스왑 마우스 단추** -마우스 오른쪽 단추에 대 한 마우스 왼쪽된 단추 기능을 교체 하려면이 옵션을 사용 합니다. (이 경우 특히 유용 왼손 사용자에 대 한 원격 PC가 구성 되었지만 오른손 마우스를 사용 합니다.)
  - **관리자 세션에 연결** -Windows server 관리 콘솔 세션에 연결 하려면이 옵션을 사용 합니다.
  - 원격 PC에 원격 파일 시스템으로 로컬 저장소를 탑재 **로컬 저장소로 이동** -합니다.
4. **저장**을 누릅니다.

이러한 설정을 편집할 필요가? 바탕 화면 이름 옆에 있는 오버플로 메뉴 (**...**) 누르고 누른 다음 **편집**합니다.

연결을 삭제 하려면? 마찬가지로 오버플로 메뉴 (**...**)를 누르고 **제거**합니다.

>[!TIP]
> 0xf07 오류에 대 한 경우 "(우리 수 없습니다" 연결 원격 PC에 연결 된 사용자 계정의 암호 만료 되었으므로) 잘못 된 암호, 암호를 변경 하 고 다시 시도 합니다.

### 원격 리소스를 추가 합니다.
원격 리소스는 RemoteApp 프로그램, 세션 기반의 데스크톱 및 가상 데스크톱 RemoteApp 및 데스크톱 연결을 사용 하 여 게시 합니다.

에 원격 리소스를 추가 합니다.

1. 탭 연결 센터 화면에서 **+** 를 이동한 다음 **원격 리소스 피드**를 탭 합니다. 
2. 원격 리소스에 대 한 정보를 입력 합니다.
   - **전자 메일 또는 URL** -RD 웹 액세스 서버의 URL입니다. 전자 메일 주소와 관련 된 RD 웹 액세스 서버에 대 한 검색 하려면 클라이언트를 인지를 –이 필드에 회사 메일 계정에 입력할 수 있습니다.
   - **사용자 이름** -에 연결 하는 RD 웹 액세스 서버에 사용할 사용자 이름입니다.
   - **암호** -RD 웹 액세스 서버에 연결 하는 데 암호입니다.
3. **저장**을 누릅니다.

원격 리소스 연결 가운데에 표시 됩니다.


원격 리소스를 삭제 합니다.

1. 연결 센터에서 원격 리소스 옆의 오버플로 메뉴 (**...**)를 누릅니다.
2. **제거**를 누릅니다.
3. 삭제를 확인 합니다.

### 위젯 – 핀 홈 화면에 저장 된 데스크톱

원격 데스크톱 응용 프로그램 Android 위젯 기능을 사용 하 여 홈 화면에 고정 연결을 지원 합니다. 위젯을 추가 하는 방법은 사용 하는 Android 디바이스와 운영 체제의 유형에 따라 다릅니다. 위젯 추가 하는 가장 일반적인 방법은 다음과 같습니다. 

1. **응용 프로그램** 바로 앱 메뉴를 탭 합니다.
2. **위젯을**탭 합니다.
3. 살짝 밀기는 위젯 통해 및 설명, "Pin 원격 데스크톱"를 사용 하 여 원격 데스크톱 아이콘
4. 탭 하 고 해당 원격 데스크톱 위젯 홈 화면으로 이동 합니다.
5. 아이콘을 놓으면 저장 된 원격 데스크톱 표시 됩니다. 홈 화면에 저장 하려는 연결을 선택 합니다.

이제 눌러 홈 화면에서 직접 원격 데스크톱 연결을 시작할 수 있습니다.

> [!NOTE]
> 원격 데스크톱 응용 프로그램에서 데스크톱 연결의 이름을 바꾸면이 고정 된 원격 데스크톱의 레이블을 업데이트 되지 않습니다.

## 내부 자산에 액세스할 수 있는 RD 게이트웨이에 연결

원격 데스크톱 게이트웨이 (RD 게이트웨이)를 사용 하면 어디에서 나 회사 네트워크에서 원격 컴퓨터에 연결에서 인터넷 합니다. 만들 수 있으며 원격 데스크톱 클라이언트를 사용 하 여 게이트웨이 관리.

새로운 게이트웨이를 설정 하려면:

1. 연결 중앙에서 **설정을 gt_ 게이트웨이**누릅니다. 탭 **+** 새로운 게이트웨이 추가 합니다.
2. 다음 정보를 입력 합니다.
  - **서버 이름** -게이트웨이 사용 하려는 컴퓨터의 이름입니다. 이 Windows 컴퓨터 이름, 인터넷 도메인 이름으로 또는 IP 주소일 수 있습니다. 서버 이름으로 포트 정보를 추가할 수도 있습니다 (예: **RDGateway:443** 또는 **10.0.0.1:443**).
  - **사용자 이름** -사용자 이름 및 암호를 연결 하는 원격 데스크톱 게이트웨이를 사용 해야 합니다. **데스크톱 사용자 계정을 사용 하 여** 원격 데스크톱 연결에 사용 되는 것과 동일한 자격 증명을 사용 하 여 선택할 수 있습니다.

## 사용자 계정 관리

데스크톱 또는 원격 리소스에 연결할 때 다시에서 선택 하도록 사용자 계정을 저장할 수 있습니다. 또한 사용자 데이터를 저장 하 여 데스크톱에 연결할 때가 아니라 클라이언트 자체의 사용자 계정을 정의할 수 있습니다.

새 사용자 계정을 만들려면:

1. 연결 센터에서 **설정**, 누르고 **사용자 계정**.
2. 탭 **+** 새 사용자 계정을 추가 합니다.
3. 다음 정보를 입력 합니다.
   - **사용자 이름** -원격 연결에 사용 하기 위해 저장 하는 사용자의 이름입니다. 다음 형식 중 하나로 사용자 이름을 입력할 수 있습니다: 사용자 _, domain\user_name, 또는 user_name@domain.com 합니다.
   - **암호** -지정한 사용자에 대 한 암호입니다. 원격 연결에 사용 하도록 저장할 하려는 모든 사용자 계정 암호 연결 되어 있어야 합니다.
4. **저장**을 누릅니다.

사용자 계정을 삭제 합니다.

1. 연결 센터에서 탭 **gt_ 사용자 계정을 설정**합니다.
2. 선택 목록에 사용자 계정의 눌러. 여러 사용자가 선택할 수 있습니다.
3. 선택한 사용자를 삭제 하는 휴지통을 탭 합니다.

## 원격 데스크톱 세션 이동
원격 데스크톱 연결을 시작할 때 사용할 수 있는 세션 이동 하는 데 사용할 수 있는 도구가 있습니다.

### 원격 데스크톱 연결을 시작 합니다.

1. 세션을 시작에 대 한 원격 데스크톱 연결을 누릅니다. 
2. 원격 데스크톱에 대 한 인증서를 확인 하 라는 메시지가 표시, **연결**을 누릅니다. **이 컴퓨터에 연결에 대 한 다시 묻지 않음** 항상 인증서를 수락 하도록 선택할 수 있습니다.

### 전역 앱 설정 관리

Android 클라이언트에서 다음 전역 설정을 설정할 수 있습니다.

- **데스크톱 미리 보기 표시** -에 연결 하기 전에 연결 중심의 데스크톱의 미리 보기를 볼 수 있습니다. 기본적으로이 **on**으로 설정 됩니다.
- **확대/축소를 손가락 모으기** -손가락을 모아 확대/축소 제스처를 사용할 수 있습니다. 원격 데스크톱을 통해 사용 중인 앱 멀티 터치 (Windows 8에서 도입 됨)를 지 원하는 경우이 설정을 켠 **꺼져**있습니다.
- **원격 데스크톱을 개선 하는 데 도움이** -익명 데이터를 Microsoft에 보냅니다. 클라이언트를 개선 하기 위해이 데이터를 사용 합니다. 이 익명, 개인 데이터를 처리 하는 방법에 대 한 자세한 정보, [원격 데스크톱 클라이언트 개인정보취급방침](https://www.microsoft.com/privacystatement/RemoteApp/Default.aspx)볼 수 있습니다. **기본적으로이 설정은 켜져 있습니다.**
- **표시** -디스플레이 대 한 전역 설정이 두 가지가 있습니다.
   - **방향** -사용자 세션에 대 한 기본 방향 (가로 또는 세로)을 설정 합니다. 
   >[!NOTE]
   > Windows 8 또는 이전 버전의 Windows 실행 하는 PC에 연결 하면 세션 올바르게 조정 되지 않습니다. PC에서 분리 한 다음 다시 사용 하려는 방향에 가장 좋습니다. 최소한 PC를 업그레이드 하는 훨씬 더 나은 옵션은 Windows 8.1.

   - **해상도** -전체적으로 데스크톱 연결에 사용 하려는 해상도 설정 합니다. 개별 앱 또는 연결에 대 한 사용자 지정 해결 이미 설정한 경우에이 설정을 변경 되지 않습니다.
   >[!NOTE]
   >디스플레이 설정 중 하나를 변경 하는 경우에 적용 새 연결 그 시점부터에 합니다. 이미 연결을 끊은 후 다시 연결 하는 세션의 변경을 확인할 수 있습니다.

### 연결 표시줄

연결 표시줄은 추가 탐색 컨트롤에 대 한 액세스 제공 합니다. 기본적으로 연결 표시줄 화면 맨 위에 있는 가운데에 배치 됩니다. 두 번 탭 하 고 끌어서 바 왼쪽 또는 오른쪽으로 이동 합니다.

- **이동 컨트롤**: 팬 컨트롤을 사용 하면 화면을 확대 하 고 이동 합니다. 참고 이동 컨트롤은 직접 터치를 사용 하 여 사용할 수만 있습니다.
   - 활성화 / 이동 컨트롤을 사용 하지 않도록 설정: 화면을 확대/축소 및 이동 컨트롤을 표시 하려면 연결 표시줄에 이동 아이콘을 탭 합니다. 연결 표시줄 숨기기를 기존 픽셀 화면 돌아갑니다에 팬 아이콘을 탭 합니다.
   - 이동 컨트롤을 사용 하 여: 팬 누르고 제어 하 고 끌어서 화면을 이동 방향으로 합니다.
   - 이동 컨트롤을 이동: 화면에 컨트롤을 이동 컨트롤을 길게 두 번 탭 합니다.
- **추가 옵션**: 세션 선택 항목이 표시 바 및 명령 표시줄 (아래 참조)을 추가 옵션 아이콘을 탭 합니다.
- **키보드**: 키보드 아이콘을 표시 하거나 숨기려면 키보드를 탭 합니다. 이동 컨트롤이 키보드 표시 될 때 자동으로 표시 됩니다.
- **연결 표시줄 이동**: 탭 및 보류 연결 표시줄 및 다음 끌어서 놓기를 새 위치로 화면 맨 위에 있는 합니다.


### 명령 모음

화면의 오른쪽에 있는 명령 모음을 표시 하려면 연결 모음을 누릅니다. 마우스 모드 (직접 터치 및 마우스 포인터) 간에 전환할 수 있습니다. 명령 모음에서 연결 센터로 돌아가려면 홈 단추를 사용 합니다. 또는 동일한 작업에 대해 뒤로 단추를 사용할 수 있습니다. 활성 사용자 세션을 중단 합니다. 


### 원격 세션에서 사용 하 여 직접 터치 제스처 및 마우스 모드

클라이언트는 표준 터치 제스처를 사용합니다. 또한 원격 데스크톱에서 마우스 동작을 복제할 터치 제스처를 사용할 수 있습니다. 마우스 모드를 사용할 수 있는 아래 표에 정의 됩니다.

> [!NOTE]
> Windows 8와의 상호 작용 또는 최신 직접 터치 모드에서 기본 터치 제스처 지원 됩니다. 

| 마우스 모드    | 마우스 조작      | 제스처                                                               |
|---------------|----------------------|-----------------------------------------------------------------------|
| 직접 터치  | 마우스 왼쪽 단추로 클릭           | 1 손가락으로 탭 하기                                                          |
| 직접 터치  | 마우스 오른쪽 단추로 클릭          | 1 손가락으로 탭 하기 및 누르고 있기                                                 |
| 마우스 포인터 | Zoom                 | 확대 또는 축소 손가락 떨어져 이동 손가락 모으기 및 2 손가락을 사용 합니다. |
| 마우스 포인터 | 마우스 왼쪽 단추로 클릭           | 1 손가락으로 탭 하기                                                          |
| 마우스 포인터 | 왼쪽 클릭 후 끌기  | 두 번 탭 1 손가락으로 누른 다음 끕니다.                               |
| 마우스 포인터 | 마우스 오른쪽 단추로 클릭          | 2 손가락으로 탭 하기                                                          |
| 마우스 포인터 | 오른쪽 클릭 후 끌기 | 2 손가락으로 두 번 탭 하기 및 고정, 다음 끕니다.                               |
| 마우스 포인터 | 마우스 휠          | 2 손가락으로 탭 하 고, 계속 누르고 위 또는 아래로 끌기                           |
