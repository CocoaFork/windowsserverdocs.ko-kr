# <a name="troubleshooting-web-application-proxy"></a>웹 응용 프로그램 프록시 문제 해결

>적용 대상: Windows Server 2016

**이 콘텐츠는 웹 응용 프로그램 프록시의 온-프레미스 버전에 적합 합니다. 클라우드를 통해 온-프레미스 응용 프로그램에 대 한 보안 액세스를 사용 합니다 [Azure AD 응용 프로그램 프록시 콘텐츠](https://azure.microsoft.com/documentation/articles/active-directory-application-proxy-get-started/)합니다.**  
  
이 섹션에서는 이벤트 설명 및 솔루션을 포함 하 여 웹 응용 프로그램 프록시에 대 한 문제 해결 절차를 제공 합니다. 오류가 표시 되는 세 장소가 있습니다.  
  
-   웹 응용 프로그램 프록시 관리자 콘솔에서  
  
    관리자 콘솔에 나열 된 각 이벤트 ID는 Windows 이벤트 뷰어 및 해당 설명을 볼 수 있습니다 하 고 솔루션 아래에 있습니다.  
  
    이벤트 뷰어를 열고 아래에 있는 웹 응용 프로그램 프록시에 관련 된 이벤트를 검색할 **Applications and Services Logs** > **Microsoft** > **Windows**  >  **웹 응용 프로그램 프록시** > **관리자**  
  
    ![](media/Troubleshooting-Web-Application-Proxy/WebApplicationProxyTroubleshooting.png)  
  
    필요한 경우 자세한 로그 분석 설정 및 디버깅 로그에서 사용할 수 하 고 아래의 Windows 이벤트 뷰어에 있는 웹 응용 프로그램 프록시 세션 로그를 켜면 \ Microsoft \ Windows \ 웹 응용 프로그램 프록시 \ 관리자  
  
-   PowerShell 오류  
  
    구성 하는 동안 발생 하는 문제에 대 한 이벤트는 PowerShell에 표시 됩니다.  
  
    모든 오류는 표준 PowerShell 오류 메시지를 사용 하 여 PowerShell 사용자에 게 표시 됩니다. 모든 PowerShell 명령 이벤트로 기록 됩니다. PowerShell에서 발생 하는 모든 이벤트 12016, ID 번호를 사용 하 여 Windows 이벤트 뷰어에서 나와 아래 PowerShell 섹션에서 정의 됩니다.  
  
-   Best Practices Analyzer에서  
  
    이러한 이벤트에 설명 합니다 [웹 응용 프로그램 프록시에 대 한 모범 사례 분석기](assetId:///995f5cea-e334-4ce6-a14c-6a2d03c0c79a)
