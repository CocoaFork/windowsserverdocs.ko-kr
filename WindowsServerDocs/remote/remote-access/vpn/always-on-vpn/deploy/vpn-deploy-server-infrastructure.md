# 2단계. 서버 인프라 구성

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **이전:** 1 단계입니다. 항상 켜져 VPN 배포 계획](always-on-vpn-deploy-planning.md)<br>
& #187;  [ **다음:** 3 단계입니다. VPN에 대 한 원격 액세스 서버를 항상 구성](vpn-deploy-ras.md)


이 단계에서 설치 및 VPN을 지 원하는 데 필요한 서버 측 구성 요소를 구성 합니다. 서버 쪽 구성 요소는 사용자, VPN 서버 및 NPS 서버에서 사용 되는 인증서를 배포할 PKI 구성 포함 됩니다.  RRAS IKEv2 연결 및 VPN 연결에 대 한 인증을 수행 하는 NPS 서버를 지원 하도록 구성할 수도 있습니다.

## 그룹 정책에서 인증서 자동 등록 구성
이 절차에서는 도메인 구성원은 자동으로 사용자 및 컴퓨터 인증서를 요청할 수 있도록 도메인 컨트롤러에서 그룹 정책을 구성 합니다. 이렇게 하면 VPN 사용자에 게를 요청 하 고 VPN 연결을 자동으로 인증 된 사용자 인증서를 검색할 수 있습니다. 마찬가지로,이 정책을 통해 서버를 요청 하는 NPS 서버 인증 인증서 자동으로 합니다. 

수동으로 VPN 서버에서 인증서를 등록합니다.

>[!TIP]
>비 domained 가입된 컴퓨터 [CA 구성을 비-도메인 가입 컴퓨터](#ca-configuration-for-non-domain-joined-computers) 아래를 참조 하세요. RRAS 서버 도메인에 가입 되지 않으므로 VPN 게이트웨이 인증서 등록에 자동 등록을 사용할 수 없습니다.  따라서 오프 라인 인증서 요청 절차를 사용 합니다. 


1.  도메인 컨트롤러에서 그룹 정책 관리를 엽니다.

2.  탐색 창에서 도메인 (예를 들어 corp.contoso.com)를 마우스 오른쪽 단추로 클릭 하 고**이 도메인에서 GPO를 만들고 여기에 링크**를 클릭 합니다.

3.  새 GPO 대화 상자에서 **자동 등록 정책**를 입력 하 고 **확인**을 클릭 합니다.

4.  탐색 창에서 **자동 등록 정책**마우스 오른쪽 단추로 클릭 하 고 **편집**을 클릭 합니다.

5.  그룹 정책 관리 편집기에서 컴퓨터 인증서 자동 등록을 구성 하려면 다음 단계를 완료 합니다.

    1.  탐색 창에서 **컴퓨터 Configuration\\Policies\\Windows Settings\\Security Settings\\Public 키 정책**를 클릭 합니다.

    2.  세부 정보 창에서**인증서 서비스 클라이언트-자동 등록**마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다.

    3.  인증서 서비스 클라이언트-자동 등록 속성 대화 상자의 **구성 모델** **사용**을 클릭 합니다.

    4.  **만료 된 인증서 갱신, 보류 중인 인증서를 업데이트 하 고, 해지 된 인증서 제거**선택하 고**인증서 템플릿을 사용 하는 인증서를 업데이트**합니다.

    5.  **확인**을 클릭 합니다.

6.  그룹 정책 관리 편집기에서 구성 사용자 인증서 자동 등록 하려면 다음 단계를 완료 합니다.

    1.  탐색 창에서 **사용자 Configuration\\Policies\\Windows Settings\\Security Settings\\Public 키 정책**를 클릭 합니다.

    2.  세부 정보 창에서**인증서 서비스 클라이언트-자동 등록**마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다.

    3.  인증서 서비스 클라이언트-자동 등록 속성 대화 상자의 **구성 모델** **사용**을 클릭 합니다.

    4.  **만료 된 인증서 갱신, 보류 중인 인증서를 업데이트 하 고, 해지 된 인증서 제거**선택하 고**인증서 템플릿을 사용 하는 인증서를 업데이트**합니다.

    5.  **확인**을 클릭 합니다.

    6.  그룹 정책 관리 편집기를 닫습니다.

7.  닫기 그룹 정책 관리 합니다.

### 비-도메인 가입된 컴퓨터에 대 한 CA 구성

RRAS 서버 도메인에 가입 되지 않으므로 VPN 게이트웨이 인증서 등록에 자동 등록을 사용할 수 없습니다.  따라서 오프 라인 인증서 요청 절차를 사용 합니다. 

1. RRAS 서버에서 예제 인증서 정책에 따라 호출된 **VPNGateway.inf** (0 섹션) 부록에 제공 된 요청 하 고 다음 항목을 사용자 지정 파일을 생성 합니다. 

   - [새 요청] 섹션에서 선택한 [_고객_] VPN 끝점 FQDN으로 주체 이름에 사용 되는 vpn.contoso.com를 대체 합니다.

   - [확장] 섹션에서 선택한 [_고객_] VPN 끝점 FQDN으로 주체 대체 이름에 사용 되는 vpn.contoso.com을 바꿉니다.

2. 저장 하거나 선택한 위치 **VPNGateway.inf** 파일을 복사 합니다.

3. 관리자 권한 명령 프롬프트에서 **VPNGateway.inf** 파일 및 형식을 포함 된 폴더로 이동 합니다.

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

4. 인증 기관 서버 또는 권한 있는 액세스 워크스테이션 (발)을 새로 만든된 **VPNGateway.req** 출력 파일을 복사 합니다. 

5. 저장 하거나 인증 기관 서버 또는 권한 있는 액세스 워크스테이션 (발)에서 선택한 위치 **VPNGateway.req** 파일을 복사 합니다.

6. 관리자 권한 명령 프롬프트에서 입력 하 고 이전 단계에서 만든 VPNGateway.req 파일이 포함 된 폴더로 이동 합니다. 

   ```
   certreq -attrib “CertificateTemplate:[Customer]VPNGateway” -submit VPNgateway.req VPNgateway.cer
   ```

7. 인증 기관 목록 창에서 메시지를 표시 하는 경우 인증서 요청을 처리 하기 위해 적절 한 엔터프라이즈 CA를 선택 합니다.

8. RRAS 서버로 새로 만든된 **VPNGateway.cer** 출력 파일을 복사 합니다. 

9. 저장 하거나 RRAS 서버에서 선택한 위치 **VPNGateway.cer** 파일을 복사 합니다.

10. 관리자 권한 명령 프롬프트에서 입력 하 고 이전 단계에서 만든 VPNGateway.cer 파일이 포함 된 폴더로 이동 합니다.
   
   ```
   certreq -accept VPNGateway.cer
   ```

11. 으로 설명 [여기](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in) **컴퓨터 계정** 옵션을 선택 하면 인증서 MMC 스냅인을 실행 합니다.

12. 다음 속성을 사용 하 여 RRAS 서버에 대 한 유효한 인증서가 있는지 확인 합니다.

   - **용도:** 서버 인증, 중간 IKE IP 보안 

   - **인증서 템플릿:** [_고객_] VPN 서버

#### 예: VPNGateway.inf 스크립트

여기의 대역의 프로세스를 사용 하 여 VPN 게이트웨이 인증서를 요청 하는 데 사용 되는 인증서 요청 정책 예제 스크립트를 볼 수 있습니다.

>[!TIP]
>인증서 요청 정책 폴더 아래의 VPN 제공 IP 키트 VPNGateway.inf 스크립트의 복사본을 찾을 수 있습니다. 고객의 특정 값을 사용 하 여 '제목' 및 '\_continue\_'를 업데이트 합니다.

```
[Version] 

Signature="$Windows NT$"

[NewRequest]
Subject = "CN=vpn.contoso.com"
Exportable = FALSE   
KeyLength = 2048     
KeySpec = 1          
KeyUsage = 0xA0      
MachineKeySet = True
ProviderName = "Microsoft RSA SChannel Cryptographic Provider"
RequestType = PKCS10 

[Extensions]
2.5.29.17 = "{text}"
_continue_ = "dns=vpn.contoso.com&"

```

## VPN 사용자, VPN 서버 및 NPS 서버 그룹 만들기

이 절차에서는 사용자가 VPN을 사용 하 여 조직의 네트워크에 연결할 수 있는 새 Active Directory (AD) 그룹을 추가할 수 있습니다.

이 그룹에는 두 가지 용도가 있습니다.

-   VPN 필요한 사용자 인증서에 대 한 자동으로 등록할 수 있는 사용자를 정의 합니다.

-   NPS VPN 액세스에 대 한 권한을 부여 있는 사용자를 정의 합니다.

사용자 지정 그룹을 사용 하 여 사용자의 VPN 액세스를 해지 하려는 경우에서 제거할 수 있습니다 해당 사용자 그룹.

VPN 서버를 포함 하는 그룹 및 NPS 서버를 포함 하는 다른 그룹 추가할 수도 있습니다. 해당 멤버를 인증서 요청을 제한 하기 위해 이러한 그룹을 사용 합니다.

>[!NOTE] 
>VPN 권장 DMA/주변에 있는 서버를 도메인에 가입 된를 수 없습니다. 그러나 더 나은 관리 효율성에 대 한 도메인에 가입 된 VPN 서버 하려는 경우 (그룹 정책, 백업/모니터링 에이전트 로컬 사용자를 관리 하 고 등) AD 그룹 VPN 서버 인증서 템플릿에 다음 추가 합니다.

### VPN 사용자 그룹 구성

1.  도메인 컨트롤러에서 Active Directory 사용자 및 컴퓨터를 엽니다.

2.  컨테이너 또는 조직 구성 단위를 단추로 클릭 하 고 **새로** 다음 **그룹**을 클릭 합니다.

3.  **그룹 이름** **는 VPN 사용자**입력 하 고 **확인**을 클릭 합니다.

4.  **VPN 사용자**마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다.

5.  VPN 사용자 속성 대화 상자의 **구성원** 탭에서 **추가**클릭 합니다.

6.  사용자 선택 대화 상자에서 추가 VPN 해야 하는 모든 사용자가 액세스 하 고 **확인**을 클릭 합니다.

7.  Active Directory 사용자 및 컴퓨터를 닫습니다.

### VPN 서버와 NPS 서버 그룹 구성

1.  도메인 컨트롤러에서 Active Directory 사용자 및 컴퓨터를 엽니다.

2.  컨테이너 또는 조직 구성 단위를 단추로 클릭 하 고 **새로** 다음 **그룹**을 클릭 합니다.

3.  **그룹 이름** **VPN 서버**입력 하 고 **확인**을 클릭 합니다.

4.  **VPN 서버**마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다.

5.  VPN 서버 속성 대화 상자의 **구성원** 탭에서 **추가**클릭 합니다.

6.  **개체**유형 **컴퓨터** 확인란을 선택 하 고 **확인**을 클릭 합니다.

7.  **선택할 개체 이름 입력**, VPN 서버의 이름을 입력 하 고 **확인**을 클릭 합니다.

8.  VPN 서버 속성 대화 상자를 닫으려면 **확인** 클릭 합니다.

9.  NPS 서버 그룹에 대 한 이전 단계를 반복 합니다.

10. Active Directory 사용자 및 컴퓨터를 닫습니다.

## 사용자 인증 템플릿 만들기

이 절차에서는 사용자 지정 클라이언트-서버 인증 템플릿을 구성할 수 있습니다. 업그레이드 된 호환성 수준을 선택 하 여 인증서의 전반적인 보안을 강화 하려는 때문에이 템플릿을 필요 하 고 Microsoft 플랫폼 암호화 공급자를 선택 합니다. 이 마지막 변경 인증서를 보호 하 고 클라이언트 컴퓨터에 TPM를 사용할 수 있습니다. TPM의 개요를 [신뢰할 수 있는 플랫폼 모듈 기술 개요](https://docs.microsoft.com/windows/device-security/tpm/trusted-platform-module-overview)를 참조 하세요.

**절차:**

1.  CA에서 인증 기관을 엽니다.

2.  탐색 창에서**인증서 템플릿**을 마우스 오른쪽 단추로 클릭 하 고**관리**를 클릭 합니다.

3.  인증서 템플릿 콘솔에서**사용자**마우스 오른쪽 단추로 클릭 하 고**템플릿 복제**를 클릭 합니다.

    >[!WARNING]
    >언제 든 지 10 단계 전에 **적용** 또는 **확인** 을 클릭 하지 마십시오.  모든 매개 변수를 입력 하기 전에 이러한 단추를 클릭 하면 고정 및 더 이상 편집 가능한 많은 선택 됩니다. 예를 들어 **암호화** 탭의 _레거시 암호화 저장소 공급자_ 공급자 범주 필드에 표시 하는 경우 해지면 변경 방지 더 이상 사용 하지 않도록 설정 합니다. 유일한 대체 하 서식 파일을 삭제 하 고 다시입니다.  

4.  **일반**새 템플릿의 속성 대화 상자에서탭에서 다음 단계를 완료 합니다.

    1.  **템플릿 표시 이름**, **VPN 사용자 인증**을 입력 합니다.

    2.  **Active Directory에 인증서 게시** 확인란의 선택을 취소 합니다.

5.  **보안**탭에서 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다.

    2.  사용자 선택, 컴퓨터, 서비스 계정 또는 그룹 대화 상자에서 **VPN 사용자**입력 하 고 **확인**을 클릭 합니다.

    3.  **그룹 또는 사용자 이름** **는 VPN 사용자**를 클릭 합니다.

    4.  **사용자가 VPN에 대 한 권한**을 **허용** 열에서 **등록** 및 **자동 등록** 확인란을 선택 합니다.

       >[!TIP]
       >읽기 확인란을 선택한 상태로 유지 해야 합니다. 즉, 등록에 대 한 읽기 권한이 필요합니다. 

    5.  **그룹 또는 사용자 이름**, **도메인 사용자**를 클릭 하 고 **제거**를 클릭 합니다.

6.  **호환성** 탭에서 다음 단계를 수행 합니다.

    1.  **인증 기관** **Windows Server2012R2**를 클릭 합니다.

    2.  **결과 변경 내용** 대화 상자에서 **확인**을 클릭 합니다.

    3.  **인증서 받는 사람** **Windows8.1/Windows Server2012R2**를 클릭 합니다.

    4.  **결과 변경 내용** 대화 상자에서 **확인**을 클릭 합니다.

7.  **요청 처리** 탭에 **개인 키를 내보낼 수 있음** 확인란의 선택을 취소 합니다.

8.  **암호화** 탭에서 다음 단계를 수행 합니다.

    1.  **공급자 범주** **키 저장소 공급자**를 클릭 합니다.

    2.  클릭 **요청 다음 공급자 중 하나를 사용 해야 합니다**.

    3.  **Microsoft 플랫폼 암호화 공급자** 확인란을 선택 합니다.

9.  **주체 이름** 탭에서 모든 사용자 계정에 나열 된 메일 주소를 설정 하지 않은 경우 확인란의 선택을 취소 **전자 메일** 이름과 **주체 이름에 전자 메일 이름을 포함** 합니다.

10. **확인** 하 여 VPN 사용자 인증 인증서 템플릿을 저장을 클릭 합니다.

## VPN 서버 인증 템플릿 만들기

이 절차에서는 VPN 서버에 대 한 새 서버 인증 템플릿을 구성할 수 있습니다. IP 보안 (IPsec) IKE 중간 응용 프로그램 정책을 사용 하면 필터 인증서 서버가 둘 이상의 인증서 서버 인증을 사용 하 여 사용할 수 있는 경우 추가 확장 키 사용 합니다.

>[!IMPORTANT] 
>VPN 클라이언트 공용 인터넷에서이 서버에 액세스 하기 때문에 제목과 대체 이름은 내부 서버 이름 다릅니다. 결과적으로, 권한의이 인증서를 VPN 서버에 없습니다.

**필수 조건:**<p>
도메인에 가입 된 VPN 서버

**절차:** 

1.  CA에서 인증 기관을 엽니다.

2.  탐색 창에서 **인증서 템플릿**을 마우스 오른쪽 단추로 클릭 하 고**관리**를 클릭 합니다.

3.  인증서 템플릿 콘솔에서**RAS 및 IAS 서버를**마우스 오른쪽 단추로 클릭 하 고**템플릿 복제**를 클릭 합니다.

4.  **일반**새 템플릿의 속성 대화 상자에서탭에서 **템플릿 표시 이름**, 입력 예를 들어, **VPN 서버 인증이** 나 **RADIUS 서버로**VPN 서버에 대 한 설명이 포함 된 이름을 합니다.

5.  **확장**에서탭에서 다음 단계를 완료 합니다.

    1.  **응용 프로그램 정책**클릭 하 고**편집**을 클릭 합니다.

    2.  **응용 프로그램 정책 확장 편집** 대화 상자에서**추가**클릭 합니다.

    3.  **응용 프로그램 정책 추가** 대화 상자에서 **IP 보안 IKE 중간**클릭 하 고 **확인**을 클릭 합니다.<p>IP 추가 보안 IKE EKU 중간 VPN 서버에 서버 인증 인증서가 여러 개 있는 경우 도움이 됩니다. IP 보안 IKE 중간 있으면 IPSec 두 EKU 옵션을 사용 하 여 인증서만 사용 합니다. 이렇게 하지 않으면 IKEv2 인증 13801 오류와 함께 실패할 수 있습니다: IKE 인증 자격 증명은 사용할 수 없습니다.

    4.  **새 템플릿의 속성**돌아가려면 **확인** 클릭대화 상자.

6.  **보안**탭에서 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다.

    2.  **사용자, 컴퓨터, 서비스 계정 또는 그룹 선택** 대화 상자에서 **VPN 서버**입력 하 고 **확인**을 클릭 합니다.

    3.  **그룹 또는 사용자 이름** **VPN 서버**를 클릭 합니다.

    4.  **VPN 서버에 대 한 권한**을 **허용** 열에서 **등록** 확인란을 선택 합니다.

    5.  **그룹 또는 사용자 이름**, **RAS 및 IAS 서버를**클릭 하 고 **제거**를 클릭 합니다.

7.  **주체 이름** 탭에서 다음 단계를 수행 합니다.

    1.  **요청에서 제공**을 클릭 합니다.

    2.  **인증서 템플릿** 경고 대화 상자에서 **확인**을 클릭 합니다.

8.  (선택 사항) *VPN 연결에 대 한 조건부 액세스를 구성 하는 경우*, **요청 처리** 탭을 클릭 하 고 선택 **허용 개인 키를 내보낼 수** 클릭 합니다.

9.  **확인** 하 여 VPN 서버 인증서 템플릿을 저장을 클릭 합니다.

10. 닫기 theCertificate 템플릿 콘솔입니다.

11. Authoritysnap에서 인증의 thenavigation 창에서**인증서 템플릿을**마우스 오른쪽 단추로 클릭 하 고 **새로** 누른 다음**발급할 인증서 템플릿**.

12. 인증 기관 서비스를 다시 시작 합니다.

13. 위의 4 단계에서 선택한 이름을 선택 하 고**확인**을 클릭 합니다.

14. TheCertification 기관 스냅인을 닫습니다.

## NPS 서버 인증 템플릿 만들기

세 번째 및 마지막 인증서 템플릿을 만들 수는 NPS 서버 인증 템플릿입니다. NPS 서버 인증 템플릿은 간단한 안전 하 게이 섹션의 이전에 만든 NPS 서버 그룹은 RAS 및 IAS 서버 템플릿 복사본을입니다. 

자동 등록에 대 한이 인증서를 구성 합니다.

**절차:**

1.  CA에서 인증 기관을 엽니다.

2.  탐색 창에서 **인증서 템플릿**을 마우스 오른쪽 단추로 클릭 하 고**관리**를 클릭 합니다.

3.  인증서 템플릿 콘솔에서**RAS 및 IAS 서버를**마우스 오른쪽 단추로 클릭 하 고 **템플릿 복제**를 선택 합니다.

4.  **일반**새 템플릿의 속성 대화 상자에서 **템플릿 표시 이름**입력 **NPS 서버 인증**을 탭 합니다.

5.  **보안**탭에서 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다.

    2.  **사용자, 컴퓨터, 서비스 계정 또는 그룹 선택** 대화 상자에서 **NPS 서버**입력 하 고 **확인**을 클릭 합니다.

    3.  **그룹 또는 사용자 이름** **NPS 서버**를 클릭 합니다.

    4.  **NPS 서버에 대 한 권한**을 **허용** 열에서 **등록** 및 **자동 등록** 확인란을 선택 합니다.

    5.  **그룹 또는 사용자 이름**, **RAS 및 IAS 서버를**클릭 하 고 **제거**를 클릭 합니다.

6.  NPS 서버 인증서 템플릿을 저장 하도록**확인** 클릭 합니다.

7.  닫기 theCertificate 템플릿 콘솔입니다.

8.  Authoritysnap에서 인증의 thenavigation 창에서**인증서 템플릿을**마우스 오른쪽 단추로 클릭 하 고 **새로** 누른 다음**발급할 인증서 템플릿**.

9.  **NPS 서버 인증**을 클릭 하 고**확인**을 클릭 합니다.

10. TheCertification 기관 스냅인을 닫습니다.

## 등록 하 고 사용자 인증서의 유효성을 검사합니다

그룹 정책 권한의 사용자 인증서를 사용 하 고 있으므로 정책을 업데이트할 필요 하 고 Windows10 사용자 계정에 대 한 올바른 인증서를 자동으로 등록 됩니다. 그런 다음 인증서 콘솔에서 인증서를 확인할 수 있습니다.

**절차:**

1.  **VPN 사용자** 그룹의 구성원으로 도메인에 가입 된 클라이언트 컴퓨터에 로그인 합니다.

2.  Windows 키 + R을 누릅니다, 그리고 **gpupdate /force**입력 하 고 Enter 키를 누릅니다.

3.  시작 메뉴에서 **certmgr.msc**입력 하 고 Enter 키를 누릅니다.

4.  인증서 스냅인에서 **개인**, 아래에서 **인증서**를 클릭 합니다. 인증서의 세부 정보 창에 표시 됩니다.

5.  사용자 현재 도메인 이름을 가진 인증서를 마우스 오른쪽 단추로 클릭 하 고 **열기**를 클릭 합니다.

6.  **일반** 탭에는 **유효 기간** 아래에 나열 된 날짜는 오늘 날짜를 확인 합니다. 열려 있지 않다면 잘못 된 인증서를 선택한 수 있습니다.

7.  **확인**클릭 하 고 인증서 스냅인 닫습니다.

## 서버 인증서의 유효성을 검사 하 고 등록

사용자 인증서와 달리 수동으로 VPN 서버 인증서를 등록 해야 합니다. 등록 한 후 사용자 인증서에 사용한 것과 같은 프로세스를 사용 하 여 확인 합니다. 사용자 인증서와 같은 NPS 서버 자동으로 등록 인증 인증서의 유효성을 검사할 하기만 하면 되므로 합니다.

>[!NOTE] 
>다음이 단계를 완료 하기 전에 해당 그룹 구성원을 업데이트할 수 있도록 하려면 VPN 및 NPS 서버를 다시 시작 해야 할 수 있습니다.

### 등록 하 고 VPN 서버 인증서의 유효성을 검사합니다

1.  VPN 서버 시작 메뉴에서 **certlm.msc**입력 하 고 Enter 키를 누릅니다.

2.  **개인**을 마우스 오른쪽 단추로 클릭 하 고 **모든 작업** 을 클릭 인증서 등록 마법사를 시작 하려면 **새 인증서 요청** 차례로 클릭 합니다.

3.  시작하기 전 페이지에서 **다음**을 클릭합니다.

4.  인증서 등록 정책 선택 페이지에서 **다음**을 클릭 합니다.

5.  인증서 요청 페이지에서 선택 하도록 VPN 서버 옆에 있는 확인란을 클릭 합니다.

6.  VPN 서버 확인란에서 인증서 속성 대화 상자를 열고 다음 단계를 완료 하는 **자세한 정보가 필요 합니다** 를 클릭 합니다.

    1.  **주체** 탭을 클릭 하 고 **주체 이름** **형식**에 **일반 이름** 를 클릭 합니다.

    2.  **주체 이름** **값**예를 들어 vpn.contoso.com VPN에 연결 하는 데 사용 하는 외부 도메인 클라이언트의 이름을 입력 하 고 **추가**클릭 합니다.

    3.  **대체 이름**, **형식**에서 **DNS**를 클릭 합니다.

    4.  **대체 이름** **값**모든 이름을 클라이언트 vpn.contoso.com, vpn, 132.64.86.2 예를 들어, VPN에 연결 하는 데 사용할 서버를 입력 합니다.

    5.  각 이름을 입력 한 후 **추가** 클릭 합니다.

    6.  작업을 마쳤으면 **확인**을 클릭합니다.

7.  **등록**을 클릭합니다.

8.  **마침**을 클릭합니다.

9.  인증서 스냅인에서 **개인**, 아래에서 **인증서**를 클릭 합니다.<p>나열 된 인증서 세부 정보 창에 표시 됩니다.

10. VPN 서버의 이름을 가진 인증서를 마우스 오른쪽 단추로 클릭 하 고 **열기**를 클릭 합니다.

11. **일반** 탭에는 **유효 기간** 아래에 나열 된 날짜는 오늘 날짜를 확인 합니다. 열려 있지 않다면 잘못 된 인증서를 선택한 수 있습니다.

12. **세부 정보** 탭에서 **확장 된 키 사용**클릭 하 고 **IP 보안 중간 IKE** 및 **서버 인증이** 목록에 표시 되는지 확인 합니다.

13. 인증서를 닫으려면 **확인** 클릭 합니다.

14. 인증서 스냅인을 닫습니다.

### NPS 서버 인증서의 유효성을 검사합니다

1.  NPS 서버를 다시 시작 합니다.

2.  NPS 서버 시작 메뉴에서 **certlm.msc**입력 하 고 Enter 키를 누릅니다.

3.  인증서 스냅인에서 **개인**, 아래에서 **인증서**를 클릭 합니다.<p>나열 된 인증서 세부 정보 창에 표시 됩니다.

5.  **일반** 탭에는 **유효 기간** 아래에 나열 된 날짜는 오늘 날짜를 확인 합니다. 열려 있지 않다면 잘못 된 인증서를 선택한 수 있습니다.

6.  인증서를 닫으려면 **확인** 클릭 합니다.

7.  인증서 스냅인을 닫습니다.
