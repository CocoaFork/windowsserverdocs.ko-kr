# <a name="step-2-configure-the-server-infrastructure"></a>2단계. Server 인프라 구성

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows 10

&#171;  [**이전:** 1단계. Always On VPN 배포 계획](always-on-vpn-deploy-planning.md)<br>
&#187;  [ **Next:** 3단계. VPN에 대 한 원격 액세스 서버를 항상 구성](vpn-deploy-ras.md)


이 단계에서는 설치 및 VPN을 지 원하는 데 필요한 서버 측 구성 요소를 구성 합니다. 서버 쪽 구성 요소 사용자, VPN 서버와 NPS 서버에서 사용 된 인증서를 배포 하는 PKI를 구성 하는 포함 됩니다.  RRAS의 IKEv2 연결 및 VPN 연결에 대 한 권한 부여를 수행 하려면 NPS 서버를 지원 하도록 구성할 수도 있습니다.

## <a name="configure-certificate-autoenrollment-in-group-policy"></a>그룹 정책에서 인증서 자동 등록 구성
이 절차에서는 사용자 및 컴퓨터 인증서를 자동으로 요청 하는 도메인 구성원이 되도록 도메인 컨트롤러에서 그룹 정책을 구성 합니다. 이렇게 VPN 사용자에 게를 요청 하 고 자동으로 VPN 연결을 인증 하는 사용자 인증서를 검색할 수 있습니다. 마찬가지로,이 정책을 사용 하면 서버를 요청 하려면 NPS 서버 인증 인증서를 자동으로. 

수동으로 VPN 서버에서 인증서를 등록 합니다.

>[!TIP]
>Domained 조인 된 컴퓨터에 대 한 참조 [비-도메인에 대 한 CA 구성에 컴퓨터 가입](#ca-configuration-for-non-domain-joined-computers) 아래. RRAS 서버 도메인에 가입 된 아니므로 VPN 게이트웨이에 인증서를 등록 하려면 자동 등록을 사용할 수 없습니다.  따라서 오프 라인 인증서 요청 프로시저를 사용 합니다. 


1.  도메인 컨트롤러에서 그룹 정책 관리를 엽니다.

2.  탐색 창에서 도메인 (예: corp.contoso.com)을 마우스 오른쪽 단추로 클릭 하 고 클릭 **이 도메인에서 GPO를 만들어 여기에 연결**합니다.

3.  새 GPO 대화 상자에서 입력 **자동 등록 정책**, 클릭 **확인**합니다.

4.  탐색 창에서 마우스 오른쪽 단추로 클릭 **자동 등록 정책**, 클릭 **편집**합니다.

5.  그룹 정책 관리 편집기에서 컴퓨터 인증서 자동 등록을 구성 하려면 다음 단계를 완료 합니다.

    1.  탐색 창에서 클릭 **컴퓨터 구성\\정책을\\Windows 설정\\보안 설정\\공개 키 정책**합니다.

    2.  세부 정보 창에서 마우스 오른쪽 단추로 클릭 **인증서 서비스 클라이언트 – 자동 등록**, 클릭 **속성**합니다.

    3.  인증서 서비스 클라이언트 – 자동 등록 속성 대화 상자의 **구성 모델**, 클릭 **Enabled**합니다.

    4.  선택 **해지 된 인증서 제거 및 만료 된 인증서를 보류 중인 인증서 업데이트 갱신** 하 고 **인증서 템플릿을 사용 하는 인증서 업데이트**합니다.

    5.  클릭 **확인**합니다.

6.  그룹 정책 관리 편집기에서 사용자 인증서 자동 등록 구성 하려면 다음 단계를 완료 합니다.

    1.  탐색 창에서 클릭 **사용자 구성\\정책을\\Windows 설정\\보안 설정\\공개 키 정책**합니다.

    2.  세부 정보 창에서 마우스 오른쪽 단추로 클릭 **인증서 서비스 클라이언트 – 자동 등록**, 클릭 **속성**합니다.

    3.  인증서 서비스 클라이언트 – 자동 등록 속성 대화 상자의 **구성 모델**, 클릭 **Enabled**합니다.

    4.  선택 **해지 된 인증서 제거 및 만료 된 인증서를 보류 중인 인증서 업데이트 갱신** 하 고 **인증서 템플릿을 사용 하는 인증서 업데이트**합니다.

    5.  클릭 **확인**합니다.

    6.  그룹 정책 관리 편집기를 닫습니다.

7.  그룹 정책 관리를 닫습니다.

### <a name="ca-configuration-for-non-domain-joined-computers"></a>비-도메인에 가입 된 컴퓨터에 대 한 CA 구성

RRAS 서버 도메인에 가입 된 아니므로 VPN 게이트웨이에 인증서를 등록 하려면 자동 등록을 사용할 수 없습니다.  따라서 오프 라인 인증서 요청 프로시저를 사용 합니다. 

1. RRAS 서버의 라는 파일을 생성 **VPNGateway.inf** (섹션 0) 부록에서에 제공 된 예제에서는 인증서 정책 요청을 기반으로 하며 다음 항목을 사용자 지정 합니다. 

   - [새 요청] 섹션에서는 대체 vpn.contoso.com 선택한을 사용 하 여 주체 이름에 사용 되는 [_고객_] VPN 끝점 FQDN입니다.

   - [확장] 섹션에서는 대체 vpn.contoso.com 선택한을 사용 하 여 주체 대체 이름에 사용 되는 [_고객_] VPN 끝점 FQDN입니다.

2. 저장 하거나 복사 합니다 **VPNGateway.inf** 파일을 선택한 위치에 있습니다.

3. 관리자 권한 명령 프롬프트에서 포함 된 폴더로 이동 합니다 **VPNGateway.inf** 파일 및 형식:

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

4. 새로 생성 된 복사 **VPNGateway.req** 는 인증 기관 서버 또는 워크스테이션 PAW (Privileged Access)를 출력 파일입니다. 

5. 저장 하거나 복사 합니다 **VPNGateway.req** 인증 기관 서버 또는 워크스테이션 PAW (Privileged Access)에서 선택한 위치에 파일입니다.

6. 관리자 권한 명령 프롬프트에서 입력 하 고 이전 단계에서 만든 VPNGateway.req 파일이 포함 된 폴더로 이동 합니다. 

   ```
   certreq -attrib “CertificateTemplate:[Customer]VPNGateway” -submit VPNgateway.req VPNgateway.cer
   ```

7. 인증 기관 목록 창에서 메시지가 표시 되 면 인증서 요청을 적절 한 엔터프라이즈 CA를 선택 합니다.

8. 새로 생성 된 복사 **VPNGateway.cer** RRAS 서버에 대 한 출력 파일입니다. 

9. 저장 하거나 복사 합니다 **VPNGateway.cer** RRAS 서버의 선택한 위치에는 파일입니다.

10. 관리자 권한 명령 프롬프트에서 입력 하 고 이전 단계에서 만든 VPNGateway.cer 파일이 포함 된 폴더로 이동 합니다.
   
   ```
   certreq -accept VPNGateway.cer
   ```

11. 설명 된 대로 인증서 MMC 스냅인을 실행할 [여기](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in) 를 선택 하는 **컴퓨터 계정** 옵션입니다.

12. 다음 속성을 사용 하 여 RRAS 서버에 대 한 유효한 인증서가 있는지 확인 합니다.

   - **용도:** IP 보안 IKE 중개 서버 인증 

   - **인증서 템플릿의 경우:** [_고객_] VPN 서버

#### <a name="example-vpngatewayinf-script"></a>예: VPNGateway.inf 스크립트

여기 대역의 프로세스를 사용 하 여 VPN 게이트웨이 인증서를 요청 하는 데 사용 하는 인증서 요청 정책의 예제 스크립트를 볼 수 있습니다.

>[!TIP]
>인증서 요청 정책 폴더 아래에 있는 VPN 제공 IP 키트에서 VPNGateway.inf 스크립트 복사본을 찾을 수 있습니다. 업데이트 '제목' 및 '\_계속\_' 고객 특정 값을 사용 하 여 합니다.

```
[Version] 

Signature="$Windows NT$"

[NewRequest]
Subject = "CN=vpn.contoso.com"
Exportable = FALSE   
KeyLength = 2048     
KeySpec = 1          
KeyUsage = 0xA0      
MachineKeySet = True
ProviderName = "Microsoft RSA SChannel Cryptographic Provider"
RequestType = PKCS10 

[Extensions]
2.5.29.17 = "{text}"
_continue_ = "dns=vpn.contoso.com&"

```

## <a name="create-the-vpn-users-vpn-servers-and-nps-servers-groups"></a>VPN 사용자, VPN 서버 및 NPS 서버 그룹 만들기

이 절차에서는 조직 네트워크에 연결 하려면 VPN을 사용 하도록 허용 된 사용자를 포함 하는 새 Active Directory (AD) 그룹을 추가할 수 있습니다.

이 그룹은 두 가지 용도로 사용 됩니다.

-   VPN 필요 사용자 인증서에 대 한 자동으로 등록할 수 있는 사용자를 정의 합니다.

-   NPS는 VPN 액세스 권한을 부여 하는 사용자를 정의 합니다.

사용자 지정 그룹을 사용 하 여 사용자의 VPN 액세스를 취소 하려는 경우 제거할 수 있습니다 해당 사용자 그룹에서.

VPN 서버를 포함 하는 그룹 및 NPS 서버를 포함 하는 다른 그룹 추가할 수도 있습니다. 해당 멤버에 인증서 요청을 제한 하려면 이러한 그룹을 사용 합니다.

>[!NOTE] 
>VPN 좋습니다 DMA/경계에 있는 서버를 도메인에 가입 된를 수 없습니다. 그러나 향상 된 관리 효율성에 대 한 도메인에 가입 된 VPN 서버를 원하는 경우 (그룹 정책, 백업을 모니터링 에이전트 없는 로컬 사용자를 관리 및 등) AD 그룹을 VPN 서버의 인증서 템플릿에 다음 추가 합니다.

### <a name="configure-the-vpn-users-group"></a>VPN 사용자 그룹 구성

1.  도메인 컨트롤러에서 Active Directory 사용자 및 컴퓨터를 엽니다.

2.  컨테이너 또는 조직 구성 단위를 마우스 오른쪽 단추로 클릭, 클릭 **새로 만들기** 누릅니다 **그룹**합니다.

3.  **그룹 이름**, 형식 **VPN 사용자**를 클릭 하 고 **확인**합니다.

4.  마우스 오른쪽 단추로 클릭 **VPN 사용자**, 클릭 **속성**합니다.

5.  에 **멤버** 탭 VPN 사용자 속성 대화 상자, 클릭 **추가**합니다.

6.  사용자 선택 대화 상자를 추가 하 고 VPN 액세스 해야 하는 모든 사용자 **확인**합니다.

7.  Active Directory 사용자 및 컴퓨터를 닫습니다.

### <a name="configure-the-vpn-servers-and-nps-servers-groups"></a>VPN 서버와 NPS 서버 그룹 구성

1.  도메인 컨트롤러에서 Active Directory 사용자 및 컴퓨터를 엽니다.

2.  컨테이너 또는 조직 구성 단위를 마우스 오른쪽 단추로 클릭, 클릭 **새로 만들기** 누릅니다 **그룹**합니다.

3.  **그룹 이름**, 형식 **VPN 서버**를 클릭 하 고 **확인**합니다.

4.  마우스 오른쪽 단추로 클릭 **VPN 서버**, 클릭 **속성**합니다.

5.  에 **멤버** 탭 VPN 서버 속성 대화 상자, 클릭 **추가**합니다.

6.  클릭 **개체 유형**를 선택 합니다 **컴퓨터** 확인란을 클릭 **확인**합니다.

7.  **선택할 개체 이름을 입력 하십시오**VPN 서버의 이름을 입력 하 고 클릭 **확인**합니다.

8.  클릭 **확인** VPN 서버 속성 대화 상자를 닫습니다.

9.  NPS 서버 그룹에 대해 이전 단계를 반복 합니다.

10. Active Directory 사용자 및 컴퓨터를 닫습니다.

## <a name="create-the-user-authentication-template"></a>사용자 인증 템플릿 만들기

이 절차에서는 사용자 지정 클라이언트-서버 인증 템플릿을 구성할 수 있습니다. 이 서식 파일은 업그레이드 된 호환성 수준을 선택 하 여 인증서의 전반적인 보안을 개선 하려고 하기 때문에 필요 하 고 Microsoft 플랫폼 암호화 공급자를 선택 합니다. 이 마지막 변경을 사용 하면 클라이언트 컴퓨터에서 TPM을 사용 하 여 인증서를 보호할 수 있습니다. TPM의 개요를 보려면 [신뢰할 수 있는 플랫폼 모듈 기술 개요](https://docs.microsoft.com/windows/device-security/tpm/trusted-platform-module-overview)합니다.

**절차:**

1.  CA에서 인증 기관을 엽니다.

2.  탐색 창에서 마우스 오른쪽 단추로 클릭 **인증서 템플릿**, 클릭 **관리**합니다.

3.  인증서 템플릿 콘솔에서 마우스 오른쪽 단추로 클릭 **사용자**, 클릭 **템플릿 복제**합니다.

    >[!WARNING]
    >클릭 하지 마세요 **적용** 하거나 **확인** 언제 든 지 10 단계 전에 합니다.  모든 매개 변수를 입력 하기 전에 이러한 단추를 클릭 하면 고정 하 고 더 이상 편집할 수 많은 선택 됩니다. 예를 들어, 합니다 **암호화** 탭 _레거시 암호화 저장소 공급자_ 방지 추가적인 변경, 비활성화 됩니다 공급자 범주 필드를 보여 줍니다. 대체 방법은 템플릿을 삭제 하 고 다시 만드십시오입니다.  

4.  새 템플릿의 속성 대화 상자에서에 **일반적인** 탭에서 다음 단계를 완료 합니다.

    1.  **템플릿 표시 이름**, 형식 **VPN 사용자 인증**합니다.

    2.  선택을 취소 합니다 **Active Directory에 인증서 게시** 확인란 합니다.

5.  에 **Security** 탭에서 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다.

    2.  사용자 선택, 컴퓨터, 서비스 계정 또는 그룹 대화 상자에서 입력 **VPN 사용자**, 클릭 **확인**합니다.

    3.  **그룹 또는 사용자 이름**, 클릭 **VPN 사용자**합니다.

    4.  **VPN 사용자에 대 한 권한을**를 선택 합니다 **등록** 및 **자동 등록** 확인란을 **허용** 열입니다.

       >[!TIP]
       >보관 읽기 확인란을 선택 해야 합니다. 즉, 등록에 대 한 읽기 권한이 필요 합니다. 

    5.  **그룹 또는 사용자 이름**, 클릭 **Domain Users**, 클릭 **제거**합니다.

6.  에 **호환성** 탭에서 다음 단계를 완료 합니다.

    1.  **인증 기관**, 클릭 **Windows Server 2012 R2**합니다.

## <a name="create-the-vpn-server-authentication-template"></a>VPN 서버 인증 템플릿 만들기

이 절차에서는 VPN 서버에 대 한 새 서버 인증 템플릿을 구성할 수 있습니다. 확장 된 키 사용 (IPsec) IP 보안 IKE 중간 응용 프로그램 정책을 둘 이상의 인증서가 서버 인증을 사용 하 여 사용 가능한 필터 인증서에 서버를 사용 하면 추가 합니다.

>[!IMPORTANT] 
>VPN 클라이언트에 공용 인터넷에서이 서버에 액세스 하기 때문에 대체 이름과 주체는 내부 서버 이름을 다릅니다. 결과적으로, 먼저 자동 VPN 서버에서이 인증서입니다.

**사전 요구 사항:**<p>
도메인에 가입 된 VPN 서버

**절차:** 

1.  CA에서 인증 기관을 엽니다.

2.  탐색 창에서 마우스 오른쪽 단추로 클릭 **인증서 템플릿**, 클릭 **관리**합니다.

3.  인증서 템플릿 콘솔에서 마우스 오른쪽 단추로 클릭 **RAS 및 IAS 서버**, 클릭 **템플릿 복제**합니다.

4.  새 템플릿의 속성 대화 상자에 **일반** 탭의 **템플릿 표시 이름**, 예를 들어 VPN 서버에 대 한 설명이 포함 된 이름을 입력 **VPN 서버 인증** 나 **RADIUS 서버**합니다.

5.  에 **확장** 탭에서 다음 단계를 완료 합니다.

    1.  클릭 **응용 프로그램 정책**, 클릭 **편집**합니다.

    2.  에 **응용 프로그램 정책 확장 편집** 대화 상자, 클릭 **추가**합니다.

    3.  에 **응용 프로그램 정책 추가** 대화 상자, 클릭 **IP 보안 IKE 중개**, 클릭 **확인**합니다.<p>추가 IP 보안 IKE 중개 EKU를 VPN 서버에서 서버 인증 인증서를 둘 이상 있는 시나리오 도움이 됩니다. IP 보안 IKE 중개 존재 하는 경우 IPSec EKU 옵션을 모두 사용 하 여 인증서만 사용 합니다. 이렇게 하지 않으면 IKEv2 인증 13801 오류로 실패할 수 있습니다. IKE 인증 자격 증명 허용 되지 않습니다.

    4.  클릭 **확인** 돌아가려면 합니다 **새 템플릿의 속성** 대화 상자.

6.  에 **Security** 탭에서 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다.

    2.  에 **사용자, 컴퓨터, 서비스 계정 또는 그룹 선택** 대화 상자에서 **VPN 서버**를 클릭 하 고 **확인**합니다.

    3.  **그룹 또는 사용자 이름**, 클릭 **VPN 서버**합니다.

    4.  **VPN 서버에 대 한 권한을**를 선택 합니다 **등록** 확인란 합니다 **허용** 열.

    5.  **그룹 또는 사용자 이름**, 클릭 **RAS and IAS Servers**를 클릭 하 고 **제거**합니다.

7.  에 **주체 이름** 탭에서 다음 단계를 완료 합니다.

    1.  클릭 **요청에서 제공**합니다.

    2.  에 **인증서 템플릿** 경고 대화 상자, 클릭 **확인**합니다.

8.  (선택 사항) *VPN 연결에 대 한 조건부 액세스를 구성 하는 경우*를 클릭 합니다 **요청 처리** 탭을 클릭 **개인 키를 내보낼 수 있음** 하 여 선택 합니다.

9.  클릭 **확인** VPN 서버 인증서 템플릿을 저장 합니다.

10. 인증서 템플릿 콘솔을 닫습니다.

11. 인증 기관 스냅인의 탐색 창에서 마우스 오른쪽 단추로 클릭 **인증서 템플릿**, 클릭 **새로 만들기** 을 클릭 한 다음 **발급할 인증서 템플릿**합니다.

12. 인증 기관 서비스를 다시 시작 합니다.

13. 위의 4 단계에서 선택한 이름을 선택 하 고 클릭 **확인**합니다.

14. 인증 기관 스냅인을 닫습니다.

## <a name="enroll-and-validate-the-user-certificate"></a>등록 하 고 사용자 인증서의 유효성을 검사합니다

그룹 정책을 자동으로 등록 사용자 인증서를 사용 하는 때문에 정책 에서만 업데이트 해야 하 고 Windows 10에는 올바른 인증서에 대 한 사용자 계정을 자동으로 등록 됩니다. 인증서 콘솔에서 인증서의 유효성을 검사할 수 있습니다.

**절차:**

1.  멤버인 도메인에 가입 된 클라이언트 컴퓨터에 로그인 합니다 **VPN 사용자** 그룹입니다.

2.  Windows 키 + R, 형식 **gpupdate /force**, Enter 키를 누릅니다.

3.  시작 메뉴에서 입력 **certmgr.msc**, Enter 키를 누릅니다.

4.  인증서 스냅인에서 아래의 **개인**, 클릭 **인증서**합니다. 인증서 세부 정보 창에 나타납니다.

5.  현재 도메인 사용자 이름을 가진 인증서를 마우스 오른쪽 단추로 누른 **열려**합니다.

6.  에 **일반적인** 탭에서 아래 나열 된 날짜를 확인 **유효** 오늘 날짜입니다. 그렇지 않은 경우 잘못 된 인증서를 선택 했습니다 수 있습니다.

7.  클릭 **확인**를 하 고 인증서 스냅인을 닫습니다.

## <a name="enroll-and-validate-the-server-certificates"></a>등록 하 고 서버 인증서의 유효성 검사

사용자 인증서와는 달리 VPN 서버의 인증서를 수동으로 등록 해야 합니다. 등록 한 후 사용자 인증서에 사용한 동일한 프로세스를 사용 하 여 확인 합니다. 사용자 인증서와 같은 NPS 서버에 자동으로 등록 해당 인증 인증서를 유효성 검사는 수행 해야 합니다.

>[!NOTE] 
>다음이 단계를 완료 하기 전에 해당 그룹 멤버 자격을 업데이트할 수 있도록 하려면 VPN 및 NPS 서버를 다시 시작 해야 합니다.

### <a name="enroll-and-validate-the-vpn-server-certificate"></a>등록 하 고 VPN 서버 인증서 유효성 검사

1.  VPN 서버의 시작 메뉴에서 입력 **certlm.msc**, Enter 키를 누릅니다.

2.  마우스 오른쪽 단추로 클릭 **개인**, 클릭 **모든 작업** 을 클릭 한 다음 **새 인증서 요청** 인증서 등록 마법사를 시작 합니다.

3.  시작 하기 전에 페이지에서 클릭 **다음**합니다.

4.  인증서 등록 정책 선택 페이지에서 클릭 **다음**합니다.

5.  인증서 요청 페이지에서 선택 하도록 VPN 서버 옆의 확인란을 클릭 합니다.

6.  VPN 서버 확인란 클릭 **자세한 내용이 필요한** 인증서 속성 대화 상자를 열고 다음 단계를 완료 하려면:

    1.  클릭 합니다 **주체** 탭을 클릭 **일반 이름** 아래에 있는 **주체 이름**의 **형식**합니다.

    2.  아래 **주체 이름**의 **값**, VPN, vpn.contoso.com 예를 들어에 연결 하 고 클릭 하는 데 외부 도메인 클라이언트의 이름을 입력 **추가**합니다.

    3.  아래 **대체 이름**의 **유형**, 클릭 **DNS**합니다.

    4.  아래 **대체 이름**의 **값**, 예를 들어, VPN에 연결할 클라이언트를 사용 하는 서버 이름, vpn.contoso.com, vpn, 132.64.86.2 모두 입력 합니다.

    5.  클릭 **추가** 후 각 이름을 입력 합니다.

    6.  작업을 마쳤으면 **확인**을 클릭합니다.

7.  **등록**을 클릭합니다.

8.  **마침**을 클릭합니다.

9.  인증서 스냅인에서 아래의 **개인**, 클릭 **인증서**합니다.<p>나열 된 인증서 세부 정보 창에 나타납니다.

10. VPN 서버의 이름을 지정 하 고 클릭 하는 인증서를 마우스 오른쪽 단추로 클릭 **열려**합니다.

11. 에 **일반적인** 탭에서 아래 나열 된 날짜를 확인 **유효** 오늘 날짜입니다. 그렇지 않은 경우 잘못 된 인증서를 선택 했습니다 수 있습니다.

12. 에 **세부 정보** 탭을 클릭 **향상 된 키 용도**, 확인 **IP 보안 IKE 중개** 및 **Server 인증** 목록에 표시 합니다.

13. 클릭 **확인** 를 인증서를 닫습니다.

14. 인증서 스냅인을 닫습니다.

### <a name="validate-the-nps-server-certificate"></a>NPS 서버 인증서 유효성 검사

1.  NPS 서버를 다시 시작 합니다.

2.  NPS 서버의 [시작] 메뉴에서 입력 **certlm.msc**, Enter 키를 누릅니다.

3.  인증서 스냅인에서 아래의 **개인**, 클릭 **인증서**합니다.<p>나열 된 인증서 세부 정보 창에 나타납니다.

4.  NPS 서버의 이름을 지정 하 고 클릭 하는 인증서를 마우스 오른쪽 단추로 클릭 **열려**합니다.

5.  에 **일반적인** 탭에서 아래 나열 된 날짜를 확인 **유효** 오늘 날짜입니다. 그렇지 않은 경우 잘못 된 인증서를 선택 했습니다 수 있습니다.

6.  클릭 **확인** 를 인증서를 닫습니다.

7.  인증서 스냅인을 닫습니다.
