# 3단계. Always On VPN에 대한 원격 액세스 서버 구성

>적용 대상: Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **이전:** 2 단계입니다. 서버 인프라 구성](vpn-deploy-server-infrastructure.md)<br>
& #187;  [ **이전:** 4 단계. 설치 및 구성 (NPS (네트워크 정책 서버)](vpn-deploy-nps.md)


RRAS 다양 한 기능을 지원 하기 때문에 라우터 및 원격 액세스 서버 모두 수행 하도록 설계 되었습니다. 이러한 기능 중 일부만이 배포를 위해 필요한: IKEv2 VPN 연결 및 LAN 라우팅에 대 한 지원 합니다.

IKEv2 VPN 터널링 프로토콜에 대 한 설명을 7296 Internet Engineering Task Force\ 요청에 설명 된입니다. IKEv2의 기본 장점은에서 내부 네트워크 연결 중단을가 것입니다. 예를 들어 연결이 일시적으로 중단 또는 사용자가 다른 네트워크에서 클라이언트 컴퓨터를 이동 하는 경우 IKEv2 자동으로 복원 VPN 연결 네트워크 연결을 다시 설정할 때-사용자 개입 없이 모든 합니다.

서버 보안 공간을 줄이는 비활성화 사용 하지 않는 프로토콜 하는 동안 IKEv2 연결을 지원 하도록 RRAS 서버를 구성 합니다. 또한 VPN 클라이언트에 고정 주소 풀에서 주소를 할당 하는 서버를 구성 합니다. 풀 또는 DHCP 서버에서 실행 주소를 할당할 수 있습니다. 하지만 DHCP 서버를 사용 하 여 디자인 복잡해 및 최소한의 혜택을 제공 합니다.


>[!Important]
>것이 중요 합니다.
>- 물리적 서버에 두 개의 이더넷 네트워크 어댑터를 설치 합니다. 두 개의 외부 가상 스위치를 각 실제 네트워크 어댑터;에 대 한 만들어야 VM에서 VPN 서버를 설치 하는 경우 한 다음 한 가상 스위치에 연결 된 각 네트워크 어댑터를 사용 하 여 VM에 대 한 두 개의 가상 네트워크 어댑터를 만듭니다.
>
>- 외부 경계 네트워크에 연결 된 네트워크 어댑터와 내부 경계 네트워크에 연결 된 네트워크 어댑터를 사용 하 여 가장자리와 내부 방화벽 사이의 경계 네트워크에서 서버를 설치 합니다.


>[!Warning]
>시작 하기 전에 VPN 서버에서 i p v 6를 사용 하도록 설정 해야 합니다. 그렇지 않으면 연결을 설정할 수 및 오류 메시지가 표시 됩니다.

## 원격 액세스 RAS 게이트웨이 VPN 서버 설치

이 절차에서는 단일 테 넌 트 RAS 게이트웨이 VPN 서버 원격 액세스 역할을 설치 합니다. 자세한 내용은 [원격 액세스](../../../Remote-Access.md)를 참조하세요.


### Windows PowerShell을 사용 하 여 원격 액세스 역할을 설치 합니다.

1. **관리자 권한**으로 Windows PowerShell을 엽니다.

2. 다음 명령을 입력 하 고 **enter**키를 누릅니다.

   `Install-WindowsFeature DirectAccess-VPN -IncludeManagementTools`

   설치가 완료 되 면 Windows PowerShell에서 다음과 같은 메시지가 표시 됩니다.
    
   | 성공 | 다시 시작 필요 | 종료 코드 | 기능 결과                             |
   |---------|----------------|-----------|--------------------------------------------|
   | True    | 아니요             | 성공   | {RAS 연결 관리자 관리 키트 |
   ---

### 서버 관리자를 사용 하 여 원격 액세스 역할을 설치 합니다.

서버 관리자를 사용 하 여 원격 액세스 역할을 설치 하려면 다음 절차를 사용할 수 있습니다.

1.  서버 관리자에서 VPN 서버에서 **관리** 를 클릭 하 고 **역할 및 기능 추가**클릭 합니다. <p>추가 역할 및 기능 마법사가 열립니다.

2.  앞에서 시작 페이지를**다음**을 클릭 합니다.

3.  설치 유형 선택 페이지에서 **역할 기반 또는 기능 기반 설치** 옵션을 선택 하 고 **** 을 클릭 합니다.

4.  Select destination server 페이지에서 **서버 풀에서 서버 선택** 옵션을 선택 합니다.

5.  서버 풀에서 로컬 컴퓨터를 선택 하 고 **** 을 클릭 합니다.

6.  서버 선택 역할 페이지 **역할**을 **원격 액세스**및 다음 **다음**을 클릭 합니다.

7.  기능 선택 페이지에서 **다음**을 클릭 합니다.

8.  원격 액세스 페이지에서 **다음**을 클릭 합니다.

9.  Select 역할 서비스 페이지 **역할 서비스****DirectAccess 및 VPN (RAS)를**클릭 합니다.<p>**추가 역할 및 기능 마법사** 대화 상자가 열립니다.

10. 추가 역할 및 기능 대화 상자에서 **추가 기능** 을 클릭 하 고 **** 을 클릭 합니다.

11. 웹 서버 역할 (IIS) 페이지에서 **다음**을 클릭 합니다.

12. Select 역할 서비스 페이지에서 **다음**을 클릭 합니다.

13. 확인 설치 선택 페이지에서 선택 내용을 검토 하 고 **설치**를 클릭 합니다.

14. 설치가 완료 되 면 **닫기**를 클릭 합니다.

## 원격 액세스를 VPN 서버로 구성

이 섹션에서는 원격 액세스 VPN IKEv2 VPN 연결을 허용 하 고 다른 VPN 프로토콜에서 연결을 거부 승인 된 VPN 클라이언트를 연결 하는 데 발급의 IP 주소에 대 한 고정 IP 주소 풀 할당을 구성할 수 있습니다.

1.  서버 관리자에서 VPN 서버에서 **알림** 플래그를 클릭 합니다.

2.  **작업** 메뉴에서 **시작 마법사를 열**을 클릭 합니다.<p>원격 액세스 구성 마법사가 열립니다. 

    >[!NOTE] 
    >원격 액세스 구성 마법사 뒤 서버 관리자를 열 수 있습니다. 생각 되 면 마법사를 열고, 이동 또는 그 뒤 마법사가 있는지 확인 하려면 서버 관리자를 최소화 하는 데 너무 오래 걸립니다. 그렇지 않은 경우에 초기화 하 고 마법사를 기다립니다.

3.  **VPN에만 배포**를 클릭 합니다.<p>라우팅 및 원격 액세스 Microsoft Management Console (MMC)가 열립니다.

4.  VPN 서버를 마우스 오른쪽 단추로 클릭 하 고 **구성 하 고 사용 라우팅 및 원격 액세스를**클릭 합니다.<p>라우팅 및 원격 액세스 서버 설치 마법사가 열립니다.

5.  라우팅 및 원격 액세스 서버 설치 마법사를 시작에서 **다음**을 클릭 합니다.

6.  **구성**에서 **사용자 지정 구성**를 클릭 하 고 **** 을 클릭 합니다.

7.  **사용자 지정 구성**에서 **VPN 액세스**클릭 하 고 **** 을 클릭 합니다.<p>완료 라우팅 및 원격 액세스 서버 설치 마법사가 열립니다.

8.  마법사를 닫으면 **완료** 를 클릭 하 고 **확인** 을 클릭 라우팅 및 원격 액세스 대화 상자를 닫습니다.

9.  원격 액세스를 시작 하려면 **서비스 시작** 을 클릭 합니다.

10. 원격 액세스 MMC에서 VPN 서버를 마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다.

11. 속성에서 **보안** 탭을 클릭 하 고 작업을 수행 합니다.

    a. **인증 공급자** 를 클릭 하 고 **RADIUS 인증**을 클릭 합니다.
    
    b. **구성**을 클릭합니다.<p>RADIUS 인증 대화 상자가 열립니다.
    
    c. **추가**를 클릭합니다.<p>RADIUS 서버 추가 대화 상자가 열립니다.
    
    d. **서버 이름**조직/회사 네트워크에서 완벽 하 게 정규화 된 도메인 이름 (FQDN) NPS 서버를 입력 합니다.<p>예를 들어 NPS 서버 NetBIOS 이름은 NPS1 하는 경우 도메인 이름이 corp.contoso.com **NPS1.corp.contoso.com**를 입력 합니다.
    
    e. **공유 암호** **변경**를 클릭 합니다.<p>암호 변경 대화 상자가 열립니다.
    
    f. **새 암호**텍스트 문자열을 입력 합니다.
    
    g. **새 암호 확인**동일한 텍스트 문자열을 입력 하 고 **확인**을 클릭 합니다.

    >[!IMPORTANT] 
    >이 텍스트 문자열을 저장 합니다. 조직/회사 네트워크에서 NPS 서버를 구성할 때 RADIUS 클라이언트이 VPN 서버를 추가 합니다. 이 구성에서는 하는 동안 NPS와 VPN 서버 통신할 수 있도록이 동일한 공유 암호를 사용 합니다.

12. **RADIUS 서버 추가**에서에 대 한 기본 설정을 검토 합니다.

    - **시간 제한**
    
    - **초기 점수**
    
    - **포트**

13. 필요한 경우 사용자 환경에 대 한 요구 사항을 충족 하 고 **확인**을 클릭 값을 변경 합니다.<p>NAS 어느 정도 더 큰 네트워크에 대 한 액세스를 제공 하는 장치입니다. RADIUS 인프라를 사용 하 여 NAS RADIUS 클라이언트를 인증, 권한 부여를 위해 RADIUS 서버로 연결 요청 및 계정 메시지 보내기 및 계정 이기도 합니다.

14. **계정 공급자에**대 한 설정을 검토 합니다.

    | 원하는 경우 합니다.  | 발생하는 결과             |
    |---------------------|-------------------|
    | 원격 액세스 서버에 기록 된 원격 액세스 동작 | **Windows 계정** 선택 되어 있는지 확인 합니다.      |
    | VPN에 대 한 계정 서비스를 수행 하는 NPS   | **RADIUS 계정** 으로 **계정 공급자** 를 변경 하 고 계정 공급자로 NPS를 구성 합니다. |
    ---

15. **IPv4** 탭을 클릭 하 고 실행 합니다.

    a. **고정 주소 풀**을 클릭 합니다.
    
    b. IP 주소 풀을 구성 하려면 **추가** 클릭 합니다.<p>고정 주소 풀에서 내부 경계 네트워크 주소를 포함 해야 합니다. 이러한 주소가 회사 네트워크에 있지 VPN 서버의 내부용 네트워크 연결에 있습니다.
    
    c. **시작 IP 주소**VPN 클라이언트를 할당 하고자 하는 범위에 대 한 시작 IP 주소를 입력 합니다.
    
    d. **끝 IP 주소**, **주소 개수**또는 VPN 클라이언트를 할당 하고자 하는 범위에서 종료 IP 주소를 입력, 사용할 수 있도록 하려면 주소를 입력 합니다. 이 서브넷에 DHCP를 사용 하는 경우 DHCP 서버에서 해당 주소 제외를 구성 해야 합니다.
    
    e. (선택 사항) DHCP를 사용 하는 경우 **어댑터**클릭 하 고 결과의 목록에서 내부 경계 네트워크에 연결 된 이더넷 어댑터를 클릭 합니다.

16. (선택 사항) *VPN 연결에 대 한 조건부 액세스를 구성 하는 경우*, **SSL 인증서 바인딩**에서 **인증서** 드롭다운 목록에서 VPN 서버 인증을 선택 합니다.

17. (선택 사항) *VPN 연결에 대 한 조건부 액세스를 구성 하는 경우*에서 NPS MMC **Policies\\Network 정책** 을 확장 하 고 실행 합니다. 

    a. 오른쪽에서 **Microsoft 라우팅 및 원격 액세스 서버에 연결할** 네트워크 정책 및 **속성**선택 합니다.
    
    b. 액세스 권한 부여 **를 선택 합니다. 연결 요청이 정책에이와 일치 하는 경우 액세스 권한을 부여** 옵션입니다.
    
    c. 네트워크 액세스 서버의 유형에 드롭다운 목록에서 **원격 액세스 서버 (VPN 전화 접속)를** 선택 합니다.

3.  라우팅 및 원격 액세스 MMC에서 **포트** 를 마우스 오른쪽 단추로 클릭 하 고 **속성**을 클릭 합니다. <p>포트 속성 대화 상자가 열립니다.

4.  **WAN 미니 포트 (SSTP)를** 클릭 하 고 **구성**합니다. 구성 장치-WAN 미니 포트 (SSTP) 대화 상자가 열립니다.

    a. **원격 액세스 연결 (인바운드)** 및 **접속 라우팅 연결 (인바운드 및 아웃 바운드)** 확인란을 선택 취소 합니다.
    
    b. **확인**을 클릭합니다.

5.  **WAN 미니 포트 (L2TP)을** 클릭 하 고 **구성**합니다. 구성 장치-WAN 미니 포트 (L2TP) 대화 상자가 열립니다.

    a. **최대 포트**지 원하는 동시 VPN 연결의 최대 수와 일치 하도록 포트의 수를 입력 합니다.
    
    b. **확인**을 클릭합니다.

6.  **WAN 미니 포트 (PPTP)를** 클릭 하 고 **구성**합니다. 구성 장치-WAN 미니 포트 (PPTP) 대화 상자가 열립니다.

    a. **최대 포트**지 원하는 동시 VPN 연결의 최대 수와 일치 하도록 포트의 수를 입력 합니다.
    
    b. **확인**을 클릭합니다.
    
7. **WAN 미니 포트 (IKEv2)를** 클릭 하 고 **구성**합니다. 구성 장치-WAN 미니 포트 (IKEv2) 대화 상자가 열립니다.

    a. **최대 포트**지 원하는 동시 VPN 연결의 최대 수와 일치 하도록 포트의 수를 입력 합니다.
    
    b. **확인**을 클릭합니다.

7.  메시지가 나타나면 **예** 서버를 다시 시작을 확인 하려면 클릭 하 고 **닫기** 서버를 다시 시작을 클릭 합니다.
