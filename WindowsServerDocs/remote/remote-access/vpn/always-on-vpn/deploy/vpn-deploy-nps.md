# 4단계. 설치 및 구성 (NPS (네트워크 정책 서버)

>   적용 대상: Windows Server 2019, Windows Server (반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows 10


& #171;  [ **다음:** 3 단계입니다. VPN에 대 한 원격 액세스 서버를 항상 구성](vpn-deploy-ras.md)<br>
& #187;  [ **다음:** 5 단계입니다. DNS 및 방화벽 설정 구성](vpn-deploy-dns-firewall.md)


이 단계에서는 VPN 서버에서 전송 되는 연결 요청 처리에 대 한 네트워크 정책 서버 (NPS)를 설치 합니다.

- 사용자에 연결할 수 있는 권한이 있는지 확인 하는 인증을 수행 합니다.
- 사용자의 id를 확인 하는 인증을 수행 합니다.
- NPS에서 RADIUS 계정을 구성할 때 선택한 연결 요청의 측면을 기록 하는 계정을 수행 합니다.

이 섹션의 단계를 사용 하면 다음 항목을 완료할 수 있습니다.

1.  컴퓨터 또는 NPS 서버에 대해 계획 하 고 조직이 나 회사 네트워크에 설치 된 VM에서 NPS를 설치할 수 있습니다.

   >[!TIP] 
   >네트워크에서 하나 이상의 NPS 서버 이미 있는 경우 NPS 서버 설치를 수행할 필요가 없습니다-대신,이 항목에서는 기존 NPS 서버 구성을 업데이트를 사용할 수 있습니다.

>[!NOTE]  
Windows Server Core에서 네트워크 정책 서버 서비스를 설치할 수 없습니다.

2.  조직/회사 NPS 서버에서 VPN 서버에서 받은 연결 요청을 처리 하는 RADIUS 서버로 수행 하는 NPS를 구성할 수 있습니다.

## NPS(네트워크 정책 서버) 설치

이 절차에서는 Windows PowerShell 또는 서버 관리자 추가 역할 및 기능 마법사 중 하나를 사용 하 여 NPS를 설치 합니다. NPS는 네트워크 정책 및 액세스 서비스 서버 역할의 역할 서비스입니다.

>[!TIP] 
>기본적으로 NPS RADIUS 포트 1812, 1813, 1645 및 1646 모든 설치 된 네트워크 어댑터에 트래픽을 수신합니다. NPS 설치 고급 보안이 포함 된 Windows 방화벽을 사용 하는 경우 이러한 포트에 대 한 방화벽 예외 IPv4 및 IPv6 트래픽에 대 한 자동으로 생성 가져옵니다. 이러한 기본값 이외의 포트를 통해 RADIUS 트래픽을 보낼 네트워크 액세스 서버 구성 된 경우 고급 보안이 포함 된 Windows 방화벽에서 NPS 설치 하는 동안 만든 예외를 제거 하 고 포트에 대 한 사용에 대 한 예외 RADIUS 트래픽에 합니다.

**Windows PowerShell 용 절차:**

Windows PowerShell 관리자 권한으로 실행 되는 Windows PowerShell을 사용 하 여이 절차를 수행 하려면 다음 명령을 입력 하 고 ENTER 키를 누릅니다.

`Install-WindowsFeature NPAS -IncludeManagementTools`

**서버 관리자를 위한 절차:**

1.  서버 관리자에서 **관리**클릭 하 고 **역할 및 기능 추가**클릭 합니다. <p>추가 역할 및 기능 마법사가 열립니다.

2.  시작 하기 전에서 **다음**을 클릭 합니다.

    >[!NOTE] 
    >추가 역할 및 기능 마법사를 실행 하는 경우 이전에 **기본적으로이 페이지 건너뛰기** 을 선택한 경우에 추가 역할 및 기능 마법사의 **시작 하기 전에** 페이지가 표시 되지 않습니다.

1.  설치 유형 선택 **역할 기반 또는 기능 기반 설치** 를 선택한 **다음**을 클릭을 확인 합니다.

2.  선택 대상 서버에서 **서버 풀에서 서버 선택** 선택 되어 있는지 확인 합니다.

3.  서버 풀에서 로컬 컴퓨터를 선택 했는지 확인 하 고 **** 을 클릭 합니다.

4.  서버 역할 선택의 **역할**을 **네트워크 정책 및 액세스 서비스를**선택 합니다. 네트워크 정책 및 액세스 서비스에 필요한 기능을 추가 해야 하는 경우를 묻는 대화 상자가 열립니다.

5.  **추가 기능**을 클릭합니다 하 고 **** 를 클릭

6.  선택 기능에서 **다음**을 클릭 하 고 네트워크 정책 및 액세스 서비스에서 제공 된 정보를 검토 하 고 **** 을 클릭 합니다.

7.  선택 역할 서비스에서 **네트워크 정책 서버**를 클릭 합니다.

8.  네트워크 정책 서버에 필요한 기능에 대 한 **기능 추가** 클릭 하 고 **** 을 클릭 합니다.

9.  확인 설치 선택에서 **요구 하는 경우에 자동으로 대상 서버를 다시 시작**을 클릭 합니다.

10. **예** 를 선택, 클릭 하 고 **설치**를 클릭 합니다. <p>설치 진행률 페이지 설치 프로세스 중 상태를 표시합니다. 프로세스가 완료 되 면 메시지 "설치 성공 *컴퓨터 이름*에" 표시 됩니다 *ComputerName* 설치한 네트워크 정책 서버는 컴퓨터의 이름입니다.

11. **닫기**를 클릭합니다.

## NPS 구성

VPN 서버에서 모든 인증, 권한 부여를 처리 하는 NPS 구성 NPS를 설치 하 고 계정 의무 연결에 대 한 요청을 받습니다.

### Active Directory에서 NPS 서버 등록

이 절차에서는 연결 요청을 처리 하는 동안 사용자 계정 정보에 액세스할 수 있는 권한을 갖도록 Active Directory에 서버를 등록 합니다.

**절차:**

1.  서버 관리자에서 **도구**클릭 하 고 **네트워크 정책 서버**를 차례로 클릭 합니다. NPS 콘솔을 엽니다.

2.  NPS 콘솔에서 **NPS (로컬)** 마우스 오른쪽 단추로 클릭 하 고 선택 **서버 Active Directory에 등록** 을 클릭 합니다.<p>네트워크 정책 서버 대화 상자가 열립니다.

3.  네트워크 정책 서버 대화 상자에서 **확인** 을 두 번 클릭 합니다.

NPS 등록의 대체 방법에 대 한 [Active Directory 도메인에서 NPS 서버 등록](../../../../../networking/technologies/nps/nps-manage-register.md)을 참조 하세요.

### 네트워크 정책 서버 계정 구성

이 절차에서는 네트워크 정책 서버 계정 다음 로깅 유형 중 하나를 사용 하 여 구성 합니다.

-   **이벤트 로깅**. 감사 및 연결 시도 문제 해결에 주로 사용 합니다. NPS 이벤트 로깅 NPS 콘솔에서 NPS 서버 속성을 가져오는 방법으로 구성할 수 있습니다.

-   **사용자 인증 및 계정 요청 로컬 파일에 기록**합니다. 연결 분석과 청구 목적으로 주로 사용 됩니다. 또한 공격을 받은 후 악의적인 사용자의 작업을 추적 하는 방법을 제공 하기 때문에 보안 조사 도구로 사용 합니다. 로컬 파일 로깅 계정 구성 마법사를 사용 하 여 구성할 수 있습니다.

-   **사용자 인증 및 Microsoft SQL Server XML 호환 데이터베이스에 대 한 계정 요청에 기록**합니다. NPS 데이터 원본을 실행 하는 여러 서버를 허용 하는 데 사용 합니다. 또한 관계형 데이터베이스를 사용 하 여의 이점을 제공 합니다. SQL Server 로깅 계정 구성 마법사를 사용 하 여 구성할 수 있습니다.

네트워크 정책 서버 계정를 구성 하려면 [네트워크 정책 서버 계정 구성](../../../../../networking/technologies/nps/nps-accounting-configure.md)참조 하세요.

### RADIUS 클라이언트 VPN 서버 추가

[Always On VPN에 대 한 원격 액세스 서버 구성](vpn-deploy-ras.md) 섹션에서 설치 하 고 VPN 서버를 구성 합니다. VPN 서버 구성 하는 동안 VPN 서버에서 RADIUS 공유 암호를 추가 합니다. 

이 절차에서는 NPS에서 RADIUS 클라이언트 VPN 서버를 구성 하는 동일한 공유 비밀 텍스트 문자열을 사용 합니다. NPS 서버와 VPN 서버 간에 통신 실패 또는 VPN 서버에서 사용 되는 동일한 텍스트 문자열을 사용 합니다.

>[!IMPORTANT] 
>네트워크 (VPN 서버, 무선 액세스 지점, 인증 스위치 또는 전화 접속 서버) 새 네트워크 액세스 서버를 추가 하면 NPS 인식 하 고 네트워크 액세스 서버와 통신할 수 있도록 서버에서 NPS RADIUS 클라이언트로 추가 해야 합니다.

**절차:**

1.  NPS 콘솔에서 NPS 서버에서 **RADIUS 클라이언트 및 서버를**두 번 클릭 합니다.

2.  **RADIUS 클라이언트** 를 마우스 오른쪽 단추로 클릭 하 고 **새로 만들기**를 클릭 합니다. 새 RADIUS 클라이언트 대화 상자가 열립니다.

3.  **이 RADIUS 클라이언트 사용** 확인란이 선택 되어 있는지 확인 합니다.

4.  **이름**VPN 서버에 대 한 표시 이름을 입력 합니다.

5.  **주소 (IP 또는 DNS)** NAS IP 주소 또는 FQDN을 입력 합니다.<p>FQDN을 입력 하면 이름이 올바른지, 유효한 IP 주소를 확인 하려는 경우 **확인** 을 클릭 합니다.

6.  **공유 암호**다음을 수행 합니다.

    1.  **수동** 선택 되어 있는지 확인 합니다.

    2.  또한 VPN 서버에 입력 하는 강력한 텍스트 문자열을 입력 합니다.

    3.  공유 암호 확인에에서 공유 암호를 입력 합니다.

7.  **확인**을 클릭합니다. VPN 서버 RADIUS 클라이언트 NPS 서버에 구성 된 목록에 표시 됩니다.

## VPN 연결에 대 한로 반경 NPS 구성

이 절차에서는 조직 네트워크에서 RADIUS 서버로 NPS를 구성합니다. NPS에서 VPN 서버를 통해 조직/회사 네트워크에 액세스 하려면 특정 그룹에 속한 사용자만을 허용 하는 정책을 정의 해야 차례로 PEAP 인증 요청에 유효한 사용자 인증서를 사용 하는 경우에 합니다.

**절차:**

1.  표준 구성에서 NPS 콘솔에서 **전화 접속 또는 VPN 연결에 대 한 RADIUS 서버** 선택 되어 있는지 확인 합니다.

2.  **VPN 또는 전화 접속 구성**을 클릭 합니다.<p>VPN 구성 또는 전화 접속 마법사가 열립니다.

3.  **가상 개인 네트워크 (VPN) 연결**을 클릭 하 고 **** 을 클릭 합니다.

4.  지정 전화 접속 또는 RADIUS 클라이언트에서 VPN 서버에서 이전 단계에서 추가한 VPN 서버 이름을 선택 합니다.<p>예를 들어 VPN 서버 NetBIOS 이름을 RAS1 이면 **RAS1**를 클릭 합니다.

5.  **다음**을 클릭합니다.

6.  인증 방법 구성에서 다음 단계를 완료 합니다.

    1.  **Microsoft 암호화 인증 버전 2 (Ms-chapv2)** 확인란의 선택을 취소 합니다.

    2.  선택 **확장할 수 있는 인증 프로토콜** 확인란을 클릭 합니다.

    3.  유형 (액세스 및 네트워크 구성 방법에 따라)에서 클릭 **Microsoft: 보호 된 EAP (PEAP)**, **구성**을 클릭 합니다.<p>보호 된 EAP 속성 편집 대화 상자가 열립니다.

    4.  보안 암호 (EAP-있을 v2) EAP 형식을 제거 하려면 **제거** 를 클릭 합니다.

    5.  **추가**를 클릭합니다. EAP 추가 대화 상자가 열립니다.

    6.  **스마트 카드 또는 다른 인증서를**클릭 하 고 **확인**을 클릭 합니다.

    7.  보호 된 EAP 속성 편집 닫힐 때까지 **확인** 클릭 합니다.

7.  **다음**을 클릭합니다.

8.  사용자 지정 그룹의 다음 단계를 완료 합니다.

    1.  **추가**를 클릭합니다. 사용자, 컴퓨터, 서비스 계정 또는 그룹 선택 대화 상자가 열립니다.

    2.  **VPN 사용자** 입력 하 고 **확인**을 클릭 합니다.

    3.  **다음**을 클릭합니다.

9.  지정 IP 필터에서 **다음**을 클릭 합니다.

10. 암호화 설정 지정에서 **다음**을 클릭 합니다. 변경 하지 않습니다.<p>이러한 설정은이 시나리오를 지원 하지 않는 Microsoft 지점 간 암호화 (MPPE) 연결에만 적용 됩니다.

11. 영역 이름 지정에서 **다음**을 클릭 합니다.

12. 마법사를 닫습니다 **완료** 를 클릭 합니다.

## NPS 서버 인증서 자동 등록

이 절차에서는 하면 그룹 정책 새로 고침 로컬 NPS 서버에서 수동으로 합니다. 때 그룹 정책 새로 고침, 인증서 자동 등록 구성 되 고 로컬 컴퓨터는 제대로 작동 하는 경우 자동 등록 인증서 인증 기관 (CA).

>[!NOTE]  
>그룹 정책 도메인 구성원 컴퓨터를 다시 시작할 때 또는 사용자가 도메인 구성원 컴퓨터에 로그온 할 때 자동으로 새로. 또한 그룹 정책 정기적으로 새로 고칩니다. 기본적으로이 정기적으로 새로 고침 최대 30 분의 임의 오프셋을 사용 하 여 90 분 마다 발생합니다.

**관리자**또는 이와 동등한의 구성원 이어야가이 절차를 완료 하는 데 필요한 최소입니다.

**절차:**

1.  NPS, Windows PowerShell을 엽니다.

2.  Windows PowerShell 프롬프트에 **gpupdate**입력 하 고 enter 키를 누릅니다.
