# <a name="hyper-v-network-virtualization-overview-in-windows-server-2016"></a>Windows Server 2016에에서 hyper V 네트워크 가상화 개요

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

Windows Server 2016 및 관리자 가상 컴퓨터에서 Microsoft 종단 간 네트워크 가상화 솔루션을 제공합니다.  Microsoft의 네트워크 가상화 솔루션을 구성 하는 5 가지 주요 구성 요소 가지가 있습니다.  
  
-   **Windows Server에 대 한 Windows Azure 팩** 가상 네트워크 만드는 포털 및 가상 네트워크 관리를 관리 포털 향하도록 테 제공 합니다.  
  
-   **가상 컴퓨터 관리자** (VMM) 네트워크 패브릭 중앙 집중식된 관리 기능을 제공 합니다.  
  
-   **Microsoft 네트워크 컨트롤러** 관리, 구성 하 고, 모니터, 데이터 센터에 인프라 가상 및 실제 네트워크 문제를 해결 하려면 자동화의 중앙 집중식, 프로그래밍 포인트를 제공 합니다.  
  
-   **Hyper-v 네트워크 가상화** 교통 네트워크 가상화 하는 데 필요한 인프라를 제공 합니다.  
  
-   **Hyper-v 네트워크 가상화 게이트웨이** 사이의 물리적 및 가상 네트워크 연결을 제공 합니다.  
  
이 항목 개념을 소개 하 고 주요 이점 및 Windows Server 2016에 Hyper-v 네트워크 가상화 (전체 네트워크 가상화 솔루션 한 부분) 기능에 설명 합니다. 네트워크 가상화 두 개인 클라우드 서비스 (IaaS)로 엔터프라이즈 작업 통합 및 공공 클라우드 서비스 공급자의 인프라 찾고를 활용 하는 방법을 설명 합니다.

## <a name="BKMK_OVER"></a>기능 설명  
Hyper-v 네트워크 가상화 가상 컴퓨터 (하이퍼바이저) 서버 virtualization "가상 컴퓨터" 운영 체제를 제공 하는 방법을 유사한 "가상 네트워크" (VM 네트워크 라고도 함)를 제공 합니다. 네트워크 가상화 가상 네트워크 실제 네트워크 인프라를 분리 하 고 가상 컴퓨터 프로비저닝 VLAN 및 계층 IP 주소 할당 제한 제거 합니다. 이러한 유연성 IaaS 구름으로 이동 하는 고객을 위한 쉽고 효율적인 호스팅 업체와 데이터 센터 관리자 필요한 다중 테 격리, 보안 요구 사항을 관리 하는 동안 인프라를 관리할 수 있으며 지원 겹치는 가상 컴퓨터의 IP 주소 있습니다.  
  
고객은 클라우드로 자녀의 데이터 센터를 확장 원활 하 게 하고자 합니다. 오늘 이러한 원활 하 게 하이브리드 클라우드 아키텍처 만드는에 기술 문제가 있습니다. 가장 큰 문제 고객 얼굴 중 하나는 자녀의 기존 네트워크 토폴로지 (서브넷, IP 주소, 네트워크 서비스 및 등) 클라우드에서 다시 사용 하 고 클라우드 리소스의 온-프레미스 리소스와 브리지.  Hyper-v 네트워크 가상화 기본 실제 네트워크의 독립적인 VM 네트워크의 개념을 제공 합니다. 하나 이상의 가상 서브넷 이루어져 VM 네트워크에 대 한이 개념을 가상 네트워크에 연결 된 모든 가상 컴퓨터의 실제 네트워크의 정확한 위치 가상 네트워크 토폴로지에서 분리 됩니다. 결과적으로, 고객을 쉽게 이동할 수의 가상 서브넷 클라우드 기존 서비스 계속 작업할 수 있도록 자녀의 기존 IP 주소와 토폴로지 클라우드에 유지 하는 동안 서브넷의 실제 위치를 인식 하지 못하는 합니다. 즉, Hyper-v 네트워크 가상화 원활 하 게 하이브리드 클라우드를 수 있습니다.  
  
하이브리드 클라우드 뿐 아니라 많은 조직 데이터 센터 통합 하 고 개인 구름 클라우드 아키텍처의 효율성 및 확장성 혜택을 내부적으로 만드는 됩니다. Hyper-v 네트워크 가상화 사용할 수 있게 유연성과 효율성 더 나은 (으로 만들어 가상) 비즈니스 장치의 분리 하 여 개인 구름 네트워크 토폴로지 실제 실제 네트워크 토폴로지에서 합니다. 이런 방법으로 부서 수 쉽게 서로 격리 되는 동안 내부 개인 클라우드 공유를 계속 기존 네트워크 토폴로지 유지 합니다. Datacenter 운영 팀은에 배포 및 datacenter server 중단 운영 효율성 더 제공 하지 않고 전체 더 효율적 datacenter에서 아무 곳 이나 작업 동적으로 이동 하는 유연성입니다.  
  
소유자의 작업에 대 한 주요 이점은 이동할 수 있는 이제 작업 "토폴로지" 로드 클라우드에 IP 주소를 변경 하거나 해당 응용 프로그램을 다시 작성 않고입니다. 예를 들어, 일반적인 3 티어 LOB 응용 프로그램 전면 종료 계층, 비즈니스 논리가 계층 및 데이터베이스 계층 구성 됩니다. 정책을 통해 Hyper-v 네트워크 가상화 수 있게 고객의 세 가지 계층 클라우드에, 일부 또는 모든 온 보 딩 라우팅 토폴로지와 서비스 (예: 가상 컴퓨터 IP 주소)를의 IP 주소를 유지 하는 동안 응용 프로그램을 변경할 수를 않고도 합니다.  
  
Infrastructure 소유자의 가상 컴퓨터 위치에서 추가 유연성에서는 변경 가상 컴퓨터 또는 네트워크 재구성 않고 아무 곳 이나 데이터 센터의 작업을 이동할 수 있습니다. 예를 들어 네트워크 가상화 Hyper-v 가상 컴퓨터 서비스가 중단 없이 데이터 센터에 아무 곳 이나 마이그레이션할 라이브 수 있도록 크로스 서브넷 실시간 마이그레이션이 있습니다. 이전에 실시간 마이그레이션 가상 컴퓨터 찾을 수 있는 제한 같은 서브넷 수 있었습니다. 서브넷 크로스 실시간 마이그레이션 관리자 에너지 효율성 동적 리소스 요구 사항에 따라 작업 통합할 수 있습니다 및 고객 작업 속도 방해 하지 않고 infrastructure 유지 관리를 사용할 수도 있습니다.  
  
## <a name="BKMK_APP"></a>실용적인 응용 프로그램  
가상화 데이터 센터의 성공 여부, 사용 IT 조직 및 호스팅 공급자 (공급자에 게 배치 또는 실제 서버 대여가 제공) 하기 시작 주문형 server 인스턴스를 고객에 게 제공 하기 쉽게 해 주는 더 유연한 가상화 인프라를 제공 합니다. 이 새로운 종류 서비스의 Infrastructure as a Service (IaaS) 라고 부릅니다. Windows Server 2016 기업 고객이 개인 구름 및 IT 서비스 운영 모델로 전환 빌드를 사용 하도록 모두 필요한 플랫폼 기능을 제공 합니다. Windows Server 2016 2016 공개 구름 빌드를 고객에 게 IaaS 솔루션을 제공 하는 호스팅 업체를 해줍니다. 가상 컴퓨터 관리자와 Windows Azure 팩 Hyper-v 네트워크 가상화 정책을 관리를 함께 사용 하면 Microsoft는 강력한 클라우드 솔루션을 제공 합니다.  
  
Windows Server 2016 Hyper-v 네트워크 가상화 전용된 IaaS 구름을 확장 하 고 제공 클라우드 호스팅 업체 더 나은 유연성과 확장성 높은 리소스 사용량 달성 하기 위해 가상 컴퓨터 관리 하기 위한 때 기업에서 오버 얼굴을 한 관리 감소 정책 기반, 소프트웨어 제어 네트워크 가상화를 제공 합니다.  
  
가상 컴퓨터에서 다른 조직의 부문 (전용된 클라우드) 또는 다른 고객 (호스팅된 클라우드)가 IaaS 시나리오 보안 격리가 필요 합니다. 오늘 솔루션을 가상 영역 로컬 네트워크 (Vlan)이이 시나리오에서 중요 한 단점 제공할 수 있습니다.  
  
**Vlan**  
  
현재 Vlan 대부분의 조직 격리 테 넌 트 및 주소 공간 다시 사용할 수 있도록 지 원하는 데 사용 하는 장치는 합니다. VLAN는 이더넷 프레임 머리글에서 명시적 태그 (VLAN ID)을 사용 하 고 이더넷 스위치 격리를 유지 하 고 교통 동일한 VLAN id 노드가 네트워크에 대 한 제한를 사용 하 여 주요 단점 Vlan와은 다음과 같습니다.  
  
-   가상 컴퓨터 또는 격리 경계 다음 동적 데이터 센터에 이동 프로덕션 스위치 번거롭게 재구성 인해 실수로 중단 위험이 증가 합니다.  
  
-   4094 Vlan 최대 고 일반적인 스위치를 더 이상 1000 VLAN Id 지원 때문에 확장성에서 제한 됩니다.  
  
-   단일 IP 서브넷 단일 VLAN 내 노드의 수를 제한 하는 실제 위치에 따라 가상 컴퓨터의 배치를 제한 내에서 제한 됩니다. 사이트에서 Vlan 확장 가능 경우에 전체 VLAN 같은 서브넷 켜져 있어야 합니다.  
  
**IP 주소를 지정**  
  
가상 컴퓨터의 IP 주소 할당 Vlan 하 여 표시 되는 단점은 외에도 포함 하는 문제를 제공 합니다.  
  
-   네트워크 인프라 데이터 센터의에서 실제 위치 가상 컴퓨터의 IP 주소를 확인합니다. 결과적으로, 일반적으로 이동 하는 클라우드 서비스 작업의 IP 주소를 변경 해야 합니다.  
  
-   정책은 방화벽 규칙, 리소스 검색 디렉터리 서비스 등의 IP 주소에 연결 됩니다. IP 주소를 변경 하려면 관련된 된 모든 정책 업데이트 해야 합니다.  
  
-   가상 컴퓨터 배포 및 교통 격리 토폴로지에 따라 달라 집니다.  
  
Datacenter 네트워크 관리자 계획 하는 데이터 센터의 실제 레이아웃이, 여기서 서브넷 물리적으로 저장 되며 전달에 대 한 결정 하 해야 합니다. 이러한 결정 지정된 서버에 데이터 센터에서 특정 랙에 연결 되어 있는 잎 실행 가상 컴퓨터에 사용할 수 있는 잠재적 IP 주소에 영향을 주는 IP 및 이더넷 기술을 기반으로 합니다. 준비 가상 컴퓨터 되는 데이터 센터에 배치 때, 이러한 옵션을 IP 주소에 대 한 제한 사항을 준수 해야 합니다. 따라서 일반 결과 datacenter 관리자 가상 컴퓨터에 새 IP 주소를 지정 합니다.  
  
이 요구 사항에 문제는 않도록 주소, IP 주소와 연관 된 시맨틱 정보입니다. 예를 들어 한 서브넷 제공 되는 서비스를 포함 하거나 고유한 물리적 위치에 있을 수 있습니다. 방화벽 규칙, 액세스 제어 정책 및 보안 연결을 IPsec IP 주소와 일반적으로 연결 됩니다. 가상 컴퓨터 소유자 원래 IP 주소를 기초로 된 하는 모든 취급 방침이 조정 되도록 IP 주소를 변경 합니다. 이 번호 다시 매기기 오버 많은 기업 그대로 레거시 응용 프로그램을 벗어나지 클라우드에만 새로운 서비스 배포 하기로 높은입니다.  
  
Hyper-v 네트워크 가상화 실제 네트워크 infrastructure에서 가상 컴퓨터 고객에 대해 가상 네트워크 분리 합니다. 결과적으로, 실제 IP 주소 또는 VLAN Id 재구성 없이 데이터 센터에 아무 곳 이나 가상 컴퓨터 고객을 제공 하기 위해 datacenter 관리자 허용 하면서도 원래 IP 주소를 유지 하기 위해 고객 가상 컴퓨터 수 있습니다. 다음 섹션의 핵심 기능 요약 되어 있습니다.  
  
## <a name="BKMK_NEW"></a>중요 한 기능  
다음은 핵심 기능, 혜택을 및 Windows Server 2016에 네트워크 가상화 Hyper-v의 기능 목록이.  
  
-   **유연한 작업 배치 네트워크 격리 및 IP 주소 Vlan 하지 않고 다시 사용 가능**  
  
    Hyper-v 네트워크 가상화 고객의 가상 네트워크에서 내 데이터 센터 작업 위치에 대 한 자유롭게 제공, 호스팅 업체의 실제 네트워크 인프라를 분리 합니다. 가상 컴퓨터 작업 배치 내 Hyper-v 호스트 소프트웨어 정의 넌 가상화 정책에 따라 적용 되므로 IP 주소를 지정 하거나 실제 네트워크 VLAN 격리 요구 하 여 더 이상 제한 됩니다.  
  
    이제 가상 컴퓨터 겹치는 IP 주소를 가진 다른 고객의 IP 주소 계층을 위반 또는 번거롭게 VLAN 구성 필요 없이 동일한 호스트 서버에 배포할 수 있습니다. 이 호스팅 공급자, 고객이 이러한 작업 수정 가상 컴퓨터 IP 주소를 변경 하지 않고 포함 되지 않은 상태로 이동할 수 있게 공유 IaaS에 고객 작업 마이그레이션을 간소화할 수 있습니다. 호스팅 공급자에 대 한 많은 고객에 게 공유 IaaS 데이터 센터에는 기존 네트워크 주소 공간 확장할 지원 구성 하 고 한 각 고객의 잠재적으로 주소 공간에는 겹치는 공존 수 있도록 격리 Vlan 유지의 복잡 한 운동입니다. Hyper-v 네트워크 가상화 겹치는 주소를 지 원하는 더 쉽게 및 덜 네트워크 재구성 호스팅 공급자가 필요 합니다.  
  
    또한 고객 작업의 아래쪽 시간을 않고도 물리적 infrastructure 유지 관리 하 고 업그레이드를 수행할 수 있습니다. Hyper-v 네트워크 가상화 특정 호스트, 랙, 서브넷, VLAN 전체 클러스터에서 가상 컴퓨터의 IP 주소 변경 또는 주요 재구성을 않고도 마이그레이션될 수 있습니다.  
  
-   **공유 IaaS 클라우드 작업에 대 한 수 있도록 쉽게 이동**  
  
    Hyper-v 네트워크 가상화 IP 주소와 가상 컴퓨터 구성 변경 되지 않습니다. 이 통해 보다 쉽게 작업 또는 인프라 도구 및 정책을 최소화 재구성와 공유 IaaS 호스팅 공급자에 게 데이터 센터에서 작업을 이동 하려면 IT 부서 합니다. 경우에 두 개의 데이터 센터 간의 연결이 여기서 IT 관리자를 계속 사용할 수 자신의 도구 없이 실행 합니다.  
  
-   **실시간 마이그레이션 서브넷에서**  
  
    가상 컴퓨터 작업의 실시간 마이그레이션 일반적으로 제한 되어 동일한 IP 서브넷 또는 VLAN 가상 컴퓨터의 IP 주소를 변경 하려면 게스트 운영 체제 서브넷 교차 필요 하기 때문에 있습니다. 이 주소 변경 기존 통신 끊어지고 방해 가상 컴퓨터에서 실행 되는 서비스입니다. Hyper-v 네트워크 가상화 작업 IP 주소를 변경 하지 않고 다른 서브넷에 Windows Server 2016을 실행 하는 서버에 한 서브넷 Windows Server 2016을 실행 하는 서버에서 마이그레이션 작업 라이브 될 수 있습니다. Hyper-v 네트워크 가상화 실시간 마이그레이션 인해 가상 컴퓨터 위치 변화 호스트 지속적인 통신 마이그레이션된 가상 컴퓨터에서 동기화 업데이트를 설치할 수 있도록 보장 합니다.  
  
-   **분리 서버 및 네트워크 관리를 쉽게 관리할 수**  
  
    마이그레이션 및 작업의 배치 기본 실제 네트워크 구성 독립적인 되므로 서버 작업 배치 간단해 집니다. 서비스 및 서버, 관리 서버 관리자 집중할 수 하 고 네트워크 관리자 전체 네트워크 인프라와 교통 관리에 집중할 수 있습니다. 배포 하 고 가상 컴퓨터 가상 컴퓨터의 IP 주소를 변경 하지 않고 마이그레이션하 datacenter server 관리자가 있습니다. 여기 감소 오버 Hyper-v 네트워크 가상화 가상 컴퓨터 위치에 관계 없이 네트워크 토폴로지 되려면 격리 경계를 변경할 수 있는 위치에 포함 네트워크 관리자에 대 한 필요성을 줄일 수 있기 때문에 합니다.  
  
-   **네트워크를 간소화 하 고 서버/네트워크 리소스 사용량 개선**  
  
    Vlan 흔들면서 및 실제 네트워크 인프라에 배치 가상 컴퓨터의 종속성 overprovisioning 및 사용률 발생합니다. 종속성을 분리 하 여 작업 배치 가상 컴퓨터의 증가 유연성 네트워크 관리를 간소화 하 고 서버 및 네트워크 리소스 사용량 개선 수 있습니다. 참고 Hyper-v 네트워크 가상화 실제 데이터 센터의 상황에 맞는의 Vlan를 지원 합니다. 예를 들어, 데이터 센터에 특정 VLAN 모든 Hyper-v 네트워크 가상화 교통을 않도록 할 수 있습니다.  
  
-   **기존 인프라와 신흥 기술과 호환**  
  
    데이터 센터 "평평한 네트워크" 기술 신흥와 호환 되는 아직 Hyper-v 네트워크 가상화 오늘 데이터 센터에 배포 될 수 있습니다.  
  
    예를 들어, Windows Server 2016에 HNV VXLAN 캡슐화 형식 및 지원 열기 vSwitch 데이터베이스 관리 프로토콜 (OVSDB)로 SouthBound 인터페이스 (SBI).  
  
-   **에코 시스템 및 상호 운용성 준비를 위한 제공**  
  
    Hyper-v 네트워크 가상화 프레미스 연결, 저장 영역 네트워크 (SAN), 가상화 되지 않은 리소스 액세스 등에 간와 같은 기존 리소스와 통신을 위해 여러 구성을 지원합니다. Microsoft 지원 하 고 향상 Hyper-v 네트워크 가상화 성능, 확장성, 및 관리 하는 경험이 에코 시스템 파트너와 협력 하 최선을 다하고 있습니다.  
  
-   **정책 기반 구성**  
  
    Windows Server 2016에 네트워크 가상화 정책은 Microsoft 네트워크 컨트롤러를 통해 구성 됩니다. 네트워크 컨트롤러에 RESTful northbound API를 Windows PowerShell 인터페이스 정책 구성 하 고 있습니다. Microsoft 네트워크 컨트롤러에 대 한 자세한 내용은 참조 [네트워크 컨트롤러](../../../sdn/technologies/network-controller/../../../sdn/technologies/network-controller/Network-Controller.md)합니다.
