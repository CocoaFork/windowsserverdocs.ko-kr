# <a name="manage-qos-policy"></a>QoS 정책 관리

>적용 대상: Windows Server (반기 채널), Windows Server 2016

QoS 정책 마법사를 사용 하 여 만들기, 편집 또는 QoS 정책을 삭제에 대해 자세히 알아보려면이 항목에서는 사용할 수 있습니다.

>[!NOTE]
>  이 항목 외에 다음 QoS 정책 관리 설명서는 사용할 수 있습니다.
> 
>  - [QoS 정책 이벤트 및 오류](qos-policy-errors.md)

Windows 운영 체제에서 QoS 정책을 그룹 정책의 관리 효율성을 사용 하 여 표준 기반 QoS의 기능을 결합 합니다. 이 조합의 구성 QoS 정책 쉽게 적용할 그룹 정책 개체를 만듭니다. Windows는 다음 작업을 수행할 수 있도록 QoS 정책 마법사를 포함 합니다.

-  [QoS 정책 만들기](#bkmk_createpolicy)

-  [보기, 편집 또는 QoS 정책을 삭제합니다](#bkmk_editpolicy)

##  <a name="bkmk_createpolicy"></a>QoS 정책 만들기

QoS 정책을 만들기 전에 네트워크 트래픽을 관리 하는 데 사용 되는 두 개의 키 QoS 컨트롤을 파악 하는 것:

- DSCP 값

-   스로틀 속도

### <a name="prioritizing-traffic-with-dscp"></a>DSCP 사용 하 여 트래픽 우선 순위 지정

이전 기간 업무 응용 프로그램 예제에서 설명한 대로, 사용 하 여 아웃 바운드 네트워크 트래픽의 우선 순위를 정의할 수 있습니다 **DSCP 값 지정** 특정 DSCP 값으로 QoS 정책을 구성 합니다. 

RFC 2474에 설명 된, DSCP 값을 0에서 63 IPv4 패킷의 TOS 필드 및 ipv6에서 트래픽 클래스 필드 내에서 지정할 수 있습니다. 네트워크 패킷을 분류 하 고 적절 하 게 대기 네트워크 라우터는 DSCP 값을 사용 합니다.
  
> [!NOTE]
>  기본적으로 Windows 트래픽을 DSCP 값이 0에 있습니다.
  
큐의 개수 및 우선 순위 지정 동작은 사용자 조직이 갖춘 QoS 전략의 일환으로 정해져야 합니다. 예를 들어, 조직 수도 큐가 5: 대기 시간이 중요 한 트래픽, 제어 트래픽, 비즈니스에 중요 한 트래픽, 최상의 노력이 드는 트래픽, 및 대량 데이터 전송 트래픽입니다.  
  
### <a name="throttling-traffic"></a>트래픽 제한

DSCP 값과 함께 제한는 네트워크 대역폭을 관리 하기 위한 또 다른 주요 컨트롤입니다. 앞에서 설명한 대로 사용할 수는 **스로틀 속도 지정** 아웃 바운드 트래픽에 대 한 특정 스로틀 속도 사용 하 여 QoS 정책을 구성 하는 설정입니다. 제한를 사용 하 여 QoS 정책은 나가는 네트워크 트래픽을 지정 된 스로틀 속도를 제한 합니다. 트래픽을 효과적으로 관리하기 위해 DSCP 표시와 제한을 함께 사용할 수도 있습니다.

>[!NOTE]
>기본적으로 **스로틀 속도 지정** 확인란은 선택되어 있지 않습니다.

QoS 정책을 만들려면의 그룹 정책 개체 (GPO)에서 그룹 정책 관리 콘솔 (GPMC) 도구 내에서 설정을 편집 합니다. 그런 다음 GPMC 그룹 정책 개체 편집기를 엽니다.

QoS 정책에는 고유한 이름을 지정해야 합니다. 서버 및 최종 사용자가 정책의 적용 방법 QoS 정책을 그룹 정책 개체 편집기에서 저장 위치에 따라 달라 집니다.

- 컴퓨터 구성 Settings\QoS 정책에서 QoS 정책은 현재 로그온 한 사용자에 관계 없이 컴퓨터에 적용 됩니다. 일반적으로 서버 컴퓨터에는 컴퓨터 기반 QoS 정책을 사용합니다.

- 사용자 구성 Settings\QoS 정책에서 QoS 정책 로그온을 컴퓨터에 관계 없이 로그온 한 후 사용자에 게 적용 됩니다.

#### <a name="to-create-a-new-qos-policy-with-the-qos-policy-wizard"></a>QoS 정책 마법사를 사용 하 여 새 QoS 정책을 만들려면

-   그룹 정책 개체 편집기에서 단추로 중 하나를 클릭 합니다 **QoS 정책을** 노드 및 클릭 한 다음 **새 정책 만들기**합니다.

### <a name="wizard-page-1---policy-profile"></a>마법사 페이지 1-정책 프로필

QoS 정책 마법사의 첫 번째 페이지에서 정책 이름을 지정 하 고 QoS 나가는 네트워크 트래픽을 제어 하는 방식을 구성할 수 있습니다.

#### <a name="to-configure-the-policy-profile-page-of-the-qos-based-policy-wizard"></a>QoS 기반 정책 마법사의 정책 프로필 페이지를 구성하려면

1. **정책 이름**에 QoS 정책의 이름을 입력합니다. 이름의 정책을 고유 하 게 식별 해야 합니다.

2. 필요에 따라 **DSCP 값 지정** DSCP 표시를 사용 하도록 설정 하 고 0에서 63 사이의 DSCP 값을 구성 합니다.

3. 선택적으로 트래픽 제한을 사용하고 스로틀 속도를 구성하는 데 **스로틀 속도 지정**을 사용할 수 있습니다. 스로틀 속도 값은 1 보다 커야 하 고 초당 킬로바이트 단위를 지정할 수 있습니다 \(KBps\) 또는 초당 메가바이트 \(MBps\)합니다.

4. **다음**을 클릭합니다.

### <a name="wizard-page-2---application-name"></a>마법사 페이지 2-응용 프로그램 이름

QoS 정책 마법사의 두 번째 페이지에서 해당 실행 파일 이름, 경로 및 응용 프로그램 이름, 또는 특정 URL에 대 한 요청을 처리 하는 HTTP 서버 응용 프로그램을 식별 한 대로 특정 응용 프로그램의 모든 응용 프로그램에 정책을 적용할 수 있습니다.

- **모든 응용 프로그램** 을 QoS 정책 마법사의 첫 번째 페이지의 트래픽 관리 설정이 모든 응용 프로그램에 적용 하도록 지정 합니다.

- **이 실행 파일 이름이 있는 응용 프로그램만** QoS 정책 마법사의 첫 번째 페이지의 트래픽 관리 설정이 특정 응용 프로그램에 대 한 지정 합니다. 실행 파일 이름은 확장명이.exe로 끝나야 합니다.

- **이 URL에 대 한 요청에 응답 HTTP 서버 응용 프로그램만** 을 QoS 정책 마법사의 첫 번째 페이지의 트래픽 관리 설정이 특정 HTTP 서버 응용 프로그램에만 적용 하도록 지정 합니다.

선택적으로 응용 프로그램의 경로를 입력할 수 있습니다. 응용 프로그램 경로를 지정하려면 경로에 응용 프로그램 이름을 포함하십시오. 경로 환경 변수를 포함할 수 있습니다. 경로는 예를 들어 %ProgramFiles%\My Application Path\MyApp.exe나 c:\program files\my application path\myapp.exe가 될 수 있습니다.

>[!NOTE]
>응용 프로그램 경로는 기호화 된 링크를 확인 하는 경로 포함할 수 없습니다.

URL을 따라야 합니다 [RFC 1738](https://tools.ietf.org/html/rfc1738), 형태로 `http[s]://<hostname\>:<port\>/<url-path>`합니다. 와일드 카드를 사용할 수 있습니다 `‘*’`에 대 한 `<hostname>` 및/또는 `<port>`예 `https://training.\*/, https://\*.\*`, 와일드 카드의 부분 문자열을 나타낼 수 없습니다 하지만 `<hostname>` 또는 `<port>`합니다.

즉, 둘 다 `https://my\*site/` 나 `https://\*training\*/` 유효 합니다. 

확인할 수 있습니다 **하위 디렉터리 및 파일 포함** 모든 하위 디렉터리 및 파일 URL 뒤에서 일치를 수행 합니다. 예를 들어가이 옵션을 선택 하 고 URL이 `https://training`에 QoS 정책에 대 한 요청을 고려 합니다` https://training/video` 적합 합니다.

#### <a name="to-configure-the-application-name-page-of-the-qos-policy-wizard"></a>QoS 정책 마법사의 응용 프로그램 이름 페이지를 구성 하려면

1. **이 QoS 정책이 적용 되**, 선택 **모든 응용 프로그램** 하거나 **이 실행 파일 이름이 있는 응용 프로그램만**합니다.

2. **이 실행 파일 이름이 있는 응용 프로그램만**을 선택한다면 .exe 파일 이름 확장명으로 끝나는 실행 파일 이름을 지정합니다.

3. **다음**을 클릭합니다.

### <a name="wizard-page-3---ip-addresses"></a>마법사 페이지 3-IP 주소

QoS 정책 마법사의 세 번째 페이지에서 다음을 포함 하는 QoS 정책에 대 한 IP 주소 조건을 지정할 수 있습니다.

- 모든 원본 IPv4 또는 IPv6 주소나 특정 원본 IPv4 또는 IPv6 주소

- 모든 대상 IPv4 또는 IPv6 주소나 특정 대상 IPv4 또는 IPv6 주소

**다음 원본 IP 주소에만** 또는 **다음 대상 IP 주소에만**을 선택하면 다음 중 한 항목을 입력해야 합니다.

- IPv4 주소와 같은 `192.168.1.1`

- 와 같은 네트워크 접두사 길이 표기법을 사용 하 여 IPv4 주소 접두사 `192.168.1.0/24`

- 와 같은 IPv6 주소 `3ffe:ffff::1`

- IPv6 주소 접두사와 같은 `3ffe:ffff::/48`

모두 선택 하면 **다음 원본 IP 주소에만** 하 고 **다음 대상 IP 주소에 대해서만**, 주소나 주소 접두사를 모두 하거나 IPv4 또는 IPv6 기반 이어야 합니다.

이전 마법사 페이지에서 HTTP 서버 응용 프로그램에 대 한 URL을 지정한 경우이 마법사 페이지에서 QoS 정책에 대 한 원본 IP 주소는 회색 알 수 있습니다. 

원본 IP 주소는 HTTP 서버 주소 및 여기는 구성 가능한 때문에 그렇습니다. 반면, 대상 IP 주소를 지정 하 여 여전히 정책을 지정할 수 있습니다. 이렇게 하면 동일한 HTTP 서버 응용 프로그램을 사용 하 여 다른 클라이언트에 대 한 다른 정책을 만들 수 있습니다.

#### <a name="to-configure-the-ip-addresses-page-of-the-qos-policy-wizard"></a>QoS 정책 마법사의 IP 주소 페이지를 구성 하려면

1. **이 QoS 정책이 적용 되** (원본) 선택 **모든 원본 IP 주소** 하거나 **다음 IP에 대해서만 원본 주소**합니다.

2. 선택한 경우 **다음 IP 원본 주소만**접두사 또는 IPv4 또는 IPv6 주소를 지정 합니다.

3. **이 QoS 정책이 적용 되** (대상) 선택 **모든 대상 주소** 또는 **다음 대상 IP 주소에 대해서만 합니다.**

4. 선택한 경우 **다음 대상 IP 주소에 대해서만**, IPv4 또는 IPv6 주소 또는 주소 유형에 해당 하는 접두사 또는 원본 주소에 지정 된 접두사를 지정 합니다.

5.  **다음**을 클릭합니다.  

### <a name="wizard-page-4---protocols-and-ports"></a>마법사 페이지 4-프로토콜 및 포트

QoS 정책 마법사의 네 번째 페이지에서 마법사의 첫 번째 페이지의 설정에 의해 제어 되는 포트 및 트래픽의 유형을 지정할 수 있습니다. 다음과 같이 지정할 수 있습니다.  
-   TCP 트래픽이나 UDP 트래픽(또는 둘 다)  

-   모든 원본 포트, 원본 포트의 범위, 특정 원본 포트

-   모든 대상 포트, 대상 포트 범위 또는 특정 대상 포트  

#### <a name="to-configure-the-protocols-and-ports-page-of-the-qos-policy-wizard"></a>QoS 정책 마법사의 프로토콜 및 포트 페이지를 구성 하려면

1. **이 QoS 정책이 적용되는 프로토콜 선택**에서 **TCP**, **UDP** 또는 **TCP 및 UDP**를 선택합니다.

2. **원본 포트 번호 지정**에서 **모든 원본 포트부터** 또는 **이 원본 포트 번호부터**를 선택합니다.

3. 선택한 경우 **이 원본 포트 번호에서**, 1에서 65535 사이의 포트 번호를 입력 합니다.

     형식의 포트 범위를 지정할 수는 필요에 따라 "*낮은*:*높은*," 위치 *낮은* 하 고 *높은* 하한값을 나타내는 및 상한 포트의 범위를 까지입니다. *낮은* 하 고 *높은* 각각 1에서 65535 사이의 숫자 여야 합니다. 콜론 (:) 문자와 숫자 사이에는 공백이 없어야 합니다.

4. **대상 포트 번호 지정**에서 **모든 대상 포트까지** 또는 **이 대상 포트 번호까지**를 선택합니다.

5. 이전 단계에서 **이 대상 포트 번호부터**를 선택한 경우 1에서 65535 사이의 숫자를 써서 포트 번호를 입력합니다.

새로운 QoS 정책 만들기를 완료 하려면 클릭 **완료** 에 **프로토콜 및 포트** QoS 정책 마법사의 페이지입니다. 완료 되 면 새 QoS 정책은 그룹 정책 개체 편집기의 세부 정보 창에 나열 됩니다.  
  
사용자 또는 컴퓨터에 QoS 정책 설정을 적용 하려면 QoS 정책이, 도메인, 사이트 또는 조직 구성 단위 (OU) 같은 Active Directory Domain Services 컨테이너에 배치 되는 GPO를 연결 합니다.  
  
##  <a name="bkmk_editpolicy"></a>보기, 편집 또는 QoS 정책을 삭제합니다

QoS 정책 마법사 설명 페이지를 보거나 정책의 속성을 편집할 때 표시 되는 속성 페이지에 이전에 해당 합니다.  
  
### <a name="to-view-the-properties-of-a-qos-policy"></a>QoS 정책의 속성을 보려면  
  
-   그룹 정책 개체 편집기의 세부 정보 창에서 정책 이름을 마우스 오른쪽 단추로 누른 **속성**합니다.  
  
     그룹 정책 개체 편집기는 다음과 같은 탭을 사용 하 여 속성 페이지를 표시합니다.  
  
    -   정책 프로필  
  
    -   Application Name  
  
    -   IP 주소

### <a name="BKMK_precedencerules"></a>QoS 정책 우선 순위 규칙

GPO의 우선 순위와 마찬가지로 QoS 정책을 트래픽의 특정 집합에 여러 QoS 정책을 적용 하는 경우 충돌을 해결할 우선 순위 규칙을 갖습니다. 아웃 바운드 TCP 또는 UDP 트래픽의 QoS 정책을 스로틀 속도 합계 하는 위치와 같은 누적 효과 없는 즉 한 번에 하나만 QoS 정책을 적용할 수 있습니다.

일반적으로 가장 일치 하는 조건 사용 하 여 QoS 정책이 적용 됩니다. 규칙은 세 가지 범주로 나뉩니다 여러 QoS 정책이 적용 되는 경우: 사용자 수준 및 컴퓨터 수준; 응용 프로그램 네트워크 다섯 번; 비교 및 네트워크 간에 quintuple입니다.

하 여 *네트워크 quintuple*, 원본 IP 주소, 대상 ip, 원본 포트, 대상 포트 및 프로토콜 의미 \(TCP/UDP\)합니다.  

 **사용자 수준 QoS 정책은 컴퓨터 수준 QoS 정책 보다 우선**

이 규칙에는 특히 사용자 그룹 기반 정책에 대 한 QoS Gpo의 네트워크 관리자의 관리를 크게 용이해 집니다. 예를 들어, 네트워크 관리자에서 사용자 그룹에 대 한 QoS 정책을 정의 하려는 경우 이러한 수을 만들고 해당 그룹에 GPO를 배포 합니다. 걱정할 필요가 있는 컴퓨터에 대 한 해당 사용자가 로그온 해 있고 여부를 해당 컴퓨터를 정의 하는 QoS 정책이 충돌 하는 충돌이 있는 경우 사용자 수준 정책이 항상 우선 하기 때문에.

> [!NOTE]
>  사용자 수준 QoS 정책을 해당 사용자가 생성 되는 트래픽에 적용 됩니다. 특정 컴퓨터 및 컴퓨터 자체의 다른 사용자는 해당 사용자에 대해 정의 된 모든 QoS 정책이 적용 되지 않습니다.

 **응용 프로그램 특수성 및 네트워크 다섯 번 우선 수행**

여러 QoS 정책을 특정 트래픽에 맞게에 구체적인 정책이 적용 됩니다. 응용 프로그램을 식별 하는 정책 중에서 보내는 응용 프로그램의 파일 경로 포함 하는 정책이 응용 프로그램 이름 (경로 없음)만 하 게 식별 하는 다른 정책 보다 더 구체적으로 간주 됩니다. 응용 프로그램을 사용 하 여 여러 정책을 적용 하는 경우 선행 규칙은 정확히 일치를 찾으려고 quintuple 네트워크를 사용 합니다.

또는 여러 QoS 정책을 겹치지 않는 조건을 지정 하 여 동일한 트래픽에 적용 될 수 있습니다. 응용 프로그램의 조건 사이의 네트워크 다섯 번 응용 프로그램을 지정 하는 정책을 더 구체적인 것으로 간주 됩니다 하 고 적용 됩니다. 

예를 들어 policy_A (app.exe) 응용 프로그램 이름을 지정 하 고 policy_B 대상 IP 주소 192.168.1.0/24를 지정 합니다. 이 QoS 정책이 충돌 하는 경우 \(app.exe 192.168.4.0/24 범위 내의 IP 주소에 트래픽을 보내는\), policy_A 적용 됩니다.

 **보다 구체적인 쿼리 지정 quintuple 네트워크 내에서 우선 순위 사용**

네트워크 다섯 번 내에서 정책 충돌에 대 한 가장 일치 하는 조건 사용 하 여 정책을 우선 적용 됩니다. 예를 들어 policy_C "모두", 대상 IP 주소 10.0.0.1 원본 IP 주소, 원본 포트 "any", 대상 포트 "모두", "TCP" 프로토콜을 지정 합니다. 

이번에 policy_D "any" 원본 IP 주소, 대상 IP 주소 10.0.0.1, 원본 포트 "모두", 대상 포트 80 및 "TCP" 프로토콜을 지정 합니다. 그런 다음 policy_C 및 policy_D 대상 10.0.0.1: 80에 대 한 연결을 일치합니다. QoS 정책을 가장 구체적인 일치 조건 사용 하 여 정책 적용, policy_D는이 예제의 우선 합니다.  
  
그러나 QoS 정책을 동일한 수의 조건 있을 수 있습니다. 예를 들어, 여러 정책을 지정할 수 있습니다 각 하나만 (하지만 동일 하지 않음) 네트워크 다섯 부분입니다. 네트워크 다섯 간에 다음 순서가 높은 항목에서 낮은 우선 순위에서:

- 원본 IP 주소

- 대상 IP 주소

- 원본 포트

- 대상 포트

- 프로토콜(TCP 또는 UDP)
