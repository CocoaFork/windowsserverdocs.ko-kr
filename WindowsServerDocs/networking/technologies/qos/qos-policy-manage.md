##  <a name="bkmk_createpolicy"></a>정책 QoS 만들기

QoS 정책을 만들기 전에 네트워크 교통 관리 하는 데 사용 하는 두 가지 주요 QoS 컨트롤을 파악 하는 다음과 같습니다.

- DSCP 값

-   속도 제한

### <a name="prioritizing-traffic-with-dscp"></a>DSCP와 함께 교통 체증 우선 순위를 지정

이전 줄 비즈니스 응용 프로그램 예에서 언급 했 듯이, 사용 하 여 네트워크 교통 아웃 바운드의 우선 순위를 정의할 수 있습니다 **DSCP 값 지정** DSCP 값을 QoS 정책을 구성할 수 있습니다. 

RFC 2474에 설명 된 대로 DSCP 0에서에서 값을 63 및 ipv6에서 교통 클래스 필드 내 IPv4 패킷 TOS 분야에서 지정할 수 있습니다. 네트워크 라우터가 네트워크 패킷을 분류 하 고을 적절 하 게 큐에 DSCP 값을 사용 합니다.
  
> [!NOTE]
>  기본적으로 Windows 교통에 0 DSCP 값 합니다.
  
QoS 전략 조직의의 일환으로 설계 큐과 우선 순위 지정 동작 수 있어야 합니다. 예를 들어 조직 5 큐 하도록 선택할 수 있습니다: 대기 민감한 교통, 컨트롤 교통, 업무용 교통, 최선을 교통 및 데이터 전송 대량 교통 합니다.  
  
### <a name="throttling-traffic"></a>교통 제한

DSCP 값 함께 제한 네트워크 대역폭 관리에 대 한 또 다른 키 컨트롤은 합니다. 앞에서 언급 한 것 처럼 사용할 수 있는 **조절 속도 지정** 설정을 QoS 정책을 아웃 바운드 교통에 대 한 특정 조절 속도와 구성에 합니다. 조절를 사용 하 여 QoS 정책 지정된 조절 속도를 거는 기능은 발신 네트워크 교통량을 제한 합니다. DSCP 표시 및 조절 교통 효과적으로 관리 함께 사용할 수 있습니다.

>[!NOTE]
>기본적으로는 **조절할 속도 지정** 확인란이 선택 되어 있지 합니다.

QoS 정책을 만들려면의 그룹 정책 개체 (GPO)에서 그룹 정책을 콘솔 GPMC (관리) 도구에서 설정을 편집 합니다. 그런 다음 GPMC 그룹 정책 개체 편집기를 엽니다.

QoS 정책 이름은 고유한 해야 합니다. 서버와 최종 사용자에 게 정책이 적용 되는 방식을 QoS 정책이 그룹 정책 개체 편집기에에서 저장 된 위치에 따라 다릅니다.

- 컴퓨터에 현재 로그온 한 사용자 관계 없이 컴퓨터 구성 Settings\QoS 정책에서 QoS 정책이 적용 됩니다. 일반적으로 컴퓨터 기반 QoS 정책을 서버 컴퓨터에 대 한 사용 합니다.

- QoS 정책 사용자 구성할 Settings\QoS 정책에 로그온 하기는 컴퓨터에 관계 없이에 로그인 한 후 사용자에 게 적용 됩니다.

### <a name="wizard-page-1---policy-profile"></a>마법사 페이지 1-정책 프로필

QoS 정책 마법사의 첫 번째 페이지에서 정책 이름을 지정 하 고 QoS 거는 기능은 발신 네트워크 교통량을 제어 하는 방법을 구성할 수 있습니다.

#### <a name="to-configure-the-policy-profile-page-of-the-qos-based-policy-wizard"></a>정책 QoS 기반 마법사의 정책 프로필 페이지 구성 하려면

1. **정책 이름**, QoS 정책에 대해 이름을 입력 합니다. 이름의 정책을 고유 하 게 식별 해야 합니다.

2. 필요에 따라 **DSCP 값 지정** DSCP 표시 활성화 하 고 다음 0 사이의 63 DSCP 값을 구성 합니다.

3. 필요에 따라 **조절 속도 지정** 교통 제한을 설정 하 고 조절 속도 구성 합니다. 조절 속도 값 1 보다 커야 및 단위 k b 두 번째 \(KBps\) / 또는 m b 두 번째 \(MBps\) / 지정할 수 있습니다.

4. 클릭 **다음**합니다.

### <a name="wizard-page-2---application-name"></a>마법사 페이지 2-응용 프로그램 이름

두 번째 페이지 QoS 정책 마법사의 경로를 실행 파일 이름 및 응용 프로그램 이름 또는 특정 URL에 대 한 요청을 처리 하는 HTTP 서버 응용 프로그램을 식별 정책을 특정 응용 프로그램의 모든 응용 프로그램에 적용할 수 있습니다.

- **응용 프로그램을 모두** 모든 응용 프로그램에 적용 QoS 정책 마법사의 첫 번째 페이지에서 교통 관리 설정을 지정 합니다.

- **응용 프로그램에만이 실행 되는 이름의** 특정 응용 프로그램에 대 한 첫 번째 페이지 QoS 정책 마법사의 트래픽 관리 설정 되어 있는지를 지정 합니다. 실행 파일 이름 지정 및.exe 파일 이름 확장명이 종료 해야 합니다.

- **HTTP 서버 응용 프로그램에만이 URL에 대 한 요청에 응답** 특정 HTTP 서버 응용 프로그램에만 적용 QoS 정책 마법사의 첫 번째 페이지에서 교통 관리 설정을 지정 합니다.

필요에 따라 응용 프로그램 경로 입력할 수 있습니다. 응용 프로그램 경로 지정 하려면 응용 프로그램의 이름 사용 하 여 경로 포함 합니다. 경로 환경 변수 포함 될 수 있습니다. 예를 들어, 응용 프로그램 Path\MyApp.exe %ProgramFiles%\My 하거나 c:\program files\my application path\myapp.exe 합니다.

>[!NOTE]
>응용 프로그램 경로 기호 링크를 확인 하는 경로로 포함할 수 없습니다.

URL을 따라야 합니다 [RFC 1738](http://tools.ietf.org/html/rfc1738), 형태의 `http[s]://<hostname\>:<port\>/<url-path>`합니다. 와일드 카드를 사용할 수 `‘*’`에 대 한 `<hostname>`및/또는 `<port>`, 예를 들어 `http://training.\*/, https://\*.\*`, 와일드 카드의 부분을 나타낼 수 없는 하지만 `<hostname>`또는 `<port>`합니다.

즉, 둘 다 `http://my\*site/`않으며 `https://\*training\*/`유효 합니다. 

확인할 수 있습니다 **닫은 다음 파일을 포함** 일치 하는 모든 하위 및 URL 다음 파일을 수행할 수 있습니다. 예를 들어가이 옵션을 선택 하 고 URL이 `http://training`, QoS 정책에 대 한 요청을 고려 합니다` http://training/video` 적합 합니다.

#### <a name="to-configure-the-application-name-page-of-the-qos-policy-wizard"></a>정책 QoS 마법사의 응용 프로그램 이름이 페이지 구성 하려면

2. 선택 하면 **응용 프로그램에만이 실행 되는 이름의**, 끝나는.exe 파일 이름 확장명은 실행 이름을 지정 합니다.

3. 클릭 **다음**합니다.

### <a name="wizard-page-3---ip-addresses"></a>마법사 페이지 3-IP 주소

QoS 정책 마법사의 세 번째 페이지에서 다음을 포함 하 여 QoS 정책에 대 한 IP 주소 조건 지정할 수 있습니다.

- 모든 소스 V4 또는 IPv6 주소 또는 특정 소스 IPv4 또는 IPv6 주소

- 모든 대상 V4 또는 IPv6 주소 또는 특정 대상 V4 또는 IPv6 주소

선택 하는 경우 **다음 소스 IP 주소에 대 한** 또는 **다음 대상 IP 주소에 대 한**에서 다음 중 하나를 입력 해야 합니다.

- IPv4 주소 `192.168.1.1`

- 와 같은 네트워크 접두사 길이 표기법을 사용 하는 IPv4 주소 접두사 `192.168.1.0/24`

- IPv6 주소 `3ffe:ffff::1`

- IPv6 주소 접두사와 같은 `3ffe:ffff::/48`

모두 선택 하는 경우 **다음 소스 IP 주소에 대 한** 및 **다음 대상 IP 주소에 대 한**, 주소 또는 주소 접두사 중 IPv4 또는 IPv6 기반 여야 합니다.

HTTP 서버 응용 프로그램에 대 한 URL 이전 마법사 페이지에 지정 하면 마법사가 페이지에 QoS 정책에 대 한 소스 IP 주소 회색 볼 수 있습니다. 

원본 IP 주소 HTTP 서버 주소 이며 여기는 구성할 수 하므로 마찬가지입니다. 반면에 대상 IP 주소를 지정 하 여 정책을 계속 지정할 수 있습니다. 이렇게 하면 다른 클라이언트에 대 한 다양 한 정책을 동일한 HTTP 서버 응용 프로그램을 사용 하 여 만들 수 있습니다.

#### <a name="to-configure-the-ip-addresses-page-of-the-qos-policy-wizard"></a>정책 QoS 마법사의 IP 주소 페이지 구성 하려면

1. **이 QoS 정책이 적용 되** (소스)를 선택 **출처 IP 주소는 하나** 또는 **다음 IP에 대해서만 원본 주소**합니다.

2. 선택한 경우 **다음 IP 소스 주소만**접두사 또는 IPv4 또는 IPv6 주소를 지정 합니다.

3. **이 QoS 정책이 적용 되** (대상) 선택 **대상 주소** 또는 **다음 IP 목적지 주소에 대해서만 합니다.**

4. 선택한 경우 **다음 IP 목적지 주소에 대 한**, V4 또는 IPv6 주소 또는 주소 유형을에 해당 하는 접두사 또는 원본 주소에 지정 된 접두사 지정 합니다.

5.  클릭 **다음**합니다.  

### <a name="wizard-page-4---protocols-and-ports"></a>마법사 페이지 프로토콜 및 포트 4

QoS 정책 마법사 네 번째 페이지에서 교통량 하 고 마법사의 첫 번째 페이지에서이 설정을 제어 하는 포트 유형의 지정할 수 있습니다. 지정할 수 있습니다.  
-   TCP 교통, UDP 한 트래픽 또는 둘 다  

-   포트, 다양 한 소스 포트 또는 특정 원본 포트 모든 소스

-   모든 목적지 포트, 다양 한 목적지 포트 또는 특정 대상 포트  

#### <a name="to-configure-the-protocols-and-ports-page-of-the-qos-policy-wizard"></a>프로토콜와 포트 페이지 QoS 정책 마법사의 구성 하려면

1. **이 QoS 정책이 적용 되는 프로토콜**선택 **TCP**, **UDP**, 또는 **UDP 및 TCP**합니다.

2. **소스 포트 번호를 지정**선택 **소스 포트에서** 또는 **이 소스 포트 번호를**합니다.

3. 선택한 경우 **이 소스 포트 번호를**, 1 사이의 65535 포트 번호를 입력 합니다.

     형식의 포트 범위를 필요에 따라 지정할 수 있습니다 "*낮은*:*높은*," 여기서 *낮은* 및 *높은* 고 포트 범위 형식의 포괄적 나타냅니다. *낮은* 및 *높은* 각각 1과 65535 사이의 숫자 여야 합니다. 공백 없음으로 (:) 문자와 숫자가 사이 허용 됩니다.

4. **대상 포트 번호를 지정**선택 **모든 대상 포트** 또는 **이 대상 포트 번호를**합니다.

5. 선택한 경우 **이 대상 포트 번호를** 이전 단계에서 65535 1 사이의 포트 번호를 입력 합니다.

새 QoS 정책 작성을 완료 하려면 클릭 **완료** 에 **프로토콜 및 포트** QoS 정책 마법사의 페이지 합니다. 완료 되 면 그룹 정책 개체 편집기의 세부 정보 창에서 해당 새 QoS 정책이 표시 됩니다.  
  
QoS 정책 설정을 사용자 또는 컴퓨터에 적용할 GPO QoS 정책을 된 Active Directory Domain Services 컨테이너 도메인, 사이트나 조직을 (OU)와 같은 위치에 연결 합니다.

### <a name="advanced-settings-for-roaming-and-remote-users"></a>로밍 하 고 원격 사용자에 대 한 고급 설정  
QoS 정책을 사용 하 여 목표는 엔터프라이즈 네트워크에 대 한 교통량을 관리 하는 것입니다. 모바일 시나리오 엔터프라이즈 네트워크 켜거나 사용자 교통을 보낼 될 수 있습니다. QoS 정책을 엔터프라이즈 네트워크 외부에 있는 동안 관련 되지 않으므로 QoS 정책은 한 네트워크 인터페이스 Windows 8, Windows 7 또는 Windows Vista에 대 한 엔터프라이즈에 연결 되어 있는 경우에 활성화 됩니다.  
  
예를 들어, 사용자 커피숍에서 가상 개인 네트워크 VPN ()를 통해 자신의 엔터프라이즈 네트워크에 자신의 노트북을 연결 될 수 있습니다. VPN (예: 무선) 실제 네트워크 인터페이스 QoS 정책이 적용 없을 것입니다. 그러나 VPN 인터페이스 QoS 정책을 회사에 연결 하기 때문에 적용 되는 수 있습니다. 사용자가 AD DS 신뢰 관계 하지 않은 다른 엔터프라이즈 네트워크를 나중에 입력 QoS 정책을 사용할 수 있습니다.  
  
참고 모바일 이러한 시나리오 서버 작업에 적용 되지 않습니다. 예를 들어, 여러 네트워크 어댑터를 사용 하 여 서버는 엔터프라이즈 네트워크의 가장자리에 앉아 될 수 있습니다. IT 부서 엔터프라이즈; 들어가는 QoS 정책 조절 교통량을 선택할 수 있습니다. 그러나이 네트워크 어댑터가 송신 교통 전송 하는 엔터프라이즈 네트워크에 다시 반드시 연결 되지 않습니다. 이러한 이유로 QoS 정책 Windows Server 2012를 실행 하는 컴퓨터의 모든 네트워크 인터페이스에서 항상 활성화 됩니다.  
  
> [!NOTE]
>  선택적 기능 부여 옆이 문서에서 설명한 QoS 고급 설정을 아니라 QoS 정책에만 적용 됩니다.  
  
### <a name="advanced-qos-settings"></a>고급 QoS 설정

QoS 고급 설정을 네트워크 소비 컴퓨터와 DSCP 표시 관리 IT 관리자에 대 한 추가 컨트롤을 제공 합니다. 컴퓨터와 사용자 수준에서 QoS 정책이 적용 될 수 있는 반면 컴퓨터 수준만 고급 QoS 설정이 적용 됩니다.

#### <a name="to-configure-advanced-qos-settings"></a>고급 설정 QoS 구성 하려면

1.  클릭 **컴퓨터 구성**을 차례로 클릭 하 고 **그룹 정책에서 Windows 설정을**합니다.
  
2.  마우스 오른쪽 단추로 클릭 **QoS 정책**을 차례로 클릭 하 고 **QoS 고급 설정**합니다.

     다음 그림에서는 두 개의 고급 설정 탭 QoS: **TCP 교통 돌아오는** 및 **DSCP 표시 재정의**합니다.
  
> [!NOTE]
>  고급 QoS 설정이 컴퓨터 레벨 그룹 정책 설정 됩니다.
  
#### <a name="advanced-qos-settings-inbound-tcp-traffic"></a>고급 설정 QoS: TCP 교통 돌아오는

**돌아오는 TCP 교통** QoS 정책은 아웃 바운드 UDP 및 TCP 교통 영향을 반면 TCP 대역폭 수신기의 측면에서 제어 됩니다. 

낮은 처리량에 수준 설정으로 **TCP 교통 돌아오는** 탭 TCP 떨어집니다 TCP 수신 창의 크기는 광고 합니다. 이 설정 적용 처리 속도가 증가 되며 활용도 TCP 연결 대역폭 높은 또는 대기 시간이 (대역폭 지연 제품)에 대 한 링크입니다. 기본적으로 Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 및 Windows Vista를 실행 하는 컴퓨터 최대 처리량 수준으로 설정 됩니다.
  
TCP 수신 창이 Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 및 Windows Vista에서 이전 버전의 Windows에서 변경 합니다. 이전 버전의 Windows Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 및 Windows Vista 받기 측 창 동적 크기 반면 64 (kb)를 최대 TCP 받기 측 창 제한 최대 16 크기 (MB). 인바인드 TCP 교통 컨트롤에 최대 값 TCP 수신 창 자랄 수를 설정 하 여 인바인드 처리량 수준을 제어할 수 있습니다. 수준 다음 최대 값으로 해당합니다. 
  
|돌아오는 처리량 수준|최대|  
|------------------------|-------|  
|0|KB 64|
|1|256KB|
|2|1MB|
|3|16 M B|

실제 창의 크기와 같은 또는 네트워크 상태에 따라 최대 보다 작은 값 수 있습니다.

###### <a name="to-set-the-tcp-receive-side-window"></a>TCP 받을 측 창 설정 하려면

1. 그룹 정책 개체 편집기 클릭 **로컬 컴퓨터 정책**, 클릭 **Windows 설정**를 마우스 오른쪽 단추로 클릭 **QoS 정책**, 클릭 한 다음 **QoS 고급 설정**합니다.
  
2. **TCP 받는 처리량**선택 **TCP 받는 처리량 구성**, 누른 다음 원하는 처리량 수준을 선택 합니다.

3.  GPO를 연결 합니다.

#### <a name="advanced-qos-settings-dscp-marking-override"></a>고급 설정 QoS: DSCP 재정의 표시

응용 프로그램을 지정할 수 있도록 제한 DSCP 표시 재정의-또는 "표시" 등 DSCP 값 QoS 정책에 명시 아닌 다른 합니다. 응용 프로그램 DSCP 값을 설정할 수 있도록 지정 하 여 응용 프로그램 DSCP 0이 아닌 값 설정할 수 있습니다. 

지정 하 여 **무시**, QoS Api를 사용 하는 응용 프로그램은 해당 DSCP 0으로 설정 크고 QoS 정책에 대 한 DSCP 값을 설정할 수 있습니다. 

기본적으로 Windows Server 2016, Windows 10, Windows Server 2012 R2, Windows 8.1을 실행 하는 컴퓨터 Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 및 Windows Vista 허용 응용 프로그램을 DSCP 지정 합니다. 응용 프로그램 및 장치의 QoS Api를 사용 하지 않는 무시 되지 않습니다.

##### <a name="wireless-multimedia-and-dscp-values"></a>무선 멀티미디어 및 DSCP 값

[Wi Fi Alliance](https://go.microsoft.com/fwlink/?LinkId=160769) 무선 멀티미디어 인증 설정한 Wi\ Fi 무선 네트워크에 액세스 아니요로 정의 \(WMM\) \(WMM_AC\) 네트워크 교통 우선순위를 전송 합니다. 액세스 범주 포함 \ (에서 가장 높은-에-최저 priority\ 순서 대로): 음성, 동영상, 좋은 노력 및 배경; 각각 VO, VI, 및 BK 약어 합니다. WMM 사양 어떤 DSCP 정의 값 각 네 액세스 범주와 일치 합니다.
  
|DSCP 값|WMM 액세스 범주|
|----------|-------------------|
|48-63|음성 (VO)|
|32-47|비디오 (VI)|
|24-31, 0-7|최상의 (BE)|
|8-23|백그라운드 (BK)|

이러한 DSCP 값을 WMM 무선 어댑터에 대 한 Wi\-Fi Certified™와 노트북 우선 순위가 지정 된 처리 WMM 액세스 포인트에 대 한 Wi\ Fi 인증와 연결 된 경우 받도록 사용 하는 QoS 정책 만들 수 있습니다.
  
### <a name="BKMK_precedencerules"></a>우선 순위 규칙 QoS 정책

유사한 GPO의 우선 순위를 QoS 정책을 충돌을 해결 특정 교통 집합에 여러 QoS 정책이 적용 우선 규칙을 갖습니다. 아웃 바운드 TCP 또는 UDP 교통량 QoS 정책을 조절 속도 합계 될 것 같은 누적 효과 설치 되지 않은 것을 의미 하는 한 번에 하나의 QoS 정책 적용할 수 있습니다.

일반적으로 가장 일치 하는 조건으로 QoS 정책이 적용 됩니다. 규칙은 다음 세 가지 범주에 속합니다 여러 QoS 정책이 적용: 사용자 수준 컴퓨터 수준; 정보 네트워크 다섯 번;와 응용 프로그램 및 quintuple 네트워크 중입니다.

하 여 *다섯 배 네트워크*, 소스 IP 주소, 대상 IP 주소, 소스 포트, 대상 포트 의미 하며 \(TCP/UDP\) 프로토콜 했습니다.  

 **사용자 액세스 제어 QoS 정책 우선 순위가 컴퓨터 수준의 QoS 정책**

이 규칙 쉽게 QoS Gpo의 사용자 그룹 기반 정책에 대 한 특히 네트워크 관리자 관리 크게 수 있습니다. 예를 들어 네트워크 관리를 사용자 그룹에 대 한 QoS 정책이 정의 하려는 경우 수만 만들고 GPO 해당 그룹을 배포 합니다. 자녀가 걱정할 필요가에 로그온 해당 사용자가 컴퓨터에 대 한 고 여부 컴퓨터가 충돌 하는 QoS 정책이 정의 충돌 하는 경우 사용자 수준 정책 항상 우선 때문에입니다.

> [!NOTE]
>  사용자 액세스 제어 QoS 정책만 해당 사용자가 생성 되는 교통 적용 됩니다. 다른 사용자가 특정 컴퓨터와 자체의 해당 사용자에 대해 정의 된 모든 QoS 정책이 적용 되지 않습니다.

 **응용 프로그램 특이성 및 다섯 번 네트워크 보다 우선 촬영**

여러 QoS 정책을 특정 교통와 일치 하는 경우 자세한 정책이 적용 됩니다. 정책 응용 프로그램을 식별 하는 중 파일 경로가 보낸 응용 프로그램을 포함 하는 정책만 응용 프로그램의 이름을 (경로 없음)를 식별 하는 다른 정책 보다 특정 간주 됩니다. 응용 프로그램을 사용 하 여 여러 개의 정책을 계속 적용 되는 경우 우선 순위 규칙 다섯 배 네트워크를 사용 하 여 가장 일치 항목을 찾습니다.

또는 여러 개의 QoS 정책은 조건을 겹치지 지정 하 여 동일한 교통에 적용 될 수 있습니다. 응용 프로그램의 상태와 네트워크 다섯 번 정책 응용 프로그램 지정 하는 특정 것으로 간주 됩니다 고 적용 됩니다. 

예를 들어, policy_A만는 응용 프로그램 이름 (app.exe) 지정 하 고 policy_B는 대상 IP 주소 192.168.1.0 월 24 지정 합니다. 이러한 QoS 정책이 충돌 때 \ (app.exe 보냅니다 교통 IP 주소 192.168.4.0/24\ 범위 내에), policy_A 적용 됩니다.

 **더 많은 특이성은 네트워크 보다 우선 다섯 배**

내 네트워크 다섯 번 정책 충돌을 가장 일치 하는 조건으로 정책을 우선 적용 됩니다. 예를 들어 "모든" 원본 IP 주소, 대상 IP 원본 포트 10.0.0.1 대상 포트 "모든", "모든" 해결 하 고 "TCP" 프로토콜 policy_C 지정 합니다. 

이번에 policy_D 대상 IP 주소 원본 포트 10.0.0.1 "모든," 대상 포트 80을 프로토콜 "TCP", "모든" 원본 IP 주소를 지정 합니다. 그런 다음 policy_C 및 policy_D 10.0.0.1:80 목적지에 대 한 연결와 일치합니다. QoS 정책 특정 가장 일치 하는 조건으로 정책을 적용을 하기 때문에 policy_D 우선이 예제 합니다.  
  
그러나 QoS 정책을 조건 동일한 수가 있을 수 있습니다. 예를 들어 몇 가지 정책 각 지정할 수 하나만 (있지만 같지) 네트워크 다섯 번입니다. 네트워크 다섯 번 간에 다음 순서 아래로 우선 순위가 높은에서 다음과 같습니다.

- 원본 IP 주소

- 대상 IP 주소

- 원본 포트

- 대상 포트

- 프로토콜 (TCP 또는 UDP)

IP 주소와 같은 특정 조건에서 구체적인 IP 주소를 취급에 우선 순위가 높은; 예를 들어, IP 주소 192.168.4.1 구체적인 192.168.4.0 월 24 보다입니다.

QoS 정책을 디자인 구체적으로 조직의 어떤 정책이 적용 됩니다 이해 하는 기능을 간단 하 게 수 있습니다.
