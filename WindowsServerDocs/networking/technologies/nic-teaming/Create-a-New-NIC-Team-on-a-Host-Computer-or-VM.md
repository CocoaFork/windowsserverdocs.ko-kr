# <a name="create-a-new-nic-team-on-a-host-computer-or-vm"></a>호스트 컴퓨터 또는 VM에서 새 NIC 팀 만들기

>적용 대상: Windows Server (반기 채널), Windows Server 2016

이 항목에서는 호스트 컴퓨터 또는 Windows Server 2016을 실행 중인 Hyper-v 가상 컴퓨터 (VM)에서 새 NIC 팀을 만듭니다.  

## <a name="network-configuration-requirements"></a>네트워크 구성 요구 사항  
새 NIC 팀을 만들기 전에 서로 다른 실제 스위치에 연결 하는 두 개의 네트워크 어댑터를 사용 하 여 Hyper-v 호스트를 배포 해야 합니다. 또한 동일한 IP 주소 범위에 있는 IP 주소를 사용 하 여 네트워크 어댑터를 구성 해야 합니다.  

물리적 스위치, Hyper-v 가상 스위치, 로컬 영역 네트워크 (LAN) 및 VM에서 NIC 팀을 만들기 위한 요구 사항 NIC 팀 다음과 같습니다.  

-   Hyper-v를 실행 하는 컴퓨터 네트워크 어댑터 두 개 이상 있어야 합니다.  

-   여러 물리적 스위치에 네트워크 어댑터에 연결 하는 경우 물리적 스위치도 동일한 계층 2 서브넷에 있어야 합니다.  

-   두 개의 외부 Hyper-v 가상 스위치를 만들려면 Hyper-v 관리자 또는 Windows PowerShell을 사용 해야 합니다, 그리고 각각 서로 다른 실제 네트워크 어댑터에 연결 합니다.  

-   VM은 만든 두 외부 가상 스위치에 연결 해야 합니다.  

-   Windows Server 2016에서 NIC 팀을 Vm의 두 멤버를 사용 하 여 팀을 지원 합니다. 규모가 큰 팀을 만들 수 있지만 지원 되지 않습니다.

-   VM에서 NIC 팀을 구성 하는 경우에 선택 해야는 **팀 모드** 의 _독립 스위치_ 와 **부하 분산 모드** 의 _주소 해시_. 

## <a name="step-1-configure-the-physical-and-virtual-network"></a>1단계. 실제 및 가상 네트워크를 구성 합니다.  
이 절차에서는 두 개의 외부 Hyper-v 가상 스위치, 스위치를 VM에 연결할 만들고 스위치에 VM 연결을 구성한 합니다.  

### <a name="prerequisites"></a>사전 요구 사항

멤버 여야 **관리자**, 또는 그와 동등 합니다.  

### <a name="procedure"></a>절차

1.  Hyper-v 호스트에서 Hyper-v 관리자를 열고 작업에서 클릭 **가상 스위치 관리자**합니다.  

   ![Virtual Switch Manager](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hv.jpg)  

2.  가상 스위치 관리자에서 했는지 **외부** 를 선택한 다음 클릭 **가상 스위치 만들기**합니다.  

   ![가상 스위치 만들기](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hv_02.jpg)  

3.  가상 스위치 속성에서 입력을 **이름** 가상 스위치 및 추가 대 한 **메모** 필요에 따라 합니다.  

4.  **연결 유형**의 **외부 네트워크**, 가상 스위치에 연결 하려는 실제 네트워크 어댑터를 선택 합니다.  

5.  배포에 대 한 추가 스위치 속성을 구성 하 고 클릭 **확인**합니다.  

6.  이전 단계를 반복 하 여 두 번째는 외부 가상 스위치를 만듭니다. 다른 네트워크 어댑터에 두 번째 외부 스위치를 연결 합니다.  

7.  Hyper-v 관리자에서 아래의 **Virtual Machines**를 구성 하 고 클릭 하려는 VM을 마우스 오른쪽 단추로 클릭 **설정**합니다.  

   VM **설정을** 대화 상자가 열립니다.

8.  VM 시작 되지 않은 것을 확인 합니다. 시작 된 VM을 구성 하기 전에 종료를 수행 합니다.  

8.  **하드웨어**, 클릭 **네트워크 어댑터**합니다.  

   ![네트워크 어댑터](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hvs_01.jpg)  

9. **네트워크 어댑터** 속성을 이전 단계에서 만든 가상 스위치 중 하나를 선택 하 고 클릭 **적용**합니다.  

    ![가상 스위치를 선택 합니다.](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hvs_02.jpg)  

10. **하드웨어**를 클릭 하 여 옆에 더하기 기호 (+)를 확장 **네트워크 어댑터**합니다. 

11. 클릭 **고급 기능** 그래픽 사용자 인터페이스를 사용 하 여 NIC 팀을 사용 하도록 설정 합니다. 

    >[!TIP]
    >Windows PowerShell 명령을 사용 하 여 NIC 팀을 사용할 수도 있습니다. 
    >
    >```PowerShell
    >Set-VMNetworkAdapter -VMName <VMname> -AllowTeaming On
    >```

    a. 선택 **동적** MAC 주소에 대 한 합니다. 

    b. 선택 하려면 클릭 **보호 된 네트워크**합니다. 

    c. 선택 하려면 클릭 **게스트 운영 체제에서 팀의 일부가 되려면이 네트워크 어댑터를 사용 하도록 설정**합니다. 

    d. **확인**을 클릭합니다.  

    ![네트워크 어댑터를 팀에 추가](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hvs_05.jpg)  

13. Hyper-v 관리자에서 두 번째 네트워크 어댑터를 추가할 **Virtual Machines**동일한 VM을 마우스 오른쪽 단추로 클릭 한 다음 클릭 **설정**합니다.  

    VM **설정을** 대화 상자가 열립니다.

14. **추가 하드웨어**, 클릭 **네트워크 어댑터**를 클릭 하 고 **추가**합니다.  

   ![네트워크 어댑터를 추가 합니다.](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hvs_06.jpg)  

15. **네트워크 어댑터** 속성을 이전 단계에서 만든 두 번째 가상 스위치를 선택한 다음 클릭 **적용**합니다.  

   ![가상 스위치를 적용 합니다.](../../media/Create-a-New-NIC-Team-in-a-VM/nict_hvs_07.jpg)  

16. **하드웨어**를 클릭 하 여 옆에 더하기 기호 (+)를 확장 **네트워크 어댑터**합니다. 

17. 클릭 **고급 기능**, 아래로 스크롤하여 **NIC 팀**를 클릭 하 여 선택 하 고 **게스트 운영 체제에서 팀의 일부가 되려면이 네트워크 어댑터를 사용 하도록 설정**합니다. 

18. **확인**을 클릭합니다.  

_**축 하!** _  실제 및 가상 네트워크를 구성 했습니다.  이제 새 NIC 팀 만들기를 진행할 수 있습니다.  

## <a name="step-2-create-a-new-nic-team"></a>2단계. 새 NIC 팀 만들기

새 NIC 팀을 만들 때 NIC 팀 속성을 구성 해야 합니다.  

-   팀 이름  

-   멤버 어댑터  

-   팀 모드  

-   부하 분산 모드  

-   대기 중인 어댑터  

또한 필요에 따라 주 팀 인터페이스를 구성 하 고 가상 LAN (VLAN) 숫자를 구성할 수 있습니다.  

이러한 설정에 대 한 자세한 내용은 참조 하세요. [NIC 팀 설정](nic-teaming-settings.md)합니다.

### <a name="procedure"></a>절차

1. 서버 관리자에서 **로컬 서버**를 클릭합니다.  

2. 에 **속성** 창, 첫 번째 열에서 찾을 **NIC 팀**를 클릭 하 고는 **사용 안 함** 링크.  

   합니다 **NIC 팀** 대화 상자가 열립니다.

   ![NIC 팀 대화 상자](../../media/Create-a-New-NIC-Team/nict_02_nicteaming.jpg)

3. **어댑터 및 인터페이스**, NIC 팀에 추가 하려는 하나 이상의 네트워크 어댑터를 선택 합니다.  

4. 클릭 **태스크**, 클릭 **새 팀을 추가할**합니다.  

   합니다 **새 팀** 대화 상자가 열리고 네트워크 어댑터와 팀 멤버가 표시 됩니다.

5. **팀 이름**새 NIC 팀에 대 한 이름을 입력 하 고 클릭 **추가 속성**합니다.  

6. **추가 속성**에 대 한 값을 선택 합니다.

   - **팀 모드**합니다. 팀 모드에 대 한 옵션은 **독립 스위치** 하 고 **스위치 종속**합니다. 스위치 종속 모드 포함 **정적 팀** 하 고 **집계 제어 프로토콜 (LACP (링크)** 합니다.

     - **스위치 종속 됩니다.** [!INCLUDE [switch-dependent-shortdesc-include](../../includes/switch-dependent-shortdesc-include.md)]

LACP 팀 모드를 구성한 경우 NIC 팀 항상 간단한 타이머를 사용 하 여 LACP의 활성 모드에서 작동 됩니다.  없음 옵션은 타이머를 수정 하거나 LACP 모드를 변경 하려면 현재 사용할 수 있습니다. |

   - **부하 분산 모드**합니다. 부하 분산 배포 모드에 대 한 옵션은 **주소 해시**하십시오 **Hyper-v 포트**, 및 **동적**합니다.

   - **대기 중인 어댑터**합니다. 대기 어댑터에 대 한 옵션은 **없음 (모든 어댑터 활성)** 또는 Standby 어댑터로 작동 하는 NIC 팀에서 특정 네트워크 어댑터를 선택 합니다.

   -   TNIC 인터페이스 이름을 제공 합니다.  

   -   VLAN 멤버 자격을 구성 합니다:를 클릭 **특정 VLAN** 한 VLAN 정보를 입력 합니다. 예를 들어, 회계 VLAN에이 NIC 팀을 추가 하려는 경우에 44에서 형식 회계 44-VLAN 번호입니다.   

9. **확인**을 클릭합니다.  

_**축 하!** _  호스트 컴퓨터 또는 VM에서 새 NIC 팀을 만들었습니다.
