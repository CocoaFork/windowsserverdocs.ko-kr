## <a name="bkmk_teaming"></a>팀 모드 선택  
팀 구성 모드에 대 한 옵션은 **스위치 독립**, **정적 팀**, 및 **링크 집합 제어 프로토콜 (LACP)**합니다. 고정 팀과 LACP 스위치 종속 모드 됩니다. 최상의 성능을 세 가지 팀 구성 모드와 NIC 팀, 부하 분산 동적 메일 모드를 사용 하는 좋습니다.  
  
**스위치 독립**  
  
전환 하 여 독립 모드, 스위치를 또는 스위치를 NIC 팀 멤버 연결 NIC 팀의 존재 인식 하 고 네트워크 교통 NIC 팀원-에 게 배포 하는 방법을 결정 하지는 않습니다 대신 NIC 팀 배포 인바인드 네트워크 교통 NIC 팀 멤버 합니다.  
  
스위치 독립 모드 동적 메일을 사용 하면 수정 하 여 동적 균형 알고리즘 TCP 포트 주소 해시에 따라 네트워크 교통 로드 배포 됩니다. 동적 균형 알고리즘 재배포 흐름을 개별 흐름 전송 한 활성 팀원에서 다른 위치로 이동할 수 있도록 팀 멤버 대역폭 사용을 최적화 합니다. 알고리즘을 교통 재배포 일으킬 수 패킷, 주문 아웃 배달 단계 가능성을 최소화 하는 데 걸리는 하므로 작은 가능성을 고려 합니다.  
  
**종속 전환**  
  
종속 전환 모드 스위치를 NIC 팀 멤버 연결 되어 있는 인바인드 네트워크 교통 NIC 팀원 들을 배포 하는 방법을 결정 합니다. 스위치는 NIC 팀 멤버 네트워크 트래픽을 배포 하는 방법을 확인 하려면 완료 독립을 있습니다.  
  
> [!IMPORTANT]  
> 동일한 실제 스위치에 연결 된 모든 팀원 또는 다중 섀시 전환 하는 필요한 스위치 종속 팀 중에서 다양 한 섀시 스위치 ID를 공유 합니다.  
  
고정 팀 구성 스위치 및 팀에서 링크를 식별 하는 호스트 수동으로 구성 하도록 요구 합니다. 정적으로 구성 된 솔루션을 이므로 스위치를 지원 없이 추가 프로토콜 않으며 올바르게 식별 호스트 전원 케이블이 나 다른 오류 수행 하지 않아야 하는 팀 일으킬 수 있는 합니다. 이 모드는 일반적으로 서버 등급 스위치가에서 지원 됩니다.  
  
고정 팀과 달리 모드 LACP 팀 호스트 스위치와 연결 된 링크 동적으로 식별 합니다. 이 일상 전송 또는 LACP 패킷 피어 엔터티에서 수신 하 여 자동 만들기를 팀을 이론적으로 만들어졌지만 보너스 거의 연습, 확장 및 감소 팀에 있습니다. 모든 서버 등급 스위치 LACP를 지원 하 고 스위치 포트에서 LACP 관리 방식으로 사용할 수 있도록 네트워크 운영자 모든 필요 합니다. 팀 구성 모드 LACP 구성할 때 짧은 타이머와 함께 LACP의 활성 모드로 작동 항상 NIC 팀 합니다.  없음 옵션은 현재 타이머를 수정 하거나 변경 LACP 모드를 사용할 수 있습니다.  
  
스위치 종속 모드 동적 메일을 사용 하면 수정 하 여 동적 균형 알고리즘 TransportPorts 주소 해시에 따라 네트워크 교통 로드 배포 됩니다.  동적 부하 분산 알고리즘 재배포 흐름을 팀 멤버 대역폭 사용을 최적화 합니다. 개별 흐름 전송 간에 이동할 수 활성 팀 멤버에서 동적 메일의 일환으로 합니다. 알고리즘을 교통 재배포 일으킬 수 패킷, 주문 아웃 배달 단계 가능성을 최소화 하는 데 걸리는 하므로 작은 가능성을 고려 합니다.  
  
모든 스위치 종속 구성 된 하 스위치 인바인드 교통 여 팀원 들을 배포 하는 방법을 결정 합니다.  스위치 팀 멤버 교통량 분산 적절 한 작업을 수행 해야 하는 했으나는 어떻게 결정을 완료 독립 있습니다.  
  
## <a name="bkmk_lb"></a>균형 모드 선택  
배포 부하 분산 모드에 대 한 옵션은 **주소 해시**, **Hyper-v 포트**, 및 **동적**합니다.  
  
**주소 해시**  
  
이 부하 분산 모드 해시 패킷 주소 구성 요소에 따라를 만듭니다. 다음 사용 가능한 어댑터 중 하나를 해당 해시 값 패킷을 할당 합니다. 일반적으로이 장치 에서만 사용할 수 있는 어댑터에서 적절 한 균형 만들 수 있는 충분 한입니다.  
  
Windows PowerShell를 사용 하 여 다음 해시 함수 구성 요소에 대 한 값을 지정할 수 있습니다.  
  
-   원본와 목적지 TCP 포트 및 원본과 대상 IP 주소 합니다. 선택 하면 이것이 기본 **주소 해시** 부하 분산 모드로 합니다.  
  
-   원본 및 대상 IP 주소만 합니다.  
  
-   원본 및 대상 MAC 주소만 합니다.  
  
TCP 포트 해시 독립적으로 이동할 수 NIC 팀 멤버 간에 더 작게 스트림을 교통 스트림의 가장 세부적으로 메일을 만듭니다. 그러나를 사용할 수 없습니다 TCP 포트 해시 TCP 하지 않은 교통 또는 UDP 기반 위치 UDP 및 TCP 포트 숨겨집니다 스택에서 같은 IPsec 암호로 보호 된 교통와 또는 합니다. 이러한 경우 대체로 해시 자동으로 IP 주소 해시 사용 또는 교통량 IP 교통 없으면 MAC 주소 해시 사용 됩니다.  
  
**Hyper-v 포트**  
  
Hyper-v 호스트에서이 구성 NIC 팀에 대 한 포트 Hyper-v 모드를 사용 하 여에 이점이 있습니다. Vm 독립 MAC 주소를 있으므로 VM의 MAC 주소 또는 Hyper-v의 가상 스위치에 VM 연결 되어 포트 네트워크 교통 NIC 팀 멤버 사이 있는 기반 될 수 있습니다.  
  
> [!IMPORTANT]  
> NIC 팀 Vm 내에서 만든 Hyper-v 포트 부하 분산 모드도 구성할 수 없습니다. 사용 하 여 주소 해시 대신 분산 모드를 로드 합니다.  
  
옆에 있는 스위치를 항상 한 포트에서 특정 MAC 주소를 확인, 하므로 스위치 MAC (VM MAC) 주소 대상에 따라 여러 링크에서 수신 로드 (트래픽 스위치에서 호스트)을 배포 합니다. 특히 유용 큐 가상 컴퓨터 (VMQs)를 사용 하는 때문에 도착 하는 데 필요한 교통량 곳 특정 NIC 큐 나 배치할 수 있습니다.  
  
그러나 호스트 몇 Vm만 있으면이 모드 잘 균형 배포 달성 하기 위해 충분히 세밀 되지 않을 수 있습니다. 이 모드는 항상도 대역폭 단일 인터페이스에서 사용할 수 있는 단일 VM (즉, 단일 스위치 포트에서 교통량)를 제한 합니다. NIC 팀에서 경우에 따라 VM 스위치 포트에서 둘 이상의 MAC 주소를 사용 구성 수 있기 때문에 소스 MAC 주소를 사용 하는 대신 식별자로 Hyper-v의 가상 스위치를 포트 사용 합니다.  
  
**동적**  
  
이 균형 모드 하는 최고의 측면에 각 다른 두 가지 모드를 사용 하 고 단일 모드 결합 하 여 다음과 같습니다.  
  
-   아웃 바운드 로드 하는 TCP 포트 및 IP 주소 콘텐츠의 해시에 따라 배포 됩니다.  동적 모드 지정된 아웃 바운드 흐름 팀원 간에 전환 이동할 수 있습니다도 실시간으로 로드 재정렬할 합니다.  
  
-   Hyper-v 포트 모드와 같은 방식으로 인바인드 로드 배포 됩니다.  
  
이 모드에서는 아웃 바운드 로드 flowlets의 개념에 따라 균형 동적으로 됩니다. 실제 음성 자연 나누기 단어 및 문장 끝에 있는 것 처럼 TCP 흐름 (TCP 통신 스트림을) 자연스럽 발생 나누기 한도 있습니다. 이러한 두 가지 나누기 사이의 TCP 흐름 부분을 flowlet 라고 합니다.  
  
동적 모드 알고리즘을 flowlet 경계 실행 되었습니다-TCP 흐름-에 충분 한 길이의 마시며 휴식 발생 했을 때와 같은 감지 알고리즘 자동으로 다른 팀원을 흐름을 재정렬할 필요한 경우 합니다.  경우에 따라 알고리즘 수도 주기적으로 균형을 다시 조정 흐름 모든 flowlets 포함 되어 있지 않은 합니다. 이 때문 TCP 흐름과 팀 멤버 사이 선호도 따라 동적 분산 알고리즘 하는 작업 팀원을 지속적으로 언제 든 지 변경할 수 있습니다.  
  
팀은 스위치 독립 또는 모드 스위치 종속 중 하나 구성 되어 있는지 여부를 최적 성능에 대 한 동적 메일 모드를 사용 하는 것이 좋습니다.  
  
NIC 팀에는 두 가지 팀원, 스위치 독립 모드로 구성 되어 있으며 활성/대기 모드로 활성 NIC 사용 하도록 설정 하 고 다른 대기 모드에 대해 구성 하는 경우에이 규칙는 예외입니다. NIC 팀 구성과이 메일 주소 해시 배포 동적 보다 약간 더 나은 성능의 제공합니다.  
  
## <a name="bkmk_standby"></a>대기 어댑터 설정 선택  
대기 어댑터에 대 한 옵션은 없음 (모든 어댑터 활성) 또는 특정 네트워크 어댑터는 역할을 대기 어댑터를 선택 하지 않은 다른 팀원은 활성 NIC 팀에서 선택 합니다. 없는 네트워크 교통량을 전송 되거나 하지 않는 한 어댑터에 의해 처리 대기 어댑터로 NIC를 구성할 때까지 활성 NIC 실패 한 경우 현재 하 고 있습니다. 현재 NIC 실패 한 후 대기 NIC 활성화 되 고 프로세스 네트워크 교통 합니다. 서비스에 모든 팀원은 복원 경우 대기 팀원 대기 상태를 반환 됩니다.  
  
2 NIC 팀 있으며 대기 어댑터로 NIC 구성 하도록 선택할 경우 두 가지 활성 Nic 된 있는 집계 이점 대역폭 손실 있습니다.  
  
> [!IMPORTANT]  
> 결함 허용; 달성 하기 위해 대기 어댑터 지정 해야 결함 허용은 항상 NIC 팀에서 두 개 이상 네트워크 어댑터는 때마다 포함 되어 있습니다.

![새 인터페이스 팀 대화 상자](../../media/Create-a-New-NIC-Team-on-a-Host-Computer-or-VM/nict_newteaminterface.jpg)
