# <a name="connection-request-policies"></a>연결 요청 정책

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

서버를 구성 NPS RADIUS 서버, RADIUS 프록시 또는 둘 다로 NPS 연결 요청 정책을 사용 하는 방법을 자세히 알아보려면이 항목을 사용할 수 있습니다.

>[!NOTE]
>이 항목 외에 다음과 같은 연결 요청 정책 설명서 제공 됩니다.
> - [구성 연결 요청 정책](nps-crp-configure.md)
> - [원격 RADIUS 서버 그룹 구성](nps-crp-rrsg-configure.md)

연결 요청 정책은 집합 조건 및 네트워크 관리자가 어떤 RADIUS(Remote Authentication Dial-In User Service) (RADIUS) 서버 네트워크 NPS 정책 서버 ()를 실행 하는 서버 RADIUS 클라이언트에서 받아 연결 요청 인증과 수행 하 여 지정 하도록 허용 하는 설정입니다. 연결 요청 정책 RADIUS 계정에 사용 되는 RADIUS 서버 지정를 구성할 수 있습니다.

연결 만들 수 있습니다 요청 정책 RADIUS 클라이언트에서 보낸 일부 RADIUS 요청 메시지를 로컬로 처리 되도록 (NPS RADIUS 서버도 사용 됩니다.) (NPS RADIUS 프록시 사용이) 다른 RADIUS 서버에 다른 종류의 메시지 전달 됩니다.

연결 요청 정책을 사용 하 여 NPS RADIUS 서버 또는 다음과 같은 요인에 따라 RADIUS 프록시도 사용할 수 있습니다. 

- 시간 및 요일
- 연결 요청 영역 이름
- 요청 하 고 연결 유형
- RADIUS 클라이언트의 IP 주소

RADIUS 액세스 요청 메시지 처리 또는 구성 NPS 서버에 연결 요청 정책을 중 하나 이상을 수신 메시지 설정 일치 하는 경우에 NPS로 전달 됩니다. 

정책 설정을 일치 정책에 따라 NPS 서버 메시지를 처리 하는 경우 인증 연결 요청을 승인 하 고 NPS RADIUS 서버,으로 작동 합니다. 정책 설정을 일치 하는 경우 정책에 따라 NPS 서버 메시지를 전달 NPS RADIUS 프록시 역할을 하 고 연결 요청을 처리 하기 위해 원격 RADIUS 서버 전달 합니다.

수신 RADIUS 액세스 요청 메시지의 설정 연결 요청 정책 중 하나 이상을 일치 하지 않는 경우 RADIUS 클라이언트 액세스 거부 메시지가 전송 되 고 사용자 또는 컴퓨터 네트워크에 연결 하려고 할 때 액세스 거부 되었습니다.

## <a name="configuration-examples"></a>구성 예시

다음과 같은 구성 예 연결 요청 정책을 사용 하는 방법을 설명 합니다.

### <a name="nps-as-a-radius-server"></a>NPS RADIUS 서버

기본 연결 요청 정책을 구성이입니다. 여기에서 NPS RADIUS 서버도 구성 되어 및 모든 연결 요청 로컬 NPS 서버에서 처리 됩니다. NPS 서버 인증 고 NPS 서버 도메인 도메인에 있는 하 고 신뢰할 수 있는 도메인에 있는 계정이 있는 사용자에 게 권한을 부여 수 있습니다.


### <a name="nps-as-a-radius-proxy"></a>NPS RADIUS 프록시도

기본 연결 요청 정책을, 삭제 하 고 서로 다른 두 도메인 앞으로 요청에 두 가지 새로운 연결 요청 정책을 만들어집니다. 여기에서 NPS RADIUS 프록시도 구성 됩니다. NPS 로컬 서버에 연결 요청을 처리 하지 않습니다. 대신, NPS 또는 원격 RADIUS 서버 그룹의 회원으로 구성 된 다른 RADIUS 서버에 연결 요청 전달 합니다.


### <a name="nps-as-both-radius-server-and-radius-proxy"></a>NPS RADIUS 서버와 RADIUS 프록시

기본 연결 요청 정책을 뿐 아니라 NPS 나 다른 RADIUS 서버 신뢰할 수 없는 도메인에 연결 요청을 전달 하는 새 연결 요청 정책을 만들어집니다. 여기에서 프록시 정책 정책 정렬된 목록에서 첫 번째 표시 됩니다. 연결 요청 일치 하는 프록시 정책을 연결 요청 원격 RADIUS 서버 그룹에 RADIUS 서버에 게 전달 됩니다. 연결 요청 프록시 정책 일치 하지 않는 기본 연결 요청 정책을 일치 하는 경우, NPS 로컬 서버에 연결 요청을 처리 합니다. 연결 요청 중 하나 정책 일치 하지 않을 경우 삭제 됩니다.

### <a name="nps-as-radius-server-with-remote-accounting-servers"></a>NPS RADIUS 서버 계정 서버를 원격으로

여기에서 로컬 NPS 서버 계정 작업을 수행 하도록 구성 되지 않은 및 기본 연결 요청 정책을 하도록 수정 원격 RADIUS 서버 그룹의 다른 RADIUS 서버 또는 NPS RADIUS 계정 메시지 전달 됩니다. 계정 메시지를 전달 있지만 인증과 메시지를 전달 되지 않습니다 및 로컬 NPS 서버 로컬 도메인에 대 한 이러한 기능을 수행 하 고 신뢰할 수 있는 모든 도메인.


### <a name="nps-with-remote-radius-to-windows-user-mapping"></a>NPS RADIUS Windows 사용자 매핑을 원격으로

여기에서 NPS RADIUS 서버를 원격으로 인증 요청 인증을 위해 Windows 로컬 사용자 계정을 사용 하는 동안 전달 하 여 RADIUS 서버 및 각 개별 연결 요청에 대 한 프록시 RADIUS로 작동 합니다. 이 구성은 RADIUS Windows 사용자 매핑 특성을 원격 연결 요청 정책 조건으로 구성 구현 합니다. (뿐만 아니라 사용자 계정을 만들어야 로컬로 원격 RADIUS 서버 인증을 수행 하는 원격 사용자 계정으로 동일한 이름의 합니다.)

## <a name="connection-request-policy-conditions"></a>연결 요청 정책 조건

연결 요청 정책 조건과 하나 이상의 RADIUS 특성 수신 RADIUS 액세스 요청 메시지의 특성을 비교입니다. 여러 조건을 인 경우 연결에 나열 된 상황의 모든 요청 메시지 및 연결 요청 정책 nps 적용 정책에 대 한 순서 대로 일치 해야 합니다.

다음은 연결 요청 정책에서 구성 수 있는 조건 특성 합니다.

### <a name="connection-properties-attribute-group"></a>연결 속성 특성 그룹

연결 속성 특성 그룹 다음과 같은 특성 포함 되어 있습니다.

- **프레임 프로토콜**합니다. 들어오는 패킷 프레이밍 유형의 지정 하는 데 사용 합니다. 예로 (지점간 프로토콜), 직렬 줄 인터넷 프로토콜 (지연), 프레임 릴레이 x. 25 합니다.
- **서비스 유형**합니다. 유형의 요청 되는 서비스를 지정 하는 데 사용 합니다. 예를 들면 프레임 (예: PPP 연결) (예: 연결 텔넷) 로그인 합니다. RADIUS 서비스 형식에 대 한 자세한 내용은 참조 2865 RFC "원격 인증 전화 접속 사용자 서비스 (RADIUS)입니다."
- **타이핑 터널**합니다. 요청 하는 클라이언트에서 만들고 터널 유형의 지정 하는 데 사용 합니다. 터널 형식은 지점 간 터널링 (PPTP 프로토콜)와 계층 두 터널링 프로토콜 (L2TP) 포함 됩니다.

## <a name="connection-request-policy-settings"></a>연결 요청 정책 설정

연결 요청 정책 설정은 집합 속성 RADIUS 수신 메시지에 적용 되는입니다. 속성 다음 그룹 설정 구성 됩니다.

- 인증
- 계정 관리
- 특성 조작
- 전달 요청
- 고급

다음 섹션에서는 이러한 설정에 대 한 추가 세부 정보를 제공 합니다.

### <a name="authentication"></a>인증

이 설정을 사용 하 여 인증 설정을 모든 네트워크 정책에서 구성 된를 무시 하 고는 인증 방법 및 네트워크에 연결 하는 데 필요한 형식 지정할 수 있습니다.

>[!IMPORTANT]
>인증 방법에는 인증 방법인 네트워크 정책에서 구성 보다 안전 하지 않은 연결 요청 정책에서 구성한 경우 네트워크 정책에서 구성 더 안전 하 게 인증 방법을 재정의 됩니다. 예를 들어, 보호 Extensible 인증 프로토콜 Microsoft Challenge Handshake 인증 프로토콜 버전 2 사용 해야 하는 네트워크 정책을 둘 \ (PEAP 암호 v2\) 보안 무선에 대 한 암호 기반 인증 방법인 및 인증 되지 않은 액세스할 수 있도록 연결 요청 정책을 구성할 수도 결과 클라이언트가 v2 PEAP 암호를 사용 하 여 인증 필요 하다는 합니다. 네트워크에 연결 된 모든 클라이언트가 예제에서 부여 인증 되지 않은 액세스 합니다.

### <a name="accounting"></a>계정 관리

이 설정을 사용 하 여 원격 RADIUS 서버 그룹 수행 계정 수 있도록 계정 정보 NPS 나 원격 RADIUS 서버 그룹의 다른 RADIUS 서버에 전달 하도록 요청 정책을 연결을 구성할 수 있습니다.

>[!NOTE]
>여러 RADIUS 서버 하는 경우 계정 정보 한 중앙 RADIUS 계정 데이터베이스에 저장 된 모든 서버에 대 한 모든 NPS에 서버 또는 계정 서버로 지정 된 다른 RADIUS 서버에서 앞으로 계정 데이터를 정책 각 RADIUS 서버에 요청 연결 정책 계정 설정을 사용할 수 있습니다.

계정 연결 요청 정책 설정 독립적 로컬 NPS 서버의 계정 구성을으로 작동 합니다. 즉, 로컬 NPS 서버 로컬 파일 또는 Microsoft SQL Server 데이터베이스에 RADIUS 계정 정보를 구성 하는 경우 계정 메시지 원격 RADIUS 서버 그룹에 전달 하도록 요청 연결 정책을 구성 있는지 여부에 관계 없이 작업 이루어집니다.

계정 정보를 로컬로 하지 원격으로 기록, 원하는 경우 하지 계정도 계정 전달 계정 데이터를 원격 RADIUS 서버 그룹에 연결 요청 정책에서 구성 하는 동안 작업을 수행 하려면 로컬 NPS 서버를 구성 해야 합니다.

### <a name="attribute-manipulation"></a>특성 조작

다음과 같은 특성 중 하나는 텍스트 문자열을 조작 하는 찾기 바꾸기 규칙 집합이 구성할 수 있습니다.

- 사용자 이름
- 라고 스테이션 ID
- 전화 스테이션 ID

찾기 바꾸기 규칙 처리 RADIUS 메시지가 인증과 계정 설정 적용 되기 전에 위의 특성 중 하나에 대 한 발생 합니다. 특성 조작 규칙 하나의 특성에만 적용 됩니다. 각 특성 특성 조작 규칙 구성할 수 없습니다. 또한 조작할 수 있는 특성 중 목록은 정적 목록; 조작 사용할 수 있는 특성 중 목록에 추가할 수 없습니다.

>[!NOTE]
>암호 v2 인증 프로토콜을 사용 하는 경우 연결 요청 정책을 RADIUS 메시지를 전달 하는 데 사용 되는 경우 사용자 이름을 조작할 수 없습니다. 단, (\) 백슬래시 문자를 사용 하 고 조작 왼쪽에 있는 정보에만 적용 때 발생 합니다. 백슬래시 도메인 이름 (정보를 왼쪽 백슬래시를) (백슬래시의 오른쪽에 있는 정보) 도메인에 있는 사용자 계정 이름을 나타내기 위해 일반적으로 사용 됩니다. 이 경우에 특성 조작 규칙을 수정 하거나 도메인 이름을 바꿀 수 있습니다.

사용자 이름 특성 영역 이름 조작 하는 방법 몇 가지 항목에서 "예 조작 영역 이름의 사용자 이름에서" 섹션 참조 [NPS에서 사용 하 여 일반적인 표현](nps-crp-reg-expressions.md)합니다.

### <a name="forwarding-request"></a>전달 요청

다음을 설정할 수 전달 RADIUS 액세스 요청 메시지에 사용 되는 요청 옵션:

- **이 서버에 요청을 인증**합니다. 이 설정을 사용 하 여 NPS 연결 요청을 인증 Windows NT 4.0 도메인, Active Directory 또는 로컬 보안 계정을 관리자 (삼로) 사용자 계정 데이터베이스 사용 합니다. 또한이 설정을 전화 접속 속성 사용자 계정의 함께 구성 NPS에 해당 하는 네트워크 정책 하는 데 사용 nps 연결 요청을 승인 지정 합니다. 이 경우 NPS 서버 RADIUS 서버 서 실행 되도록 구성 됩니다.

- **다음과 같은 원격 RADIUS 서버 그룹 요청을 전달**합니다. 이 설정을 사용 하 여 NPS 연결 요청 사용자가 지정한 원격 RADIUS 서버 그룹을 전달 합니다. NPS 서버 액세스 요청 메시지에 해당 하는 유효한 액세스 허용 메시지를 받고, 인증 되 고 공인 연결 시도 간주 됩니다. 이 경우 NPS 서버 프록시 역할을 RADIUS 합니다.

- **자격 증명 유효성 검사를 않고도 사용자가 동의**합니다. 이 설정을 사용 하 여 네트워크에 연결 하려고 할 때 사용자의 신원을 확인 하지 않으며 및 사용자 또는 컴퓨터에는 네트워크에 연결할 수 있는 권한이 있는지 확인 하려면 NPS 시도 하지 않습니다. NPS 인증 되지 않은 액세스할 수 있도록 구성 되어 있고 연결 요청을 받을, NPS 즉시 보냅니다 RADIUS 클라이언트 및 사용자 또는 컴퓨터에 액세스 허용 메시지가 네트워크 액세스 권한이 부여 됩니다. 이 설정은 일부 유형의 강제 위치 액세스 클라이언트 되기 전에 터널링 사용자 자격 증명이 인증에 대 한 사용 됩니다.

>[!NOTE]
>이 인증 옵션이 인증 프로토콜 액세스 클라이언트의 암호 v2 또는 Extensible 인증 프로토콜 Transport Layer 보안 (EAP-TLS) 상호 인증을 제공 하는 때 사용할 수 없습니다. 상호 인증에서 액세스 클라이언트 증명 (NPS 서버)를 인증 서버에 유효한 액세스 클라이언트는 인증 서버를 액세스 클라이언트 올바른 인증 서버를 임을 증명 합니다. 이 인증 옵션을 사용 하는 경우 액세스 허용 메시지 반환 됩니다. 그러나 인증 서버를 액세스 클라이언트 유효성 검사를 제공 하지 않는 고 상호 인증 실패 합니다.

원격 RADIUS 서버 그룹으로 지정된 영역 이름 RADIUS 메시지를 전달 라우팅 규칙을 만드는 정기적으로 표현을 사용 하는 방법 몇 가지 항목에서 "예 프록시 서버 RADIUS 메시지를 전달" 섹션 참조 [NPS에서 사용 하 여 일반적인 표현](nps-crp-reg-expressions.md)합니다.

### <a name="advanced"></a>고급

고급 속성 RADIUS 특성을 일련 지정을 설정할 수 있습니다.

- 서버 NPS RADIUS 인증 또는 계정 서버도 사용 되 고 때 RADIUS 응답 메시지에 추가 합니다. 있는 경우 네트워크 정책 및 연결 요청 정책을에 지정 된 특성, RADIUS 응답 메시지 전송 되는 특성 두 가지 특성 집합으로 구성 됩니다.
- 서버 NPS RADIUS 인증 또는 계정 프록시도 사용 되 고 때 RADIUS 메시지에 추가 합니다. 전달 되는 메시지에 이미 특성 값 연결 요청 정책에 지정 된 특성으로 대체 됩니다. 

또한 연결에 대해 구성에 맞게 사용할 수 있는 몇 가지 특성 요청 정책 **설정** 탭의 **고급** 범주 특별 한 기능을 제공 합니다. 예를 들어 구성할 수 있습니다는 **Windows 사용자 매핑에 원격 RADIUS** 인증 분할 하려는 및 두 개의 사용자 간에 연결 요청을 승인 계정 데이터베이스 특성 합니다.

**원격 RADIUS Windows 사용자 매핑의** 특성 원격 RADIUS 서버 인증 된 사용자에 대 한 Windows 인증 발생는 지정 합니다. 즉, 원격 RADIUS 서버 원격 사용자 계정 데이터베이스의 사용자 계정에 대해 인증을 수행 하지만 NPS 서버 로컬 사용자 계정에서 로컬 사용자 계정 데이터베이스에 대 한 연결 요청에 권한을 부여 합니다. 방문자가 네트워크에 대 한 액세스를 허용 하려는 경우 유용 합니다.

예를 들어, 파트너 조직에서 방문자 자체 파트너 회사 RADIUS 서버 인증할 수 및 조직에서 Windows 사용자 계정을 사용 하 여는 게스트 lan () 네트워크에 액세스할 수 합니다.

특별 한 기능을 제공 하는 다른 특성은 다음과 같습니다.

- **MS-격리-IPFilter 및 격리 세션 시간 MS**합니다. 네트워크 액세스 차단 제어 (환경에서 NAQC) 라우팅과 원격 액세스 VPN 배포 배포 하는 경우 사용 됩니다.
- **Passport-사용자-매핑--Upn**합니다. 이 특성 사용자 계정 자격 증명™ Windows Live ID와 연결 요청을 인증할 수 있습니다.
- **태그 터널**합니다. 이 특성을 연결이 지정할 NAS 하 여 가상 영역 로컬 네트워크 (Vlan)를 배포 하는 경우 VLAN ID 번호를 지정 합니다.

## <a name="default-connection-request-policy"></a>기본 연결 요청 정책

기본 연결 요청 정책은 NPS 설치할 때 만들어집니다. 이 정책에 다음과 같은 구성 합니다.

- 인증 구성 되지 않았습니다.
- 계정 정보를 전달 원격 RADIUS 서버 그룹 계정 구성 되지 않았습니다.
- 원격 RADIUS 서버 그룹에 연결 요청을 전달 특성 조작 규칙과 함께 특성 구성 되지 않았습니다.
- 요청을 전달 연결 요청 인증 되 고 로컬 NPS 서버의 권한이 되도록 구성 됩니다.
- 고급 특성 구성 되지 않습니다.

기본 연결 요청 정책을 NPS RADIUS 서버를 사용합니다. NPS RADIUS 프록시 역할을 실행 하는 서버를 구성 하려면도 원격 RADIUS 서버 그룹을 구성 해야 합니다. 새 연결 요청 정책을 새 연결 요청 정책 마법사를 사용 하 여 만들 때 원격 새로운 RADIUS 서버 그룹을 만들 수 있습니다. 기본 연결 요청 정책의 삭제 하거나 기본 연결 요청 정책을 마지막 정책을 마지막 정책의 주문한 목록에 배치 하 여 NPS에 의해 처리 되었는지 확인 합니다.

>[!NOTE]
>NPS 한 원격 액세스 서비스 설치 된 경우 Windows 인증에 대해 구성 서비스 동일한 컴퓨터에 대 한 원격 액세스 아니며 계정 취해도 원격 액세스 인증과 계정 요청 RADIUS 서버에 전달 되도록 합니다. 이 대 한 원격 액세스 인증과 계정 요청 원격 RADIUS 서버 그룹에 전달 하도록 구성 된 연결 요청 정책을 일치 하는 경우 발생할 수 있습니다.
