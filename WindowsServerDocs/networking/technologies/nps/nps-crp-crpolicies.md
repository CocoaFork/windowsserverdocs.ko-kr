# <a name="connection-request-policies"></a>연결 요청 정책

>적용 대상: Windows Server (반기 채널), Windows Server 2016

NPS 연결 요청 정책을 사용 하 여 NPS를 RADIUS 서버로, RADIUS 프록시 또는 둘 다로 구성 하는 방법을 알아보려면이 항목에서는 사용할 수 있습니다.

>[!NOTE]
>이 항목 외에 다음 연결 요청 정책 설명서는 사용할 수 있습니다.
> - [연결 요청 정책 구성](nps-crp-configure.md)
> - [원격 RADIUS 서버 그룹 구성](nps-crp-rrsg-configure.md)

인증을 수행 하는 네트워크 관리자는 전화 접속 사용자 서비스 RADIUS (Remote Authentication) 서버를 지정할 수 있도록 설정 및 연결 권한 부여를 요청 하는 연결 요청 정책 되는 조건 집합을 네트워크 정책 (NPS 서버)를 실행 하는 서버는 RADIUS 클라이언트에서 수신 합니다. RADIUS 서버는 RADIUS 계정 사용을 지정 하려면 연결 요청 정책은 구성할 수 있습니다.

정책을 만들 수 있습니다 연결 요청 RADIUS 클라이언트에서 전송 된 일부 RADIUS 요청 메시지를 로컬로 처리 되는 (NPS를 RADIUS 서버로 사용 됨) 및 다른 유형의 메시지 (NPS를 RADIUS 프록시로 사용 됨) 다른 RADIUS 서버로 전달 됩니다.

연결 요청 정책을 사용 하 여 NPS를 RADIUS 서버로 또는 다음과 같은 요인을 기준으로 하는 RADIUS 프록시로 사용할 수 있습니다. 

- 시간 및 요일
- 연결 요청 영역 이름
- 요청 된 연결의 유형
- RADIUS 클라이언트의 IP 주소

RADIUS 액세스 요청 메시지 처리 또는 들어오는 메시지의 설정을 NPS에 구성 된 연결 요청 정책 중 하나 이상과 일치 하는 경우에 NPS를 통해 전달 됩니다. 

정책 설정이 일치 정책에 따라 NPS 메시지를 처리 하 고 인증 및 연결 요청 권한 부여 NPS를 RADIUS 서버로 작동 합니다. 정책 설정이 일치 정책에 따라 NPS 메시지를 전달 하 고 하는 경우 NPS RADIUS 프록시로 작동 하 고 처리를 위해 원격 RADIUS 서버에 연결 요청을 전달 합니다.

들어오는 RADIUS 액세스 요청 메시지의 설정을 연결 요청 정책 중 하나 이상 일치 하지 않는 경우에 액세스 거부 메시지를 RADIUS 클라이언트로 전송 되 고 사용자 또는 컴퓨터가 네트워크에 연결 하는 동안 액세스가 거부 됩니다.

## <a name="configuration-examples"></a>구성 예

다음 구성 예제에서는 연결 요청 정책을 사용 하는 방법을 보여 줍니다.

### <a name="nps-as-a-radius-server"></a>RADIUS 서버로 작동하는 NPS

기본 연결 요청 정책에만 구성 된 정책이입니다. 이 예제에서는 NPS를 RADIUS 서버로 구성 된 및 로컬 NPS에서 모든 연결 요청을 처리 합니다. NPS는 인증 하 고 NPS 도메인의 도메인 및 트러스트 된 도메인에 계정이 있는 사용자 권한을 부여할 수 있습니다.


### <a name="nps-as-a-radius-proxy"></a>RADIUS 프록시로 작동하는 NPS

기본 연결 요청 정책을 삭제 되 고, 두 개의 서로 다른 도메인에 요청을 전달 하기 위해 새 연결 요청 정책 두 개 만들어집니다. 이 예제에서는 NPS는 RADIUS 프록시로 구성 됩니다. NPS는 로컬 서버의 모든 연결 요청을 처리 하지 않습니다. 대신, NPS 또는 원격 RADIUS 서버 그룹의 구성원으로 구성 된 다른 RADIUS 서버로 연결 요청을 전달 합니다.


### <a name="nps-as-both-radius-server-and-radius-proxy"></a>RADIUS 서버와 RADIUS 프록시로 NPS

기본 연결 요청 정책의 경우 외에도 NPS 또는 트러스트 되지 않은 도메인에서 다른 RADIUS 서버에 연결 요청을 전달 하는 새 연결 요청 정책을 만들어집니다. 이 예제에서는 프록시 정책 정책의 정렬된 된 목록에서 가장 먼저 표시 됩니다. 연결 요청에 프록시 정책과 일치 하는 경우 연결 요청이 원격 RADIUS 서버 그룹의 RADIUS 서버로 전달 됩니다. 연결 요청이 프록시 정책과 일치 하지 않습니다 하지만 기본 연결 요청 정책과 일치 하는 경우 NPS는 로컬 서버의 연결 요청을 처리 합니다. 연결 요청 정책 중 하나 일치 하지 않는 경우 삭제 됩니다.

### <a name="nps-as-radius-server-with-remote-accounting-servers"></a>원격 계정 서버와 RADIUS 서버로 NPS

이 예제에서는 로컬 NPS 계정 작업을 수행 하도록 구성 되지 않은 하 고 기본 연결 요청 정책을 수정 해야 RADIUS 계정 메시지를 NPS 또는 다른 RADIUS 서버는 원격 RADIUS 서버 그룹에 전달 됩니다. 계정 메시지는 전달 하지만 인증 및 권한 부여 메시지 전달 되지 않습니다 및 로컬 도메인에 대 한 이러한 기능을 수행 하는 로컬 NPS 및 모든 트러스트 된 도메인입니다.


### <a name="nps-with-remote-radius-to-windows-user-mapping"></a>Windows 사용자 매핑으로 사용자 원격 RADIUS 사용 하는 NPS

이 예제에서는 NPS 권한 부여에 대 한 로컬 Windows 사용자 계정을 사용 하는 동안 원격 RADIUS 서버에 인증 요청을 전달 하 여 RADIUS 서버 및 각 개별 연결 요청에 대 한 RADIUS 프록시로 작동 합니다. 이 구성은 연결 요청 정책 조건으로 Windows 사용자 매핑 특성에 원격 RADIUS를 구성 하 여 구현 됩니다. (또한 사용자 계정을 만들어야 합니다 로컬로 원격 RADIUS 서버에서 인증을 수행 하는 원격 사용자 계정으로 동일한 이름을 가진.)

## <a name="connection-request-policy-conditions"></a>연결 요청 정책 조건

연결 요청 정책 조건은 들어오는 RADIUS 액세스 요청 메시지의 특성을 비교 하는 하나 이상의 RADIUS 특성입니다. 여러 조건을 경우 그런 다음 연결의 조건이 모두 요청 메시지 하 고 연결 요청 정책을 NPS에서 적용할 정책에 대 한 순서 대로 일치 해야 합니다.


다음은 연결 요청 정책에서 구성할 수 있는 사용 가능한 조건 특성입니다.

### <a name="connection-properties-attribute-group"></a>연결 속성 특성 그룹

연결 속성 특성 그룹에는 다음 특성을 포함 합니다.

- **프레임 프로토콜**합니다. 들어오는 패킷 프레이밍 형식을 지정 하는 데 사용 합니다. PPP (지점간 프로토콜), 직렬 줄 인터넷 프로토콜 (SLIP), 프레임 릴레이 및 X.25을 예로 들 수 있습니다.
- **서비스 유형을**합니다. 요청 되는 서비스의 유형을 지정 하는 데 사용 합니다. 예를 들어 프레임 (예: PPP 연결) 및 로그인 (예를 들어, 텔넷 연결). RADIUS 서비스 유형에 대 한 자세한 내용은 RFC 2865 "원격 인증 전화 접속 사용자 서비스 (RADIUS)입니다."을 참조 하세요.
- **터널 유형**합니다. 요청 하는 클라이언트에서 만들어지는 터널의 유형을 지정 하는 데 사용 합니다. 터널 종류에는 지점 간 터널링 프로토콜 (PPTP) 및 계층 2 터널링 프로토콜 (l2tp) 포함 됩니다.

### <a name="day-and-time-restrictions-attribute-group"></a>요일 및 시간 제한 특성 그룹 

요일 및 시간 제한 특성 그룹 요일 및 시간 제한 특성을 포함합니다. 이 특성을 사용 하 여는 요일 및 연결 시도의 시간을 지정할 수 있습니다. 날짜 및 시간 날짜와 NPS의 시간에 상대적입니다.

### <a name="gateway-attribute-group"></a>게이트웨이 특성 그룹

게이트웨이 특성 그룹에는 다음 특성을 포함 합니다.

- **호출 된 스테이션 ID**합니다. 네트워크 액세스 서버를의 전화 번호를 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. 지역 번호를 지정할 패턴 일치 구문을 사용할 수 있습니다.
- **NAS 식별자**합니다. 네트워크 액세스 서버 이름을 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. NAS 식별자를 지정 하는 패턴 일치 구문을 사용할 수 있습니다.
- **NAS IPv4 주소**합니다. 인터넷 프로토콜 버전 4 (IPv4)을 지정 하는 데 네트워크 액세스 서버 (RADIUS 클라이언트)의 주소입니다. 이 특성은 문자열입니다. IP 네트워크를 지정 하는 패턴 일치 구문을 사용할 수 있습니다.
- **NAS IPv6 주소**합니다. 인터넷 프로토콜 버전 6(ipv6)을 지정 하는 데 네트워크 액세스 서버 (RADIUS 클라이언트)의 주소입니다. 이 특성은 문자열입니다. IP 네트워크를 지정 하는 패턴 일치 구문을 사용할 수 있습니다.
- **NAS 포트 유형**합니다. Access 클라이언트에서 사용 된 미디어의 유형을 지정 하는 데 사용 합니다. 예로 아날로그 전화 회선 (이 비동기 이라고 함), 디지털 네트워크 ISDN (Integrated Services), 터널 또는 가상 사설망 (Vpn), IEEE 802.11 무선 및 이더넷 스위치입니다.

### <a name="machine-identity-attribute-group"></a>컴퓨터 Identity 특성 그룹

컴퓨터 Id 특성 그룹에 컴퓨터 Id 특성이 포함 되어 있습니다. 이 특성을 사용 하 여 정책에서 클라이언트를 식별 하는 메서드를 지정할 수 있습니다.

### <a name="radius-client-properties-attribute-group"></a>RADIUS 클라이언트 속성 특성 그룹

RADIUS 클라이언트 속성 특성 그룹에는 다음 특성을 포함 합니다.

- **호출 스테이션 ID**합니다. 호출자 (액세스 클라이언트)에서 사용 하는 전화 번호를 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. 지역 번호를 지정할 패턴 일치 구문을 사용할 수 있습니다.  802.1x 인증의 MAC 주소는 일반적으로 채워지고 클라이언트에서 일치 시킬 수 있습니다.  '자격 증명의 유효성을 검사 하지 않고 Accept 사용자'의 연결 요청 정책 구성 된 경우이 필드는 Mac 주소 바이패스 시나리오에 대 한 일반적으로 사용 됩니다.  
- **클라이언트 이름**합니다. 인증을 요청 하는 RADIUS 클라이언트 컴퓨터의 이름을 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. 클라이언트 이름을 지정 하려면 패턴 일치 구문을 사용할 수 있습니다.
- **클라이언트 IPv4 주소**합니다. 네트워크 액세스 서버 (RADIUS 클라이언트)의 IPv4 주소를 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. IP 네트워크를 지정 하는 패턴 일치 구문을 사용할 수 있습니다.
- **클라이언트 IPv6 주소**합니다. 네트워크 액세스 서버 (RADIUS 클라이언트)의 IPv6 주소를 지정 하는 데 사용 합니다. 이 특성은 문자열입니다. IP 네트워크를 지정 하는 패턴 일치 구문을 사용할 수 있습니다.
- **클라이언트 공급 업체**합니다. 인증을 요청 하는 네트워크 액세스 서버를 공급 업체를 지정 하는 데 사용 합니다. 라우팅 및 원격 액세스 서비스를 실행 하는 컴퓨터는 Microsoft NAS 제조업체입니다. NAS 제조업체 마다 별도 정책을 구성 하려면이 특성을 사용할 수 있습니다. 이 특성은 문자열입니다. 패턴 일치 구문을 사용할 수 있습니다.

### <a name="user-name-attribute-group"></a>사용자 이름 특성 그룹

사용자 이름 특성 그룹 사용자 이름 특성을 포함합니다. 이 특성을 사용 하 여 사용자 이름 또는 RADIUS 메시지에 대 한 액세스 클라이언트가 제공한 사용자 이름과 일치 해야 하는 사용자 이름 부분을 지정할 수 있습니다. 이 특성은 일반적으로 사용자 계정 이름 및 영역 이름을 포함 하는 문자열입니다. 사용자 이름을 지정 하려면 패턴 일치 구문을 사용할 수 있습니다.

## <a name="connection-request-policy-settings"></a>연결 요청 정책 설정

연결 요청 정책 설정은 들어오는 RADIUS 메시지에 적용 되는 속성 집합이 됩니다. 설정 속성의 다음 그룹으로 구성 됩니다.

- 인증
- 계정
- 특성 조작
- 요청 전달
- 고급

다음 섹션에서는 이러한 설정에 대 한 추가 세부 정보를 제공합니다.

### <a name="authentication"></a>인증

이 설정을 사용 하 여 모든 네트워크 정책에 구성 된 인증 설정을 재정의할 수 있습니다 및 인증 방법 및 네트워크에 연결 하는 데 필요한 형식을 지정할 수 있습니다.

>[!IMPORTANT]
>네트워크 정책에서 구성한 인증 방법 보다 덜 안전는 연결 요청 정책에서 인증 방법을 구성, 네트워크 정책에서 구성 하는 더 안전한 인증 방법을 재정의 됩니다. 예를 들어 보호 된 Extensible Authentication Protocol-Microsoft Challenge Handshake 인증 프로토콜 버전 2 사용 해야 하는 네트워크 정책을 둘 \(PEAP-2(MS-CHAP v2\)에 암호 기반 결과 클라이언트가 PEAP-2(MS-CHAP v2를 사용 하 여 인증 하는 데 필요한 무단된 액세스를 허용 하는 연결 요청 정책을 구성할 수도 보안 무선에 대 한 인증 방법을 지정 하 고 있습니다. 이 예제에서는 네트워크에 연결 하는 모든 클라이언트가 부여 된 인증 되지 않은 액세스 합니다.

### <a name="accounting"></a>계정

이 설정을 사용 하 여 원격 RADIUS 서버 그룹 계정 관리를 수행 하는 NPS 또는 다른 RADIUS 서버는 원격 RADIUS 서버 그룹에 계정 정보를 전달 하려면 연결 요청 정책을 구성할 수 있습니다.

>[!NOTE]
>여러 RADIUS 서버가 있는 경우 하나의 중앙 RADIUS 계정 데이터베이스에 저장 하는 모든 서버에 대 한 계정 정보를 원하는 설정을 사용할 수 있습니다는 연결 요청 정책 회계 각 RADIUS 서버의 정책에서에서 회계 데이터를 전달 하도록 모든 계정 서버로 지정 된 다른 RADIUS 서버 또는 NPS 서버.

연결 요청 정책 계정 설정은 로컬 NPS의 계정 구성과 무관 함수입니다. 즉, 로컬 파일 또는 Microsoft SQL Server 데이터베이스에 RADIUS 계정 정보를 기록 하도록 로컬 NPS를 구성 하는 경우이 수행을 원격 radius 계정 메시지를 전달 하는 연결 요청 정책을 구성 하는 여부에 관계 없이 서버 그룹입니다.

로컬이 아닌 원격으로 기록 하는 계정 정보를 하려는 경우에 원격 RADIUS 서버 그룹에 계정 데이터를 전달 하는 연결 요청 정책에서 계정 관리를 구성 하는 동안 계정 작업을 수행 하지 로컬 NPS를 구성 해야 합니다.

### <a name="attribute-manipulation"></a>특성 조작

다음 특성 중 하나의 텍스트 문자열을 조작 하는 찾기 및 바꾸기 규칙의 집합을 구성할 수 있습니다.

- 사용자 이름
- 호출된 된 스테이션 ID
- 호출 된 스테이션 ID

찾기 및 바꾸기 규칙 처리가 RADIUS 메시지 인증 및 계정 설정이 적용 되기 전에 앞의 특성 중 하나에 대해 발생 합니다. 특성 조작 규칙이 단일 특성에만 적용 됩니다. 각 특성에 대 한 특성 조작 규칙을 구성할 수 없습니다. 또한 조작할 수 있는 특성 목록에는 정적 목록입니다. 조작에 사용 가능한 특성의 목록에 추가할 수 없습니다.

>[!NOTE]
>2(MS-CHAP v2 인증 프로토콜을 사용 하는 경우에 RADIUS 메시지를 전달 하려면 연결 요청 정책을 사용 하는 경우 사용자 이름 특성을 조작할 수 없습니다. 유일한 예외는 경우 백슬래시 (\) 문자를 사용 하 고 조작의 왼쪽에 대 한 정보에만 영향을 줍니다. 백슬래시 문자는 일반적으로 도메인 이름 (의 정보를 왼쪽에 백슬래시 문자) 및 (백슬래시의 오른쪽에 있는 정보) 도메인 내에서 사용자 계정 이름을 나타내는 데 사용 됩니다. 이 경우에 특성 조작 하는 규칙 수정 하거나 도메인 이름 바꾸기 허용 됩니다.

사용자 이름 특성에 영역 이름을 조작 하는 방법의 예제 항목의 "사용자 이름 특성에 영역 이름 조작에 대 한 예제" 섹션을 참조 하세요 [NPS에서 정규식 사용](nps-crp-reg-expressions.md)합니다.

### <a name="forwarding-request"></a>요청 전달

다음을 설정할 수 있습니다 RADIUS 액세스 요청 메시지에 사용 되는 요청 옵션을 전달 합니다.

- **이 서버에서 요청을 인증**합니다. 이 설정을 사용 하 여 NPS 연결 요청을 인증 하는 Windows NT 4.0 도메인, Active Directory 또는 로컬 보안 계정 관리자 (SAM) 사용자 계정 데이터베이스 사용 합니다. 이 설정은 또한는 일치 하는 네트워크 정책을 NPS에서 사용자 계정에 전화 접속 속성을 함께 구성 하는 데 NPS를 통해 연결 요청을 인증을 지정 합니다. 이 경우 NPS는 RADIUS 서버로 수행 하도록 구성 됩니다.

- **다음 원격 RADIUS 서버 그룹에 요청 전달**합니다. 이 설정을 사용 하 여 NPS 지정 하는 원격 RADIUS 서버 그룹에 연결 요청을 전달 합니다. NPS에서 액세스 요청 메시지에 해당 하는 유효한 액세스 허용 메시지를 수신 하는 경우 연결 시도 권한이 부여 되었고 인증으로 간주 됩니다. 이 경우에 NPS를 RADIUS 프록시로 작동합니다.

- **자격 증명의 유효성을 검사 하지 않고 사용자를 수락할**합니다. 이 설정을 사용 하면 네트워크에 연결을 시도 하는 사용자의 id를 확인 하지 않으며 및 NPS 사용자 또는 컴퓨터에 네트워크에 연결할 수 있는 권한이 있는지 확인 하려고 시도 하지 않습니다. 인증 되지 않은 액세스를 허용 하도록 구성 된 NPS 연결 요청을 받으면를 NPS 즉시 보냅니다 RADIUS 클라이언트와 사용자 또는 컴퓨터에 대 한 액세스 허용 메시지를 네트워크 액세스 권한이 부여 됩니다. 이 설정은 일부 유형의 강제 터널링 되는 액세스 클라이언트 사용자 자격 증명을 인증 하려면 먼저 사용 됩니다.

>[!NOTE]
>액세스 클라이언트의 인증 프로토콜은 2(MS-CHAP v2 또는 확장할 수 있는 인증 프로토콜 전송 계층 보안 (EAP-TLS)는 모두 상호 인증을 제공 하는 경우에이 인증 옵션을 사용할 수 없습니다. 상호 인증에 access 클라이언트 증명 (NPS) 서버로 인증 하는 유효한 액세스 클라이언트 인증 서버 액세스 클라이언트에 유효한 인증 서버 임을 증명 합니다. 이 인증 옵션을 사용 하는 경우 액세스 허용 메시지 반환 됩니다. 그러나 인증 서버는 액세스 클라이언트를 유효성 검사를 제공 하지 않습니다 하 고 상호 인증이 실패 합니다.

항목의 "프록시 서버에서 RADIUS 메시지 전달에 대 한" 예제"섹션을 참조 하는 정규식을 사용 하 여 원격 RADIUS 서버 그룹에 지정 된 영역 이름을 사용 하 여 RADIUS 메시지를 전달 하는 라우팅 규칙을 만드는 방법의 예제 [사용 NPS의 정규식](nps-crp-reg-expressions.md)합니다.

### <a name="advanced"></a>고급

일련의 RADIUS 특성을 지정 하는 고급 속성을 설정할 수 있습니다.

- RADIUS 인증 또는 회계 서버로 NPS를 사용 하는 경우 RADIUS 응답 메시지에 추가 합니다. 네트워크 정책 및 연결 요청 정책에 지정 된 특성의 경우 RADIUS 응답 메시지에 전송 되는 특성을 특성의 두 집합의 조합이 됩니다.
- RADIUS 인증 또는 회계 프록시로 NPS를 사용 하는 경우 RADIUS 메시지에 추가 합니다. 특성에 전달 되는 메시지에 이미 있는 경우 연결 요청 정책에서 지정 된 특성의 값으로 대체 됩니다. 

연결 구성에 사용할 수 있는 일부 특성 정책을 요청 하는 또한 **설정** 탭에 **고급** 특수화 된 기능을 제공 하는 범주입니다. 예를 들어, 구성할 수 있습니다 합니다 **Windows 사용자 매핑 원격 RADIUS** 두 사용자 간의 연결 요청 권한 부여 데이터베이스 계정 및 인증을 분할 하려는 경우 특성입니다.

합니다 **Windows 사용자 매핑 원격 RADIUS** 특성 Windows 권한 부여는 원격 RADIUS 서버에서 인증 된 사용자에 대해 발생 하도록 지정 합니다. 즉, 원격 RADIUS 서버는 원격 사용자 계정 데이터베이스에서 사용자 계정에 대해 인증을 수행 하지만 로컬 NPS 연결 요청을 로컬 사용자 계정 데이터베이스에서 사용자 계정에 대해 권한을 부여 합니다. 방문자가 네트워크에 대 한 액세스를 허용 하려는 경우에 유용 합니다.

예를 들어, 방문자가 파트너 조직에서 파트너 조직의 RADIUS 서버 자체에서 인증 될 수 있습니다 및 조직에서 Windows 사용자 계정을 사용 하 여 게스트 로컬 영역 네트워크 (LAN) 네트워크에서 액세스할 수 합니다.

특수화 된 기능을 제공 하는 다른 특성은:

- **MS-격리-IPFilter 및 MS 격리-세션 제한**합니다. 이러한 특성은 라우팅 및 원격 액세스 VPN 배포를 사용 하 여 네트워크 액세스 격리 제어 환경에서 NAQC ()를 배포할 때 사용 됩니다.
- **Passport-User-Mapping-UPN-Suffix**. 이 특성을 사용 하면 Windows Live™ ID 사용자 계정 자격 증명을 사용 하 여 연결 요청을 인증할 수 있습니다.
- **터널 태그**합니다. 이 특성에는 연결 할당 해야 하는 NAS에서 가상 로컬 영역 네트워크 (Vlan)을 배포 하는 경우 VLAN ID 번호를 지정 합니다.

## <a name="default-connection-request-policy"></a>기본 연결 요청 정책

기본 연결 요청 정책은 NPS를 설치할 때 만들어집니다. 이 정책에 다음 구성 합니다.

- 인증 구성 되지 않았습니다.
- 회계 원격 RADIUS 서버 그룹에 계정 정보를 전달 하도록 구성 되지 않았습니다.
- 특성 원격 RADIUS 서버 그룹에 연결 요청을 전달 하는 특성 조작 규칙을 사용 하 여 구성 되지 않았습니다.
- 요청을 전달 하면 연결 요청을 인증 하 고 로컬 NPS에서 승인 되도록 구성 됩니다.
- 고급 특성 구성 되지 않습니다.

NPS를 RADIUS 서버로 사용 하는 기본 연결 요청 정책을 합니다. RADIUS 프록시로 NPS를 실행 하는 서버를 구성 하려면 또한 원격 RADIUS 서버 그룹을 구성 해야 합니다. 새 연결 요청 정책 마법사를 사용 하 여 새 연결 요청 정책을 만드는 동안 새 원격 RADIUS 서버 그룹을 만들 수 있습니다. 기본 연결 요청 정책을 삭제 하거나 기본 연결 요청 정책을 정책의 정렬된 된 목록에서 마지막으로 배치 하 여 NPS를 통해 처리 된 마지막 정책 인지 확인 합니다.

>[!NOTE]
>NPS 및 원격 액세스 서비스가 설치 된 경우 동일한 컴퓨터에 원격 액세스 서비스 Windows 인증을 위해 구성 되 고,이 원격 액세스 인증 및 계정 요청을 RADIUS 서버로 전달 될 수 . 원격 액세스 인증 및 계정 요청을 원격 RADIUS 서버 그룹에 전달 하도록 구성 된 연결 요청 정책에 일치 하는 경우 발생할 수 있습니다.
