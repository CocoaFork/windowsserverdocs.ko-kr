# <a name="plan-nps-as-a-radius-server"></a>NPS RADIUS 서버로 계획

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

인증 전화에서 사용자 RADIUS (원격 서비스) 서버 네트워크 정책 서버 \(NPS\) 배포할 때 NPS 기다렸다가 및 기다렸다가 신뢰 하는 도메인에 연결 요청을 승인, 인증과 계정을 수행 합니다. 간단 하 게 RADIUS 배포를 계획 다음 지침을 사용할 수 있습니다.

이러한 계획 지침 NPS RADIUS 프록시 배포 하려는 상황 포함 되지 않습니다. NPS RADIUS 프록시 배포할 때 NPS 전달 NPS 또는 기타 RADIUS 서버 원격 도메인, 신뢰할 수 없는 도메인 또는 둘 다에서 실행 하는 서버에 요청을 연결 합니다. 

네트워크에서 NPS RADIUS 서버 배포 하기 전에 배포를 계획을 다음 지침을 따릅니다.

- 서버 구성 NPS 계획입니다.

- RADIUS 클라이언트 계획입니다.

- 계획 인증 방법 사용 합니다.

- 네트워크 정책을 계획입니다.

- NPS 계정 관리를 계획입니다.

## <a name="plan-nps-server-configuration"></a>서버 구성 NPS 계획

도메인 NPS 서버가 구성원 결정 해야 합니다. 여러 도메인 환경에 대 한 NPS 서버 자격 증명 하는 것이 구성원 중 도메인에 있는 사용자 계정에 대 한 및 NPS 서버 기다렸다가 신뢰 하는 모든 도메인에 대 한 인증할 수 있습니다. NPS 서버 인증 과정에서 사용자 계정의 전화 속성 읽을 수 있도록 NPS 서버 컴퓨터 계정 각 도메인 RAS 및 NPS 서버 그룹에 추가 해야 합니다.

도메인 멤버십 NPS 서버를 확인 한 후 클라이언트 RADIUS RADIUS 프로토콜을 사용 하 여 네트워크 액세스 서버 라고도 소통 하 고 서버를 구성 합니다. 또한 구성할 수 있습니다 유형의 이벤트 NPS 기록 하 로그 하 고 서버에 대 한 설명을 입력 수 있는 경우에 있습니다.

### <a name="key-steps"></a>주요 단계

NPS 서버 구성에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- NPS 서버 RADIUS 클라이언트에서 RADIUS 메시지를 받을 때 사용 하는 RADIUS 포트를 확인 합니다. 기본 포트가 1812 및 RADIUS 인증 메시지에 대 한 1645 UDP 포트와 1813와 1646 RADIUS 계정 메시지에 대 한 포트 합니다.

- 네트워크 어댑터가 여러 NPS 서버 구성 된를 허용할 RADIUS 교통 관리할 어댑터 확인 합니다.

- NPS 이벤트 로그에 기록 하 고 싶은 이벤트의 종류를 결정 합니다. 요청이 거부 된 인증, 성공적으로 인증 요청 또는 두 가지 유형의 요청 로그인 할 수 있습니다.

- 둘 이상의 NPS 서버 배포 하는 지 여부를 결정 합니다. 를 제공 하기 위해 결함 허용 RADIUS 기반 인증과 계정에 대 한 두 개 이상의 NPS 서버를 사용 합니다. 주 RADIUS 서버 한 NPS 서버를 사용 하 고 백업으로 사용 되는 다른 합니다. 각 RADIUS 클라이언트 NPS 서버 구성 됩니다. 주 NPS 서버를 사용할 수 없는 경우 RADIUS 클라이언트 대체 NPS 서버에 액세스 요청 메시지 전송 합니다.

- 다른 NPS 서버 관리를 저장 하 고 서버 잘못 구성을 방지 한 NPS 서버 구성 복사 하는 데 사용 스크립트를 계획입니다. NPS 다른 NPS 서버에 가져오기 NPS 서버 구성 된의 전부 또는 일부를 복사할 수 있도록 Netsh 명령을 제공 합니다. 명령 프롬프트에서 Netsh 수동으로 실행할 수 있습니다. 그러나 명령 시퀀스 스크립트 파일로 저장 하면 서버 구성 변경 하려는 경우 나중에 스크립트를 실행할 수 있습니다.

## <a name="plan-radius-clients"></a>요금제 RADIUS 클라이언트

RADIUS 클라이언트는 무선 액세스 포인트 등 네트워크 액세스 서버 가상 개인 네트워크 (VPN) 서버, 802.1 X 가능 스위치 및 전화 접속 서버 합니다. 연결 RADIUS 서버에 요청 메시지를 전달 하는 RADIUS 프록시 RADIUS 클라이언트 수도 있습니다. NPS 모든 네트워크 액세스 서버를 지원 하 고 있는 RADIUS 준수 RADIUS 프록시 프로토콜 2865 RFC "원격 인증 전화 접속 사용자 서비스 (RADIUS)"에 설명 된 대로 RFC 2866 "RADIUS 계정을."

>[!IMPORTANT]
>클라이언트 컴퓨터 등 액세스 클라이언트는 RADIUS 클라이언트 되지 않습니다. 네트워크 액세스 서버 및 RADIUS 프로토콜을 지원 하는 프록시 서버 RADIUS 클라이언트 됩니다.

뿐만 아니라 무선 액세스 포인트 및 스위치 수 있어야 802.1 X 인증 합니다. 확장 인증 프로토콜 \(EAP\) 또는 Extensible 인증 프로토콜 보호 \(PEAP\) 배포 하려면 액세스 지점 수와 스위치 EAP 사용을 지원 해야 합니다.

무선 액세스 포인트에 대 한 PPP 연결에 대 한 기본 상호 운용성, 테스트 하 고 액세스 지점과 인증 프로토콜 PAP (암호)를 사용 하 여 액세스 클라이언트를 구성 합니다. 네트워크 액세스를 위해 사용 하는 기능을 테스트 될 때까지 PEAP 같은 추가 PPP 기반 인증 프로토콜을 사용 합니다.

### <a name="key-steps"></a>주요 단계

RADIUS 클라이언트에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 문서 공급 업체 특정 특성 (Vsa) NPS 구성 해야 합니다. 네트워크 액세스 서버 Vsa를 요구 하는 경우 네트워크 정책 NPS에서 구성한 경우 나중에 사용할 VSA 정보를 기록 합니다. 

- 모든 장치의 구성을 간소화 하기 위해 RADIUS 클라이언트 및 서버 NPS의 IP 주소를 기록 합니다. RADIUS 클라이언트 배포할 때 인증 서버를 입력 한 NPS 서버 IP 주소와 RADIUS 프로토콜을 사용 하도록 구성 해야 합니다. 및 NPS 스냅인에 RADIUS 클라이언트 IP 주소를 입력 해야 NPS RADIUS 클라이언트 통신할 수를 구성 합니다.

- 스냅인 NPS RADIUS 클라이언트 켜고 구성에 맞게 공유 암호를 만듭니다. 암호, 공유 또는 암호를 입력할 NPS 스냅인 NPS RADIUS 클라이언트 구성 하는 동안에 RADIUS 클라이언트를 구성 해야 합니다.

## <a name="plan-the-use-of-authentication-methods"></a>계획 인증 방법 사용

NPS 모두 암호 및 인증서 기반 인증 방법을 지원합니다. 그러나 모든 네트워크 액세스 서버 동일한 인증 방법을 지원합니다. 경우에 따라 액세스 네트워크의 종류에 따라 다른 인증 방법을 배포 하는 것이 좋습니다.

예를 들어, 하려는 경우 무선 고 조직에 대 한 액세스 VPN 배포 되지만 다른 인증 방법을 각 종류의 액세스를 사용 하 여: EAP-TLS EAP Transport Layer Security (EAP-TLS)를 제공 하는 강력한 보안 인해 VPN 연결 및 암호 PEAP v2 802.1 X 무선으로 연결 합니다.

Microsoft 인증 프로토콜 버전 (PEAP 암호 v2) 2 빠른 이라는 기능이 제공 된 PEAP 특히 노트북 및 다른 무선 디바이스를 사용 하기 위해 설계 된를 다시 연결 합니다. 빠른 새로운 액세스 포인트에 연결 될 때마다 검색 하지 않고 동일한 네트워크에 무선 액세스 포인트 간에 이동할 수 있도록 무선 클라이언트 다시 연결 합니다. 무선 사용자 더 나은 환경을 제공 하 고 자격 증명을 다시 입력할 필요 없이 액세스 지점 간에 이동할 수 있습니다.
패스트로 인해 다시 연결 하 고 무선 연결을 위한 인증 방법으로 암호 PEAP v2 보안 PEAP 암호 v2에서 제공 하는 논리 선택 합니다.

VPN 연결에 대 한 EAP-TLS 조직 VPN 서버에 home 또는 모바일 컴퓨터에서 인터넷을 통해 전송 될 때에 네트워크 교통량을 보호 하는 강력한 보안을 제공 하는 인증서 기반 인증 방법입니다.

### <a name="certificate-based-authentication-methods"></a>인증 인증서 기반 방법

가지 옵션이 인증 방법을 강력한 보안을 제공 하 고 이용 고 보다 암호 기반 인증 방법을 배포 하기는 어렵습니다 더 되 고 단점은 가집니다.

암호 PEAP v2 및 EAP-TLS 가지 옵션이 인증 방법에는 이지만 많은 차이점을 및 배포 되는 방식입니다.

### <a name="eap-tls"></a>EAP-TLS

EAP-TLS 클라이언트 및 서버 인증을 위한 인증서를 사용 하 여 있으며 조직의 공개 키 infrastructure (PKI)을 배포 합니다. 배포 PKI 복잡 수 있으며 NPS RADIUS 서버로 사용에 대 한 계획 관계 계획 단계 필요 합니다.

Eap-tls, NPS 서버 서버 인증서를 등록 인증서 스토어에서 로컬 컴퓨터에서 \(CA\), 그리고 인증서 인증 기관에서 저장 합니다. 인증 하는 동안 서버 인증 경우 NPS 서버 해당 id를 액세스 클라이언트 증명 액세스 클라이언트는 서버 인증서를 보냅니다. 액세스 클라이언트 검사 다양 한 인증서 속성을 인증서 유효 있는지 확인 하는 동안 서버 인증 사용 하기에 적합 합니다. 서버 인증서 최소 서버 인증서 요구 사항을 충족 발급 캘리포니아 액세스 클라이언트 신뢰 하는 경우, NPS 서버 클라이언트 성공적으로 인증 됩니다.

마찬가지로, 클라이언트 인증 클라이언트 클라이언트 인증서 NPS 서버에 해당 id를 증명 NPS 서버를 보냅니다 때 인증 도중 발생 합니다. 액세스 클라이언트는 NPS 서버 성공적으로 인증 클라이언트 인증서 인증서 클라이언트 최소 요구 사항을 충족 NPS 서버 신뢰 하는 캘리포니아에서 발급 있으면 및 NPS 서버 인증서를 검사 합니다.

이지만, 서버 인증서 인증서 저장소 NPS 서버에 저장 된 필요 클라이언트 또는 사용자 인증서에에서 저장할 수 중 인증서 저장소 클라이언트 또는 스마트 카드에 있습니다.

성공 하려면이 인증 프로세스에 대 한 것이 필요한 모든 컴퓨터 로컬 컴퓨터와 현재 사용자에 대 한 신뢰할 수 있는 인증 기관 인증서 스토어에서 선택 된 인증서 조직의 되었는지 합니다.

### <a name="peap-ms-chap-v2"></a>암호 PEAP v 2

PEAP 암호 v2 서버 인증 및 암호 기반 자격 증명 사용자 인증 인증서를 사용 합니다. 서버 인증을 위한 인증서를 사용할 수 PKI PEAP 암호 v 2를 사용 하기 위해 구축 필요 하지 않습니다. 암호 PEAP v2 배포 하는 경우는 다음과 같은 두 가지 방법 중 하나에 NPS 서버에 대 한 서버 인증서를 얻을 수 있습니다.

- NPS 서버로 Active Directory 인증서 서비스 (광고 CS)을 선택한 다음 자동 등록 인증서를 설치할 수 있습니다. 이 방법을 사용 하는 경우 캐나다 인증서 NPS 서버에 발급 된 인증서를 신뢰 하 않도록 네트워크에 연결 하는 클라이언트 컴퓨터를도 등록 해야 합니다.

- 서버 인증서 VeriSign 등 공개 캘리포니아에서 구입할 수 있습니다. 이 방법을 사용 하는 경우 클라이언트 컴퓨터에서 이미 신뢰할 수 있는 캘리포니아 선택 했는지 확인 합니다. 클라이언트 컴퓨터 신뢰를 캘리포니아 있는지 여부를 확인 하려면 클라이언트 컴퓨터에서 인증서 Management Console MMC (Microsoft) 스냅인 열고 로컬 컴퓨터에 대 한 현재 사용자 신뢰할 수 있는 인증 기관 스토어를 확인 합니다. 캘리포니아 이러한 인증서 스토어에서의 인증서를 클라이언트 컴퓨터는 캘리포니아 신뢰 하 고는 캐나다에서 발급 된 인증서를 신뢰 따라서 됩니다.

암호 PEAP v 2 인증 과정 서버 인증 경우 NPS 서버 서버 인증서 클라이언트 컴퓨터를 보냅니다. 액세스 클라이언트 검사 다양 한 인증서 속성을 인증서 유효 있는지 확인 하는 동안 서버 인증 사용 하기에 적합 합니다. 서버 인증서 최소 서버 인증서 요구 사항을 충족 발급 캘리포니아 액세스 클라이언트 신뢰 하는 경우, NPS 서버 클라이언트 성공적으로 인증 됩니다.

사용자 인증 사용자가 네트워크 종류 암호 기반 자격 증명에 연결 하려고 할 때 발생 하 고 로그온 하려고 합니다. NPS 자격 증명을 수신 하 고 인증과 수행 합니다. 사용자가 인증를 성공적으로 허가 있고 클라이언트 컴퓨터 NPS 서버 인증 연결 요청 부여 됩니다.

### <a name="key-steps"></a>주요 단계

인증 방법의 사용에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 네트워크 액세스를 제공, 무선, VPN, 802.1 등 X 지원 전환 하려는 유형과 전화 접속 연결을 확인 합니다.

- 인증 방법을 또는 각 종류의 액세스에 사용 하려는 하는 방법을 확인 합니다. 강력한 보안을 제공 하는 인증서 기반 인증 방법을 사용 하는 것이 좋습니다. 그러나 그 수행할 수 없습니다 인증 방법을 다른 네트워크에 필요한의 원활을 제공할 수 있도록 PKI 배포할 수 있습니다.

- EAP-TLS 배포 하는 경우 PKI 배포를 계획 합니다. 계획 컴퓨터 인증서 클라이언트 및 서버 인증서를 사용 하는 것 인증서 템플릿이 포함 됩니다. 도메인 구성원 및 비 도메인 구성원 컴퓨터 스마트 카드를 사용할 것인지 여부를 결정 하는 인증서를 등록 하는 방법을 결정 포함 되어 있습니다.

- 암호 PEAP v2를 배포 하는 광고 CS NPS 서버 또는 VeriSign 등 공개 캐나다에서 서버 인증서 구입 하려는 문제 서버 인증서를 설치 하려면 여부를 결정 합니다.

### <a name="plan-network-policies"></a>요금제 네트워크 정책

네트워크 정책 NPS RADIUS 클라이언트에서 받은 연결 요청을 실시 권 자 여부는 결정 하기 위해 사용 됩니다. NPS에서는 승인 결정을 사용자 계정의 전화 속성 있습니다.

네트워크 정책 NPS 스냅인에 표시 되는 순서 대로 처리 됩니다을 하기 때문에 정책 목록에서 가장 제한적인 정책 먼저 배치 예정입니다. 각 연결 요청에 대 한 NPS 연결 요청 속성와 정책 조건과 일치 하려고 합니다. 일치 하는 항목을 찾을 때까지 NPS 순서 대로 각각 네트워크 정책을 검사 합니다. 일치 하는 항목을 찾을 수 없으면 연결 요청이 거부 됩니다.

### <a name="key-steps"></a>주요 단계

네트워크 정책에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 제한이 관리자에서 네트워크 정책 기본 NPS 처리 순서를 확인 합니다.

- 정책 상태를 확인 합니다. 정책 상태 사용 하거나 사용 하지 않도록 설정 하는 가치에 있을 수 있습니다. 이 설정을 NPS 정책을 인증을 수행 하는 동안 평가 합니다. 정책 사용 하지 않는 경우 평가 되지 않습니다.

- 정책 형식을 확인 합니다. 정책 정책 조건과 연결 요청 하거나 여부 정책을 하도록 정책 조건과 연결 요청 일치 하는 때 액세스 거부 일치 하는 경우 액세스 권한을 부여 하도록 되어 있는지 여부를 확인 해야 합니다. 예를 들어 Windows 그룹 구성원을 무선 액세스 거부 하려면 그룹, 무선 연결 방법 및에 액세스 거부의 설정을 입력 정책이 지정 하는 네트워크 정책을 만들 수 있습니다.

- 그룹 정책을 삼고의 회원 사용자 계정의 전화 속성을 무시 하도록 NPS 있는지 여부를 결정 합니다. 이 설정을 활성화 되지 않은 경우 전화 접속 속성의 사용자 계정 네트워크 정책에서 구성 된 설정을 재정의 합니다. 예를 들어, 사용자에 대 한 액세스 권한을 부여 하는 네트워크 정책 구성 하는 경우 전화 접속 속성 해당 사용자에 대 한 사용자 계정 액세스 거부 하려면 설정의 사용자는 액세스할 수 없습니다. 하지만 같은 사용자 네트워크에 대 한 액세스를 얻은 정책 유형 무시 사용자 계정 전화에서 설정을 속성을 설정한 경우 합니다.

- 정책 정책을 원본 사용 하는지 확인 합니다. 이 설정을 쉽게 모든 액세스 요청에 대 한 소스를 지정할 수 있습니다. 가능한 한 소스는 Terminal Services 게이트웨이 (TS 게이트웨이), (VPN 또는 접속) 원격 액세스 서버, DHCP, 무선 액세스 포인트 및 건강 등록 기관 서버 있습니다. 또는 공급 업체 특정 소스를 지정할 수 있습니다.

- 네트워크 정책 적용 되도록 하기 위해에서 일치 해야 하는 조건 확인 합니다.

- 네트워크 정책 조건과 연결 요청 일치 하는 경우 적용 되는 설정을 확인 합니다.

- 사용 하 여, 수정 또는 삭제 기본 네트워크 정책 것인지 여부를 결정 합니다.

## <a name="plan-nps-accounting"></a>요금제 NPS 계정 관리

NPS RADIUS 계정 데이터, 사용자 인증과 계정 요청 같은 세 가지 형식 로그인 할 수 있게: IAS 형식, 데이터베이스 호환 형식 및 Microsoft SQL Server 로깅 합니다. 

IAS 및 데이터베이스 호환 텍스트 파일 형식에는 로컬 NPS 서버에 로그 파일을 만듭니다. 

SQL Server 로깅 SQL Server 2000 또는 경우도 로깅 이점을 활용할 수 있도록 RADIUS 계정을 확장 SQL Server 2005 XML 호환 데이터베이스에 로그인 하는 기능을 제공 합니다.

### <a name="key-steps"></a>주요 단계

NPS 계정에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 로그 파일 또는 SQL Server 데이터베이스 NPS 계정 데이터를 저장 것인지 여부를 결정 합니다.

### <a name="nps-accounting-using-local-log-files"></a>로컬 로그 파일을 사용 하 여 NPS 계정

녹음/녹화 로그 파일에서 인증과 계정 요청 사용자 주로 연결 분석 및 청구 목적을 위해 사용 및 공격 이후 악의적인 사용자의 활동을 추적 하기 위한 방법으로 사용자에 게 제공 하는 보안 조사 도구도 큰 도움이 됩니다.

### <a name="key-steps"></a>주요 단계

로컬 로그 파일을 사용 하 여 NPS 계정에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- NPS 로그 파일에 사용 하려는 텍스트 파일 형식을 확인 합니다.

- 로그인에 원하는 정보를 선택 합니다. 계정 요청, 인증 요청 및 정기 상태 로그인 할 수 있습니다.

- 하드 디스크 위치 로그 파일을 저장 하려는 장소를 확인 합니다.

- 로그 파일 백업 솔루션을 디자인 합니다. 로그 파일을 저장 하는 하드 디스크 위치 데이터를 백업 쉽게 할 수 있는 위치에 있어야 합니다. 또한, 하드 디스크 위치 로그 파일을 저장할 폴더에 대 한 액세스를 제어 목록 (ACL) 구성 보호 해야 합니다.

- 새 로그 파일을 원하는 주파수를 확인 합니다. 파일 크기에 따라 만들어야 로그 파일 하려는 경우 새 로그 파일을 nps 생성 하기 전에 수 있는 최대 파일 크기를 확인 합니다.

- 하드 디스크 공간이 부족 한 경우 이전 로그 파일을 삭제 하려면 NPS 있는지 여부를 결정 합니다.

- 응용 프로그램 또는 사용할 계정 데이터를 보고 보고서를 작성 하는 응용 프로그램을 확인 합니다.

### <a name="nps-sql-server-logging"></a>NPS SQL Server 로깅

NPS SQL Server 로깅 세션 상태 정보 보고서를 만들고 분석 데이터에 대 한 및 집중 하 고 계정 데이터를 관리를 간소화 하 필요한 경우 사용 됩니다.

NPS는 SQL Server 녹화 사용자 인증 기록 하 고 계정 SQL Server 2000 나중에 Microsoft SQL Server 데스크톱 엔진 \(MSDE 2000\) 또는 SQL Server의 모든 버전을 실행 하는 컴퓨터에 데이터 소스 하나 이상의 네트워크 액세스 서버에서 받은 요청을 사용 하는 기능을 제공 합니다.

계정 데이터도 전달 됩니다 NPS에서 XML 형식에서 모두 구조적된 쿼리 언어를 지원 하는 데이터베이스에 저장된 프로시저 \(SQL\) 및 XML \(SQLXML\) 합니다. 사용자 인증을 녹화/녹음 하 고 계정 XML 호환 SQL Server 데이터베이스에 요청 여러 NPS 서버 하나의 데이터 소스를 수 있습니다.

### <a name="key-steps"></a>주요 단계

계획 하는 동안은 NPS 계정에 대 한 NPS SQL Server 로깅 사용 하 여, 다음과 같은 단계를 사용할 수 있습니다.

- 또는 회사의 다른 구성원 SQL Server 2000 또는 SQL Server 2005 경우도 개발 환경에 있는지 확인 하 고 이러한 제품 만들고, 수정 하 고, 관리, SQL Server 데이터베이스 관리를 사용 하는 방법을 파악 합니다.

- SQL Server NPS 서버 또는 원격 컴퓨터에 설치 되어 있는지 확인 합니다.

- 디자인 프로시저 SQL Server 데이터베이스에 NPS 계정 데이터를 포함 하는 수신 XML 파일을 처리 하는 데 사용 됩니다.

- 데이터베이스 복제 구조 SQL Server 및 흐름 디자인 합니다.

- 응용 프로그램 또는 사용할 계정 데이터를 보고 보고서를 작성 하는 응용 프로그램을 확인 합니다.

- 모든 계정 요청 Class 특성 전송 하는 네트워크 액세스 서버를 사용 하 여 예정입니다. Class 특성 RADIUS 클라이언트는 액세스 허용 메시지에 게 전송 하 고 계정 요청 메시지 인증 세션으로 연결 하는 데 유용 합니다. Class 특성 계정 요청 메시지의 네트워크 액세스 서버에 전송 하는 경우 계정 및 인증 기록에 맞게 사용할 수 있습니다. 서비스-다시 부팅 시간을 및 서버 주소 특성 고유-일련 번호의 조합 서버를 허용 하는 각 인증을 위한 고유 id 해야 합니다.

- 네트워크 액세스 서버 임시 계정을 지 원하는 사용 하 여 예정입니다.

- 계정 설정 / 해제 메시지 보내기 네트워크 액세스 서버를 사용 하 여 예정입니다.

- 네트워크 액세스 지 원하는 서버에 저장 하 고 및 전달 계정 데이터를 사용 하려면 예정입니다. 네트워크 액세스 서버 NPS 서버 통신할 수 없는 경우이 기능을 지 원하는 네트워크 액세스 서버 계정 데이터를 저장할 수 있습니다. NPS 서버를 사용할 수 있는 네트워크 액세스 서버를 통해이 기능을 제공 하지 않는 네트워크 액세스 서버 계정의 안정성을 제공 하는 NPS 서버에 저장 된 기록 전달 합니다.

- 항상 계정 중간 간격 특성 네트워크 정책에서 구성 예정입니다. 계정 중간 간격 특성 네트워크 액세스 서버에 전송 하는 중간 업데이트 각 사이의 초에서 간격을 설정 합니다. 한번에 따라 계정 중간 간격 특성 값 60 초 하거나 1 분 미만의 야 하며 600 초 또는 10 분 미만의 수 없습니다. 자세한 내용은 참조 한번 "RADIUS 확장 합니다."

- 로그인은 정기적으로 상태 NPS 서버에서 활성화 되었는지 확인 합니다.
