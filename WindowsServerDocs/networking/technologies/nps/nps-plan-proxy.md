# <a name="plan-nps-as-a-radius-proxy"></a>NPS RADIUS 프록시 계획

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

네트워크 NPS 정책 서버 () 사용자 서비스에서 전화 인증 원격 \(RADIUS\) 프록시 배포할 때 NPS 네트워크 액세스 서버 등 RADIUS 클라이언트의 연결이 요청 또는 기타 RADIUS 프록시 받아서 다음 전달 NPS RADIUS 서버 다른를 실행 하는 서버에 대 한 이러한 연결 요청 합니다. 간단 하 게 RADIUS 배포를 계획 다음 지침을 사용할 수 있습니다.

이러한 계획 지침 NPS RADIUS 서버 배포 하려는 상황 포함 되지 않습니다. NPS RADIUS 서버, 배포할 때 NPS 기다렸다가 및 기다렸다가 신뢰 하는 도메인에 연결 요청을 승인, 인증과 계정을 수행 합니다.

네트워크에서 NPS RADIUS 프록시 배포 하기 전에 배포를 계획을 다음 지침을 따릅니다.

- 서버 구성 NPS 계획입니다.

- RADIUS 클라이언트 계획입니다.

- 원격 RADIUS 서버 그룹 계획입니다.

- 메시지를 전달 특성 조작 규칙 계획입니다.

- 연결 요청 정책 계획입니다.

- NPS 계정 관리를 계획입니다.

## <a name="plan-nps-server-configuration"></a>서버 구성 NPS 계획

NPS RADIUS 프록시도 사용 하 여 NPS 전달 NPS 서버 또는 처리 하기 위해 다른 RADIUS 서버에 연결 요청 합니다. 이 인해 NPS 프록시 도메인 멤버십 적합 하지 않습니다. 프록시 전화 접속 속성의 사용자 계정에 액세스 하지 않아도 Active Directory 도메인 서비스 \(AD DS\)에 등록할 필요는 없습니다. 또한 연결 요청에 대 한 프록시 인증을 수행 하지 않는 때문에 NPS 프록시에서 네트워크 정책을 구성 필요는 없습니다. NPS 프록시 도메인 구성원 또는 없이 도메인 멤버 자격으로 독립 실행형 서버 될 수 있습니다.

NPS는 RADIUS 프로토콜을 사용 하 여 네트워크 액세스 서버 라고도 RADIUS 클라이언트와 통신 하도록 설정 해야 합니다. 또한 구성할 수 있습니다 유형의 이벤트 NPS 기록 하 로그 하 고 서버에 대 한 설명을 입력 수 있는 경우에 있습니다.

### <a name="key-steps"></a>주요 단계

프록시 구성을 NPS에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 프록시 NPS RADIUS 클라이언트의 RADIUS 메시지를 받을 수 및 원격 RADIUS 서버 그룹의 회원 RADIUS 메시지를 보낼 수를 사용 하 여 RADIUS 포트를 확인 합니다. 기본 사용자 데이터 그램 Protocol (UDP) 포트가 1812 및 RADIUS 인증 메시지 및 UDP 포트 1813 및 RADIUS 계정 메시지에 대 한 1646 1645 않습니다.

- NPS 프록시 여러 네트워크 어댑터를 구성 된 경우 허용할 RADIUS 교통 관리할 어댑터를 확인 합니다.

- NPS 이벤트 로그에 기록 하 고 싶은 이벤트의 종류를 결정 합니다. 요청이 거부 된 연결이, 성공 연결 요청 중 하나 또는 둘 다 로그인 할 수 있습니다.

- 둘 이상의 NPS 프록시 배포 하는 지 여부를 결정 합니다. 결함 허용을 제공 하기 위해 두 개 이상의 NPS 프록시 사용 합니다. 하나 NPS 프록시 기본 RADIUS 프록시도 사용 되 고 백업으로 사용 되는 다른 합니다. 그런 다음 각 RADIUS 클라이언트 두 NPS 프록시 구성 되어 있습니다. 기본 NPS 프록시 사용할 수 없게 되 RADIUS 클라이언트 메시지를 보낼 액세스 요청을 대체 NPS 프록시.

- 한 NPS 프록시 구성을 다른 NPS 프록시 서버 잘못 구성 방지 하 고 관리를 저장 하도록에 복사 하는 데 사용 스크립트를 계획입니다. NPS는 가져오기 다른 NPS 프록시 끌어다에 대 한 NPS 프록시 구성의 전부 또는 일부를 복사할 수 있도록 Netsh 명령을 제공 합니다. 명령 프롬프트에서 Netsh 수동으로 실행할 수 있습니다. 그러나 명령 시퀀스 스크립트 파일로 저장 하면 프록시 구성을 변경 하려는 경우 나중에 스크립트를 실행할 수 있습니다.

## <a name="plan-radius-clients"></a>요금제 RADIUS 클라이언트

RADIUS 클라이언트는 무선 액세스 포인트 등 네트워크 액세스 서버 가상 개인 네트워크 \(VPN\) 서버 802.1 X 가능 스위치 및 전화 접속 서버 합니다. 연결 RADIUS 서버에 요청 메시지를 전달 하는 RADIUS 프록시 RADIUS 클라이언트 수도 있습니다. NPS 지원 모든 네트워크 액세스 서버 및 RADIUS 프로토콜을 준수 RADIUS 프록시 RFC 2865에 설명 된 대로 "사용자의 전화 원격 인증 \(RADIUS\), 서비스"와 RFC 2866 "RADIUS 계정을."

뿐만 아니라 무선 액세스 포인트 및 스위치 수 있어야 802.1 X 인증 합니다. 확장 인증 Protocol (EAP) 또는 보호 Extensible 인증 프로토콜 (PEAP) 배포 하려면 액세스 지점 수와 스위치 EAP 사용을 지원 해야 합니다.

무선 액세스 포인트에 대 한 PPP 연결에 대 한 기본 상호 운용성, 테스트 하 고 액세스 지점과 인증 프로토콜 PAP (암호)를 사용 하 여 액세스 클라이언트를 구성 합니다. 네트워크 액세스를 위해 사용 하는 기능을 테스트 될 때까지 PEAP 같은 추가 PPP 기반 인증 프로토콜을 사용 합니다.

### <a name="key-steps"></a>주요 단계

RADIUS 클라이언트에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 문서 공급 업체 특정 특성 (Vsa) NPS 구성 해야 합니다. 사용자 Nas Vsa를 요구 하는 경우 네트워크 정책 NPS에서 구성한 경우 나중에 사용할 VSA 정보를 기록 합니다.

- 모든 장치 구성을 간소화 하기 위해 RADIUS 클라이언트 및 NPS 프록시의 IP 주소를 기록 합니다. RADIUS 클라이언트 배포할 때 인증 서버를 입력 한 NPS 프록시 IP 주소와 RADIUS 프로토콜을 사용 하도록 구성 해야 합니다. 및 NPS 스냅인에 RADIUS 클라이언트 IP 주소를 입력 해야 NPS RADIUS 클라이언트 통신할 수를 구성 합니다.

- 스냅인 NPS RADIUS 클라이언트 켜고 구성에 맞게 공유 암호를 만듭니다. 암호, 공유 또는 암호를 입력할 NPS 스냅인 NPS RADIUS 클라이언트 구성 하는 동안에 RADIUS 클라이언트를 구성 해야 합니다.

## <a name="plan-remote-radius-server-groups"></a>원격 RADIUS 서버 그룹 계획

원격 RADIUS 서버 그룹 NPS 프록시에서 구성한 NPS 프록시 요청 메시지 액세스 서버 네트워크 및 프록시 NPS 또는 기타 RADIUS 프록시에서 수신 하는 일부 또는 전체 연결을 보낼을 알리는 됩니다.

NPS RADIUS 프록시 전달 연결 하도록 하나를 요청 하거나 더 멀리 RADIUS 서버 그룹 하 고 각 그룹 하나 또는 여러 개의 RADIUS 서버 포함 될 수 있습니다 사용할 수 있습니다. 메시지를 전달 하를 여러 그룹 NPS 프록시 하려는 경우 그룹별로 하나 연결 요청 정책의 구성 합니다. 연결 요청 정책을 특성 조작 규칙 NPS 프록시 원격 RADIUS 서버 그룹 정책에 지정 된을 보내려면 메시지를 확인 하는 등의 추가 정보를 포함 되어 있습니다.

원격 RADIUS 서버 그룹 nps Netsh 명령을 사용 하 여, 구성 그룹 직접 NPS 끌기 기능에서 원격 RADIUS 서버 그룹 아래 하거나 여 실행 하 고 새 연결 요청할 정책 마법사 구성할 수 있습니다.

### <a name="key-steps"></a>주요 단계

원격 RADIUS 서버 그룹에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 연결 요청을 전송 하도록 NPS 프록시 RADIUS 서버 포함 되어 있는 도메인 확인 합니다. 이러한 도메인을 통해 배포할 RADIUS 클라이언트 네트워크에 연결 하는 사용자의 사용자 계정 포함 됩니다.

- 여기서 RADIUS 배포 되지 않은 이미 도메인에 있는 새로운 RADIUS 서버를 추가 하려면 필요한 지 확인 합니다.

- 원격 RADIUS 서버 그룹에 추가할 RADIUS 서버의 IP 주소를 기록 합니다.

- 원격 개수 RADIUS 서버 그룹 만들어야 확인 합니다. 경우에 따라 도메인 마다 하나의 원격 RADIUS 서버 그룹 만들고 RADIUS 서버 도메인에 대 한 그룹에 추가 하는 것이 좋습니다. 하지만 많은 양의 리소스를 도메인, 도메인 컨트롤러의 많 및 많은 RADIUS 서버 많은 수의 사용자 계정으로 사용자가 포함 하 여 한 도메인에 있는 하는 경우 있을 수 있습니다. 또는 도메인 넓은 지역을 일으키는 네트워크 액세스 서버 및 RADIUS 서버 서로 떨어져 있는 위치에 포함 될 수 있습니다. 이러한 및 그 외의 경우에서 각 도메인 여러 원격 RADIUS 서버 그룹을 만들 수 있습니다.

- 원격 RADIUS 서버와 NPS 프록시에서 구성에 맞게 공유 암호를 만듭니다.

## <a name="plan-attribute-manipulation-rules-for-message-forwarding"></a>메시지를 전달 특성 조작 규칙 계획

연결 요청 정책에서 구성 된 특성 조작 규칙 액세스 요청 메시지 특정 원격 RADIUS 서버 그룹에 전달 하 고 싶은 확인할 수 있습니다.

원격 한 RADIUS 서버 그룹 특성 조작 규칙을 사용 하지 않고 모든 연결 요청을 전송 하도록 NPS 구성할 수 있습니다.

그러나 여러 위치 연결 요청을 전송 하도록 하려는 경우 해야 만든 각 위치에 대 한 연결 요청 정책을 다음 정책을 NPS 메시지를 전달 알려주는 특성 조작 규칙 뿐만 아니라를 메시지를 전달 하 원격 RADIUS 서버 그룹으로 구성 합니다.

규칙 다음과 같은 특성을 만들 수 있습니다.

- 라는 스테이션-id 네트워크 액세스 서버 (NAS)의 전화 번호입니다. 이 특성 가치 문자열을입니다. 지역 번호를 지정할 구문을 패턴 검색을 사용할 수 있습니다.

- 전화 스테이션 id 발신자에서 사용 하는 전화 번호입니다. 이 특성 가치 문자열을입니다. 지역 번호를 지정할 구문을 패턴 검색을 사용할 수 있습니다.

- 사용자 이름입니다. 액세스 클라이언트에서 제공 하는 및 RADIUS 액세스 요청 메시지의 NAS가 포함 된 사용자 이름입니다. 이 특성 값은 일반적으로 영역 이름과 사용자 계정 이름이 포함 된 문자 문자열 합니다.

올바르게 교체 변환 영역 이름 연결 요청의 사용자 이름에서, 특성 조작 규칙 사용자 이름에 대 한 적절 한 연결 요청 정책에서 구성 해야 합니다.

### <a name="key-steps"></a>주요 단계

특성 조작 규칙에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 메시지 RADIUS 서버에 메시지를 전달 하는 논리 경로 받았는지 확인 하기 원격 RADIUS 서버에에서 프록시를 통해 NAS을 경로 계획입니다.

- 하나 이상의 특성 각 연결 요청 정책에 대 한 사용 하려는 확인 합니다.

- 각 연결 요청 정책에 대 한 사용 하려는 특성 조작 규칙 및 메시지는 전달 원격 RADIUS 서버 그룹 규칙 일치 합니다.

## <a name="plan-connection-request-policies"></a>계획 연결 요청 정책

기본 연결 요청 정책은 NPS RADIUS 서버도 사용 하는 경우 구성 됩니다. 추가 연결 요청 정책은 특정 조건 정의 하 고 조작 규칙 NPS RADIUS 서버 그룹 원격에 전달 하 고 고급 특성을 지정할 되는 메시지 알려주는 특성 만들어 사용할 수 있습니다. 새 연결 요청 정책 마법사를 사용 하 여 일반적인 또는 사용자 지정 연결 요청 정책 만들 수 있습니다.

### <a name="key-steps"></a>주요 단계

연결 요청 정책에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 기본 연결 요청 정책을 NPS RADIUS 프록시로만 작동 하 실행 중인 각 서버에서 삭제 합니다.

- 이 정보를 결합 하 여 원격 RADIUS 서버 그룹 정책에 대 한 계획 특성 조작 규칙와 각 정책에 필요한 설정 및 추가 조건을 계획 있습니다.

- 디자인 일반적인 연결 요청 정책 모든 NPS 프록시에 배포를 계획 합니다. 정책 여러 NPS 프록시 일반적인 한 NPS 서버에 만들고 사용 하 여 nps Netsh 명령을에서 다른 모든 프록시 연결 요청 정책 및 서버 구성 가져올 수 있습니다.

## <a name="plan-nps-accounting"></a>요금제 NPS 계정 관리

NPS RADIUS 프록시도를 구성할 때 로깅 NPS 로그 파일 형식, 데이터베이스 호환 형식 로그 파일 또는 NPS SQL Server를 사용 하 여 계정 RADIUS 수행 하도록 구성할 수 있습니다.

또한 계정을 로깅 형식 다음 중 하나를 사용 하 여 수행 하는 원격 RADIUS 서버 그룹 계정 메시지를 전달할 수 있습니다.

### <a name="key-steps"></a>주요 단계

NPS 계정에 대 한 계획 하는 동안 다음 단계를 사용할 수 있습니다.

- 원하는 수행 계정 서비스 하거나 계정에 대 한 원격 RADIUS 서버 그룹 계정 메시지를 전달 NPS 프록시 있는지 여부를 결정 합니다.

- 계정 메시지 다른 서버에 전달 하도록 하려는 경우 계정 로컬 NPS 프록시 사용 하지 않도록 설정 예정입니다.

- 계정 메시지 다른 서버에 전달 하도록 하려는 경우 연결 요청 정책 구성 단계 계획입니다. NPS 프록시 로컬 계정 사용 하지 않도록 설정 하면 해당 프록시에서 구성 연결 요청 정책을 각 회계 메시지 전달 사용할 수 있고 제대로 구성 있어야 합니다.

- 사용 하려는 로깅 형식을 확인: IAS 포맷 로그 파일, 데이터베이스 호환 형식 로그 파일 또는 NPS SQL Server 로그인 합니다.

부하 분산 NPS RADIUS 프록시도에 대 한 참조 하십시오 [NPS 프록시 서버 부하 분산](nps-manage-proxy-lb.md)합니다.
