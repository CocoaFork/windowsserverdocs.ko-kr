# <a name="configure-firewalls-for-radius-traffic"></a>RADIUS 트래픽에 대한 방화벽 구성

>적용 대상: Windows Server (반기 채널), Windows Server 2016

방화벽을 허용할지 또는 차단할지를 컴퓨터 또는 방화벽 실행 되는 장치에서 IP 트래픽 유형을 구성할 수 있습니다. 방화벽은 RADIUS 클라이언트 간에 RADIUS 트래픽을 허용 하도록 올바로 구성 되지 않으면, RADIUS 프록시 및 RADIUS 서버, 네트워크 액세스 인증 사용자가 네트워크 리소스에 액세스 하지 못하도록 실패할 수 있습니다. 

두 가지 유형의 RADIUS 트래픽을 허용 하도록 방화벽을 구성 해야 합니다.

- 네트워크 정책 (NPS 서버)를 실행 하는 로컬 서버에서 고급 보안이 포함 된 Windows Defender 방화벽입니다.
- 다른 컴퓨터 또는 하드웨어 장치에서 실행 되는 방화벽입니다.

## <a name="windows-firewall-on-the-local-nps"></a>로컬 NPS에서 Windows 방화벽

기본적으로 NPS 보냅니다 및 사용자 데이터 그램 프로토콜을 사용 하 여 RADIUS 트래픽을 수신 \(UDP\) 포트 1812, 1813, 1645 및 1646 합니다. Windows Defender 방화벽 NPS에는이 RADIUS 트래픽을 보내고 받을 수 있도록 NPS 설치 하는 동안 예외를 사용 하 여 자동으로 구성 됩니다.

따라서 기본 UDP 포트를 사용 하는 경우 NPSs에서 RADIUS 트래픽을 허용 하도록 Windows Defender 방화벽 구성을 변경할 필요가 없습니다.

일부 경우에 NPS RADIUS 트래픽에 사용 하는 포트를 변경 하는 것이 좋습니다. 기본값 이외의 포트에서 RADIUS 트래픽을 수신 하는 NPS 및 네트워크 액세스 서버를 구성한 경우 다음을 수행 해야 합니다.

- 기본 포트에서 RADIUS 트래픽을 허용 하는 예외를 제거 합니다.
- 새 포트에서 RADIUS 트래픽을 허용 하는 새 예외를 만듭니다.

자세한 내용은 [NPS UDP 포트 정보 구성](nps-udp-ports-configure.md)합니다.

## <a name="other-firewalls"></a>다른 방화벽

가장 일반적인 구성에서 방화벽은 인터넷에 연결 하 고 NPS는 경계 네트워크에 연결 된 인트라넷 리소스.

인트라넷의 도메인 컨트롤러에 도달 하려면 NPS 있을 수 있습니다.

- 경계 네트워크와 인트라넷 인터페이스의 인터페이스 (IP 라우팅는 사용 되지 않음). 
- 경계 네트워크에는 단일 인터페이스입니다. 이 구성에서는 NPS를 인트라넷 경계 네트워크에 연결 하는 다른 방화벽을 통해 도메인 컨트롤러와 통신 합니다.

## <a name="configuring-the-internet-firewall"></a>인터넷 방화벽 구성

인터넷 인터페이스에 대 한 입력 및 출력 필터를 사용 하 여 인터넷에 연결 된 방화벽을 구성 해야 합니다 \(및 필요에 따라 네트워크 경계 인터페이스\), NPS 간의 RADIUS 메시지를 전달할 수 있도록 하려면 RADIUS 클라이언트와 인터넷 프록시입니다. 경계 네트워크에 웹 서버, VPN 서버 및 다른 유형의 서버에 트래픽 전달 하도록 허용 하려면 추가 필터를 사용할 수 있습니다.

입력 및 출력 패킷 인터넷 인터페이스와 경계 네트워크 인터페이스에서 필터를 구성할 수 있습니다를 구분 합니다.

### <a name="configure-input-filters-on-the-internet-interface"></a>인터넷 인터페이스에서 입력된 필터를 구성 합니다.

다음과 같은 유형의 트래픽 허용 하도록 방화벽의 인터넷 인터페이스에서 다음 입력된 패킷 필터를 구성 합니다.

- 경계 네트워크 인터페이스와 NPS (0x714) 1812 UDP 대상 포트의 대상 IP 주소입니다.  이 필터는 NPS로 RADIUS 인터넷 기반 클라이언트에서 RADIUS 인증 트래픽을 허용합니다. RFC 2865에 정의 된 대로 NPS에서 사용 되는 기본 UDP 포트입니다. 다른 포트를 사용 하는 경우 포트 번호 1812를 대체 합니다.
- 경계 네트워크 인터페이스와 NPS (0x715) 1813 UDP 대상 포트의 대상 IP 주소입니다. 이 필터는 NPS로 RADIUS 인터넷 기반 클라이언트의 RADIUS 계정 관리 트래픽의 허용합니다. RFC 2866에 정의 된 대로 NPS에서 사용 되는 기본 UDP 포트입니다. 다른 포트를 사용 하는 경우를 1813 포트 번호를 대체 합니다.
- \(선택적\) 경계 네트워크 인터페이스와 1645 UDP 대상 포트의 대상 IP 주소 \(0x66D\) NPS입니다. 이 필터는 NPS로 RADIUS 인터넷 기반 클라이언트에서 RADIUS 인증 트래픽을 허용합니다. 이전 버전의 RADIUS 클라이언트에서 사용 되는 UDP 포트입니다.
- \(선택적\) 경계 네트워크 인터페이스 및 1646 UDP 대상 포트의 대상 IP 주소 \(0x66E\) NPS입니다. 이 필터는 NPS로 RADIUS 인터넷 기반 클라이언트의 RADIUS 계정 관리 트래픽의 허용합니다. 이전 버전의 RADIUS 클라이언트에서 사용 되는 UDP 포트입니다.

- \(선택적\) 1645 UDP 원본 포트 및 경계 네트워크 인터페이스의 원본 IP 주소 \(0x66D\) NPS입니다. 이 필터는 RADIUS 인터넷 기반 클라이언트를 NPS에서 RADIUS 인증 트래픽을 허용합니다. 이전 버전의 RADIUS 클라이언트에서 사용 되는 UDP 포트입니다.
