# <a name="configure-network-policies"></a>네트워크 정책 구성

>적용 대상: Windows Server (반기 채널), Windows Server 2016

NPS에서 네트워크 정책을 구성 하려면이 항목에서는 사용할 수 있습니다.

## <a name="add-a-network-policy"></a>네트워크 정책 추가

네트워크 정책 서버 \(NPS\) 네트워크 정책 및 연결 요청을 네트워크에 연결할 수 있는 권한이 있는지 확인 하려면 사용자 계정 전화 접속 속성을 사용 합니다.

NPS 콘솔 또는 원격 액세스 콘솔에서 새 네트워크 정책을 구성 하려면이 절차를 사용할 수 있습니다.

### <a name="performing-authorization"></a>권한 부여를 수행합니다.

NPS 연결 요청 권한 부여를 수행할 때 첫 번째 정책과 시작한 후 구성 된 정책 목록을 아래로 이동 정책의 정렬된 된 목록에서 각 네트워크 정책 사용 하 여 요청을 비교 합니다. NPS 연결 요청을 일치 하는 조건이 정책의 찾으면 NPS 일치 정책 및 사용자 계정 전화 접속 속성 권한 부여 하는 데 사용 합니다. 사용자 계정의 전화 접속 속성 네트워크 정책을 통해 액세스 하거나 제어 액세스 권한을 부여 하도록 구성 된 연결 요청이 승인 되 고 NPS 연결에 네트워크 정책에서 구성 된 설정을 적용 합니다.

NPS 연결 요청을 일치 하는 네트워크 정책에서 찾을 수 없는 경우 액세스 권한을 부여할 사용자 계정에 전화 접속 속성 설정 되지 않을 경우 연결 요청이 거부 됩니다.

사용자 계정의 전화 접속 속성은 액세스를 거부 하도록 설정 하는 경우 NPS를 통해 연결 요청이 거부 됩니다.

### <a name="key-settings"></a>키 설정

새 네트워크 정책 마법사에서 지정 하는 값을 네트워크 정책을 만들려면 사용 하는 경우 **네트워크 연결 방법** 자동으로 구성 하는 데 사용 되는 **정책 유형** 조건: 

- Unspecified의 기본값을 유지 하는 경우 모든 유형의 네트워크 액세스 서버 (NAS)를 사용 하는 모든 네트워크 연결 형식에 대해 만든 네트워크 정책은 NPS를 통해 평가 됩니다.
- 네트워크 연결 메서드를 지정 하는 경우 NPS 연결 요청 형식을 지정 하는 네트워크 액세스 서버에서 발생 하는 경우에 네트워크 정책을 평가 합니다.

에 **액세스 권한을** 선택 해야 합니다 페이지 **부여 된 액세스** 정책을 사용자가 네트워크에 연결할 수 있도록 하는 경우. 사용자가 선택한 네트워크에 연결 하지 못하도록 정책을 **액세스가 거부 되었습니다**합니다. 

Active Directory에서 사용자 계정 전화 접속 속성에서 결정 해야 하는 액세스 권한을 원하는 경우&reg; Domain Services \(AD DS\)를 선택할 수 있습니다는 **사용자 전화 접속 속성따라결정됩니다** 확인란입니다.

**Domain Admins**의 구성원이거나 이에 준하는 자격이 있어야 이 절차를 완료할 수 있습니다.

### <a name="to-add-a-network-policy"></a>네트워크 정책을 추가 하려면 

1. NPS 콘솔을 열고 두 번 클릭 **정책을**합니다.

2. 콘솔 트리에서 마우스 오른쪽 단추로 클릭 **네트워크 정책**, 클릭 **새로 만들기**합니다. 새 네트워크 정책 마법사가 열립니다.

3. 새 네트워크 정책 마법사를 사용 하 여 정책을 만들어야 합니다.

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>전화 접속 또는 VPN 마법사에 대 한 네트워크 정책 만들기

이 절차를 사용 하 여 연결 요청 정책을 만들고 네트워크 정책을 전화 접속 서버 또는 가상 사설 네트워크를 배포 하는 데 필요한 \(VPN\) 서버를 원격 인증 전화 접속 사용자 서비스 \(RADIUS\) NPS RADIUS 서버에 클라이언트입니다.

>[!NOTE]
>랩톱 컴퓨터 및 클라이언트 운영 체제를 실행 하는 다른 컴퓨터 등의 클라이언트 컴퓨터는 RADIUS 클라이언트가 아닙니다. RADIUS 클라이언트는 네트워크 액세스 서버-무선 액세스 지점, 802.1x 인증 스위치, 가상 사설망 \(VPN\) 서버 및 전화 접속 서버-이러한 장치 통신 하는 RADIUS 프로토콜을 사용 하기 때문에 NPSs와 같은 RADIUS 서버입니다.

이 절차에는 NPS에서 새 전화 접속 또는 가상 사설망 연결 마법사를 여는 방법을 설명 합니다.

마법사를 실행 한 후에 다음 정책 생성 됩니다.

- 연결 요청 정책
- 하나의 네트워크 정책

전화 접속 서버 및 VPN 서버에 대 한 새 정책을 만들 때마다 새 전화 접속 또는 가상 사설망 연결 마법사를 실행할 수 있습니다.

NPS RADIUS 클라이언트로 전화 접속 또는 VPN 서버를 배포 하는 데 필요한 단계만 아닙니다 새 전화 접속 또는 가상 사설망 연결 마법사를 실행 합니다. 네트워크 액세스 메서드가 모두 추가 하드웨어 및 소프트웨어 구성 요소를 배포 해야 합니다.

**Domain Admins**의 구성원이거나 이에 준하는 자격이 있어야 이 절차를 완료할 수 있습니다.

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>마법사를 사용 하 여 전화 접속 또는 VPN에 대 한 정책을 만들려면

1. NPS 콘솔을 엽니다. 아직 선택 하지 않은 경우 클릭 **NPS \(로컬\)** 합니다. 원격 NPS에서 정책을 만들려는 서버를 선택 합니다.

2. **Getting Started** 하 고 **표준 구성**를 선택 **전화 접속 또는 VPN 연결용 RADIUS 서버**합니다. 텍스트 및 텍스트 변경 사항을 반영 하기 위해 아래에 링크 합니다.

3. 클릭 **구성 VPN 또는 전화 접속 마법사로**합니다. 새 전화 접속 또는 가상 사설망 연결 마법사가 열립니다.

4. 새 정책 만들기를 완료 하는 마법사의 지시를 따릅니다.

## <a name="create-network-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>네트워크 정책 802.1 X 유선 또는 무선 마법사로 만들기

이 절차를 사용 하 여 연결 요청 정책을 만들고 802.1x 인증 스위치 또는 802.1 X 무선 액세스 지점 NPS로 전화 접속 사용자 서비스 RADIUS (Remote Authentication) 클라이언트로 배포 하는 데 필요한 네트워크 정책 RADIUS 서버입니다.

이 절차에는 NPS에서 새 IEEE 802.1 X 보안 유선 및 무선 연결 마법사를 시작 하는 방법을 설명 합니다.

마법사를 실행 한 후에 다음 정책 생성 됩니다.

- 연결 요청 정책
- 하나의 네트워크 정책

802.1x 액세스에 대 한 새 정책을 만들 때마다 새 IEEE 802.1 X 보안 유선 및 무선 연결 마법사를 실행할 수 있습니다.

802.1 X 스위치 및 무선 액세스 지점 NPS로 RADIUS 클라이언트로 인증을 배포 하는 데 필요한 단계만 아닙니다 새 IEEE 802.1 X 보안 유선 및 무선 연결 마법사를 실행 합니다. 네트워크 액세스 메서드가 모두 추가 하드웨어 및 소프트웨어 구성 요소를 배포 해야 합니다.

**Domain Admins**의 구성원이거나 이에 준하는 자격이 있어야 이 절차를 완료할 수 있습니다.

2. 아직 선택 하지 않은 경우 클릭 **NPS \(로컬\)** 합니다. 원격 NPS에서 정책을 만들려는 서버를 선택 합니다.

3. **Getting Started** 하 고 **표준 구성**를 선택 **802.1 X 무선 또는 유선 연결용 RADIUS 서버**합니다. 텍스트 및 텍스트 변경 사항을 반영 하기 위해 아래에 링크 합니다.

4. 클릭 **802.1 X 마법사를 사용 하 여 구성**합니다. 새 IEEE 802.1 X 보안 유선 및 무선 연결 마법사가 열립니다.

## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>사용자 계정 전화 접속 속성을 무시 하도록 NPS 구성

권한 부여 프로세스 동안 Active Directory의 사용자 계정 전화 접속 속성을 무시 하는 NPS 네트워크 정책을 구성 하려면이 절차를 따르십시오. Active Directory 사용자 및 컴퓨터에서 사용자 계정에 하지 않는 한 권한 부여 프로세스 동안 계산 되는 NPS는 전화 접속 속성을 **네트워크 액세스 권한** 사용자 계정의 속성 **컨트롤 NPS 네트워크 정책을 통해 액세스**합니다. 

두 가지 환경이 Active Directory의 사용자 계정 전화 접속 속성을 무시 하는 NPS를 구성 하려는:

- 때 네트워크 정책을 사용 하 여 NPS 권한 부여를 단순화 하려고 했지만 모든 사용자 계정의 합니다 **네트워크 액세스 권한** 속성이로 설정 **NPS 네트워크 정책을 통해 액세스를 제어할**합니다. 예를 들어 일부 사용자 계정은 있을 수 있습니다 합니다 **네트워크 액세스 권한** 사용자 계정 속성을 설정 **액세스를 거부할** 또는 **액세스를 허용**합니다.

- 사용자 계정 전화 접속 속성 다른 네트워크 정책에서 구성 된 연결 형식에 적용할 수 없는 경우입니다. 이외의 다른 예를 들어, 속성을 **네트워크 액세스 권한** 설정은 전화 접속 또는 VPN 연결의 경우에 적용할 수 있으 나 만들면 네트워크 정책 무선 또는 인증 스위치 연결 합니다.

사용자 계정 전화 접속 속성을 무시 하도록 NPS를 구성 하려면이 절차를 사용할 수 있습니다. NPS는 사용자 또는 컴퓨터 네트워크;에 액세스할 권한이 있는지 확인 하는 데 사용자 계정 전화 접속 속성을 사용 하지 않습니다 연결 요청을이 확인란을 선택 하는 위치에 네트워크 정책 일치 하는 경우 권한 부여를 확인 하려면 네트워크 정책에서 설정에 대해서만 사용 됩니다.

멤버 자격이 **관리자**, 또는 이와 동등한 최소한이이 절차를 완료 합니다.

1. NPS를 서버 관리자에서 클릭 **도구**를 클릭 하 고 **네트워크 정책 서버**합니다. NPS 콘솔이 열립니다.

2. 두 번 클릭 **정책을**, 클릭 **네트워크 정책**를 차례로 클릭 한 다음 세부 정보 창에서 구성 하려는 정책입니다.

3. 정책에서 **속성** 대화 상자의 합니다 **개요** 탭의 **액세스 권한을**를 선택는 **사용자 계정 전화 접속 속성 비교무시**확인란을 선택한 다음 클릭 **확인**합니다.

## <a name="configure-nps-for-vlans"></a>Vlan에 대 한 NPS 구성

Windows Server 2016에서 VLAN 인식 네트워크 액세스 서버 및 NPS를 사용 하 여 해당 보안 권한에 대 한 적절 한 네트워크 리소스에만 액세스를 사용 하 여 사용자 그룹을 제공할 수 있습니다. 예를 들어 제공할 수 있습니다 방문자 무선 액세스를 사용 하 여 인터넷에 조직 네트워크에 대 한 액세스를 허용 하지 않고 있습니다. 

또한 Vlan을 사용 하면 논리적으로 그룹 네트워크 리소스에 서로 다른 물리적 위치 또는 서로 다른 물리적 서브넷에 존재 합니다. 예를 들어, 판매 부서와 클라이언트 컴퓨터, 서버 및 프린터와 같은 해당 네트워크 리소스의 멤버는 조직에서 여러 건물에서 있을 수 있습니다. 하지만 동일한 IP를 사용 하는 하나의 VLAN에 이러한 모든 리소스를 배치할 수 있습니다. 주소 범위입니다. VLAN 다음 함수는 단일 서브넷으로 최종 사용자 관점에서 합니다.

다른 사용자 그룹의 네트워크를 분리 하려는 경우에 Vlan을 사용할 수 있습니다. 그룹을 정의 하려는 방법을 확인 한 후 있습니다 Active Directory 사용자 및 컴퓨터 스냅인에서 보안 그룹 만들기를 추가한 다음 그룹에 멤버를 추가 합니다.

### <a name="configure-a-network-policy-for-vlans"></a>Vlan에 대 한 네트워크 정책 구성

VLAN에 사용자를 할당 하는 네트워크 정책을 구성 하려면이 절차를 사용할 수 있습니다. 라우터, 스위치 및 액세스 컨트롤러와 같은 VLAN 인식 네트워크 하드웨어를 사용 하는 경우 특정 Vlan에 특정 Active Directory 그룹의 멤버를 배치할 액세스 서버에 지시 하려면 네트워크 정책을 구성할 수 있습니다. 이 기능은 논리적으로 Vlan 사용 하 여 네트워크 리소스 그룹 유연 하 게 디자인 하 고 네트워크 솔루션을 구현를 제공 합니다.

Vlan을 사용 하 여 사용 하기 위한 NPS 네트워크 정책 설정을 구성한 경우에 특성을 구성 해야 합니다 **터널 중간 형식당**를 **터널 Pvt-그룹 ID**, **터널 종류**, 및 **터널 태그**합니다. 

이 절차는;으로 제공 됩니다. 네트워크 구성에는 아래 설명 된 것 보다 다양 한 설정이 필요할 수 있습니다.

멤버 자격이 **관리자**, 또는 이와 동등한 최소한이이 절차를 완료 합니다.

### <a name="to-configure-a-network-policy-for-vlans"></a>Vlan에 대 한 네트워크 정책을 구성 하려면

1. NPS를 서버 관리자에서 클릭 **도구**를 클릭 하 고 **네트워크 정책 서버**합니다. NPS 콘솔이 열립니다.

2. 두 번 클릭 **정책을**, 클릭 **네트워크 정책**를 차례로 클릭 한 다음 세부 정보 창에서 구성 하려는 정책입니다.

3. 정책의 **속성** 대화 상자에서 클릭 합니다 **설정** 탭 합니다.

4. 정책에서 **속성**의 **설정**의 **RADIUS 특성**, 되도록 **표준** 을 선택 합니다.

5. 세부 정보 창에서의 **특성**의 **Service-type** 특성의 기본값을 사용 하 여 구성 된 **프레임이**합니다. 기본적으로 VPN 및 전화 접속 액세스 메서드를 사용 하 여 정책에 대 한 합니다 **프레임이 프로토콜** 특성의 값으로 구성 됩니다 **PPP**합니다. Vlan에 필요한 추가 연결 속성을 지정 하려면 클릭 **추가**합니다. 합니다 **표준 RADIUS 특성 추가** 대화 상자가 열립니다.

6. **표준 RADIUS 특성 추가**, 특성, 아래로 스크롤하여 다음 특성을 추가 합니다.

    - **Tunnel-Medium-Type**. 정책에 대해 수행한 경우 이전 선택 사항에 적합 한 값을 선택 합니다. 예를 들어 네트워크 정책을 구성 하는 무선 정책 인 경우 선택 **값: (모든 802 포함 미디어 및 이더넷 정규 형식) 802**합니다.

    - **Tunnel-Pvt-Group-ID**. 그룹 구성원 할당 된 VLAN 번호를 나타내는 정수를 입력 합니다. 

    - **터널 종류**합니다. 선택 **가상 Lan (VLAN)** 합니다.


7. **표준 RADIUS 특성 추가**, 클릭 **닫기**합니다. 

8. 네트워크 액세스 서버 (NAS)를 사용 해야 하는 경우는 **터널 태그** 특성을 추가 하려면 다음 단계를 사용 합니다 **터널 태그** 네트워크 정책에 특성입니다. NAS 설명서는이 특성을 언급 하지 않습니다, 정책에 추가 하지 마십시오. 필요한 경우 아래와 같이 특성을 추가 합니다.

    - 정책의 **속성**의 **설정**의 **RADIUS 특성**, 클릭 **공급 업체 특정**합니다. 

    - 세부 정보 창에서 클릭 **추가**합니다. 합니다 **공급 업체 특정 특성 추가** 대화 상자가 열립니다.

    - **특성**, 아래로 스크롤하여 선택 **터널 태그**를 클릭 하 고 **추가**합니다. 합니다 **특성 정보** 대화 상자가 열립니다. 

    - **특성 값**, 해당 하드웨어 설명서에서 가져온 값을 입력 합니다.

## <a name="configure-the-eap-payload-size"></a>EAP 페이로드 크기를 구성 합니다.

일부 경우, 라우터 또는 방화벽에서 조각화를 필요로 하는 패킷을 삭제 하도록 구성 되어 있으므로 패킷을 삭제 합니다.

Extensible Authentication Protocol을 사용 하는 네트워크 정책을 사용 하 여 NPS를 배포 하는 경우 \(EAP\) with Transport Layer Security \(TLS\), 또는 EAP-TLS, 최대 기본 인증 방법으로 전송 단위 \(MTU\) EAP 페이로드를 사용 하 여 NPS는 1500 바이트입니다. 

EAP 페이로드에 대 한이 최대 크기 라우터 또는 방화벽 NPS 사이의 RADIUS 클라이언트에서 조각화를 필요로 하는 RADIUS 메시지를 만들 수 있습니다. 이 경우 라우터 또는 방화벽 RADIUS 클라이언트와 NPS 사이 인증 실패 및 액세스 클라이언트 네트워크에 연결할 수 없음을 일부 조각을 자동으로 취소 될 수 있습니다.

다음 절차를 사용 하 여 NPS 1344 보다 크지 않은 값으로 네트워크 정책에서 프레임이 MTU 특성을 조정 하 여 EAP 페이로드를 사용 하는 최대 크기를 줄입니다.

멤버 자격이 **관리자**, 또는 이와 동등한 최소한이이 절차를 완료 합니다.

### <a name="to-configure-the-framed-mtu-attribute"></a>프레임이 MTU 특성을 구성 하려면

1. NPS를 서버 관리자에서 클릭 **도구**를 클릭 하 고 **네트워크 정책 서버**합니다. NPS 콘솔이 열립니다.
 
2. 두 번 클릭 **정책을**, 클릭 **네트워크 정책**를 차례로 클릭 한 다음 세부 정보 창에서 구성 하려는 정책입니다.

3. 정책의 **속성** 대화 상자에서 클릭 합니다 **설정** 탭 합니다.

4. **설정을**의 **RADIUS 특성**, 클릭 **표준**합니다. 세부 정보 창에서 클릭 **추가**합니다. 합니다 **표준 RADIUS 특성 추가** 대화 상자가 열립니다.

5. **특성**, 아래로 스크롤하여 클릭 **프레임이 MTU**를 클릭 하 고 **추가**합니다. 합니다 **특성 정보** 대화 상자가 열립니다.

6. **특성 값**, 작거나 같은 값을 입력 보다 **1344**합니다. 클릭 **확인**, 클릭 **닫기**를 클릭 하 고 **확인**합니다.
