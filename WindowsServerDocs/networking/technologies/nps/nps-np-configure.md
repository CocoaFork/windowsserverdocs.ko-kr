# <a name="configure-network-policies"></a>네트워크 정책을 구성합니다

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

네트워크 정책 NPS에서 구성 하려면이 항목을 사용할 수 있습니다.

## <a name="add-a-network-policy"></a>네트워크 정책을 추가합니다

네트워크 정책 서버 \(NPS\) 네트워크 정책을 사용 하 여 및 네트워크에 연결 하려면 전화 접속 속성의 사용자 계정 연결 요청 하는지 여부를 결정 합니다.

새 네트워크 정책 NPS 콘솔 또는 원격 Access 콘솔에서 구성 하려면이 절차를 사용할 수 있습니다.

### <a name="performing-authorization"></a>인증을 수행합니다.

NPS 연결 요청을 승인을 수행할 때 시작 첫 번째 정책으로 이동한 다음 구성 된 정책 목록 아래로 정책의 정렬된 목록에서 각 네트워크 정책으로 요청을 비교 합니다. NPS 해당 조건이 연결 요청 일치 정책을 찾으면 NPS 일치 하는 정책 및 사용자 계정의 전화 속성 권한을 부여 하 사용 합니다. 사용자 계정의 전화 속성 네트워크 정책을 통해 액세스 또는 제어 권한을 부여 하도록 구성 된 경우 연결 요청이 승인 되 NPS 연결 하는 네트워크 정책에서 구성 된 설정을 적용 합니다.

NPS 연결 요청와 일치 하는 네트워크 정책을 찾지 못하면 전화 접속 속성 사용자 계정에 액세스 권한을 부여할 설정 되어 있지 않으면 연결 요청이 거부 됩니다.

전화 접속 속성 사용자 계정 액세스 거부 설정, nps 연결 요청이 거부입니다.

### <a name="key-settings"></a>키 설정

때 사용 하 여 새 네트워크 정책 마법사 네트워크 정책을에 지정 된 값 만들려면 **네트워크 연결이 방법** 자동으로 구성 하는 데 사용 되는 **정책 유형** 조건: 

- 지정 하지 않음 기본값을 유지 하는 경우 네트워크 정책 사용자가 만든 모든 종류의 네트워크 NAS 액세스 서버 ()를 사용 하는 모든 네트워크 연결 형식에 대 한 nps 평가 됩니다.
- 네트워크 연결 방법을 지정 NPS 연결 요청 사용자 지정 하는 네트워크 액세스 서버 유형의에서 발생 하는 경우에 네트워크 정책을 평가 합니다.

에 **액세스 권한을** 선택 해야 페이지 **액세스 허용** 사용자가 네트워크에 연결할 수 있도록 정책을 하려는 경우입니다. 사용자가 선택 하 고 네트워크에 연결 하지 못하도록 하려면 정책을 하려는 경우 **액세스 거부**합니다. 

사용자 계정에 전화 속성 Active Directory에 의해 결정 액세스 권한을 하려는 경우&reg; 도메인 서비스 \ (AD DS \)을 선택할 수는 **액세스 속성 사용자 전화에 따라 결정 됩니다** 확인란을 선택 합니다.

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

### <a name="to-add-a-network-policy"></a>네트워크 정책을 추가 하려면 

1. NPS 콘솔을 연 다음 두 번 클릭 **정책**합니다.

2. 콘솔 트리에서 마우스 오른쪽 단추로 클릭 **네트워크 정책**를 클릭 하 고 **새로**합니다. 새 네트워크 정책 마법사가 열립니다.

3. 새 네트워크 정책 마법사를 사용 하 여 정책을 만듭니다.

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>전화 접속 또는 마법사에 VPN 연결에 대 한 네트워크 정책을 만들기

이 절차를 연결 요청 정책 만들고 NPS RADIUS 서버에 RADIUS(Remote Authentication Dial-In User Service) \(RADIUS\) 클라이언트로 전화 접속 서버 또는 가상 개인 네트워크 \(VPN\) 서버를 배포 하는 데 필요한 네트워크 정책 사용할 수 있습니다.

>[!NOTE]
>클라이언트 운영 체제를 실행 하는 다른 컴퓨터, 랩톱 컴퓨터 등 클라이언트 컴퓨터 RADIUS 클라이언트 되지 않습니다. RADIUS 클라이언트는 네트워크 액세스 서버-무선 액세스 포인트, 802.1 X 인증 스위치, 가상 개인 네트워크 \(VPN\) 서버 전화 접속 서버 등-이러한 디바이스는 RADIUS 프로토콜을 사용 하 여와 같은 서버 NPS RADIUS 서버 통신할 수 있습니다.

이 절차 새 전화 접속 또는 가상 개인 네트워크 연결이 마법사 NPS에서 여는 방법에 설명 합니다.

마법사를 실행 한 후 다음 정책은 생성 됩니다.

- 하나 연결 요청 정책
- 하나 네트워크 정책

전화 접속 서버 및 VPN 서버의 대 한 새로운 정책을 해야 할 때마다 새로운 전화 접속 또는 가상 개인 네트워크 연결이 마법사를 실행할 수 있습니다.

서버 NPS RADIUS 클라이언트로 전화 접속 또는 VPN 서버에 배포 하는 데 필요한만 단계 아닙니다 새 전화 접속 또는 가상 개인 네트워크 연결이 마법사를 실행 합니다. 네트워크 액세스 방법 두 배포한 하드웨어 및 소프트웨어 구성 요소를 추가 해야 합니다.

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>전화 접속 또는 VPN에 대 한 정책을 마법사를 만들려면

1. NPS console을 엽니다. 이미 선택 하지 않으면 클릭 **NPS \(Local\)**합니다. 정책 NPS 원격 서버에서 만들려는 서버를 선택 합니다.

2. **시작** 및 **표준 구성**선택 **전화 접속 또는 VPN 연결에 대 한 RADIUS 서버**합니다. 텍스트와 적절 하 게 텍스트 변경 아래 링크입니다.

3. 클릭 **구성 VPN 또는 마법사 전화 접속**합니다. 새 전화 접속 또는 가상 개인 네트워크 연결이 마법사가 열립니다.

4. 새 정책을 생성 완료 하 고 마법사의 지침을 따릅니다.

## <a name="create-network-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>802.1에 대 한 네트워크 정책 유선 또는 무선 마법사에 X 만들기

이 절차를 연결 요청 정책 및 NPS RADIUS 서버에 RADIUS(Remote Authentication Dial-In User Service) (RADIUS) 클라이언트로 802.1 X 인증 스위치 또는 802.1 X 무선 액세스 포인트를 배포 하는 데 필요한 네트워크 정책 만드는 사용할 수 있습니다.

이 절차 NPS에서 새 IEEE 802.1 X 유선 보안 및 무선 연결 마법사를 시작 하는 방법에 설명 합니다.

마법사를 실행 한 후 다음 정책은 생성 됩니다.

- 하나 연결 요청 정책
- 하나 네트워크 정책

802.1 X 액세스 대 한 새로운 정책을 해야 할 때마다 새로운 IEEE 802.1 X 유선 보안 및 무선 연결 마법사를 실행할 수 있습니다.

802.1 X 인증 서버 NPS RADIUS 클라이언트로 스위치 및 무선 액세스 포인트 배포 하는 데 필요한만 단계 아닙니다 새로운 IEEE 802.1 X 유선 보안 및 무선으로 연결 마법사 실행 합니다. 네트워크 액세스 방법 두 배포한 하드웨어 및 소프트웨어 구성 요소를 추가 해야 합니다.

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

### <a name="to-create-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>유선 또는 무선 마법사에 802.1 X에 대 한 정책을 만들려면

1. 서버 관리자에서 NPS 서버에서 클릭 **도구**을 차례로 클릭 하 고 **네트워크 정책 서버**합니다. NPS console을 엽니다. 

2. 이미 선택 하지 않으면 클릭 **NPS \(Local\)**합니다. 정책 NPS 원격 서버에서 만들려는 서버를 선택 합니다.

3. **시작** 및 **표준 구성**선택 **802.1 X Wireless 또는 유선 연결을 위한 RADIUS 서버**합니다. 텍스트와 적절 하 게 텍스트 변경 아래 링크입니다.

4. 클릭 **802.1 X 마법사를 사용 하 여 구성**합니다. 새 IEEE 802.1 X 유선 보안 및 무선 연결 마법사가 열립니다.

5. 새 정책을 생성 완료 하 고 마법사의 지침을 따릅니다.

## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>사용자 계정에 전화 속성을 무시 하도록 NPS 구성

이 절차를 인증 과정에서의 사용자 계정 Active Directory 전화 속성 무시 NPS 네트워크 정책 구성할 수 있습니다. 사용자 계정 Active Directory 사용자 및 컴퓨터 있는 하지 않는 한 NPS 인증 과정에서 평가 하는 전화 접속 속성의 **네트워크 액세스 권한을** 사용자 계정의 속성을 설정한 **NPS 네트워크 정책을 통해 액세스를 제어**합니다. 

두 가지 환경이 NPS 전화 접속 속성의 사용자 계정 Active Directory를 무시 하도록 구성을 저장할 수 있습니다.

- 때 NPS 승인 네트워크 정책을 사용 하 여 간단 하 게 하려면 하지만 사용자 계정의 모든 하지는 **네트워크 액세스 권한을** 속성으로 설정 **NPS 네트워크 정책을 통해 액세스를 제어**합니다. 일부 사용자 계정 예를 들어 있을 수 있습니다는 **네트워크 액세스 권한을** 속성 사용자 계정으로 설정 **액세스 거부** 또는 **액세스 허용**합니다.

- 되 면 사용자 계정의 다른 전화 접속 속성 네트워크 정책에서 구성 된 연결 유형을에 적용 됩니다. 예를 들어, 속성 아닌 다른는 **네트워크 액세스 권한을** 설정을 전화 접속 또는 VPN 연결에만 해당 하지만 만들 되 네트워크 정책 무선 또는 인증 스위치 연결에 대 한 합니다.

이 절차를 무시 사용자 계정이 전화 접속 속성 NPS 구성에 사용할 수 있습니다. NPS는 사용자 또는 컴퓨터, 네트워크에 액세스할 수 있는 권한이 있는지 확인 하려면 전화 접속 속성 사용자 계정의 사용 하지 않는 연결 요청이 확인란이 선택 되어 네트워크 정책 일치 하는 경우 네트워크 정책에서 설정에 대해서만 사용 하는 데 사용 합니다.

회원 **관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

1. 서버 관리자에서 NPS 서버에서 클릭 **도구**을 차례로 클릭 하 고 **네트워크 정책 서버**합니다. NPS console을 엽니다.

2. 두 번 클릭 **정책**, 클릭 **네트워크 정책**, 다음 세부 정보 창에서 정책 구성할에 두 번 클릭 합니다.

3. 정책에 **속성** 대화 상자의 **개요** 탭에 **액세스 권한을**는 **사용자 계정에 전화 속성 무시** 확인란을 선택한 다음 클릭 **확인**합니다.

## <a name="configure-nps-for-vlans"></a>NPS Vlan에 대 한 구성

Windows Server 2016에 VLAN 인식 네트워크 액세스 서버 및 NPS을 사용 하 여 해당 보안 권한을에 적합 한 네트워크 리소스에만 액세스할 수 있는 사용자 그룹을 제공할 수 있습니다. 예를 들어, 제공할 수 있습니다 방문자 무선에 액세스할 수 있는 인터넷 하 여 회사 네트워크에 대 한 액세스를 않고도 합니다. 

또한 Vlan 하면 논리적 그룹 네트워크 리소스를 또는 다른 실제 서브넷 서로 다른 위치에 있는 있습니다. 예를 들어, 판매 성와 해당 네트워크 리소스 클라이언트 컴퓨터, 서버, 프린터 등의 회원 조직에서 여러 가지 건물에 있을 수 있습니다 하지만 이러한 리소스가 모두 동일한 IP 주소 범위를 사용 하는 VLAN에 배치할 수 있습니다. VLAN 다음 최종 사용자 관점 단일 서브넷으로 기능입니다.

다른 사용자 그룹 사이의 네트워크를 분리 하 고 싶은 경우에 Vlan 사용할 수 있습니다. 어떻게 그룹 정의 것인지를 확인 한 후 Active Directory 사용자 및 컴퓨터 스냅인에서 보안 그룹을 만들 수 있으며 그룹 구성원을 추가 합니다.

### <a name="configure-a-network-policy-for-vlans"></a>네트워크 정책을 Vlan에 대 한 구성

이 절차 VLAN에 사용자 지정 하는 네트워크 정책을 구성에 사용할 수 있습니다. 라우터, 스위치를 액세스 컨트롤러 등 VLAN 인식 네트워크 하드웨어를 사용 하는 경우 네트워크 정책 액세스 서버 특정 Active Directory 그룹의 회원 특정 Vlan에 배치 하 않도록 하려면 구성할 수 있습니다. 이 기능을 Vlan으로 논리적 그룹 네트워크 리소스 디자인 하 고 네트워크 솔루션 구현 하는 경우 유연성을 제공 합니다.

사용에 대 한 NPS 네트워크 정책 설정 Vlan와를 구성할 때 특성 구성 해야 **터널 중간 종류**, **Pvt 그룹 ID 터널**, **터널 종류**, 및 **터널 태그**합니다. 

이 절차는 예를 들어, 제공 네트워크 구성 아래에 설명 된 것 보다 다양 한 설정 필요할 수 있습니다.

회원 **관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

### <a name="to-configure-a-network-policy-for-vlans"></a>네트워크 정책 Vlan에 대 한 구성 하려면

1. 서버 관리자에서 NPS 서버에서 클릭 **도구**을 차례로 클릭 하 고 **네트워크 정책 서버**합니다. NPS console을 엽니다.

2. 두 번 클릭 **정책**, 클릭 **네트워크 정책**, 다음 세부 정보 창에서 정책 구성할에 두 번 클릭 합니다.

3. 정책에 **속성** 대화 상자를 클릭 하 고 **설정** 탭 합니다.

4. 정책에서 **속성**에 **설정**에 **RADIUS 특성**, 되도록 **표준** 을 선택 합니다.

5. 세부 정보 창에서에 **특성**, **서비스 유형** 특성 기본값을로 구성 **프레임이**합니다. 기본적으로 정책을 VPN 및 전화 접속, 액세스 방법으로는 **프레임이 프로토콜** 특성 값으로 구성 되어 **PPP**합니다. Vlan에 필요한 추가 연결 속성을 지정 하려면 클릭 **추가**합니다. **표준 RADIUS 속성 추가** 대화 상자를 엽니다.

6. **표준 RADIUS 속성 추가**의 특성을 아래로 스크롤하여 다음과 같은 특성을 추가 합니다.

    - **터널 중간 종류**합니다. 이전에 정책에 대 한 내용을 선택한에 적절 한 값을 선택 합니다. 예를 들어 구성 하 고 네트워크 정책 무선 정책 이면 선택 **값: 802 (모든 802 포함 미디어 + 이더넷 정식 format)**합니다.

    - **Pvt 그룹 ID 터널**합니다. 그룹 구성원 할당 될 VLAN 번호 나타내는 입력 합니다. 

    - **터널 종류**합니다. 선택 **가상 Lan (VLAN)**합니다.


7. **표준 RADIUS 속성 추가**, 클릭 **닫기**합니다. 

8. 네트워크 액세스 서버 (NAS)를 사용 해야 하는 경우는 **터널 태그** 특성을 추가 하려면 다음 단계를 수행은 **터널 태그** 네트워크 정책으로 특성 합니다. NAS 설명서가이 특성 언급 하지 않는, 정책에 추가 하지 마십시오. 필요한 경우 다음과 같은 특성을 추가 합니다.

    - 정책에서 **속성**에 **설정**에 **RADIUS 특성**, 클릭 **공급 업체 특정**합니다. 

    - 세부 정보 창에서 클릭 **추가**합니다. **공급 업체 추가 특정 특성** 대화 상자를 엽니다.

    - **특성**때까지 아래로 스크롤하고 **터널 태그**을 차례로 클릭 하 고 **추가**합니다. **특성 정보** 대화 상자를 엽니다. 

    - **특성 값**, 하드웨어 설명서에서 얻은 값을 입력 합니다.

## <a name="configure-the-eap-payload-size"></a>구성 EAP 페이로드 크기

경우에 따라 라우터 또는 방화벽 핀 패킷을 조각화 필요한 패킷을 삭제 구성 되어 있기 때문에 있습니다.

인증 방법으로 확장 인증 프로토콜 \(EAP\) Transport Layer Security \(TLS\) 또는 EAP-TLS을 사용 하는 네트워크 정책으로 NPS 배포 하는 경우 기본 최대 전송 단위 \(MTU\) NPS EAP 페이로드를 사용 하는 1500 바이트 합니다. 

이 최대 크기 EAP 페이로드를 라우터 또는 방화벽 NPS 서버 RADIUS 하는 클라이언트와 조각화 해야 하는 RADIUS 메시지를 만들 수 있습니다. 이 경우 라우터 또는 RADIUS 클라이언트 NPS 서버와 배치 방화벽 되 고 액세스 클라이언트의 네트워크에 연결 하지 못하는 인증 오류 몇 가지 조각 자동으로 삭제 될 수도 있습니다.

다음 절차를 사용 하 여 NPS 1344 보다 더 큰 값을 하는 네트워크 정책 프레임이 MTU 특성을 조정 하 여 EAP 페이로드를 사용 하 고 최대 크기를 줄입니다.

회원 **관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

### <a name="to-configure-the-framed-mtu-attribute"></a>프레임이 MTU 특성 구성 하려면

1. 서버 관리자에서 NPS 서버에서 클릭 **도구**을 차례로 클릭 하 고 **네트워크 정책 서버**합니다. NPS console을 엽니다.
 
2. 두 번 클릭 **정책**, 클릭 **네트워크 정책**, 다음 세부 정보 창에서 정책 구성할에 두 번 클릭 합니다.

3. 정책에 **속성** 대화 상자를 클릭 하 고 **설정** 탭 합니다.

4. **설정**에 **RADIUS 특성**, 클릭 **표준**합니다. 세부 정보 창에서 클릭 **추가**합니다. **표준 RADIUS 속성 추가** 대화 상자를 엽니다.

5. **특성**, 아래로 스크롤하여 클릭 **프레임이 MTU**을 차례로 클릭 하 고 **추가**합니다. **특성 정보** 대화 상자를 엽니다.

6. **특성 값**, 낮거나 값을 입력 보다 **1344**합니다. 클릭 **확인**, 클릭 **닫기**을 차례로 클릭 하 고 **확인**합니다.



네트워크 정책에 대 한 자세한 내용은 참조 [네트워크 정책](nps-np-overview.md)합니다.

NPS에 대 한 자세한 내용은 참조 [네트워크 NPS 정책 서버 ()](nps-top.md)합니다.
