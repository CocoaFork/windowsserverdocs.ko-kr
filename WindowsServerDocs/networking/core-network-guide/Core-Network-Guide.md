## <a name="BKMK_about"></a>이 가이드 정보
이 가이드는 새로운 네트워크를 설치 하는 사람 또는 작업 그룹으로 구성 된 네트워크를 교체 하는 도메인 기반 네트워크 작성 네트워크 및 시스템 관리자 용 설계 되었습니다. 이 가이드에 제공 해 배포 시나리오 네트워크에 더 많은 서비스와 기능 나중에 추가 해야 예상 되는 경우에 특히 유용 합니다.

디자인을 검토 하 고 배포 안내 각각 서비스와 구성에이 가이드에서는 제공 하는지 여부를 결정 하는 데 도움이 되도록이 배포 시나리오에서 사용 되는 기술에 대 한 것이 좋습니다.

핵심 네트워크의 네트워크 하드웨어, 디바이스, 컬렉션 이며 조직의 IT 정보 기술 ()에 대 한 기본 서비스를 제공 하는 소프트웨어 필요 합니다.

Windows Server의 핵심 네트워크는 다음을 포함 한 여러 가지 이점을 제공 합니다.

-   컴퓨터와 다른 TCP/IP (TCP/IP) 호환 디바이스 간에 네트워크 연결에 대 한 핵심 프로토콜 합니다. TCP/IP는 컴퓨터를 연결 하 고 네트워크 빌드에 대 한 프로토콜 표준 제품군입니다. TCP/IP는 네트워크 프로토콜 Microsoft Windows 운영 체제와 함께 제공 되는 소프트웨어 및 구현 tcp/ip 프로토콜을 지원 합니다.

-   DHCP(Dynamic Host Configuration Protocol) (DHCP) 자동 IP 주소 할당 컴퓨터와 DHCP 클라이언트도 구성 된 기타 장치에 있습니다. 네트워크의 모든 컴퓨터의 IP 주소를 수동으로 구성 시간이 및 덜 동적으로 DHCP를 사용 하 여 IP 주소 구성 된 컴퓨터 및 기타 장치를 제공 하는 보다 효율적입니다.

-   시스템 DNS (도메인 이름) 해상도 서비스의 이름을 지정 합니다. DNS는 사용자가, 컴퓨터, 응용 프로그램 및 서비스 네트워크에서 완벽 하 게 된 도메인의 이름을 컴퓨터나 디바이스를 사용 하 여 컴퓨터와 디바이스의 IP 주소를 찾을 수 있습니다.

-   숲-동일한 클래스 및 특성 정의 (스키마), (구성) 하는 사이트 및 복제 정보 및 숲 검색 기능 (드)를 공유 하는 하나 이상의 Active Directory 도메인은 합니다.

-   새 숲 속의 첫 번째 도메인 숲 루트 도메인에 만들었습니다. 숲 속 전체 관리 그룹 인는 엔터프라이즈 관리자 및 스키마 관리자 그룹 숲 루트 도메인에 있습니다. 또한 이렇게, 다른 도메인와 마찬가지로 모음입니다 컴퓨터, 사용자 및 그룹 개체 Active Directory Domain Services (AD DS)에서 관리자 권한으로 정의 됩니다. 이러한 물체 일반적인 디렉터리 데이터베이스 및 보안 정책이 공유합니다. 또한 증가 하는 조직 도메인을 추가 하는 경우 보안 관계 다른 도메인 공유할 수 있습니다. 디렉터리 서비스는 또한 디렉터리 데이터를 저장 하 고 권한이 부여 된 컴퓨터, 응용 프로그램 및 사용자 데이터에 액세스할 수 있게 합니다.

-   컴퓨터와 사용자 계정 데이터베이스 합니다. 디렉터리 서비스는 사람 및 리소스 응용 프로그램, 데이터베이스, 공유 파일 및 폴더, 프린터 등 네트워크와 액세스 네트워크에 연결 하려면 실시 권 자는 컴퓨터에 대 한 사용자와 컴퓨터 계정을 만들 수 있게 해 주는 중앙된 사용자 계정 데이터베이스를 제공 합니다.

핵심 네트워크 조직 성장 하 고 IT 요구 사항을 변경 때이 네트워크에 맞게 수도 있습니다. 예를 들어, core 네트워크 도메인, IP 서브넷, 원격 액세스 서비스, 무선 서비스 및 기타 기능 및 Windows Server 2016에서 제공 하는 서버 역할을 추가할 수 있습니다.

-   자녀가 해당 클라이언트 및 서버 운영 체제에 대 한 최소 하드웨어 요구 사항을 충족 하는 컴퓨터 합니다.

## <a name="technology-overviews"></a>기술 개요
다음 섹션 핵심 네트워크 만드는 배포 되는 필요한 기술에 대 한 간략 한 개요를 제공 합니다.

### <a name="active-directory-domain-services"></a>Active Directory Domain Services
디렉터리는 소식 개체에 대 한 정보를 사용자와 컴퓨터와 같은 네트워크에 저장 합니다. AD DS 등 디렉터리 서비스를 제공 디렉터리 데이터를 저장 하 고 네트워크 사용자에 게와 관리자에 게이 데이터를 사용할 수 있게 하는 방법입니다. 예를 들어, AD DS 이름, 메일 주소, 암호 및 전화 번호를 포함 하 여 사용자 계정에 대 한 정보를 저장 하 고 다른 권한이 있는 사용자가 동일한 네트워크에 수 있도록이 정보에 액세스 합니다.

### <a name="dns"></a>DNS
DNS는 조직의 네트워크 또는 인터넷 등의 TCP/IP 네트워크 이름을 확인 프로토콜입니다. DNS 서버 섬에는 클라이언트 컴퓨터를 사용할 수 있는 정보 및 서비스를 쉽게 확인할 인식 서로 통신할를 사용 하 여 컴퓨터의 IP 주소를 영숫자 DNS 이름을 합니다.

### <a name="dhcp"></a>DHCP
DHCP IP 표준 호스트 IP 구성 관리를 간소화에 대 한입니다. DHCP를 사용 하도록 클라이언트 네트워크에 대 한 동적 할당의 IP 주소 및 기타 관련된 구성 세부 정보를 관리 하는 방법으로 DHCP 서버의 사용할 표준 DHCP 제공 합니다.

DHCP 동적으로 컴퓨터나 로컬 네트워크의 프린터 등 기타 장치에는 IP 주소를 지정 하려면 DHCP를 사용할 수 있습니다. TCP/IP 네트워크에 있는 모든 컴퓨터의 IP 주소와 관련된 서브넷 마스크 호스트 컴퓨터와 연결 되어 있는 컴퓨터 서브넷을 식별 하기 때문에 고유한 IP 주소를 있어야 합니다. DHCP를 사용 하 여 DHCP 클라이언트로 구성 되어 있는 모든 컴퓨터의 네트워크 위치와 서브넷 적합 IP 주소를 받을 수 있도록 하며 등 기본 게이트웨이 DNS 서버, DHCP 옵션을 사용 하 여 제공할 수 있습니다 자동으로 DHCP 클라이언트 네트워크에서 제대로 작동 하는 데 필요한 정보를 사용 합니다.

네트워크 TCP 기반 DHCP 복잡 하 고 컴퓨터를 다시 구성에 포함 된 관리 작업의 양을 줄입니다.

### <a name="tcpip"></a>TCP/IP
Windows Server 2016에 TCP/IP 다음과 같습니다.

-   네트워킹 소프트웨어 산업 표준을 네트워킹 프로토콜에 따라 합니다.

-   Windows 기반 컴퓨터 (lan) 및 다양 한 영역 (네트워크) 환경을 연결을 지 원하는 경로 조정 가능 enterprise 네트워킹 프로토콜 합니다.

-   핵심 기술 및 Windows 기반 컴퓨터 정보를 공유 하기 위해 다른 시스템을 사용 하 여 연결에 대 한 유틸리티입니다.

-   파일 전송 FTP (프로토콜)와 World Wide Web 서버 등 글로벌 인터넷 서비스에 대 한 액세스 하기 위한 기초 합니다.

-   강력한, 확장, 플랫폼 간 클라이언트/서버 프레임 워크 합니다.

TCP/IP은 Windows 기반 컴퓨터에 연결 하 고 다른 Microsoft 및 Microsoft가 아닌 타사 시스템과 포함 하 여 정보를 공유할 수 있도록 하는 기본 TCP/IP 유틸리티를 제공 합니다.

-   Apple Macintosh 시스템

-   VM 시스템 열기

-   네트워크에서 사용할 프린터

-   태블릿 및 휴대폰 전화기 이더넷 유선된 또는 무선 802.11 기술을 사용

##### <a name="router"></a>라우터
이 배포 가이드 핵심 네트워크를 구분 하는 라우터에서 DHCP 전달을 사용할 수 있는 두 서브넷 배포 하기 위한 지침을 제공 합니다. 그러나 2 계층 스위치, Layer 3 스위치 또는 요구 사항 및 리소스에 따라 허브에 배포할 수, 있습니다. 스위치를 배포 하는 경우 DHCP 전달 스위치 수 있어야 또는 DHCP 서버 각 서브넷에 두어야 합니다. 허브 배포 하는 경우 단일 서브넷 배포 하는 하 고 DHCP 전달을 또는 두 번째 범위 DHCP 서버에 필요 하지 않습니다.

##### <a name="static-tcpip-configurations"></a>TCP/IP 구성 고정
이 배포 서버 고정 IPv4 주소도 구성 됩니다. 클라이언트 컴퓨터 IP 주소 임대 DHCP 서버에서 수신 하는 기본적으로 구성 됩니다.

##### <a name="active-directory-domain-services-global-catalog-and-dns-server-dc1"></a>드 Active Directory Domain Services와 DNS 서버 d c 1
모두 이름은 디렉터리를 제공 하는 d c 1이이 서버에 설치 되어 이름 모든 컴퓨터 및 네트워크에서 디바이스를 확인 서비스 (AD DS) Active Directory Domain Services 및 시스템 DNS (도메인 이름).

##### <a name="dhcp-server-dhcp1"></a>DHCP 서버 DHCP1
라는 DHCP1, DHCP 서버를 컴퓨터에 IP (인터넷 프로토콜) 주소 임대 로컬 서브넷을 제공 하는 범위도 구성 됩니다. DHCP 서버 라우터에서 DHCP 전달 구성 된 컴퓨터에 IP 주소 임대 다른 서브넷 제공 하기 위해 추가 범위를 구성할 수도 있습니다.

##### <a name="client-computers"></a>컴퓨터

### <a name="bkmk_NetFndtn_Pln_Subnt"></a>서브넷 계획
TCP/IP (TCP/IP) 네트워킹에서 라우터 interconnect 하드웨어 및 소프트웨어 다른 물리적 네트워크 세그먼트 서브넷 라는에 사용 되는 데 사용 됩니다. 라우터의 IP 패킷을 간에 각 서브넷 전달 하도 사용 됩니다. 실제 레이아웃이 라우터 하 고 지침에이 가이드를 계속 하기 전에 필요한 서브넷 개수를 포함 하 여 네트워크를 결정 합니다.

또한 고정 IP 주소를 사용 하 여 하면 네트워크에서 서버를 구성 하 핵심 네트워크 서버가 있는 서브넷 사용할 IP 주소 범위를 확인 해야 합니다. 이 가이드에서는 개인 IP 주소 범위 10.0.0.1-10.0.0.254 및 10.0.1.1-10.0.1.254 예제로 사용 되지만 선호 하는 모든 개인 IP 주소 범위를 사용할 수 있습니다.

> [!IMPORTANT]
> 각 서브넷 사용할 IP 주소 범위를 선택한 후에 IP 주소 로부터 라우터를 설치한 서브넷에 사용 된 것과 동일한 IP 주소 범위를 라우터를 구성 하는 있는지 확인 합니다. 예를 들어, 라우터 192.168.1.1의 IP 주소를 가진 기본적으로 구성 되어 있지만 라우터의 IP 주소 범위 10.0.0.0 월 24 서브넷에 설치 하는, 경우 10.0.0.0 월 24 IP 주소 범위의 IP 주소를 사용 하 여 라우터를 재구성 해야 합니다.

다음 인식에 대 한 RFC 1918 범위 인터넷 요청 하 여 지정 된 개인 IP 주소:

-   10.0.0.0 - 10.255.255.255

-   172.16.0.0 - 172.31.255.255

-   192.168.0.0 - 192.168.255.255

RFC 1918에 지정 된 대로 개인 IP 주소 범위를 사용 하면 직접 하므로 이러한 주소에서 보내고 받는 요청 인터넷 서비스 공급자 라우터에서 자동으로 삭제 됩니다 개인 IP 주소를 사용 하 여 인터넷에 연결할 수 없습니다. 나중에 인터넷에 연결 핵심 네트워크에 추가 하려면 공용 IP 주소를 획득 하는 ISP는와 계약 해야 있습니다.

> [!IMPORTANT]
> 개인 IP 주소를 사용 하는 경우 인터넷에 전달 수 있는 공개 IP 주소를 로컬 네트워크의 개인 IP 주소 범위 변환 몇 가지 유형의 프록시 또는 네트워크 주소 NAT (변환) 서버를 사용 해야 합니다. 대부분의 라우터 NAT 수 있는 라우터 선택 매우 간단 이어야 NAT 서비스를 제공 합니다.

### <a name="bkmk_NetFndtn_Pln_DomAccess"></a>계획 도메인 액세스
도메인에 로그온 할 때 켜져 있어야 하는 도메인 회원 컴퓨터와 사용자 계정에에서 만들어야 AD DS 로그온 시도가 하기 전에 합니다.

> [!NOTE]
> Windows를 실행 하는 개별 컴퓨터는 로컬 사용자 및 그룹 사용자 계정 보안 계정을 관리자 (삼로) 사용자 계정 데이터베이스 라고 하는 데이터베이스 합니다. 사용자 계정 삼로 데이터베이스에 로컬 컴퓨터에서를 만들 때 로컬 컴퓨터에 로그온 할 수 있지만 도메인에 로그온 할 수 없습니다. 사용자 도메인 계정 Active Directory 사용자 및 컴퓨터 Microsoft Management Console (MMC) 로컬 사용자 및 그룹 로컬 컴퓨터에서 사용 하지 도메인 컨트롤러에서 생성 됩니다.

도메인 로그온 자격 증명으로 성공한 첫 번째 로그온 한 후 로그온 설정을 유지 하지 않으면 로그온 설정을 수동으로 변경 또는 컴퓨터에서 도메인 제거 됩니다.

이전에 로그온 할 도메인.

-   Active Directory 사용자 및 컴퓨터 사용자 계정 만들기. 각 사용자 Active Directory 사용자 및 컴퓨터 Active Directory Domain Services 사용자 계정이 있어야 합니다. 자세한 내용은 참조 [Active Directory 사용자 및 컴퓨터 사용자 계정 만들기](#BKMK_createUA)합니다.

-   잘못 구성 된 IP 주소를 확인 합니다. 컴퓨터에서 도메인에 가입, 하려면 컴퓨터의 IP 주소가 있어야 합니다. 이 가이드에서는 서버가 고정 IP 주소도 구성 하 고 클라이언트 컴퓨터 DHCP 서버에서 IP 주소 임대 수신 합니다. 이런 이유이 때문에 대 한 클라이언트 도메인에 연결 하기 전에 DHCP 서버를 배포 합니다. 자세한 내용은 참조 [배포 DHCP1](#BKMK_deployDHCP01)합니다.

-   컴퓨터에서 도메인에 가입 합니다. 네트워크 리소스에 액세스 하거나 제공 하는 모든 컴퓨터가 도메인에 가입 있어야 합니다. 자세한 내용은 참조 [도메인 및에 로그온 서버 컴퓨터 참여](#BKMK_joinlogserver) 및 [도메인 및에 로그인 하는 클라이언트 컴퓨터 참여](#BKMK_joinlogclients)합니다.

### <a name="bkmk_NetFndtn_Pln_DHCP-01"></a>DHCP1 배포를 계획
다음 주요 계획 단계 DHCP 서버 역할 DHCP1에 설치 하기 전에 합니다.

#### <a name="planning-dhcp-servers-and-dhcp-forwarding"></a>DHCP 서버와 DHCP 전달을 계획
DHCP 메시지는 되므로 알림된 메시지를 라우터에서 서브넷 전달 되지 않습니다. 여러 개의 서브넷 있고 각 서브넷 DHCP 서비스를 제공 하려는 경우 다음 중 하나를 수행 해야 합니다.

-   DHCP를 서브넷 각에 설치

-   라우터 서브넷 전체 DHCP 알림된 메시지를 전달 하 고 여러 범위 DHCP 서버의 서브넷 마다 하나의 범위 구성를 구성 합니다.

대부분의 경우 라우터에서 DHCP 알림된 메시지를 전달 하도록 구성는 네트워크의 각 영역에 DHCP를 배포 보다 더 많은 비용 효과적인 합니다.

#### <a name="planning-ip-address-ranges"></a>IP 주소 범위 계획
각 서브넷 자체 고유한 IP 주소 범위가 있어야 합니다. 이 범위 범위가 DHCP 서버에 표시 됩니다.

범위 서브넷 DHCP 서비스를 사용 하는 컴퓨터에 IP 주소를의 관리 그룹입니다. 관리자 먼저 각 물리적 서브넷 범위를 만들고 범위를 사용 하는 클라이언트에서 사용 되는 매개 정의 합니다.

범위에는 다음과 같습니다.

-   IP 주소를 포함 하거나 서비스 임대 DHCP 사용 하는 주소를 제외 하 합니다.

-   서브넷 마스크 서브넷 접두사 IP 주소를 지정된를 결정 합니다.

-   만들 때 할당 범위 이름입니다.

-   임대 기간 값 동적 할당된 IP 주소를 수신 하는 클라이언트 DHCP에 지정 합니다.

-   DHCP 클라이언트 라우터/기본 게이트웨이 IP 주소, DNS 서버 IP 주소 등에 할당에 대 한 구성 된 DHCP 범위 옵션입니다.

-   예약은 원하는 경우 DHCP는 클라이언트 항상 동일한 IP 주소를 받을 사용 됩니다.

서버를 배포 하기 전에 서브넷 및 각 서브넷 사용할 IP 주소 범위 나열 합니다.

#### <a name="planning-subnet-masks"></a>서브넷 마스크 계획
네트워크 Id 및 IP 주소 호스트 Id 서브넷 마스크를 사용 하 여 구분 됩니다. 각 서브넷 마스크 32 비트 숫자 그룹의 모든 기능을 사용 하는 네트워크를 식별 (1)의 IP 주소 호스트 ID 일부를 식별 하기 위해 ID와 모두 0 (0).

예를 들어, 131.107.16.200 IP 주소를 정상적으로 사용 되는 서브넷 마스크 다음 32 비트 이진 번호는 다음과 같습니다.

```
11111111 11111111 00000000 00000000
```

이 서브넷 마스크 번호는 16 1 비트 16 0 비트 이어서이 IP 주소 네트워크 ID와 호스트 ID 섹션의 길이 모두 16 비트가 합니다. 일반적으로이 서브넷 마스크 255.255.0.0 소수점 표기에 표시 됩니다.

다음 표에서 서브넷 마스크 인터넷 주소 클래스를 표시합니다.

|주소 클래스|서브넷 마스크 비트|서브넷 마스크|
|-----------------|------------------------|---------------|
|클래스 A|11111111 00000000 00000000 00000000|255.0.0.0|
|클래스 B|11111111 11111111 00000000 00000000|255.255.0.0|
|클래스 C|11111111 11111111 11111111 00000000|255.255.255.0|

DHCP에서 범위를 만들고 IP 주소 범위 범위를 입력 하는 경우 DHCP는 이러한 기본값 서브넷 마스크 제공 합니다. 일반적으로 서브넷 마스크 기본값은 없이 특별 요구 사항이 있는 대부분의 네트워크 하 고 각 IP 네트워크 세그먼트 실제 네트워크에 해당 하는 어디에 적합 합니다.

경우에 따라 정의 서브넷 마스크 IP 서브넷 구현 사용할 수 있습니다. IP 서브넷 IP 주소를 지정 서브넷 세부 원래 네트워크 클래스 기반 id가 기본 호스트 ID 부분을 나눌 수 있습니다.

서브넷 마스크 길이 사용자 지정 하 여 실제 호스트 id 사용 되는 비트의 수를 줄일 수 있습니다.

주소 및 라우팅 문제를 방지 하려면 모든 TCP/IP 컴퓨터에는 네트워크 세그먼트 같은 서브넷 마스크 사용 하 여 각 컴퓨터 또는 디바이스의 고유한 IP 주소가 있는지 확인 해야 합니다.

#### <a name="planning-exclusion-ranges"></a>계획 제외 범위
DHCP 범위를 만들 때 IP 주소 같은 컴퓨터 및 기타 장치 DHCP 클라이언트로 임대 DHCP 서버 허용 되는 모두 포함 된 IP 주소 범위를 지정 합니다. 다음으로 이동 하 고 일부 서버 수동으로 구성 및 DHCP 서버를 사용 하는 동일한 IP 주소 범위에서 고정 된 다른 디바이스 IP 주소, IP 주소 충돌 실수로 만들 수 있습니다와 DHCP 서버 있는 모두 할당 동일한 IP 주소 서로 다른 장치를.

이 문제를 해결 하기 위해 DHCP 범위에 대 한 제외 범위를 만들 수 있습니다. 제외 범위 DHCP 서버를 사용할 수 없는 범위의 IP 주소 범위 내 연속 IP 주소입니다. 제외 범위를 만든 경우 DHCP 서버 수동으로 IP 주소가 충돌 만들지 않고도이 주소를 지정 하도록 허용 하는 범위 내에 주소를 지정 하지 않습니다.

각 영역에 대 한 제외 범위 만들어 DHCP 서버에 IP 주소 배포에서 제외할 수 있습니다. 고정 IP 주소를으로 구성 된 모든 디바이스에 대 한 제외를 사용 해야 합니다. 제외 주소 다른 서버, DHCP 비 클라이언트, 워크스테이션, 디스크 또는 라우팅 및 원격 액세스 PPP 클라이언트를 수동으로 할당 모든 IP 주소를 포함 해야 합니다.

향후 네트워크 증가 맞게 추가 주소로 제외 범위 구성 하는 것이 좋습니다. 다음 표에서과 10.0.0.1-10.0.0.254과 255.255.255.0 서브넷 마스크의 IP 주소 범위 범위에 대 한 예제 제외 범위를 제공합니다.

|구성 항목|예 값|
|-----------------------|------------------|
|IP 주소 시작 제외 범위|10.0.0.1|
|제외 범위 종료 IP 주소|10.0.0.25|

#### <a name="planning-tcpip-static-configuration"></a>TCP/IP 고정 구성 계획
특정 디바이스 DNS 서버, DHCP 서버 라우터 등에서 고정 IP 주소를 구성 합니다. 또한 추가 장치를 항상 되도록 하려면 되는 프린터 등 동일한 IP 주소가 할 수도 있습니다. 구성 하려면 정적 각 서브넷 장치 하 고 DHCP 서버 정적으로 구성 된 디바이스의 IP 주소 임대 하지 않는 되도록 DHCP 서버에서 사용 하려는 제외 범위 계획을 수립 합니다. 제외 범위는 제한 된 시퀀스 DHCP 서비스에서 제외 범위 내에 IP 주소입니다. 제외 범위 보증할 DHCP 클라이언트 네트워크에 이러한 범위에 있는 주소 서버에서 제공 되지 않습니다.

예를 들어, 서브넷 IP 주소 범위는 192.168.0.254 통해 192.168.0.1에서 고정 IP 주소를 구성 하 고 싶은 10 장치가 있는 경우에 192.168.0 제외 범위를 만들 수 있습니다. *x* 10 개 이상의 IP 주소를 포함 하는 범위: 192.168.0.15 통해 192.168.0.1 합니다.

여기에서 고정 IP 주소와 서버 및 다른 디바이스를 구성 하 10 제외 IP 주소를 사용 하 고 5 추가 IP 주소가 앞으로 추가 하려는 경우 하는 새 디바이스에 정적 구성에 맞게 사용할 수 있는 남아 있습니다. 이 제외 범위가 DHCP 서버 주소 풀 192.168.0.254 통해 192.168.0.16의 그대로 유지 됩니다.

다음 표에서에 AD DS 및 DNS에 대 한 자세한 예제 구성 항목 제공 됩니다.

|구성 항목|예 값|
|-----------------------|------------------|
|네트워크 연결 바인딩을|이더넷|
|DNS 서버 설정|DC1.corp.contoso.com|
|기본 설정된 DNS 서버 IP 주소|10.0.0.2|
|범위 대화 상자가 값 추가<br /><br />1. 범위 이름<br />2. IP 주소를 시작합니다.<br />3. 종료 IP 주소<br />4. 서브넷 마스크<br />5. 기본 게이트웨이 (선택 사항)<br />6. 임대 기간|1. 기본 서브넷<br />2.  10.0.0.1<br />3.  10.0.0.254<br />4.  255.255.255.0<br />5.  10.0.0.1<br />6. 8 일|

#### <a name="BKMK_ip"></a>고정 IP 주소를 구성
이 항목의 네트워크 연결의 인터넷 프로토콜 버전 4 (IPv4) 속성와 Windows Server 2016을 실행 하는 컴퓨터에 고정 IP 주소를 구성 하는 절차를 사용할 수 있습니다.

> [!NOTE]
> Windows PowerShell를 사용 하 여이 절차를 수행, PowerShell 열고 하 고 다른 줄에 다음 cmdlet 입력 한 다음 ENTER 키를 누릅니다. 인터페이스 이름 및이 이때에 IP 주소를 사용 하 여 컴퓨터 구성 원하는 값으로도 교체 해야 합니다.
>
> `New-NetIPAddress -IPAddress 10.0.0.2 -InterfaceAlias "Ethernet" -DefaultGateway 10.0.0.1 -AddressFamily IPv4 -PrefixLength 24`
>
> `Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses 127.0.0.1`

###### <a name="to-configure-a-static-ip-address-on-computers-running-windows-server-2016-windows-server-2012-r2-and-windows-server-2012"></a>Windows Server 2016, Windows Server 2012 R2 및 Windows Server 2012를 실행 하는 컴퓨터에서 고정 IP 주소를 구성 하려면

1.  작업 표시줄의 네트워크 아이콘을 마우스 오른쪽 단추로 클릭 한 다음 **공개 네트워크 및 공유 센터**합니다.

2.  **네트워크 및 공유 센터**, 클릭 **어댑터 설정 변경**합니다. **네트워크 연결** 폴더를 열고 사용 가능한 네트워크 연결을 표시 합니다.

3.  **네트워크 연결**를 구성 하 고 다음을 클릭 하려면 연결을 마우스 오른쪽 단추로 클릭 **속성**합니다. 네트워크 연결 **속성** 대화 상자를 엽니다.

4.  네트워크 연결에 **속성** 대화 상자의 **이 연결에 다음 항목 사용**, **인터넷 프로토콜 버전 4 (TCP/IPv4)**을 차례로 클릭 하 고 **속성**합니다. **인터넷 프로토콜 버전 4 (TCP/IPv4) 속성** 대화 상자를 엽니다.

5.  **인터넷 프로토콜 버전 4 (TCP/IPv4) 속성**에 **일반** 탭을 클릭 **다음 IP 주소를 사용 하 여**합니다. **IP 주소**를 사용할 IP 주소를 입력 합니다.

6.  Tab 키를 눌러에 커서를 놓습니다 **서브넷 마스크**합니다. 서브넷 마스크에 대 한 기본값은 자동으로 입력 됩니다. 기본 서브넷 마스크 수락 하거나 사용 하려는 서브넷 마스크 입력 합니다.

7.  **기본 게이트웨이**, 기본 게이트웨이의 IP 주소를 입력 합니다.

    > [!NOTE]
    > 구성 해야 **기본 게이트웨이** 라우터에의 (lan)에 사용 하는 동일한 IP 주소를 사용 합니다. 예를 들어,는 인터넷도 LAN 등 다양 한 영역 WAN 네트워크에 연결 되어 있는 라우터를 사용 하는 경우 인터페이스를 구성 LAN는 다음으로 지정 하는 동일한 IP 주소에서 **기본 게이트웨이**합니다. 다른 예에서 라우터 두 Lan LAN A 사용 하 여 주소 범위 10.0.0.0 월 24 LAN B 사용 하 여 주소 범위 192.168.0.0 월 24 장소와에 연결 되어 있는 경우 해당 주소 범위 10.0.0.1 등의 주소와 LAN A 라우터의 IP 주소를 구성 합니다. 또한이 주소로 범위 내에 대 한 DHCP 범위를 구성할 **기본 게이트웨이** 10.0.0.1 IP 주소를 사용 합니다. LAN b LAN B 라우터 인터페이스 등 192.168.0.1, 해당 주소 범위에서 주소를 구성 하 고 다음는 LAN B 범위 192.168.0.0 월 24로 구성는 **기본 게이트웨이** 192.168.0.1 값 합니다.

8.  **기본 설정 DNS 서버**, DNS 서버의 IP 주소를 입력 합니다. 로컬 컴퓨터 기본 설정된 DNS 서버를 사용 하려는 경우 로컬 컴퓨터의 IP 주소를 입력 합니다.

9. **대체 DNS 서버**를 있는 경우 대체 DNS 서버의 IP 주소를 입력 합니다. 로컬 컴퓨터 대체 DNS 서버를 사용 하려는 경우 로컬 컴퓨터의 IP 주소를 입력 합니다.

10. 클릭 **확인**을 차례로 클릭 하 고 **닫기**합니다.

> [!NOTE]
> 다른 Microsoft 운영 체제를 실행 하는 컴퓨터에서 고정 IP 주소를 구성 하는 방법에 대 한 내용은 참조 [부록 B-고정 IP 주소 구성](#BKMK_B)합니다.

사용자 도메인 그룹의 모든 구성원이 수 컴퓨터에 로그온 모든 클라이언트 도메인에 가입한 후 합니다.

사용자 계정에는 사용자가 컴퓨터에 로그온 할 수 있는 날짜 및 시간을 지정 구성할 수 있습니다. 각 사용자가 사용 하 여 사용할 수 있는 컴퓨터 지정할 수 있습니다. 이러한 설정을 구성할, Active Directory 사용자 및 컴퓨터 엽니다, 구성 하려면 사용자 계정 찾아 계정을 두 번 클릭 합니다. 사용자 계정에서 **속성**를 클릭는 **계정** 탭을 클릭 한 다음 **로그온 시간** 또는 **에 로그**합니다.

#### <a name="BKMK_installAD-DNS"></a>AD DS 및 DNS 새 숲에 설치

DNS Active Directory Domain Services (AD DS)를 설치 하 고 새 숲 속의 새 도메인 만들 다음 절차 중 하나를 사용할 수 있습니다.

**AD DS 및 DNS Windows PowerShell를 사용 하 여 설치**

설치 및 구성 AD DS 및 DNS 뒤 다음 명령을 사용할 수 있습니다. 도메인에 대 한 사용 하려는 값이 예제 도메인 이름을 바꿔야 합니다.

>[!NOTE]

>- [Install-WindowsFeature](https://technet.microsoft.com/itpro/powershell/windows/server-manager/install-windowsfeature)
>- [Install-ADDSForest](https://technet.microsoft.com/itpro/powershell/windows/adds/deployment/install-addsforest)

`Install-WindowsFeature AD-Domain-Services -IncludeManagementTools`


    **SafeModeAdministratorPassword:**

    **SafeModeAdministratorPassword 확인 합니다.**

    이 작업을 계속 하 시겠습니까?

##### <a name="BKMK_reverse"></a>DNS 역방향 조회 영역이 구성

회원 **도메인 관리자** 이 절차를 수행 하는 데 필요한 최소한 합니다.

> [!NOTE]

> -   Windows PowerShell를 사용 하 여이 절차를 수행 하려면 PowerShell 하 고 열고 한 줄에 다음 cmdlet 입력 한 다음 ENTER 키를 누릅니다. 사용 하려는 된 값이 예제 DNS 역방향 조회 영역이 및 zonefile 이름도 교체 해야 합니다. 역방향 영역 이름에 대 한 네트워크 ID 반전 있는지 확인 합니다. 예를 들어 네트워크 ID 192.168.0 이면 만들 역방향 조회 영역이 이름 **0.168.192.in addr.arpa**합니다.
>
> `Add-DnsServerPrimaryZone 0.0.10.in-addr.arpa -ZoneFile 0.0.10.in-addr.arpa.dns`

###### <a name="to-configure-a-dns-reverse-lookup-zone"></a>DNS 역방향 조회 영역이 구성 하려면

1.  서버 관리자 d c 1 클릭 **도구**을 차례로 클릭 하 고 **DNS**합니다. DNS MMC을 엽니다.

2.  DNS에서 이미 확장 되지 않은 경우 이름을 두 번 서버 트리를 확장 합니다. 예를 들어, DNS 서버 이름 d c 1 경우 두 번 클릭 **d c 1**합니다.

3.  선택 **역방향 조회 영역이**를 마우스 오른쪽 단추로 클릭 **역방향 조회 영역이**을 차례로 클릭 하 고 **새 영역**합니다. 새로운 영역 마법사가 열립니다.

4.  **새로운 영역 마법사**, 클릭 **다음**합니다.

5.  **영역 종류**선택 **주 영역**합니다.

6.  쓰기 도메인 컨트롤러, DNS 서버 경우 되도록 **영역 Active Directory에 저장** 을 선택 합니다. 클릭 **다음**합니다.

7.  **Active Directory 영역 복제 범위**선택 **이 도메인에 있는 도메인 컨트롤러에서 실행 중인 모든 DNS 서버를**다른 옵션을 선택 하는 특별 한 이유가 않은 경우, 합니다. 클릭 **다음**합니다.

8.  첫 번째에서 **역방향 조회 영역이 이름** 페이지를 선택 하 고 **IPv4 역방향 조회 영역이**합니다. 클릭 **다음**합니다.

    -   **네트워크 ID**, IP 주소 범위의 네트워크 ID를 입력 합니다. 예를 들어, IP 주소 범위 10.0.0.254 통해 10.0.0.1 면 입력 **10.0.0**합니다.

    -   **역방향 조회 영역이 이름**, 사용자 IPv4 역방향 조회 영역이 이름이 자동으로 추가 됩니다. 클릭 **다음**합니다.

10. **동적 업데이트**을 허용 하려는 동적 업데이트 유형을 선택 합니다. 클릭 **다음**합니다.

11. **새로운 영역 마법사를 완료**선택 내용을 검토 한 클릭 한 다음, **완료**합니다.

#### <a name="BKMK_deployDHCP01"></a>DHCP1 배포

배포 하는 컴퓨터 DHCP1 DHCP(Dynamic Host Configuration Protocol) (DHCP) 서버 역할을 실행 중인 다음 주문에서 이러한 단계를 완료 해야 합니다.

-   [DHCP(Dynamic Host Configuration Protocol) (DHCP) 설치](#BKMK_installDHCP)

-   [홈 그룹을 만들고 새로운 DHCP 범위를 정품 인증](#BKMK_newscopeDHCP)

> [!NOTE]
> Windows PowerShell를 사용 하 여이 절차를 수행, PowerShell 열고 하 고 다른 줄에 다음 cmdlet 입력 한 다음 ENTER 키를 누릅니다. 사용 하려는 값으로도 범위 이름, IP 주소 시작 및 끝 범위, 서브넷 마스크 및이 예제 다른 값을 교체 해야 합니다.
>
> `Install-WindowsFeature DHCP -IncludeManagementTools`
>
> `Add-DhcpServerv4Scope -name "Corpnet" -StartRange 10.0.0.1 -EndRange 10.0.0.254 -SubnetMask 255.255.255.0 -State Active`
>
> `Add-DhcpServerv4ExclusionRange -ScopeID 10.0.0.0 -StartRange 10.0.0.1 -EndRange 10.0.0.15`
>

네트워크 NPS 정책 서버 ()를 사용 하면 중앙 구성 하 고 다음과 같은 기능을 사용 하 여 네트워크 정책을 관리: RADIUS(Remote Authentication Dial-In User Service) (RADIUS) 서버 및 RADIUS 프록시 합니다.

NPS 선택적 구성 요소는 핵심 네트워크의 이지만 다음 중 하나에 NPS 설치 해야 합니다.

-   Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 또는 Windows Server 2008 및 라우팅과 원격 액세스 서비스, Terminal Services 게이트웨이 또는 원격 데스크톱 게이트웨이의 실행 하는 컴퓨터 등 RADIUS 프로토콜와 호환 되는 원격 액세스 서버를 포함 하도록 네트워크 확장 계획입니다.


-   무선 액세스 또는 유선에 대 한 802.1 X 인증을 배포할 예정입니다.

##### <a name="bkmk_NetFndtn_Pln_NPS-01"></a>NPS1 배포를 계획
네트워크 액세스 서버 무선 액세스 포인트 또는 VPN 서버의 등 핵심 네트워크를 배포한 후 배포 하려는 경우 NPS 배포 하는 것이 좋습니다.

NPS (RADIUS) RADIUS(Remote Authentication Dial-In User Service) 서버를 사용 하면 NPS 인증과 네트워크 액세스 서버 통해 연결 요청을 수행 합니다. NPS 수도 있습니다 중앙 구성 하 고 네트워크 정책을 관리 네트워크에 액세스할 수 있는 네트워크를 액세스 하는 방식 및 때 네트워크에 액세스할 수 있습니다 결정 하는 합니다.

다음은 NPS 설치 하기 전에 주요 계획 단계입니다.

- 사용자 계정 데이터베이스를 계획입니다. 기본적으로 NPS Active Directory 도메인을 실행 하는 서버에 가입 하면 NPS 수행 AD DS 사용자 계정 데이터베이스를 사용 하 여 인증과 합니다. 경우에 따라와 같은 다른 RADIUS 서버에 연결 요청을 전송 하도록 NPS RADIUS 프록시도 사용 하는 큰 네트워크 하려는 경우 NPS 비 도메인 회원 컴퓨터에 설치 합니다.

- RADIUS 계정을 계획입니다. NPS는 SQL Server 데이터베이스에 또는 텍스트 파일 로컬 컴퓨터에 계정 데이터를 기록 수 있습니다. SQL Server 로깅 사용 하려는 경우 설치 및 구성 SQL Server 실행 하는 서버 계획입니다.
