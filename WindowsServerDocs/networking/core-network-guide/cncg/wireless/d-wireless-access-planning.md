# <a name="wireless-access-deployment-planning"></a>무선 액세스 배포 계획

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

무선 액세스를 배포 하기 전에 다음과 같은 항목을 계획 해야 합니다.

- 무선 액세스를 설치 하면 네트워크에서 \(APs\) 포인트

- 무선 클라이언트 구성과 액세스

다음 섹션에서는이 계획 단계에 자세히 설명 합니다.

## <a name="planning-wireless-ap-installations"></a>무선 계획 AP 설치
무선 네트워크 액세스 솔루션을 만들 때 다음을 수행 해야 합니다.

1. 무선 Ap 지원 해야 하는 어떤 표준 확인
2. 무선 서비스를 제공 하려는 영역과 확인
3. 무선 Ap 찾으려는 확인

또한, IP 주소 구성표 무선 AP 계획 하 고 클라이언트 무선 해야 합니다. 섹션을 참조 **무선 AP의 NPS에서 구성 계획** 아래에 대 한 관련 정보 합니다.

### <a name="verify-wireless-ap-support-for-standards"></a>표준 AP 지원 무선 확인
일관성 목적으로 간편 하 게 배포 및 AP 관리에 대 한 동일한 브랜드 및 모델의 무선 앱을 배포 하는 것이 좋습니다.

무선 Ap 배포 하는 다음을 지원 해야 합니다.

- **IEEE 802.1 X**

- **RADIUS 인증**

- **무선 인증 및 암호입니다.** 최소을 선호 하는 가장 순서 대로 나열 되어:

    1.  AES와 WPA2\ 엔터프라이즈

    2.  WPA2\ 엔터프라이즈 TKIP 사용

    3.  AES와 WPA\ 엔터프라이즈

    4.  WPA\ 엔터프라이즈 TKIP 사용

>[!NOTE]
>W p a 2를 배포 하는 무선 네트워크 어댑터와 무선 Ap도 지원 w p a 2를 사용 해야 합니다. 그렇지 않으면 WPA\ Enterprise를 사용 합니다.

또한를 제공 하기 위해 강화 된 보안 네트워크에 대 한 무선 Ap 다음과 같은 보안 옵션 지원 해야 합니다.

- **DHCP를 필터링 합니다.** 무선 클라이언트가 DHCP 구성 되어 있는 경우 해당 알림된 메시지 DHCP 전송 되지 않도록 하려면 IP 포트에서 무선 AP 필터링 해야 합니다. 무선 AP IP 패킷을 UDP 포트 68 네트워크에 보내는 클라이언트를 차단 해야 합니다.

- **DNS 필터링 합니다.** 무선 AP 클라이언트 DNS 서버를 수행 하지 않도록 하려면 IP 포트 필터링 해야 합니다. 무선 AP TCP 또는 UDP IP 패킷을 포트 네트워크에 53 보내지 클라이언트를 차단 해야 합니다.

- **클라이언트 격리** 위장 이용 가능한 확인 프로토콜 주소 \(ARP\) 방지 기능을 사용 해야 경우 무선 액세스 포인트 클라이언트 격리 기능을 제공 합니다.

### <a name="identify-areas-of-coverage-for-wireless-users"></a>무선 사용자를 위한 범위가 식별
각 빌드에 대 한 각 층의 아키텍처 드로잉을 사용 하 여 무선 보도 제공 하려는 영역을 식별. 예를 들어 사무실 적절 한, 회의 룸, 로비, 식당, 또는 courtyards 식별 합니다.

그리기, 의료 장비, 무선 비디오 카메라, 2.4 g h z 산업, 공학용 및 의료 \(ISM\) 범위 2.5를 통해에서 작동 하는 무선 전화기 등 무선 신호를 방해할 하는 디바이스 및 Bluetooth\ 가능한 장치를 나타냅니다.

그리기에서; 무선 신호를 방해할 수 있는 건물의 측면 표시 건설 건물에 사용 되는 금속 물체에서 무선 신호의 영향을 줄 수 있습니다. 다음과 같은 일반적인 개체 신호 전파 방해가 될 수 있습니다 예를 들어,: 승강기를 air\ 조절 덕트 및 girders 구체적인 지원 합니다.

해당 AP 제조업체 무선 AP 무선 주파수 감쇠 발생할 수 있는 원본에 대 한 정보를 참조 하세요. 대부분 Ap 신호 강도, 평가, 오류 및 데이터 처리에 대 한 확인 하는 데 사용할 수 있는 테스트 소프트웨어를 제공 합니다.

### <a name="determine-where-to-install-wireless-aps"></a>무선 앱을 설치 하는 위치 결정
아키텍처 드로잉에서 무선 Ap을 닫기 기능을 제공 하도록 충분히 무선 있을 만큼 있지만 떨어져 서로에 방해가 되지 않는 충분히 함께 찾습니다.

필요한 간격 Ap AP의 종류에 따라 달라 집니다 및 AP 안테나를 통해 건물 측면을 차단 하는 무선 신호를 및 기타 소스의 간섭을입니다. 무선 각 탭에서 옆에 있는 모든 무선 AP 300 피트 이상 되지 않도록 무선 AP 위치를 표시할 수 있습니다. 위치에 대 한 지침와 AP 사양에 대 한 무선 AP 제조업체의 설명서를 참조 합니다.

일시적으로 아키텍처 드로잉에 지정 된 위치에 무선 앱을 설치 합니다. 그런 다음 무선 어댑터가 장착 802.11 무선 어댑터 일반적으로 제공 되는 사이트 설문 조사 소프트웨어와 랩톱을 사용 신호 강도 검사 각 영역 내에서 결정 합니다.

신호 강도 낮습니다 영역과, 신호 강도 검사 영역에 대 한 향상, 추가 무선 Ap 필요한 보도 제공 재배치 또는 신호 간섭을 뽑고를 설치 하기 위해 AP을 놓습니다.  

모든 무선 Ap 최종 위치를 나타내는 아키텍처 드로잉 업데이트 합니다. 정확한 AP 배치 지도 데 업그레이드 하거나 교체 Ap 하려는 경우 또는 나중 작업 문제를 해결 하는 동안 하는 데 도움이 됩니다.

### <a name="plan-wireless-ap-and-nps-radius-client-configuration"></a>탭 하 고 NPS RADIUS 클라이언트 구성 무선 계획
개별적으로 또는 그룹 무선 Ap 구성 NPS 사용할 수 있습니다.

많은 앱을 포함 하 여 큰 무선 네트워크를 배포 하는 경우에 훨씬 더 쉽게 그룹 Ap 구성할 수 있습니다. Ap NPS RADIUS 클라이언트 그룹을 추가 하려면 이러한 속성 Ap 구성 해야 합니다.

- 무선 Ap IP 주소에서 동일한 IP 주소 범위 구성 됩니다.

- 무선 Ap 모두 동일한 공유 암호도 구성 됩니다.

### <a name="plan-the-use-of-peap-fast-reconnect"></a>계획 PEAP 빠른 연결을 사용 하 여
802.1 X 인프라, 무선 액세스 포인트 RADIUS 클라이언트로 RADIUS 서버 구성 됩니다. PEAP 패스트를 다시 연결 되 면, 배포 두 개 이상의 액세스 지점 간 로밍 하는 무선 클라이언트를 새로 연결할 때마다 인증 필요 하지 않습니다.

다시 연결 PEAP 빠른 새 액세스 지점에서 인증과 클라이언트 연결 요청에 대 한 원래 실행 NPS 서버 인증 요청 전달 되기 때문에 클라이언트와 인증 간에 인증에 대 한 응답 시간을 줄입니다.

캐시 된 Transport Layer Security \(TLS\) 연결 속성 PEAP 클라이언트와 이전에 사용 하 여 둘 다 NPS 서버 하기 때문에 \ (중 컬렉션의 TLS handle\ 이름은), NPS 서버 클라이언트가 다시 연결에 대 한 권한이 있는지를 신속 하 게 확인할 수 있습니다.

>[!IMPORTANT]
>제대로 작동 하려면 다시 연결 패스트, Ap 같은 서버 NPS RADIUS 클라이언트로 구성 합니다.

원래 NPS 서버를 사용할 수 또는 클라이언트 다른 RADIUS 서버에 RADIUS 클라이언트도 구성 된는 액세스 지점으로 이동 하는 경우 전체 인증은 클라이언트와 새로운 인증 발생 해야 합니다.

### <a name="wireless-ap-configuration"></a>무선 AP 구성
다음 목록에서 항목 802.1X\에 일반적으로 구성 된 요약-지원 무선 앱:

> [!NOTE]
> 항목 이름의 브랜드와 모델에 따라 다를 수 하 고 다음 목록에 다를 수 있습니다. 특정 configuration\ 세부 정보에 대 한 무선 AP 설명서를 참조 하십시오.

- **서비스 식별자를 설정 \(SSID\)**합니다. 무선 네트워크의 이름 \ (예를 들어 ExampleWlan\) 및 무선 클라이언트를 알리는 이름을 합니다. 혼란을 줄이기 위해 광고를 선택 하는 SSID 수신 무선 네트워크가 범위 내에 있는 모든 무선 네트워크에서 방송 하는 SSID 일치 하지 해야 합니다.

    동일한 무선 네트워크의 일환으로 여러 무선 Ap 배포 되는 경우에서와 동일한 SSID 각 무선 AP을 구성 합니다. 동일한 무선 네트워크의 일환으로 여러 무선 Ap 배포 되는 경우에서와 동일한 SSID 각 무선 AP을 구성 합니다.  

    다른 무선 네트워크 비즈니스 특정 요구에 맞게 배포 해야 하는 있는 경우에 무선 AP의 네트워크에는 SSID 하면 다른 network\(s\) 보다 다른 SSID 브로드캐스트 해야 합니다. 예를 들어, 필요한 경우 별도 무선 네트워크 직원 및 게스트, 구성할 수 있습니다 비즈니스 네트워크에 대 한 무선 Ap SSID 브로드캐스트으로 설정 된 **ExampleWLAN**합니다. 각 무선 AP SSID 브로드캐스트 설정 다음 게스트 네트워크에 대 한 **GuestWLAN**합니다. 이런 방법으로 직원와 게스트 불필요 한 혼동 없이 의도 네트워크에 연결할 수 있습니다.  

    > [!TIP]  
    > 무선 일부 AP 여러 ssid의에 맞게 multi\ 네트워크 배포 하는 기능이 있습니다. 배포 및 유지 관리 작업 비용 무선 AP의 여러 SSID 브로드캐스트 수를 줄일 수 있습니다.  

- **무선 인증 및 암호화**합니다.

    무선 인증은 무선 클라이언트 무선 액세스 포인트를 연결할 때 사용 되는 보안 인증 합니다.  

    무선 암호화 무선 인증 무선 AP 간의 무선 클라이언트 전송 커뮤니케이션 보호 하기 위해 사용 되는 보안 암호화 암호입니다.  

- **AP IP 주소 \(static\) 무선**합니다. 각 무선 AP 고유한 고정 IP 주소를 구성 합니다. 서브넷 서비스를 제공 DHCP 서버, DHCP 서버 다른 컴퓨터 또는 디바이스에 동일한 IP 주소를 발급 하려고 하지 되도록 모든 AP IP 주소 DHCP 제외 범위 내 속하는 확인 합니다. 제외 범위에 "만들고 새 DHCP 범위를 정품 인증"을 절차에 설명 되어는 [Core 네트워크 가이드](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide)합니다. 그룹 NPS에 의해 RADIUS 클라이언트로 Ap 구성 하려면 계획 하는 경우 각 탭에 있는 그룹에 IP 주소 로부터 동일한 IP 주소 범위 있어야 합니다.

- **DNS 이름**합니다. 일부 무선 Ap DNS 이름으로 구성할 수 있습니다. 각 무선 AP 고유한 이름의을 구성 합니다. 예를 들어, 배포 무선 Ap multi\ 이야기 건물 있는 경우 처음 세 개 무선 Ap 세 번째 바닥 AP3\ 01, AP3\ 02 AP3\ 03에 배포 되는 이름을 수 있습니다.

- **무선 AP 서브넷 마스크**합니다. 마스크 부분을 IP 주소는 네트워크 ID 및 IP 주소는 일부는 호스트 지정를 구성 합니다.

- **AP DHCP 서비스**합니다. 무선 AP에 DHCP built\에 서비스를 사용 하지 않도록 설정 합니다.

- **공유 암호 RADIUS**합니다. 사용 하 여 고유한 RADIUS NPS RADIUS 클라이언트 그룹-중인 상황에서 구성 해야 모든 Ap 그룹에서 공유 동일한 암호를 구성할 계획인 하지 않으면 무선 각 탭에 대 한 암호를 공유 합니다. 임의 이상 22 자를 모두 대문자로 사용 하 여 소문자, 숫자와 문장 부호 공유 암호가 있어야 합니다. 임의성 되도록 공유 암호를 만들 수 임의 문자 세대 프로그램을 사용할 수 있습니다. 무선 각 탭에 대 한 공유 암호를 기록 하 고 안전 office 등의 안전한 장소에 저장 것이 좋습니다. NPS 콘솔에서 RADIUS 클라이언트를 구성할 때 가상 버전 각 탭을 만듭니다. 공유 암호 NPS에서 가상 각 탭에서 구성 하는 실제 탭에 공유 암호를 일치 해야 합니다.

- **RADIUS 서버 IP 주소**합니다. 인증이 액세스 포인트에 대 한 연결 요청을 승인 하는 데 사용할 NPS 서버의 IP 주소를 입력 합니다.

- **UDP port\(s\)**합니다. 기본적으로 NPS RADIUS 인증 메시지 및 UDP 포트 1813 및 RADIUS 계정 메시지에 대 한 1646 1812 및 1645 UDP 포트 사용합니다. 기본 RADIUS UDP 포트 설정을 변경 하지 않는 것이 좋습니다.

- **Vsa**합니다. 일부 무선 Ap 전체 무선 AP 기능을 제공 하도록 vendor\ 특성 \(VSAs\) 필요 합니다.

- **DHCP 필터링**합니다. 무선 Ap IP 패킷을 UDP 포트 68 네트워크에 보내는 무선 클라이언트 차단를 구성 합니다. 무선 AP DHCP 필터링 구성에 대 한 설명서를 참조 하세요.

- **DNS 필터링**합니다. 무선 Ap TCP 또는 UDP 포트 53 IP 패킷을 네트워크에 보내는 무선 클라이언트 차단를 구성 합니다. 무선 AP DNS 필터링 구성에 대 한 설명서를 참조 하세요.

## <a name="planning-wireless-client-configuration-and-access"></a>계획 무선 클라이언트 구성과 액세스

802.1X\ 배포를 계획 하는 경우-인증 무선 액세스 여러 client\ 관련 요인 고려해 야 합니다.

- **여러 표준에 대 한 지원이 계획**합니다.

    여부 무선 컴퓨터 모두 사용 하는 동일한 버전의 Windows 또는 다른 운영 체제를 실행 하는 컴퓨터의 혼합 하는지 여부를 결정 합니다. 다른 경우 간의 차이점을 운영 체제에서 지원 되는 표준 이해 하는 있는지 확인 합니다.

    무선 네트워크 어댑터에서 무선 클라이언트 컴퓨터의 모든 모든 동일한 무선 표준을 지원 하는지 여부 또는 다양 한 표준을 지원 해야 하는지 확인 합니다. 예를 들어 지원 여부를 확인 일부 네트워크 어댑터 하드웨어 드라이버 WPA2\ Enterprise 및 AES를 지원 하지만 다른 WPA\ Enterprise 및 TKIP 합니다.

- **계획 클라이언트 인증 모드**합니다. 인증 모드 Windows 클라이언트 도메인 자격 증명을 처리 하는 방법을 정의 합니다. 무선 네트워크 정책에서 다음 세 가지 네트워크 인증 모드 중에서 선택할 수 있습니다.  

    1. **사용자 re\ 인증**합니다. 이 모드 인증 하는 컴퓨터의 현재 상태에 따라 보안 자격 증명을 사용 하 여 수행 항상 지정 합니다. 사용자가 컴퓨터에 로그온 하는 경우 컴퓨터가 자격 증명을 사용 하 여 인증이 수행 됩니다. 사용자가 컴퓨터에 로그온 하는 경우 사용자 자격 증명을 사용 하 여 언제 든 지이 인증이 수행 됩니다.  

    2. **컴퓨터에만**합니다. 컴퓨터는 컴퓨터 자격 증명을 사용 하 여 모드 지정 인증만 항상 수행 됩니다.  

    3.  **사용자 인증**합니다. 사용자 인증 모드 인증 하는 사용자가 컴퓨터에 로그온 할 때 수행만 지정 합니다. 컴퓨터에 로그온 한 사용자 인 경우 인증 시도 수행 되지 않습니다.  

- **무선 제한 계획**합니다. 모든 무선 사용자에 게 동일한 수준의 무선 네트워크에 대 한 액세스를 제공 하려는 또는 무선 사용자의 일부에 대 한 액세스를 제한 하려면 있는지 확인 합니다. 무선 사용자의 특정 그룹에 대 한 제한 NPS에 적용할 수 있습니다.  예를 들어 수 정의한 특정 날짜 및 시간 특정 그룹 무선 네트워크에 액세스할 수 있게 됩니다.  

- **새 컴퓨터에 무선 추가 하기 위한 방법 계획**합니다. 에 가입 된 도메인 무선 네트워크를 배포 하기 전에 컴퓨터 802.1 X로 보호 되지 않은 유선된 네트워크 영역에 연결 되어 있으면 wireless\ 지원 컴퓨터에 대 한 무선 구성 설정이 자동으로 적용 됩니다 무선 네트워크를 구성 하 고 나면 \ (IEEE 802.11\) 정책 도메인 컨트롤러에서 및 후 그룹 정책 무선 클라이언트를 새로 고쳐집니다.  

    그러나 이미 도메인에 가입 하지 않은 하는 컴퓨터에 대 한 계획 해야 802.1X\에 필요한 설정을 적용 하는 방법-인증 액세스 합니다. 예를 들어 다음 방법 중 하나를 사용 하 여 컴퓨터에서 도메인에 가입 것인지 여부를 결정 합니다.

    1.  컴퓨터에서 도메인에 가입 다음 802.1 X으로 보호 되지 않은 유선된 네트워크 영역에 컴퓨터를 연결 합니다.

    2.  컴퓨터에서 도메인에 가입 할 수 있게 고유한 무선 부트스트랩 프로필을 추가 하는 데 필요한 설정 단계와 무선 사용자에 게 제공 합니다.

    3.  IT 직원이 무선 클라이언트 도메인에 가입를 할당 합니다.

### <a name="planning-support-for-multiple-standards"></a>여러 표준에 대 한 계획 지원

무선 네트워크 \ (IEEE 802.11\) 정책 확장 그룹 정책에서 다양 한 구성 옵션을 지 원하는 다양 한 배포 옵션을 제공 합니다.

를 지원 하 고 표준으로 구성 된 무선 Ap 배포 하 고 다음 무선 네트워크에서 여러 무선 프로필 구성 \ (IEEE 802.11\) 정책, 각 프로필 지정 한 일련의 필요한 표준 합니다.

예를 들어 네트워크 WPA2\ Enterprise 및 AES WPA\ Enterprise 및 AES를 지 원하는 다른 컴퓨터, WPA\ Enterprise 및 TKIP을 지 원하는 다른 컴퓨터를 지 원하는 무선 컴퓨터에 것인지 여부를 결정 해야 합니다.

- 무선 컴퓨터의 모든 약한 암호화 방법을 사용 하 여 지원 하기 위해 한 프로필 구성는 컴퓨터의 모든-이 경우 WPA\ Enterprise 및 지원 TKIP 합니다.  
- 무선 각 컴퓨터에서 지원 되는 최상의 가능한 한 보안을 제공 하기 위해 두 개의 프로필을 구성 합니다. 이 경우에 가장 강력한 암호화 지정 하는 하나의 프로필 구성할 수 \ (WPA2\ Enterprise 및 AES\) 및 약한 WPA\ Enterprise 및 TKIP 암호화를 사용 하는 하나의 프로필 합니다. 여기에서 반드시 WPA2\ Enterprise 및 AES 높은 우선 순위를 사용 하 여 프로필을 배치 합니다. WPA2\ Enterprise 및 AES 사용할 수 없는 컴퓨터에는 자동으로 다음 프로 우선 순위를 건너뛰고 WPA\ Enterprise 및 TKIP 지정 프로필 처리 합니다.

> [!IMPORTANT]
> 연결 컴퓨터를 사용 하 여 수 있는지는 첫 번째 프로필을 사용 하기 때문에 프로필 주문한 목록에는 가장 안전 표준 높은 프로필을 두어야 합니다.

### <a name="planning-restricted-access-to-the-wireless-network"></a>무선 네트워크에 대 한 액세스를 제한 된 계획

대부분의 경우 무선 사용자 다양 한 수준의 무선 네트워크에 대 한 액세스를 제공 해야 할 수 있습니다. 예를 들어, 이번 주 매일 하루 중 시간 일부 사용자에 게 무제한 액세스 허용 하 시겠습니까 수 있습니다. 다른 사용자에 대 한 액세스를 허용할 월요일부터 금요일 핵심 시간 시간과에 대 한 액세스 거부 원하는 될 수 있습니다.

이 가이드 무선 리소스를 일반적인 액세스 권한이 있는 그룹의 모든 사용자에 게 무선 장소 액세스 환경을 만들 수는 지침을 제공 합니다. 무선 사용자가 보안 그룹 Active Directory 사용자 및 컴퓨터 snap\에서 만들고 해당 그룹의 회원 무선 액세스 권한을 부여 하려는 사람에 대 한 모든 사용자를 확인 합니다.

NPS 네트워크 정책, 구성 NPS 인증 결정할 때 처리 개체 보안 그룹 무선 사용자 지정 합니다.

그러나 다양 한 수준의 액세스에 대 한 지원이 필요한 경우 사용자만 다음을 수행 해야 합니다.  

1. 그룹을 만들려면 무선 추가 보안 Active Directory 사용자 및 컴퓨터 둘 이상의 무선 사용자가 보안 그룹을 만듭니다. 예를 들어, 일반적인 작업 시간에 용량이 액세스할 수만 있는 사람에 대 한 그룹 및 다른 사용자의 요구 사항에 맞는 조건에 맞는 다른 그룹 전체 액세스 권한이 있는 사용자를 포함 하는 그룹을 만들 수 있습니다.

2. 사용자가 만든 보안 적절 한 그룹에 추가 합니다.

3. 각 무선 추가 보안 그룹에 대 한 추가 NPS 네트워크 정책을 구성 하 고 정책을 조건 및 각 그룹에 대 한 필요한 제약 적용 하지 구성 합니다.

### <a name="planning-methods-for-adding-new-wireless-computers"></a>새 컴퓨터에 무선 추가 대 한 계획을 수립

도메인 한 다음 도메인 로그온 무선 새 컴퓨터에 가입 하는 기본 방법 도메인 컨트롤러에 액세스할 수 있는 802.1 X 인증 이더넷 스위치도 보호 되지 않는 LAN의 부분이 유선된 연결을 사용 하 여는 합니다.

그러나 경우에 따라 아닐 유선된 연결을 사용 하 여 컴퓨터에서 도메인 또는 첫 번째 로그온 시도가 도메인에 가입한 이미 컴퓨터를 사용 하 여 유선된 연결을 사용 하 여 사용자에 대 한 연결 하기에 적합 합니다.

도메인에 로그온 할 때 사용자에 게 전화 걸기 또는 무선 연결을 사용 하 여 컴퓨터에서 도메인에 가입 하는 domain\ 가입한 컴퓨터와 무선 연결을 사용 하 여 처음 무선 클라이언트 해야 먼저 설정 다음 방법 중 하나를 사용 하 여 네트워크 도메인 컨트롤러에 액세스할 수 있는 세그먼트에서 무선 네트워크에 연결 합니다.

1. **IT 팀의 구성원 무선 컴퓨터가 도메인에 가입 하 고 단일 로그인을 부트스트랩 무선 프로필을 구성 합니다.** 이 방법으로 IT 관리자가 무선 컴퓨터는 유선된 이더넷 네트워크에 연결 하 고 컴퓨터가 도메인에 연결 합니다. 그런 다음 관리자 컴퓨터 사용자를 배포합니다. 컴퓨터를 시작할 때 사용자 로그온 프로세스에 대 한 직접 지정 도메인 자격 증명 무선 네트워크에 연결 하는 도메인 로그온 사용 됩니다.  

2. **사용자 수동으로 부트스트랩 무선 프로필로 무선 컴퓨터를 구성 하 고를 도메인에 추가 합니다.** 이 방법으로 사용자 IT 관리자 로부터 지침에 따라 부트스트랩 무선 프로필로 무선 컴퓨터 수동으로 구성 합니다. 무선 부트스트랩 프로필 무선 연결을 설정 하 고 다음 컴퓨터가 도메인에 가입 사용자 수 있습니다. 컴퓨터가 도메인에 가입한 후 컴퓨터를 다시 시작 사용자 수 도메인에 사용 하 여 로그온 무선 연결 및 해당 도메인 계정 자격 증명 합니다.

무선 액세스를 배포 참조 [무선 액세스 배포](e-wireless-access-deployment.md)합니다.
