# <a name="deploy-password-based-8021x-authenticated-wireless-access"></a>802.1 X 인증된 무선 액세스 Password\ 기반 배포

>적용 대상: Windows Server (세미콜론 연간 채널) Windows Server 2016

이것은 Windows Server에 대 한 도우미 가이드&reg; 2016 핵심 네트워크 가이드 합니다. 핵심 네트워크 가이드 계획 및 배포 완벽 하 게 작동 네트워크와 새 숲 속의 새 Active Directory® 도메인에 필요한 구성 요소에 대 한 지침을 제공 합니다.

이 가이드에서는 전기 연구소 및 전자 제품 엔지니어 \(IEEE\) 802.1X\ 배포 하는 방법에 대 한 지침을 제공 하 여 핵심 네트워크 시 작성 하는 방법을 설명-인증 보호 Extensible 인증 프로토콜-Microsoft 인증 프로토콜 버전 2 사용 하 여 무선 액세스 IEEE 802.11 \ (PEAP\ 기능 MS\ 기능을 제공 v2\).

인증 과정 사용자가 제공 인증서를 보다는 password\ 기반 자격 증명을 PEAP\ 기능 MS\ 기능을 제공 v2 하므로 일반적으로 쉽고 저렴 EAP\ TLS 또는 PEAP\ TLS 보다 배포 하는 합니다.

>[!NOTE]
>이 가이드에서는 IEEE 802.1 X 인증 된 무선 액세스 PEAP\ 기능 MS\ 기능을 제공 v 2는 줄일 수 "무선" 및 "WiFi 액세스."

## <a name="about-this-guide"></a>이 가이드 정보
이 가이드를 아래에 설명 된 필수 가이드 함께에서 다음 WiFi 액세스 인프라를 배포 하는 방법에 대 한 지침을 제공 합니다.  

- 하나 이상의 802.1X\-802.11 무선 액세스 포인트 \(APs\) 가능 합니다.

- Active Directory 도메인 \(AD DS\) 사용자와 컴퓨터는 서비스입니다.

- 그룹 정책 관리 합니다.

- 하나 이상의 네트워크 정책 서버 \(NPS\) 서버 합니다.

- NPS 실행 하는 컴퓨터에 대 한 서버 인증서 합니다.

- Windows를 실행 하는 클라이언트 컴퓨터 무선&reg; 10, Windows 8.1 또는 Windows 8 합니다.

## <a name="technology-overviews"></a>기술 개요
다음은 기술을 개요 배포 무선 액세스 하기 위한 다음과 같습니다.

### <a name="ieee-8021x"></a>IEEE 802.1 X
802.1 X 표준 IEEE 정의 인증 된 네트워크 이더넷 네트워크에 대 한 액세스를 제공 하기 위해 사용 되는 네트워크 port\ 기반 액세스 제어 합니다. 이 네트워크 port\ 기반 액세스 제어 교환 LAN 인프라 물리적 특징을 사용 하 여 LAN 포트에 연결 된 장치를 인증 합니다. 인증 과정에서 오류가 발생 하는 경우 포트에 대 한 액세스 거부 될 수 있습니다. 하지만이 표준은 유선된 이더넷 네트워크 용으로 디자인 된, 무선 Lan 802.11에서 사용 하기 위해 수정 되었습니다.

### <a name="8021x-capable-wireless-access-points-aps"></a>802.1X\-지원 무선 액세스 지점 \(APs\)
이 시나리오는 하나 이상의 802.1X\ 배포 있어야-원격 인증 Dial\-In 사용자 서비스 \(RADIUS\) 프로토콜와 호환 되는 무선 Ap 가능 합니다.

802.1 X 및 RADIUS\ 준수 Ap RADIUS 인프라 NPS 서버 등 RADIUS 서버에 배포 때 이라고 *RADIUS 클라이언트*합니다.

### <a name="wireless-clients"></a>무선 클라이언트
이 가이드 802.1 X 인증된 액세스 domain\ 회원 사용자에 게 Windows 10, Windows 8.1 및 Windows 8을 실행 하는 무선 클라이언트 컴퓨터 된 네트워크에 연결에 대 한 제공할 포괄적인 구성 세부 정보를 제공 합니다. 컴퓨터는 성공적으로 인증 된 액세스를 설정 하기 위해 도메인에 가입 수 있어야 합니다.

### <a name="support-for-ieee-80211-standards"></a>Ieee 802.11 표준을 지원합니다
지원 되는 Windows와 Windows Server 운영 체제 802.11 무선 네트워크에 대 한 built\에서 지원을 제공합니다. 이러한 운영 체제 설치 된 802.11 무선 네트워크 어댑터가 네트워크 및 공유 센터에서 무선 네트워크 연결이으로 표시 됩니다. 

있지만 built\ 기능에 대 한 지원은 802.11 무선 네트워킹, Windows의 무선 구성 요소는 다음에 따라 달라 집니다.

- 무선 네트워크 어댑터의 기능입니다. 설치 된 무선 네트워크 어댑터에서 무선 LAN 또는 필요한 무선 보안 표준 지원 해야 합니다. 예를 들어, 무선 네트워크 어댑터 Wi\ Fi 보호 액세스 \(WPA\)를 지원 하지 않는 경우 사용 하도록 설정 하거나 WPA 보안 옵션을 구성 수 없는 있습니다.

- 무선 네트워크 어댑터 드라이버의 기능입니다. 무선 네트워크 어댑터에 대 한 드라이버에 무선 네트워크 옵션 구성할 수 있도록 해당 기능을 모두 보고 Windows를 지원 해야 합니다. 무선 네트워크 어댑터에 대 한 드라이버는 운영 체제 기능에 대 한 기록 있는지 확인 합니다. 또한 드라이버 무선 네트워크 어댑터가 공급 업체의 웹 사이트 또는 Microsoft 업데이트를 확인 하 여 최신 버전 인지 확인 합니다.

### <a name="network-policy-server"></a>네트워크 정책 서버
네트워크 정책 서버 \(NPS\) 중앙 구성 하 고 네트워크 정책 RADIUS 프록시 원격 인증 Dial\-In 사용자 서비스 \(RADIUS\) 서버를 사용 하 여 관리할 수 있습니다. 802.1 X 무선 액세스 배포할 때 NPS 필요 합니다.

802.1 X 무선 액세스 포인트에 NPS RADIUS 클라이언트로 구성할 때 NPS Ap 전송한 연결 요청을 처리 됩니다. 연결 요청을 처리 하는 동안 NPS 인증과 수행합니다. 인증은 클라이언트가 올바른 자격 증명 제출한 있는지 여부를 결정 합니다. NPS 요청 하는 클라이언트를 성공적으로 인증 여부는 클라이언트 실시 권 자 요청한 연결 하기에 및 중 하나를 허용 하거나 연결 거부 NPS 결정 합니다. 이 자세히 설명 다음과 같습니다.

#### <a name="authentication"></a>인증

성공 상호 PEAP\ 기능 MS\ 기능을 제공 v2 인증 두 부분으로 구성 사항은 다음과 같습니다.

1.  클라이언트는 NPS 서버를 인증합니다.  이 단계 상호 인증을 NPS 서버 클라이언트 인증서 NPS 서버의 id를 확인할 수 있도록 클라이언트 컴퓨터로 서버 인증서를 보냅니다. NPS 서버 인증을 하려면 클라이언트 컴퓨터 NPS 서버 인증서 캘리포니아 신뢰 해야 합니다. 캐나다의 인증서가 신뢰할 수 있는 인증 기관 인증서 저장소 클라이언트 컴퓨터에 있는 경우이 캘리포니아 신뢰 하는 클라이언트 합니다.

    자신의 개인 캘리포니아 배포 하는 경우 도메인 회원 클라이언트 컴퓨터에서 그룹 정책을 고치면 캘리포니아 인증서 신뢰할 수 있는 인증 기관 인증서 저장소 및 로컬 컴퓨터에 대 한 현재 사용자에 자동으로 설치 됩니다. 공용 캘리포니아에서 서버 인증서 배포할 하려는 경우 공용 캘리포니아 인증서가 신뢰할 수 있는 인증 기관 인증서 스토어에서 이미 확인 합니다.  

2.  NPS 서버에서 사용자를 인증 합니다. 클라이언트 NPS 서버를 성공적으로 인증 후 클라이언트 \(AD DS\) Active Directory 도메인 서비스에서에서 사용자 계정 데이터베이스와 비교 사용자의 자격을 확인 하는 NPS 서버에 사용자의 password\ 기반 자격 증명을 보냅니다.

자격 증명 유효 하 고 인증에 성공 하면, NPS 서버 연결 요청을 처리 하는 인증 단계를 시작 합니다. 자격 증명은 잘못 인증 오류가 발생 하는 경우 NPS 액세스 거부 메시지 보내고 연결 요청이 거부 되었습니다.  

#### <a name="authorization"></a>인증

NPS 실행 하는 서버 인증을 다음과 같이 수행 합니다.  

1. NPS에서 사용자 또는 컴퓨터 계정에서 dial\ 속성의 제한에 대 한 확인합니다. 모든 사용자의 컴퓨터 계정과 Active Directory 사용자 및 컴퓨터 있는 것을 비롯 한 여러 속성 포함 되어 있는 **Dial\에** 탭 합니다. 이 탭에서 **네트워크 액세스 권한을**값이, **액세스 허용**, 사용자 또는 컴퓨터는 실시 권 자에 네트워크에 연결 합니다. 값이 **액세스 거부**, 사용자 또는 컴퓨터 네트워크에 연결할 수 있는 권한이 되지 않습니다. 값이 **NPS 네트워크 정책을 통해 액세스를 제어**, NPS 평가 구성 된 네트워크 정책 사용자 또는 컴퓨터가 네트워크에 연결할 수 있는 권한이 있는지 확인 합니다. 

2. 그런 다음 NPS 연결 요청와 일치 하는 정책을 찾으려면 네트워크 정책을 처리 합니다. 일치 하는 정책 발견 되 면 NPS 부여 하거나 거부 해당 정책 구성에 따라 연결 합니다.  

모두 인증과 성공와 일치 하는 네트워크 정책 액세스 권한을 부여 하는 경우 NPS 네트워크에 대 한 액세스 권한을 부여 하 고 사용자와 컴퓨터 권한이 있는 네트워크 리소스에 연결할 수 있습니다.  

>[!NOTE]
>무선 액세스를 배포 하려면 NPS 정책을 구성 해야 합니다. 이 가이드를 사용 하는 지침을 제공는 **구성 802.1 X 마법사** 802.1 X 인증 무선 액세스 NPS 정책 만드는 NPS에 있습니다.  

### <a name="bootstrap-profiles"></a>부트스트랩 프로필
802.1X\에서-무선 네트워크 인증 클라이언트 무선 네트워크에 연결 하려면 RADIUS 서버 인증 된 보안 자격 증명을 제공 해야 합니다. 보호 된 EAP \[PEAP\]\-Microsoft 인증 프로토콜 버전 2에 대 한 \[MS\-CHAP v2\], 보안 자격 증명은 사용자 이름 및 암호입니다. EAP\ Transport Layer Security \[TLS\]에 대 한 PEAP\ TLS 보안 자격 증명은 인증서 클라이언트 사용자 및 컴퓨터 인증서 등 스마트 카드 또는 합니다.

기본적으로 PEAP\ 기능 MS\ 기능을 제공 v2, PEAP\ TLS 또는 EAP\ TLS 인증을 수행 하도록 구성 된 네트워크에 연결 하는 경우 Windows 무선 클라이언트 해야 RADIUS 서버에서 보내는 컴퓨터 인증서를 확인할 수도 있습니다. 컴퓨터 인증서 모든 인증 세션에 대해 RADIUS 서버에서 전송 되는 일반적으로 서버 인증서 이라고 합니다.

앞에서 설명한 대로 실행할 수 있습니다 RADIUS 서버 서버 인증서 두 가지 방법 중 하나에: 상용 캘리포니아에서 \ (verisign, 같은 \), 네트워크에서 배포 하는 개인 캐나다에서 합니다. 클라이언트의 신뢰할 수 있는 인증 기관 인증서 스토어에서 설치한 루트 인증서가 이미 상용 캘리포니아가 발행 컴퓨터 인증서를 보내면, 사용자 RADIUS 서버, 무선 클라이언트 무선 클라이언트 Active Directory 도메인에 가입 하는 있는지 여부에 상관 없이 RADIUS 서버 컴퓨터 인증서를 확인할 수 있습니다. 이 경우 무선 클라이언트 무선 네트워크에 연결할 수 하 고 컴퓨터를 도메인에 가입 수 있습니다.

>[!NOTE]
>서버 인증서를 확인 하는 클라이언트 필요한 문제를 비활성화할 수 있습니다 하지만 서버의 인증서 유효성 검사를 사용 하지 않도록 설정 생산 환경에서 권장 되지 않습니다.

무선 부트스트랩 프로필은 무선 클라이언트 사용자가 있는 802.1X\에 연결할 수 있도록에 대 한 방식으로 구성 된 임시 프로필-인증 된 무선 네트워크 컴퓨터가 and\ 도메인에 가입 전에 / 또는 사용자가 성공적으로에 로그온 하는 도메인 처음으로 지정된 무선 컴퓨터를 사용 하 여 전에 합니다.  이 섹션은 도메인에 또는 처음으로 domain\ 가입 무선 컴퓨터에서 도메인 로그온를 사용 하 여 사용자에 대 한 무선 컴퓨터를 연결 하려고 할 때 어떤 문제가 발생 하는 요약 되어 있습니다.

사용자 또는 IT 관리자는 컴퓨터에서 도메인에 가입 하는 유선된 이더넷 네트워크에 컴퓨터를 물리적으로 연결할 수 없는 및 컴퓨터에 필요한 발급 없는 배포 루트 인증서에 설치에 대 한 해당 **신뢰할 수 있는 인증 기관** 인증서 저장소 라는 무선 연결 임시 프로필로 무선 클라이언트를 구성할 수는 *프로필 부트스트랩할* 무선 네트워크에 연결할 수 있습니다.

A *프로필 부트스트랩할* 컴퓨터 인증서 RADIUS 서버를 확인 하는 요구 사항을 제거 됩니다. 이 임시 구성 무선 사용자 컴퓨터는 시간에는 무선 네트워크 도메인에 가입 수 있게 \ (IEEE 802.11\) 정책이 적용 되 고 캐나다 적절 한 루트 인증서 컴퓨터에 자동으로 설치 됩니다.

컴퓨터에 적용 상호 인증을 요구 하는 하나 이상의 무선 연결 프로필을 적용 그룹 정책을 적용 될 때 부트스트랩 프로필은 더 이상 필요 하 고 제거 됩니다. 컴퓨터가 도메인에 가입한 후 컴퓨터를 다시 시작 사용자 도메인에 로그온 할 무선 연결을 사용할 수 있습니다.

이러한 기술을 사용 하 여 무선 액세스 배포 프로세스 개요를 참조 하세요. [무선 액세스 배포 개요](b-wireless-access-deploy-overview.md)합니다.
