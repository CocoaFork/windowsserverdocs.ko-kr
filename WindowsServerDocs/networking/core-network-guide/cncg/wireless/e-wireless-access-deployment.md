### <a name="bkmk_wirelessaps"></a>무선 Ap 구성

무선 AP 제조업체에서 제공 되는 제품 설명서 함께 다음 정보를 사용 하 여 무선 Ap 구성할 수 있습니다.

이 절차 열거 무선 AP에서 일반적으로 구성 된 항목 합니다. 항목 이름의 브랜드와 모델에 따라 다를 수 하 고 다음 목록에 다를 수 있습니다. 서비스별 세부 정보에 대 한 무선 AP 설명서를 참조 하십시오.

#### <a name="to-configure-your-wireless-aps"></a>무선 Ap 구성 하려면  

- **SSID**합니다. 무선 network\(s\)의 이름을 지정 \ (예를 들어 ExampleWLAN\). 무선 클라이언트를 알리는 이름입니다.

- **암호화**합니다. 엔터프라이즈 WPA2\ \(preferred\) 또는 WPA\ 엔터프라이즈 AES \(preferred\) 또는 따라 버전은 무선 클라이언트 컴퓨터가 네트워크 어댑터에서 지원 되는 TKIP 암호화 암호 지정 합니다.

- **AP IP 주소 \(static\) 무선**합니다. 각 AP 서브넷 DHCP 범위의 제외 범위 내에 있어야 하는 고유한 고정 IP 주소를 구성 합니다. DHCP에서 할당에서 제외 되는 주소를 사용 하 여 DHCP 서버에서 동일한 IP 주소 컴퓨터나 기타 디바이스에 할당 되지 않도록 합니다.

- **서브넷 마스크**합니다. 무선 AP 연결한 lan 서브넷 마스크 설정에 맞게 구성 합니다.  

- **DNS 이름**합니다. 일부 무선 Ap DNS 이름으로 구성할 수 있습니다. 네트워크에서 DNS 서비스 DNS 이름을 IP 주소를 확인할 수 있습니다. 이 기능을 지 원하는 무선 각 탭에서 DNS 확인을 위해 고유한 이름을 입력 합니다.  

- **DHCP 서비스**합니다. 무선 AP에 DHCP built\에 서비스를 사용 하지 않도록 설정 합니다.  

- **공유 암호 RADIUS**합니다. 앱에서 그룹별로 NPS RADIUS 클라이언트로 구성 계획인 하지 않으면 무선 각 탭에 대 한 고유한 RADIUS 공유 암호를 사용 합니다. NPS 그룹별로 Ap 구성할 하려는 경우 공유 암호에 대 한 그룹의 모든 구성원이 동일 해야 합니다. 또한 각 공유 암호를 사용 하면 이상 22 문자를 대문자로 혼합 소문자, 숫자와 문장 부호의 임의 있어야 합니다. 임의성 위해, NPS에 임의 문자 generator 등 임의 문자 generator을 사용할 수 **구성 802.1 X** 마법사 공유 암호를 만들 수 있습니다.

>[!TIP]
>무선 각 탭에 대 한 공유 암호를 기록 하 고 안전 office 등의 안전한 장소에 저장 합니다. NPS RADIUS 클라이언트를 구성할 때 무선 각 AP에 대해 공유 암호를 알고 있어야 합니다.  

- **RADIUS 서버 IP 주소**합니다. NPS 실행 하는 서버의 IP 주소를 입력 합니다.

- **UDP port\(s\)**합니다. 기본적으로 NPS 인증 메시지와 1813 및 계정 메시지에 대 한 1646 UDP 포트에 1812 및 1645 UDP 포트를 사용합니다. 하면 앱에서 이러한 동일한 UDP 포트를 사용할 수 있지만 다른 포트를 사용 하도록 설정한 경우 확인을 있습니다 새 포트 번호를 사용 하 여 Ap 구성 뿐만 아니라 모든 Ap와 동일한 포트 번호를 사용 하 여 NPS 서버를 다시 구성 것이 좋습니다. Ap 및 NPS 서버 동일한 UDP 포트도 구성 되어 있지 않으면 NPS 받을 하거나, 앱에서 연결 요청을 처리할 수 없습니다 고 무선 네트워크에서 연결이 실패 합니다.

- **Vsa**합니다. 일부 무선 Ap 전체 무선 AP 기능을 제공 하도록 vendor\ 특성 \(VSAs\) 필요 합니다. Vsa는 NPS 네트워크 정책에 추가 됩니다.

- **DHCP 필터링**합니다. 무선 Ap 무선 AP 제조업체에서 설명 된 대로 IP 패킷을 UDP 포트 68 네트워크에 보내는 무선 클라이언트 차단를 구성 합니다.

- **DNS 필터링**합니다. 무선 Ap 무선 AP 제조업체에서 설명 된 대로 IP 패킷을 TCP 또는 UDP 포트 53 네트워크에 보내는 무선 클라이언트 차단를 구성 합니다.

### <a name="bkmk_groups"></a>사용자가 보안 무선 그룹 만들기

이 절차를 사용 하 여 그룹을 만들려면 무선 보안 Active Directory 사용자 및 컴퓨터 Microsoft Management Console \(MMC\) snap\-에 있습니다.  

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

#### <a name="to-create-a-wireless-users-security-group"></a>무선 사용자가 보안 그룹을 만들려면

1. 클릭 **시작**, 클릭 **관리 도구**을 차례로 클릭 하 고 **Active Directory 사용자와 컴퓨터**합니다. Active Directory 사용자 및 컴퓨터 snap\ 열립니다. 아직 선택 하지 않은 경우 도메인에 대 한 노드를 클릭 합니다. 예를 들어 도메인 example.com를 클릭 **example.com**합니다.

2. Right\ 클릭 폴더 세부 정보 창에서 새 그룹에 추가 하려는 \ (예를 들어, right\ 클릭 **사용자가**\), 가리킨 **새로**, 클릭 한 다음 **그룹**합니다.

3. **새 개체 – 그룹**에 **그룹 이름을**, 새 그룹 이름을 입력 합니다. 예를 들어 입력 **무선 그룹**합니다.

4. **그룹 범위**, 다음 옵션 중 하나를 선택 합니다.

    - **도메인 로컬**

    - **전 세계**

    - **유니버설**

5. **그룹 종류**선택 **보안**합니다.

6. 클릭 **확인**합니다.

필요한 경우 둘 이상의 보안 그룹 무선 사용자를 위한, 추가 무선 사용자 그룹을 만들 수이 단계를 반복 합니다. 나중에 다른 조건 및 contstraints 연결 규칙와 다른 액세스 권한을 제공할 각 그룹을 적용할 NPS에서 개별 네트워크 정책 만들 수 있습니다.

### <a name="bkmk_addusers"></a>사용자가 무선 사용자가 보안 그룹에 추가

이 절차를 사용 하 여 사용자를 컴퓨터 또는 그룹 그룹에 추가할 사용자 무선 보안 Active Directory 사용자 및 컴퓨터 Microsoft Management Console \(MMC\) snap\-에 있습니다.

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

#### <a name="to-add-users-to-the-wireless-security-group"></a>사용자가 보안 무선 그룹에 추가 하려면

1. 클릭 **시작**, 클릭 **관리 도구**을 차례로 클릭 하 고 **Active Directory 사용자와 컴퓨터**합니다. MMC Active Directory 사용자 및 컴퓨터 엽니다. 아직 선택 하지 않은 경우 도메인에 대 한 노드를 클릭 합니다. 예를 들어 도메인 example.com를 클릭 **example.com**합니다.

2. 세부 정보 창 double\ 무선 보안 그룹 있는 폴더를 클릭 합니다.

3. 세부 정보 창, right\ 클릭 무선 보안 그룹 하 고 다음 클릭 **속성**합니다. **속성** 보안 그룹에 대 한 대화 상자를 엽니다.

4. 에 **회원** 탭을 클릭 **추가**, 컴퓨터를 추가 하거나 사용자 또는 그룹을 추가 하려면 다음 중 하나를 수행 하 고 있습니다.

##### <a name="to-add-a-user-or-group"></a>사용자 또는 그룹을 추가 하려면

1. **선택할 개체 이름을 입력**, 사용자의 이름을 입력 또는 그룹을 추가, 원하 및 클릭 **확인**합니다.

2. 그룹 구성원에 다른 사용자 또는 그룹을 지정 하려면이 절차의 1 단계를 반복 합니다.  

##### <a name="to-add-a-computer"></a>컴퓨터를 추가 하려면

1. 클릭 **개체 유형**합니다. **개체 유형** 대화 상자를 엽니다.

2. **개체 유형**선택 **컴퓨터**을 차례로 클릭 하 고 **확인**합니다.

3. **선택할 개체 이름을 입력**를 추가 하 고, 클릭 한 다음 하려는 컴퓨터의 이름을 입력 **확인**합니다.

4. 다른 컴퓨터에 할당 그룹 구성원이이 절차의 1\ 3 단계를 반복 합니다.

4. 다음 중 하나를 수행 합니다.

    -   **기존 domain\ 수준 GPO 편집을 위해 열려면**, 그룹 정책 개체를 관리 하려면를 포함 하는 도메인 두 번 클릭, 기본 도메인 정책 등 관리를 클릭 한 다음 원하는 right\ 클릭 도메인 정책 **편집**합니다. **그룹 정책 편집기 관리** 열립니다.

    -   **새 그룹 정책 개체를 만들고 편집 하기 위한 엽니다**, right\ 클릭 도메인에 있는 하려면 새 그룹 정책 개체, 만들고 클릭 한 다음 **GPO이이 도메인에 있는 만들고 여기에 링크**합니다.

        **새 GPO**에 **이름**, 새 그룹 정책 개체 이름을 입력 한 다음 **확인**합니다.

무선 클라이언트 보안 지원할 수 있는 가장 높은 수준의으로 연결 되도록 목록의 맨 위에 가장 제한적인 정책 놓습니다.

이 절차 무선 연결 프로필 도메인 무선 클라이언트 구성원을 무선 네트워크에 연결에 사용 되는 순서를 지정 하는 단계를 제공 합니다.

### <a name="bkmk_radiusclient"></a>무선 AP NPS RADIUS 클라이언트로 구성

>[!IMPORTANT]
>클라이언트 운영 체제를 실행 하는 다른 컴퓨터, 무선 노트북 등 클라이언트 컴퓨터 RADIUS 클라이언트 되지 않습니다. RADIUS 클라이언트는 네트워크 액세스 서버-무선 액세스 포인트, 802.1X\ 등-가능 스위치, 가상 개인 네트워크 \(VPN\) 서버 및 dial\ 접속 서버-등 서버 NPS RADIUS 서버와 통신 하도록 RADIUS 프로토콜을 사용 하기 때문에 있습니다.

회원 **도메인 관리자**, 해당 하는이 절차를 수행 하는 데 필요한 최소 또는 합니다.

#### <a name="to-add-a-network-access-server-as-a-radius-client-in-nps"></a>네트워크 액세스 서버 NPS RADIUS 클라이언트로 추가 하려면

1. NPS 서버의 **서버 관리자**, 클릭 **도구**을 차례로 클릭 하 고 **네트워크 정책 서버**합니다. snap\ NPS 열립니다.

2. Snap\에서 NPS에서 double\ 클릭 **RADIUS 클라이언트 및 서버**합니다. Right\ 클릭 **RADIUS 클라이언트**을 차례로 클릭 하 고 **새로**합니다.

3. **새 RADIUS 클라이언트**, 되어 있는지 확인는 **이 RADIUS 클라이언트를 사용 하도록 설정** 확인란을 선택 합니다.

4. **새 RADIUS 클라이언트**에 **이름**, 무선 액세스 포인트에 대 한 표시 이름을 입력 합니다.

    예를 들어 무선 액세스 추가 하려는 경우 입력 AP\ 01 라는 \(AP\) 가리킨 **AP\ 01**합니다.

5. **주소 \(IP or DNS\)**, IP 주소를 입력 하거나 정식 도메인 NAS \(FQDN\) 이름입니다.

    클릭 이름을 정확 하 고 지도 유효한 IP 주소를 확인 하기 위해 FQDN 입력 하면 **확인**를 선택한 다음 **주소를 확인**에 **주소** 필드를 클릭 **해결**합니다. 해당 NAS의 IP 주소에 자동으로 나타납니다 FQDN 이름 유효한 IP 주소를 지도, 경우 **IP 주소**합니다. FQDN 해결 되지 않는 경우에 IP 주소 이러한 호스트 알려져 나타내는 메시지가 표시 됩니다. 이 문제가 발생 하는 경우 AP 전원이 켜져 있고 네트워크에 연결 되어 있고 올바른 AP 이름이 있는지 확인 합니다.  

    클릭 **확인** 닫으려면 **확인 주소**합니다.  

6. **새 RADIUS 클라이언트**에 **공유 암호**에서 다음 중 하나를 수행 합니다.  

    -   수동으로 구성 RADIUS 공유 암호를 선택 **수동**를 선택한 다음 **공유 암호**, NAS에도 입력 강력한 암호를 입력 합니다. 공유에서 암호를 다시 입력 **공유 암호 확인**합니다.  

    -   자동으로 공유 암호를 생성 하려면는 **생성** 확인란을 선택한 다음 클릭는 **생성** 단추. 공유 생성 된 암호를 저장 한 다음 해당 금액을 사용 하 여 NAS NPS 서버와 통신할 수 있도록 구성 합니다.  

        >[!IMPORTANT]
        >RADIUS 공유 암호 AP에 대 한 사용자 가상의 NPS에서 입력 하 여 실제 무선 AP에서 구성 된 RADIUS 공유 암호 일치 해야 RADIUS 공유 암호를 생성 하는 NPS 옵션을 사용 하는 경우 NPS에서 생성 된 RADIUS 공유 암호와 일치 하는 실제 무선 AP 구성 해야 합니다.

7. **새 RADIUS 클라이언트**에 **고급** 탭에 **공급**, NAS 제조업체의 이름을 지정 합니다. NAS 제조업체의 이름을 모르는 경우 **RADIUS 표준**합니다.

8. **추가 옵션**지원 메시지 인증 특성을 사용 하 여 NAS 선택 EAP와 PEAP 이외의 인증 방법을 사용 하는 경우, **액세스 요청 메시지 Message\ 인증자 특성 있어야**합니다.

9. 클릭 **확인**합니다. 사용자 NAS RADIUS 클라이언트 NPS 서버에서 구성 된 목록에 표시 됩니다.

## <a name="bkmk_domain"></a>무선 새 컴퓨터에서 도메인에 가입
실제로 유선된 LAN의 부분이 컴퓨터를 연결 하는 무선 새 컴퓨터에서 도메인에 가입 하는 가장 쉬운 방법은 \ (받지 않는 802.1 X switch\ 세그먼트)는 컴퓨터에서 도메인에 가입입니다. 이 때문에 쉬운 무선 그룹 정책 설정을 자동으로 및 즉시 적용 하 고 컴퓨터 캘리포니아 인증서를 받은 경우 PKI 배포한를 신뢰할 수 있는 인증 기관 인증서 스토어에 배치 무선 클라이언트 기관에서 발급 한 서버의 인증서 사용 NPS 서버 신뢰할 수 있게 합니다.

마찬가지로, 무선 새 컴퓨터에서 도메인에 가입는 후 도메인에 로그온 할 때 사용자에 대 한 것이 좋습니다 네트워크에는 유선된 연결을 사용 하 여 로그에서 수행할 수 있습니다.

- **IT 직원 컴퓨터 구성**합니다. IT 팀의 구성원 무선 컴퓨터가 도메인에 가입 하 고 단일 로그인을 부트스트랩 무선 프로필을 구성 합니다. 이 방법으로 IT 관리자 무선 컴퓨터는 유선된 이더넷 네트워크에 연결 하 고 컴퓨터가 도메인에 연결 합니다. 그런 다음 관리자 컴퓨터 사용자를 배포합니다. 때 사용자 시작 둘 다에 사용 되는 유선된 연결을 수동으로 로그온 사용자 지정 하는 도메인 자격 증명을 사용 하지 않고 컴퓨터에서 도메인 로그온 및 무선 네트워크에 연결을 설정 합니다.

### <a name="bkmk_itstaff"></a>도메인 및 로그온 IT 직원 컴퓨터 구성 방법을 사용 하 여 참여

3. 또한 사용자가 로컬 컴퓨터 자격에 제공 \ (사용자 이름 및 password\) 및 도메인 자격 증명 \ (도메인 사용자 계정 이름 및 password\) 형태로 *가집니다*, "컴퓨터가 도메인에 가입" 하는 절차 뿐만 아니라 "은 도메인 로그온,"으로 Windows Server 2016에서에서 설명를 [Core 네트워크 가이드](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide)합니다.

#### <a name="to-configure-a-bootstrap-wireless-profile"></a>무선 부트스트랩 프로필 구성 하려면

1. 네트워크 관리자 또는 IT 지원 제공 자격 증명을 사용 하 여 로컬 컴퓨터의 관리자 계정 사용 하 여 컴퓨터에 로그온 할 때 전문가 합니다.

2. 데스크톱 및 클릭에서 네트워크 아이콘 Right\ 클릭 **공개 네트워크 및 공유 센터**합니다. **네트워크 및 공유 센터** 열립니다. **설정을 변경 하 여 네트워킹**, 클릭 **새 연결 또는 네트워크 설정**합니다. **연결을 설정 또는 네트워크** 대화 상자를 엽니다.

3. 클릭 **무선 네트워크에 수동으로 연결**을 차례로 클릭 하 고 **다음**합니다.

4. **무선 네트워크에 수동으로 연결**에 **네트워크 이름을**, AP SSID 이름을 입력 합니다.  

5. **보안 유형**, 관리자가 제공 하는 설정을 선택 합니다.

6. **암호화 종류** 및 **보안 키**, 선택 또는 관리자가 제공 하는 설정을 입력 합니다.

7. 선택 **자동으로이 연결 시작**을 차례로 클릭 하 고 **다음**합니다.

8. **추가 했습니다 * 나만의 네트워크 SSID*, 클릭 **연결 설정을 변경**합니다.

9. 클릭 **연결 설정을 변경**합니다. *나만의 네트워크 SSID* 무선 네트워크 속성 대화 상자를 엽니다.

10. 클릭는 **보안** 탭을 선택한 다음 **네트워크 인증 방법을 선택**선택 **보호 EAP \(PEAP\)**합니다.

11. 클릭 **설정**합니다. **보호 EAP \(PEAP\) 속성** 페이지가 열립니다.

12. 에 **보호 EAP \(PEAP\) 속성** 페이지에서 되도록 **유효성 검사 서버 인증서** 는 클릭 선택 하지 **확인** 두 번 클릭 한 다음 **닫기**합니다.

13. Windows에는 무선 네트워크에 연결 하려고 합니다. Bootstrap 무선 프로필 설정 지정 도메인 자격 증명을 제공 해야 합니다. 계정 이름 및 암호를 묻는 Windows 입력 도메인 계정 자격 증명 다음과 같은: *도메인 Name\\User 이름*, *도메인 암호*합니다.

##### <a name="to-join-a-computer-to-the-domain"></a>컴퓨터에서 도메인에 가입 하

1. 로컬 관리자 계정 사용 하 여 컴퓨터에 로그온 합니다.

2. 검색 텍스트 상자에 입력 **PowerShell**합니다. 검색 결과에서 마우스 오른쪽 단추로 클릭 **Windows PowerShell**을 차례로 클릭 하 고 **관리자 권한으로 실행**합니다. 관리자 권한 프롬프트를 Windows PowerShell 열립니다.

3. Windows PowerShell 다음 명령을 입력 하 고 ENTER 키를 누릅니다. 참여 하려면 도메인 이름으로 가변 도메인 이름을 교체 해야 합니다.
    
    Add-Computer 도메인 이름
    
4. 메시지가 표시 되 면 도메인 사용자 이름 및 암호를 입력 하 고 클릭 **확인**합니다.
5. 컴퓨터를 다시 시작 합니다.
6. 이전 섹션의 지침에 따라 [사용 하 여 Windows 10을 실행 하는 컴퓨터에서 도메인 로그온](#bkmk_w10)합니다.
