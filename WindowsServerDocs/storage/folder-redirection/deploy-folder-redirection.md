# 폴더 리디렉션과 오프 라인 파일을 사용 하 여 배포

>적용 대상: Windows 10, Windows 7, Windows 8, Windows 8.1, Windows Server 2008 R2, Windows Server 2012, Windows Server 2012 R2, Windows Server 2016, Windows Vista

이 항목에서는 Windows Server를 사용 하 여 폴더 리디렉션과 오프 라인 파일을 사용 하 여 Windows 클라이언트 컴퓨터에 배포 하는 방법을 설명 합니다.

이 항목에 최근 변경 사항 목록을 [변경 내용을](#change-history)참조 하세요.

>[!IMPORTANT]
>업데이트 했습니다 [MS16 072](https://support.microsoft.com/en-us/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)에서 보안 변경으로 인해 [3 단계: 폴더 리디렉션 GPO를 만들어](#step-3:-create-a-gpo-for-folder-redirection) 이 항목에서는 Windows 수 있도록 폴더 리디렉션 정책이 올바르게 적용 (되돌리지 영향을 받는 Pc에서 리디렉션된 폴더)의 합니다.

## 필수 구성 요소

### 하드웨어 요구 사항

폴더 리디렉션 x64 또는 x86 기반 컴퓨터; 필요 Windows® 직각에서 지원 하지 않는

### 소프트웨어 요구 사항

폴더 리디렉션 다음과 같은 소프트웨어 요구 사항에 있습니다.

- 폴더 리디렉션을 관리 하려면 도메인 관리자 보안 그룹, 엔터프라이즈 관리자 보안 그룹 또는 그룹 정책을 만든 소유자 보안 그룹의 구성원으로 로그인 해야 합니다.
- 클라이언트 컴퓨터가 Windows 10, Windows 8.1, Windows 8, Windows 7, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 또는 Windows Server 2008을 실행 해야 합니다.
- 클라이언트 컴퓨터를 관리 하는 Active Directory 도메인 서비스 (AD DS)에 가입 되어야 합니다.
- 컴퓨터 그룹 정책 관리 및 Active Directory 관리 센터가 설치를 사용 하 여 사용할 수 있어야 합니다.
- 파일 서버를 리디렉션된 폴더를 호스팅할 수 있어야 합니다.
    - DFS 네임 스페이스를 사용 하는 파일 공유를 DFS 폴더 (링크) 사용자가 서로 다른 서버에 대 한 충돌 편집한 하지 못하도록 단일 대상이 있어야 합니다.
    - DFS 복제를 사용 하 여 다른 서버를 사용 하 여 콘텐츠를 복제 하는 파일 공유, 사용자가 사용자가 서로 다른 서버에 대 한 충돌 편집한 하지 못하도록 원본 서버에만 액세스할 수 있어야 합니다.
    - 클러스터 된 파일 공유를 사용 하는 경우 폴더 리디렉션과 오프 라인 파일을 사용 하 여 성능 문제를 방지 하기 위해 파일 공유에서 지속적인 가용성을 사용 하지 않도록 설정 합니다. 또한 오프 라인 파일 수 사용자가 사용자에 게 오프 라인 파일의 항상 오프 라인 모드를 사용 하지 않는 아직 해질 수 있는 지속적으로 사용할 수 있는 파일 공유에 대 한 액세스를 잃어버린 후 3 ~ 6 분에 대 한 오프 라인 모드로 전환 되지 합니다.

>[!NOTE]
>폴더 리디렉션에서 새로운 기능은 몇 가지 추가 클라이언트 컴퓨터 및 Active Directory 스키마 요구 사항 수 있습니다. 자세한 내용은 [배포 기본 컴퓨터](deploy-primary-computers.md), [사용 하지 않도록 설정에서 오프 라인 파일 폴더](disable-offline-files-on-folders.md), [활성화 항상 오프 라인 모드](enable-always-offline.md)및 [최적화 된 폴더로 이동 사용](enable-optimized-moving.md)을 참조 하세요.

## 1 단계: 폴더 리디렉션 보안 그룹 만들기

환경 설정 하지 않으면 이미을 폴더 리디렉션, 폴더 리디렉션 정책 설정을 적용 하려는 모든 사용자를 포함 하는 보안 그룹을 만드는 첫 번째 단계는 합니다.

폴더 리디렉션에 대 한 보안 그룹을 만드는 방법은 다음과 같습니다.

1. 설치 된 Active Directory 관리 센터를 사용 하 여 컴퓨터에서 서버 관리자를 엽니다.
2. **도구** 메뉴에서 **Active Directory 관리 센터**를 선택 합니다. Active Directory 관리 센터가 표시됩니다.
3. 마우스 오른쪽 단추로 클릭 적절 한 도메인 또는 OU, **새** **그룹**을 선택 합니다.
4. **그룹 만들기** 창의 **그룹** 섹션에서 다음 설정을 지정합니다.
    - **그룹 이름**이름을 입력 보안 그룹의 예: **폴더 리디렉션 사용자**합니다.
    - **그룹 범위** **보안**을 선택한 다음 **전체**를 선택 합니다.
5. **구성원** 섹션에서 **추가**선택 합니다. 사용자, 연락처, 컴퓨터, 서비스 계정 또는 그룹 선택 대화 상자가 나타납니다.
6. 사용자 또는 배포 폴더 리디렉션 **확인**을 선택한 다음 **확인** 을 다시 선택 하려는 그룹의 이름을 입력 합니다.

## 2 단계: 리디렉션된 폴더의 파일 공유 만들기

파일 공유 리디렉션된 폴더를 아직 없는 경우 Windows Server 2012를 실행 하는 서버에서 파일 공유를 만들려면 다음 절차를 사용 합니다.

>[!NOTE]
>일부 기능은 다를 수도 다른 버전의 Windows Server를 실행 하는 서버에 파일 공유를 만드는 경우에 사용할 수 없습니다.

Windows Server 2012 및 Windows Server 2016에서 파일 공유를 만드는 방법은 다음과 같습니다.

1. 서버 관리자 탐색 창에서 **파일 및 저장소 서비스를**선택 하 고 공유 페이지를 표시 하려면 **공유** 선택 합니다.
2. **공유** 타일에서 **작업**을 선택 하 고 **새 공유**를 선택 합니다. 새로운 공유 마법사가 나타납니다.
3. **선택 프로필** 페이지에서 **빠른-SMB 공유**를 선택 합니다. 파일 서버 리소스 관리자를 설치 하 고 폴더 관리 속성을 사용 하는 경우 대신 **고급-SMB 공유**를 선택 합니다.
4. **공유 위치** 페이지에서는 공유 하려는 서버 및 볼륨을 선택 합니다.
5. **공유 이름** 페이지에서 **공유 이름** 상자에 공유 (예를 들어 **사용자가 $**)에 대 한 이름을 입력 합니다.
    >[!TIP]
    >공유를 만들 때를 배치 하 여 공유를 숨기는 ```$``` 공유 이름입니다. 이렇게 하면 적인 브라우저에서 공유를 숨겨집니다.
6. **기타 설정** 페이지에 있는 경우 사용 지속적인 가용성 확인란의 선택을 취소 하 고 필요에 따라 **액세스 기반 열거를 사용** 하 고 **암호화 데이터 액세스** 확인란을 선택 합니다.
7. **사용 권한** 페이지에서 **사용자 지정 권한을...** 를 선택 합니다. 고급 보안 설정 대화 상자가 나타납니다.
8. **상속을 사용 하지 않도록**선택 하 고 **변환에이 개체에 대 한 명시적 권한 사용 권한을 상속**을 선택 합니다.
9. 사용 권한을 그림 1에 나열 되지 않은 그룹 및 계정에 대 한 사용 권한을 제거 하 고 1 단계에서에서 만든 폴더 리디렉션 사용자 그룹에 추가 하는 특별 권한이 설명된 표 1로 설정 합니다.
    
    ![리디렉션된 폴더 공유에 대 한 권한 설정](media/deploy-folder-redirection/setting-the-permissions-for-the-redirected-folders-share.png)
    
    **그림 1** 리디렉션된 폴더 공유에 대 한 권한 설정
10. **SMB 공유-고급** 프로필을 선택한 경우 **관리 속성** 페이지에서 **사용자 파일** 폴더 사용법 값을 선택 합니다.
11. **할당량** 페이지에서 **고급-SMB 공유** 프로필을 선택한 경우 필요에 따라 공유의 사용자에 게 적용 할당량을 선택 합니다.
12. **확인** 페이지에서 선택 **만들기.**

## 3 단계: 폴더 리디렉션 GPO 만들기

폴더 리디렉션 설정에 대해 만든 GPO 아직 없는 경우 하나를 만들려면 다음 절차를 사용 합니다.

폴더 리디렉션 GPO를 만드는 방법은 다음과 같습니다.

1. 설치 그룹 정책 관리를 사용 하 여 컴퓨터에서 서버 관리자를 엽니다.
2. **도구** 메뉴에서 **그룹 정책 관리**를 선택 합니다.
3. 도메인 또는 폴더 리디렉션 설치 하려는 OU를 마우스 오른쪽 단추로 **이 도메인에서 GPO를 만들고 여기에 링크**를 선택 합니다.
4. **새 GPO** 대화 상자에서 (예를 들어, **폴더 리디렉션 설정**), GPO의 이름을 입력 하 고 **확인**을 선택 합니다.
5. 새로 만든된 GPO를 마우스 오른쪽 단추로 클릭 하 고 **링크 활성화** 확인란의 선택을 취소 합니다. 이렇게 하면 GPO를 구성 완료 될 때까지 적용 되지 않습니다.
6. GPO를 선택 합니다. **범위** 탭의 **보안 필터링** 섹션에서 **인증 된 사용자**를 선택 하 고 GPO 모두에 적용 되지 않도록 **제거** 를 선택 합니다.
7. **보안 필터링** 섹션에서 **추가**선택 합니다.
8. **사용자, 컴퓨터 또는 그룹 선택** 대화 상자에서 (예를 들어, **폴더 리디렉션 사용자**), 1 단계에서에서 만든 보안 그룹의 이름을 입력 하 고 **확인**을 선택 합니다.
9. **위임** 탭을 선택 **추가**, **인증 된 사용자**입력, **확인**을 선택한 다음 기본 읽기 권한 적용을 다시 **확인** 선택 합니다.
    
    이 단계는 [MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)보안 변경으로 인해 필요 합니다.

>[!IMPORTANT]
>변경 내용을 [MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016), 이제 사용자 지정 해야 하는 보안으로 인해 인증 된 사용자 그룹 폴더 리디렉션 GPO에 대 한 읽기 권한을 위임-그렇지 않은 경우 사용자에 게 GPO 적용 되지 않습니다 또는 이미 적용 하는 경우 GPO 제거 되, 리디렉션 로컬 PC로 다시 폴더입니다. 자세한 내용은 [배포 그룹 정책 보안 업데이트 MS16-072](https://blogs.technet.microsoft.com/askds/2016/06/22/deploying-group-policy-security-update-ms16-072-kb3163622/)를 참조 하세요.

## 4 단계: 오프 라인 파일을 사용 하 여 폴더 리디렉션 구성

폴더 리디렉션 설정에 대 한 GPO를 만든 후 다음 절차에 설명 된 대로 그룹 정책 설정을 활성화 하 고 구성 폴더 리디렉션을 편집 합니다.

>[!NOTE]
>오프 라인 파일 Windows 클라이언트 컴퓨터에서 리디렉션된 폴더에 대 한 기본적으로 활성화 되며 사용자가 변경 되지 않는 한 Windows Server를 실행 하는 컴퓨터에서 사용할 수 없습니다. 오프 라인 파일 사용 되는지 여부를 제어 하려면 그룹 정책을 사용 하려면 사용 합니다 **허용 또는 비허용 오프 라인 파일 기능을 사용** 정책 설정을 합니다.
> 다른 오프 라인 파일 그룹 정책 설정 중 일부에 대 한 정보를 [사용 하도록 설정 고급 오프 라인 파일 기능](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>)및 [오프 라인 파일에 대 한 그룹 정책 구성](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>)을 참조 하세요.

폴더 리디렉션 그룹 정책에서 구성 하는 방법은 다음과 같습니다.

1. 그룹 정책 관리 (예: **폴더 리디렉션 설정**) 만든 GPO를 마우스 오른쪽 단추로 클릭 한 다음 **편집**을 선택 합니다.
2. 그룹 정책 관리 편집기 창에서 **사용자 구성**, 다음 **정책**, 다음 **Windows 설정**및 다음 **폴더 리디렉션**이동 합니다.
3. 다음 **속성**을 선택 하 고 (예: **문서**) 리디렉션할 폴더를 마우스 오른쪽 단추로 클릭 합니다.
4. **속성** 대화 상자에서 **설정** 상자에서 **기본-동일한 위치에 모든 사용자의 폴더 리디렉션**를 선택 합니다.

    > [!NOTE]
    > Windows XP 또는 Windows Server 2003을 실행 하는 클라이언트 컴퓨터에 폴더 리디렉션 적용 **설정** 탭을 선택 하 고 선택 하 고 **에도 적용 리디렉션 정책을 Windows 2000, Windows 2000 Server, Windows XP 및 Windows Server 2003 운영 시스템** 확인란입니다.
5. **대상 폴더 위치** 섹션에서 **루트 경로 아래의 각 사용자에 대 한 폴더 만들기** 를 선택 하 고 다음 **루트 경로** 상자 예를 들어 리디렉션된 폴더를 저장할 파일 공유에 대 한 경로 입력: **\\\fs1.corp.contoso.com\\ 사용자가 $**
6. **설정** 탭을 선택 하 고 **정책 제거** 섹션에서 필요에 따라 선택 **정책이 제거 되 면 로컬 사용자 프로필 위치에 다시 폴더 리디렉션** (이 설정을 수 있도록 폴더 리디렉션 자세히 동작 규모에 대 한 adminisitrators 및 사용자).
7. **확인**선택 하 고 경고 대화 상자에서 **예** 선택 합니다.

## 5 단계: 폴더 리디렉션 GPO를 사용 하도록 설정

폴더 리디렉션 그룹 정책 설정을 구성 완료 되 면 다음 단계는 영향을 받는 사용자에 게 적용 되도록 허용 하는 GPO를 사용 하는 것입니다.

>[!TIP]
>주 컴퓨터 지원 또는 다른 정책 설정을 구현 하려는 경우 이렇게 이제 GPO를 사용 하기 전에 합니다. 이렇게 하면 사용자 데이터를 기본 컴퓨터 지원 사용 하도록 설정 하기 전에 기본이 아닌 컴퓨터에 복사 되지 않습니다.

폴더 리디렉션 GPO를 사용 하도록 설정 하는 방법은 다음과 같습니다.

1. Open 그룹 정책 관리 합니다.
2. 사용자가 만든 GPO를 마우스 오른쪽 단추로 클릭 하 고 **링크 활성화**를 선택 합니다. 메뉴 항목 옆에 있는 확인란이 표시 됩니다.

## 6 단계: 테스트 폴더 리디렉션

폴더 리디렉션을 테스트 하려면 컴퓨터에 사용 하 여 로그인 폴더 리디렉션에 대해 구성 된 사용자 계정. 그런 다음 폴더와 프로필 리디렉션 확인 합니다.

폴더 리디렉션 테스트 하는 방법은 다음과 같습니다.

1. 로그인 주 컴퓨터 (주 컴퓨터 지원 사용) 하는 경우 설정한 폴더 리디렉션 사용자 계정과 합니다.
2. 사용자는 이전 컴퓨터에 로그인을 관리자 권한 명령 프롬프트를 열고 클라이언트 컴퓨터에 최신 그룹 정책 설정을 적용 되는지 확인 하려면 다음 명령을 입력.
    
    ```PowerShell
    gpupdate /force
    ```
3. 파일 탐색기를 엽니다.
4. 리디렉션된 폴더 (예를 들어 내 문서 폴더의 문서 라이브러리)를 마우스 다음 **속성**을 선택 합니다.
5. **위치** 탭을 선택 하 고 로컬 경로 대신 지정한 파일 공유 경로 표시 되는지 확인 합니다.
