# <a name="executive-summary"></a>요약

>적용 대상: Windows Server 2012

>[!IMPORTANT] 
>다음 설명서 2013에서 쓰인 것 이며 기록의 목적 으로만 제공 됩니다.  현재이 설명서를 검토 하는 것 이므로 변경 될 수 있습니다.  현재 모범 사례를 반영 하지 않을 수 있습니다.

정보 기술 (IT) 인프라를 사용 하 여 조직이 없습니다 공격에 영향을 받지는 있지만 적절 한 정책, 프로세스 및 컨트롤, 조직의 컴퓨팅 인프라의 주요 부문을 보호 하기 위해 구현 하는 경우 수 컴퓨팅 환경의 도매 손상으로 증가 하는 보안 위험 이벤트를 방지 합니다.  
  
이 요약 조직이 Active Directory 설치의 보안을 향상 시키는 데 도움이 되는 권장 사항을 포함 하는 문서의 내용을 요약 하는 독립 실행형 문서로 유용할 것입니다. 이러한 권장 사항은 구현 하 여 조직 파악 하 고 보안 작업 우선 순위 지정, 조직의 컴퓨팅 인프라의 주요 부문을 보호 가능성을 크게 줄일 수 있는 컨트롤을 만들 수 있게 됩니다. IT 환경의 중요 한 구성 요소에 대 한 공격 성공 합니다.  
  
이 문서에서는 가장 일반적인 공격 공격 표면을 줄이기 위해 Active Directory 및 대책 설명, 하지만 완전히 손상 발생 시 복구를 위한 권장 사항을 포함 합니다. 확실 한 방법은 Active Directory의 전체 손상이 발생 한 경우 복구할 발생 하기 전에 손상에 대 한 준비가 됩니다.  
  
이 문서의 주요 섹션이 있습니다.  
  
-   손상될 작업 환경  
  
-   Active Directory 공격에 대한 취약성 줄이기  
  
-   손상 징후에 대한 Active Directory 모니터링  
  
-   손상 계획  
  
## <a name="avenues-to-compromise"></a>손상될 작업 환경  
이 섹션에서는 일부 고객의 인프라를 손상 시키는 공격자에 의해 사용 되는 가장 일반적으로 사용된 된 취약점에 대 한 정보를 제공 합니다. 처음에 고객의 인프라를 침투, 추가 시스템 손상 전파 및 결국 전체 얻으려면 Active Directory 및 도메인 컨트롤러를 대상으로 하는 데 사용 하는 방법 및 취약점으로 인 한 일반 범주 포함 조직의 포리스트 컨트롤입니다. 각 유형의는 취약성 Active Directory를 직접 대상에 사용 되지 않는 영역에서 특히 취약점을 해결 하는 방법에 대 한 자세한 권장 사항을 제공 하지 않습니다. 그러나 각 유형의 취약점으로 인 한 조직의 공격 노출 영역을 줄이고 대책을 개발 하는 데 추가 정보의 링크를 제공 했습니다.  
  
다음 주제는 포함 되어 있습니다.  
  
-   **위반 대상 초기** -대부분의 정보 보안 침해 한 번에 소량의 조직의 인프라 종종 하나 또는 두 개의 시스템 손상 시작 합니다. 이러한 초기 이벤트 또는 네트워크 진입점 수 해결 되었고 하지만 되지 않은 취약성을 악용 경우가 많습니다. 일반적으로 나타나는 취약점은 됩니다.  
  
    -   바이러스 백신 및 맬웨어 방지 프로그램 배포의 간격  
  
    -   불완전 한 패치  
  
    -   오래 된 응용 프로그램 및 운영 체제  
  
    -   잘못 된 구성  
  
    -   보안 응용 프로그램 개발 사례 부족  
  
-   **자격 증명 도난에 대 한 매력적인 계정** -자격 증명 도난 공격은 공격자가 처음에 네트워크의 컴퓨터에 대 한 권한 있는 액세스를 향상 및 다음 무료로 사용할 수 있는 도구를 사용 하 여 다른 세션에서 자격 증명을 추출 로그온 계정입니다.   
    이 섹션에 포함 된 다음과 같습니다.  
  
    -   **손상의 가능성을 높이 작업** -자격 증명 도난 대상이 일반적으로 높은 권한이 가진된 도메인 계정 및 "매우 중요 한 person"의 관리자에 대 한 중요 한 것 (VIP) 계정 자격 증명 도난 공격의 성공 가능성을 높이 작업을 선택 합니다. 이러한 활동은:  
  
        -   권한 있는 계정으로 보안이 설정 되지 않은 컴퓨터에 로그온  
  
        -   매우 강력한 권한의 계정 사용 하 여 인터넷을 검색  
  
        -   시스템에 걸쳐 동일한 자격 증명을 사용 하 여 로컬 특권된 된 계정 구성  
  
        -   권한 있는 도메인 그룹의 과부하 및 overpopulation  
  
        -   보안 도메인 컨트롤러의 부족 하 여 관리 합니다.  
  
    -   **권한 상승 및 전파** -특정 계정, 서버 및 인프라 구성 요소는 일반적으로 기본 Active Directory에 대 한 공격 대상입니다. 이러한 계정은 다음과 같습니다.  
  
        -   영구 권한 있는 계정  
  
        -   VIP 계정  
  
        -   Active Directory 계정을 "권한 연결"  
  
        -   도메인 컨트롤러  
  
        -   PKI (공개 키 인프라) 서버 및 시스템 관리 서버와 같은 id, 액세스 및 구성 관리에 영향을 주는 다른 인프라 서비스  
  
## <a name="reducing-the-active-directory-attack-surface"></a>Active Directory 공격에 대한 취약성 줄이기  
이 섹션에서는 Active Directory 설치의 공격 표면을 줄이기 위해 기술적 컨트롤에 중점을 둡니다. 다음 주제는이 섹션에 포함 합니다.  
  
-   합니다 **권한 있는 계정 및 Active Directory에서 그룹** 섹션에서는 가장 높은 권한 있는 계정 및 권한 있는 계정을 보호 되는 메커니즘과 Active Directory에서 그룹에 설명 합니다. Active Directory 내에서 세 가지 기본 제공 그룹 이며 가장 높은 권한 그룹 (Enterprise Admins, Domain Admins 및 관리자) 디렉터리에도 다양 한 추가 그룹 및 계정 보호 되어야 하지만  
  
-   합니다 **최소 권한 관리 모델 구현** 권장 사항을 제공 하는 것 외에도 매우 강력한 권한의 계정 사용 일상적인 관리를 제공 하는 위험 식별에 중점을 둡니다 섹션 위험을 줄입니다.  
  
과도 한 권한 손상 된 환경에서 Active Directory에서 발견 되지 않습니다. 조직에서 필요한 것 보다 더 많은 권한을 부여 하는 습관을 개발 하는 경우 일반적으로 전체 인프라에서 발견 되:  
  
-   Active Directory에서  
  
-   구성원 서버에서  
  
-   워크스테이션에서  
  
-   응용 프로그램에서  
  
-   데이터 저장소에서  
  
-   합니다 **보안 관리 호스트 구현** Active Directory와 연결 된 시스템 관리를 지원 하도록 구성 된 컴퓨터는 보안 관리 호스트에 설명 합니다. 이러한 호스트를 전담 하 관리 기능 및 전자 메일 응용 프로그램, 웹 브라우저 또는 생산성 소프트웨어와 같은 소프트웨어 (예: Microsoft Office) 실행 하지 마십시오.  
  
이 섹션에 포함 된 다음과 같습니다.  
  
-   **보안 관리 호스트 만들기에 대 한 원칙** -염두에 대 한 일반적인 원칙 됩니다.  
  
    -   신뢰할 수 없는 호스트에서 신뢰할 수 있는 시스템을 관리 하지 않습니다.  
  
    -   권한 있는 작업을 수행 하는 경우 단일 인증 단계에 의존 하지 않습니다.  
  
    -   물리적 보안 설계 및 보안 관리 호스트를 구현 하는 경우 잊지 마세요.  
  
-   **공격에 대 한 도메인 컨트롤러 보안** -악의적인 사용자가 도메인 컨트롤러에 대 한 권한 있는 액세스를 얻고 하는 경우 해당 사용자 수, 손상, 수정한 Active Directory 데이터베이스를 삭제 하 고 있는 계정에 확인 하 고 시스템의 모든 확장에 의해 Active Directory에서 관리 합니다.  
  
다음 주제는이 섹션에 포함 합니다.  
  
-   **도메인 컨트롤러에 대 한 물리적 보안** -데이터 센터, 지점, 원격 위치에서 도메인 컨트롤러에 대 한 물리적 보안을 제공 하기 위한 권장 사항을 포함 합니다.  
  
-   **도메인 컨트롤러 운영 체제** -도메인 컨트롤러 운영 체제를 보호 하기 위한 권장 사항을 포함 합니다.  
  
-   **보안 구성의 도메인 컨트롤러** -네이티브 및 무료로 사용할 수 있는 구성 도구 및 설정 보안 그룹 정책 개체 (이후에 적용할 수 있는 도메인 컨트롤러에 대 한 구성 기준을 만들기 위해 사용할 수 있습니다 Gpo)입니다.  
  
## <a name="monitoring-active-directory-for-signs-of-compromise"></a>손상 징후에 대한 Active Directory 모니터링  
이 섹션에서는 레거시 감사 범주 및 감사 정책 하위 범주 (하는 Windows Vista 및 Windows Server 2008에 도입 된), 고급 감사 정책 (Windows Server 2008 R2에 도입 된)에 대 한 정보를 제공 합니다. 제공 이벤트에 대 한 정보 및 모니터링 하는 개체를 해당 환경 및 Active directory는 포괄적인 감사 정책을 만드는 데 사용할 수 있는 몇 가지 추가 참조를 손상 시키려는 시도 나타낼 수 있습니다.  
  
다음 주제는이 섹션에 포함 합니다.  
  
-   **Windows 감사 정책** -Windows 보안 이벤트 로그에는 범주 및 보안 이벤트를 결정 하는 하위 범주를 추적 하 여 기록 합니다.  
  
-   **감사 정책 권장 사항** -이 섹션에서는 Windows 기본 감사 정책 설정을 설명, Microsoft 및 조직에 대 한 더 적극적으로 권장 중요 한 서버 감사를 사용 하 여 권장 되는 정책 설정을 감사 하 고 워크스테이션입니다.  
  
## <a name="planning-for-compromise"></a>손상 계획  
이 섹션에서는 조직 손상이 발생 하기 전에 준비 하는 데 도움이 되는 권장 사항, 전체 보안 위험을 발생 하기 전에 손상 이벤트를 검색 하는 사례에 대 한 대응 및 복구 지침을 제공 하는 제어 구현 디렉터리의 전체 손상이 공격자에 의해 이루어집니다. 다음 주제는이 섹션에 포함 합니다.  
  
-   **접근 방식은 재고** -원칙 및 조직 들이 가장 중요 한 자산을 넣을 수 있는 보안 환경을 만드는 데 대 한 지침을 포함 합니다. 이러한 지침은 아래와 같습니다.  
  
    -   분리 및 중요 한 자산 보호에 대 한 원칙을 식별 합니다.  
  
    -   제한, 위험 기반 마이그레이션 계획을 정의합니다.  
  
    -   필요한 경우 "nonmigratory" 마이그레이션을 활용  
  
    -   "Creative 소멸"를 구현합니다.  
  
    -   레거시 시스템 및 응용 프로그램 격리  
  
    -   최종 사용자에 대 한 보안 간소화  
  
-   **보다 보안 환경을 유지 관리** -고급 권장 사항을 지침으로 효과적인 보안 뿐만 아니라 유효 수명 주기 관리를 개발 하는 데 사용할 수를 포함 합니다. 다음 주제는이 섹션에 포함 합니다.  
  
    -   **Active Directory에 대 한 비즈니스 중심 보안 사례를 만드는** -을 효과적으로 사용자, 데이터, 응용 프로그램 및 Active Directory에서 관리 하는 시스템의 수명 주기를 관리 합니다. 이러한 원칙을 따릅니다.  
  
        -   **Active Directory 데이터에는 비즈니스 소유권을 할당할** -인프라 구성 요소는 소유권을 할당할 IT, Active Directory 도메인 서비스 (AD DS) 예를 들어 비즈니스를 지원 하기 위해 새 직원을 새 응용 프로그램에 추가 되는 데이터 및 새 정보 리포지토리, 지정 된 비즈니스 단위 또는 사용자 데이터를 사용 하 여 연결 해야 합니다.  
  
        -   **Business-Driven 수명 주기 관리를 구현** -Active Directory의 데이터에 대 한 수명 주기 관리를 구현 해야 합니다.  
  
        -   **모든 Active Directory 데이터를 분류** -비즈니스 소유자는 Active Directory의 데이터에 대 한 분류를 제공 해야 합니다. 다음 Active Directory 데이터에 대 한 분류 내 데이터 분류 모델을 포함 해야 합니다.  
  
            -   **시스템** -분류 server 채우기, 해당 운영 체제의 역할 및 IT에서 실행 중인 응용 프로그램 및 레코드의 비즈니스 소유자입니다.  
  
            -   **응용 프로그램** -기능, 사용자 기반 및 해당 운영 체제에서 응용 프로그램을 분류 합니다.  
  
            -   **사용자가** -공격자에 의해 대상으로 할 가능성이 Active Directory 설치에 계정 태그가 지정 하 고 모니터링 해야 합니다.  
  
## <a name="summary-of-best-practices-for-securing-active-directory-domain-services"></a>Active Directory Domain Services 보안에 대 한 모범 사례 요약  
다음 표에서 AD DS 설치의 보안에 대 한이 문서에 제공 된 권장 사항의 요약을 제공 합니다. 몇 가지 모범 사례 사항은 전략적인 성격 및 포괄적인 계획 및 구현 프로젝트; 필요 다른 사항은 전술적 이어서 Active Directory와 관련 된 인프라의 특정 구성 요소에 집중 합니다.  
  
사례는 우선 순위, 즉의 대략적인 순서로 나열 되어, 숫자가 낮을수록 더 높은 우선 순위를 표시 합니다. 여기서 해당 모범 사례는 예방 하거나 기본적으로 검색으로 식별 됩니다. 이러한 권장 사항의 모든 철저히 테스트 있고 조직의 특성 및 요구 사항에 대 한 필요에 따라 수정 합니다.  
  
  
||**모범 사례**|**전술적 또는 전략적**|**예방 또는 검색**|  
|-|-|-|-|  
|1|패치 응용 프로그램입니다.|전술적|예방|  
|2|운영 체제를 패치 합니다.|전술적|예방|  
|3|배포 하 고 모든 시스템 및 시도 대 한 모니터를 제거 하거나 사용 하지 않도록 하려면 바이러스 백신 및 맬웨어 방지 소프트웨어를 신속 하 게 업데이트 합니다.|전술적|Both|  
|4|수정 시도 대 한 중요 한 Active Directory 개체 및 시도 손상을 나타낼 수 있는 이벤트에 대 한 Windows 모니터링 합니다.|전술적|검색|  
|5|보호 및 중요 한 데이터에 대 한 액세스 권한이 있는 사용자에 대 한 계정 모니터링|전술적|Both|  
|6|강력한 계정 인증 되지 않은 시스템에서 사용 되지 않도록 방지 합니다.|전술적|예방|  
|7|높은 권한 있는 그룹에서 영구 멤버 자격을 제거 합니다.|전술적|예방|  
|8|필요할 때 권한 있는 그룹에서 임시 멤버 자격을 부여 하기 위한 제어를 구현 합니다.|전술적|예방|  
|9|보안 관리 호스트를 구현 합니다.|전술적|예방|  
|10|도메인 컨트롤러, 관리 호스트 및 기타 중요 한 시스템에서 응용 프로그램 허용 목록을 사용 합니다.|전술적|예방|  
|11|중요 한 자산을 파악 하 고 보안을 우선 순위를 모니터링 합니다.|전술적|Both|  
|12|디렉터리, 지원 인프라 및 도메인에 가입 된 시스템의 관리에 대 한 최소 권한, 역할 기반 액세스 제어를 구현 합니다.|전략적|예방|  
|13|레거시 시스템 및 응용 프로그램을 격리 합니다.|전술적|예방|  
|14|레거시 시스템 및 응용 프로그램을 역할 해제 합니다.|전략적|예방|  
|15|사용자 지정 응용 프로그램에 대 한 보안 개발 수명 주기 프로그램을 구현 합니다.|전략적|예방|  
|16|구성 관리를 구현 하 고 규정 준수를 정기적으로 검토 각각의 새로운 하드웨어 또는 소프트웨어 버전을 사용 하 여 설정을 평가 합니다.|전략적|예방|  
|17|엄격한 보안 및 요구 사항 모니터링을 사용 하 여 인위적인 포리스트에 중요 한 자산을 마이그레이션하십시오.|전략적|Both|  
|18|최종 사용자에 대 한 보안을 간소화 합니다.|전략적|예방|  
|19|호스트 기반 방화벽 제어 및 보안 통신을 사용 합니다.|전술적|예방|  
|20|장치를 패치 합니다.|전술적|예방|  
|21|IT 자산에 대 한 비즈니스 중심 수명 주기 관리를 구현 합니다.|전략적|해당 사항 없음|  
|22|만들거나 인시던트 복구 계획을 업데이트 합니다.|전략적|해당 사항 없음|
