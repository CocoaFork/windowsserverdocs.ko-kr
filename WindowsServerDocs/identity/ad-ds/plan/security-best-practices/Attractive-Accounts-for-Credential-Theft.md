# <a name="attractive-accounts-for-credential-theft"></a>매력적인 계정 자격 증명 도용의

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

공격자 최고 권한 (루트, 관리자가 또는 사용 중인 운영 체제에 따라 시스템) 처음 획득 도난 공격 자격 증명은에 네트워크와 무료로 사용할 다른 로그인 한 계정 세션에서 자격 증명을 추출 하 설비 사용 하 여 컴퓨터에 액세스할 수 있습니다. 시스템 구성에 따라 이러한 자격 증명 해시, 티켓 또는 일반 암호 형태의 추출 될 수 있습니다. 공격자 특정 유형의 두 계정 중 자격 증명을 확보 하 고 추가 컴퓨터로 손상 전파 네트워크의 다른 컴퓨터에 자격 증명을 제공 추수 한 자격 중 하나 (예: 창에서 관리자 계정을 또는 OSX, UNIX 또는 Linux의 루트 계정) 네트워크의 다른 컴퓨터에 있는 것으로 예상 되는 로컬 계정에 대 한 인 경우 :  

1.  (즉, Active Directory와 많은 컴퓨터에서 관리자 수준 권한이 있는 계정) 광범위 한 및 전체 권한이 있는 권한이 부여 된 도메인 계정을 합니다. 이러한 계정을 Active Directory에 있는 가장 높은 권한 그룹의 모든 구성원이 되지 않지만 될 수 권한이 부여 된 관리자 수준 권한이 도메인 또는 숲 효과적으로 권한이 있는 그룹의 회원 강력한 Active Directory에 더 많은 워크스테이션과 서버에서 합니다. 대부분의 경우 계정에서 광범위 한 Windows 인프라 양의 높은 수준의 권한 부여 된 되므로 서비스 계정, 계정 서비스 폭 및 권한 깊이 항상 평가 해야 합니다.  

2.  "아주 중요 한 사람이" (VIP) 도메인 계정을 합니다. 이 문서에서 VIP 계정 공격자 (지적 및 기타 중요 한 정보)가 정보에 액세스할 수 있는 모든 계정을 또는 계정 정보에 공격자 액세스 권한을 부여할 사용할 수 있는 합니다. 이러한 사용자 계정이 예입니다.  

    1.  해당 계정을 회사 중요 한 정보에 액세스할 수 있는 임원  

    2.  컴퓨터와 임원에서 사용 하는 응용 프로그램 관리 담당 하는 기술 지원 서비스 담당자는 계정  

    3.  계정 여부는 조직 입찰 계약 문서에 액세스할 수 있는 직원 법률에 대 한 문서는 자신의 조직 또는 클라이언트 조직에 대 한  

    4.  회사는 제품의 종류에 관계 없이 회사의 개발 중인 제품에 대 한 계획 및 사양에 액세스할 수 있는 제품 플래너 파이프라인  

    5.  계정이 연구 데이터, 제품 수행할 또는 기타 연구 공격자 관심 액세스 하는 데는 연구  

유용한 계정 자격 증명 도용의 공격에 대 한 Active Directory에 매우 권한이 있는 계정 손상 전파 하 고 조작 VIP 계정이 나 자녀가 액세스할 수 있는 데이터를 사용할 수, 되므로 Active Directory에 엔터프라이즈 관리, 도메인 관리자 및 관리자가 그룹의 회원 계정.  

도메인 컨트롤러 AD DS 데이터베이스에 대 한 저장소는 도메인 컨트롤러의 모든 데이터에 대 한 전체 액세스 Active Directory에 있는 때문 도메인 컨트롤러 손상 된 계정을 나란히 자격 증명 도용의 공격 또는 권한이 높은 Active Directory 후 여부도 손상 대상으로 합니다. 여러 (출판과 많은 공격자) 중점적 도메인 관리자 그룹 구성원 도난 공격 해시를 pass 및 기타 자격 증명을 설명 하는 경우 (에 설명 된 대로 [Active Directory 공격 줄이기](../../../ad-ds/plan/security-best-practices/Reducing-the-Active-Directory-Attack-Surface.md)), 여기에 나열 된 그룹의 모든 구성원이 계정이 손상 전체 광고 DS 설치 데 사용할 수 있습니다.

## <a name="activities-that-increase-the-likelihood-of-compromise"></a>손상 될 확률이 높아집니다 활동  
자격 증명 도용의 대상 권한이 높은 도메인 계정 및 VIP 계정을 일반적으로 되므로 관리자 자격 증명 도용의 공격 성공 가능성 증가 하는 활동을 인식 하는 것이 중요 합니다. 공격자 대상 VIP 계정을 Vip 높은 수준의 시스템 켜거나 도메인에 있는 권한 부여 하지는 경우, 하지만 도난 자격 증명 다른 유형의 맬웨어로부터 VIP 비밀 정보를 제공 하기 위해 엔지니어링 등의 공격을 필요 합니다. 또는 공격자는 VIP 자격 증명은 캐시 된 시스템에 대 한 액세스 권한이 있는 먼저 받아야 합니다. 이 때문 여기에 설명 된 자격 증명 도용의 가능성이 증가 하는 작업은 주로 권한이 높은 관리자 자격 증명 인수 방지에 집중 하면서. 이 활동은 일반적인 메커니즘 공격자는 권한 자격 증명을 가져올 시스템을 손상 시킬 수 있습니다.  

### <a name="logging-on-to-unsecured-computers-with-privileged-accounts"></a>권한이 계정 사용 하는 비보안된 컴퓨터에 로그인  
성공 하려면 자격 증명 도용의 공격 수 있도록 핵심 취약점 광범위 하 게 되는 계정을 안전 하 고 환경을 통해 깊이 권한 없는 컴퓨터에 로그온 하는 행위입니다. 이러한 로그온의 다양 한 구성 오류 여기에 설명 된 것일 수 있습니다.  

#### <a name="not-maintaining-separate-administrative-credentials"></a>관리자 자격 증명을 별도로 유지 하지 않으면  
이 다양 한 광고 DS 설치 평가 일반적인 상대적으로, IT 직원 단일 계정을 사용 하 여 모든 작업에 대 한 발견 했습니다. 계정을 하나 이상의 Active Directory에 가장 많이 권한이 있는 그룹의 회원 이며 동일한 계정에 로그온 하기 위해 자신의 워크스테이션 일 아침에 직원 사용 메일을 확인, 인터넷 사이트를 검색 및가 컴퓨터에 콘텐츠를 다운로드 합니다. 사용자가 로컬 관리자 권한이 부여 된 계정을 실행 손상 완료 로컬 컴퓨터를 노출 합니다. 해당 계정 Active Directory에 가장 권한이 있는 그룹의 회원도 경우 노출 손상, 전체 숲 쉽게 손쉽게 Active Directory 및 Windows 환경을 완전히 제어할 수 있습니다.  

마찬가지로 일부 환경에서는 동일한 사용자 이름과 암호는 계정에 사용 루트 비 Windows 컴퓨터에는 Windows 시스템 그 반대의 UNIX 또는 Linux 시스템에서 손상을 확장 하 공격자 수 있는 Windows 환경에서 사용 되 고 발견 했습니다.

#### <a name="logons-to-compromised-workstations-or-member-servers-with-privileged-accounts"></a>손상 된 워크스테이션 또는 구성원 서버 권한이 있는 계정에 로그온 할  
권한이 높은 도메인 계정이 손상된 워크스테이션 또는 멤버 서버에 대화형 로그온 할 때 사용 되 면 해당 손상 된 컴퓨터 시스템에 로그온 하는 계정에서 자격 증명을 수집 될 수 있습니다.  

#### <a name="unsecured-administrative-workstations"></a>안전 하지 않은 관리 워크스테이션  
많은 조직에서 IT 부서 여러 계정을 사용합니다. 계정을 하나 직원 워크스테이션에 로그온 할에 사용 되 고 워크스테이션에 로컬 관리자 권한이 이러한 옵션은 IT 부서에 때문에 종종 없는 합니다. UAC는 경우에 따라 사용자 이상 분할 액세스를 받을 수 있도록 설정에 로그온 할 때 토큰 왼쪽과 권한이 필요할 때 상승 해야 합니다. 이러한 사용자에이 게 유지 관리 작업을 수행 하는 경우 일반적으로 로컬에 설치 된 관리 도구를 사용 하며 자격 증명을 선택 하 여 해당 권한이 도메인 계정에 대해 제공는 **관리자 권한으로 실행** 옵션 나 메시지가 표시 되 면 자격 증명을 제공 합니다. 이 구성 적절 한 것 처럼 보일 수, 있지만을 하기 때문에 손상 환경을 제공 합니다.  

-   "일반" 사용자 계정이 직원 워크스테이션에 로그온 할 때 사용 하는 로컬 관리자 권한이, 컴퓨터가 취약 하 게 [드라이브에서 다운로드](https://www.microsoft.com/security/sir/glossary/drive-by-download-sites.aspx) 공격에는 사용자가 유효 판단 했기 가능 맬웨어를 설치 합니다.  

-   맬웨어 관리자 계정에 설치 된, 컴퓨터는 캡처할, 클립보드, 스크린샷, 입력과 메모리 상주 자격 증명을 하는 중 하나 강력한 도메인 계정 자격 증명에 노출 될 수 있습니다 이제 사용할 수 있습니다.  

이 경우 문제는 다음 두 가지입니다. 첫째, 로컬 및 도메인 관리 하기 위해 별도 계정을 사용 하지만 컴퓨터 안전 하지 않은 도난 으로부터 계정을 보호 하지 않습니다. 둘째, 일반 사용자 계정 및 관리자 계정 부여 된 과도 하 게 권한 및 사용 합니다.  

### <a name="browsing-the-internet-with-a-highly-privileged-account"></a>매우 권한 계정 사용 하 여 인터넷을 검색  
사용자에 게 인터넷 (또는 손상된 인트라넷)을 검색 한 다음 컴퓨터에서 로컬 관리자가 그룹의 회원 또는 Active Directory와 누가 권한이 있는 그룹의 회원 계정으로 컴퓨터에 로그온 할 로컬 컴퓨터와 손상 디렉터리 노출 될 수 있습니다.  

관리자 권한으로 실행 되는 브라우저와 악의적으로 만들어진된 웹 사이트에 액세스할 수 있는 공격자 권한이 있는 사용자의 컨텍스트에서 로컬 컴퓨터에서 악성 코드 보관할 수 있습니다. 사용자에 게 컴퓨터에서 관리자 권한이 로컬, 공격자 수 속이기 사용자 악성 코드를 다운로드 하거나 캐시 된 컴퓨터에서 모든 활성 사용자를 위한 자격 증명 응용 프로그램의 취약성을 활용 하 고 로컬 추출 하 여 사용자의 권한을 활용 하는 메일 첨부 파일을 열면 합니다. 사용자가 관리자 권한 구성원 엔터프라이즈 관리자, 도메인 관리자 또는 관리자가 그룹 Active directory에서에 의해 디렉터리의 경우 공격자 도메인 자격 증명을 추출 하 고 손상 숲에서 다른 컴퓨터를 않고도 전체 광고 DS 도메인 또는 숲, 손상 하는 데 사용 수 있습니다.  

### <a name="configuring-local-privileged-accounts-with-the-same-credentials-across-systems"></a>시스템 전반에서 같은 자격 증명으로 구성 로컬 권한이 있는 계정  
많은 또는 모든 컴퓨터 사용 하면 자격 증명에 도난당 삼로 데이터베이스 같은 자격 증명을 사용 하는 다른 모든 컴퓨터 손상 시킬 수 위해 한 컴퓨터에서에서 동일한 로컬 관리자 계정 이름 및 암호를 구성 합니다. 최소한 각 도메인 가입 시스템 전체에서 로컬 관리자 계정에 대 한 다른 암호를 사용 해야 있습니다. 로컬 관리자 계정 고유 하 게 라는 수 하지만 로컬 계정 권한이 부여 된 각 시스템에 대해 서로 다른 암호를 사용 하 여 다른 시스템에서 자격 증명을 사용할 수 없음을 확인 하기에 충분 합니다.  

### <a name="overpopulation-and-overuse-of-privileged-domain-groups"></a>Overpopulation 및 도메인 권한이 있는 그룹 많이  
도메인에서 EA, DA, 또는 모음 그룹 구성원에에서 권한을 부여 대상 공격자 만듭니다. 클수록 수가 이러한 그룹의 구성원을 큼 공격 가능성 권한이 있는 사용자 오용 자격 증명 고 노출 도난 자격 증명을 실수로 수 있습니다. 모든 워크스테이션 또는 서버 있는 권한이 부여 된 도메인 로그온 권한이 있는 사용자의 자격 증명 수집 하 고 광고 DS 도메인 및 숲 손상 하는 데 사용 될 수 있는 가능한 장치를 제공 합니다.  

### <a name="poorly-secured-domain-controllers"></a>보안 잘못 된 도메인 컨트롤러  
도메인 컨트롤러 도메인의 광고 DS 데이터베이스의 복제본을 쉽습니다. 읽기 전용 도메인 컨트롤러 데이터베이스의 로컬 복제 일부 기본적으로 특권된 도메인 계정이 되지 않음 디렉터리에 계정 자격 증명을 포함 됩니다. 읽기 / 쓰기 도메인 컨트롤러에서 각 도메인 컨트롤러 같은 도메인 관리자 권한이 있는 사용자에 대 한 뿐만 아니라 자격 증명을 포함 하 여 광고 DS 데이터베이스의 전체 복제 유지 하지만 도메인 컨트롤러에 도메인 컨트롤러 계정을 또는 계정 KDC 연관 된 도메인의 Krbtgt 계정 같은 권한이 있는 계정 서비스. 도메인 컨트롤러 기능에 대 한 필요 하지 않은 응용 프로그램을 추가로 도메인 컨트롤러에 설치 되어 있거나 도메인 컨트롤러 되지 않는 엄격 패치 및 보안, 공격자 패치되지 취약성을 통해 손상 될 수 있습니다 또는 기타 공격에 직접 악성 소프트웨어를 설치 하려면 활용할 수 있습니다.  

## <a name="privilege-elevation-and-propagation"></a>권한 상승 및 전파  
사용 공격 방법에 관계 없이 Windows 환경 공격 Active Directory 대상 항상, 하 고 공격자 원하는 결국 액세스를 제어 하기 때문에 있습니다. 그러나 전체 디렉터리는 대상, 것은 아닙니다. 특정 계정, 서버 및 인프라 구성 요소에 대 한 Active Directory 공격의 기본 대상이 일반적으로 됩니다. 이러한 계정은 다음과 같은 설명 되어 있습니다.  

### <a name="permanent-privileged-accounts"></a>영구 권한이 있는 계정  
때문 Active Directory 도입 된 Active Directory 숲 빌드 특권 많이 사용 하 여 계정으로 이동한 다음 대리자 권한 및 사용 권한이 적은 계정 관리 매일 수행 하는 데 필요한 가능 합니다. Active Directory에 엔터프라이즈 관리, 도메인 관리자 또는 관리자가 그룹 구성원에에서가 필요만 일시적으로 자주 및 최소 권한 일일 관리 하는 방법을 구현 하는 환경에서입니다.  

영구 권한이 있는 계정에 남아 있는 매일 및 권한이 있는 그룹에 배치 된 계정 됩니다. 조직 도메인 관리자 도메인에 대 한 그룹에 5 개 계정을 장소, 하루에 일주일 내내 5 해당 계정 대상된 24 시간 될 수 있습니다. 그러나 계정을 도메인 관리자 권한으로 사용 하 여 실제 필요는 일반적으로 특정 도메인 전체 구성에 대 한 및 짧은 시간 동안 합니다.  

### <a name="vip-accounts"></a>VIP 계정  
자주 간과 대상 Active Directory 위반에는 조직에서 "아주 중요 한 사람이" (또는 Vip)의 계정을입니다. 해당 계정에 공격자 앞부분이 섹션에 설명 된 대로를 손상 시키거나 대상된 시스템도 삭제 하면 액세스 권한을 부여할 수 있으므로 권한이 있는 계정 대상으로 합니다.  

### <a name="privilege-attached-active-directory-accounts"></a>Active Directory "권한 연결" 계정  
Active Directory 계정 "권한 연결"은 도메인 계정에 있는 가장 높은 수준의 Active Directory에 권한 하지만 높은 수준의 환경에 많은 워크스테이션과 서버에 대 한 권한이 부여 대신 그룹의 회원 설정 되지 않은 합니다. 이러한 계정 서비스 일반적으로 응용 프로그램을 실행 인프라의 큰 부분에서 도메인에 가입 시스템에서 실행 되도록 구성 가장 자주 도메인 기반 계정 됩니다. 이러한 계정을 권한이 없는 Active Directory에 많은 수의 시스템에 대 한 높은 권한이 부여 된 경우, 있지만를 손상 시키거나 더 큰 권한이 부여 된 Active Directory 계정이 손상 될 것과 같습니다 달성 인프라 부분의 삭제 사용할 수 있습니다.
