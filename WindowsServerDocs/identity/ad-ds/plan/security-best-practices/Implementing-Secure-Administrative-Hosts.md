# <a name="implementing-secure-administrative-hosts"></a>안전 하 게 관리 호스트 구현

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

안전 하 게 관리 호스트 워크스테이션 또는 도메인 컨트롤러, 시스템 도메인에 가입한 및 도메인에 가입에서 실행 되는 응용 프로그램에 있는 권한이 있는 계정 Active Directory에 또는 관리 작업을 수행할 수 있는 보안 플랫폼 만들기 위해 구성 된 서버는입니다. 이 경우 "권한이 있는 계정" Active Directory에 가장 권한이 있는 그룹의 회원 계정 뿐 아니라 위임된 권한 된 계정 및 관리 작업을 수행할 수 있는 권한을를 나타냅니다.  
  
이러한 계정의 대부분 DNS 레코드 및 영역을 관리 하는 데 사용 하는 계정 또는 계정을 구성 관리 하는 데 사용 되는 도메인의 사용자의 암호를 재설정 하는 기능이 지원 센터 계정이 될 수 있습니다. 관리 하는 기능에 안전 하 게 관리 호스트는 최선을 다하고 있으며 Microsoft Office 등 소프트웨어 생산성 소프트웨어 메일 응용 프로그램, 웹 브라우저 등을 실행 하지는 않습니다.  
  
"가장 권한" 계정과 그룹 엄격 가장 보호를 적절 하 게 될 해야 하지만 계정 및 그룹 부여 된 계정 위의 표준 사용자의 권한의을 보호 하기 위해 필요를 제거 하지 않습니다.  
  
안전 하 게 관리 호스트 원격 데스크톱 게이트웨이의 서버 역할을 실행 하 고 IT 사용자가 연결 하는 대상 호스트 관리 또는 Hyper-v 역할을 실행 하 고 관리 작업에 사용 하 여 각 IT 사용자에 대해 고유한 가상 컴퓨터를 제공 하는 서버를 수행 하는 회원 서버 관리 작업에 대 한 사용 되는 전용된 워크스테이션 될 수 있습니다. 대부분의 환경에 세 가지 접근의 조합 구현할 수 있습니다.  
  
안전 하 게 관리 호스트 구현 계획 및 조직의 크기, 관리 관행, 위험 욕구가 및 budget와 일치 구성 필요 합니다. 고려 및 안전 하 게 관리 호스트 구현 옵션을 조직에 적합 한 관리 전략을 개발 하는 데 사용 하 여 사용자에 대 한 여기 제공 됩니다.  
  
## <a name="principles-for-creating-secure-administrative-hosts"></a>안전 하 게 관리 호스트 만들기 위한 원칙  
시스템 공격 으로부터 효과적으로 보호 하기 위해 몇 가지 일반적인 원칙 지워진다는 사실에 유지 해야 합니다.  
  
1.  신뢰할 수 없는 호스트 (즉, 관리 하는 시스템와 같은 각도로에 안전 하지 않은 워크스테이션)에서 신뢰할 수 있는 시스템 (즉, 도메인 컨트롤러 등 보안 서버) 관리 절대 해야 합니다.  
  
2.  사용 하면 안 하나의 인증 비율에; 권한 있는 활동을 수행 하는 경우 즉, 사용자 이름 및 암호 조합 간주 하지 말아야 적합 한 인증만 한 배 (것을 알아야)이 표시 되므로 합니다. 자격 증명 생성 하 고 캐시 또는 관리 시나리오에 저장 된 고려해 야 합니다.  
  
3.  하지만 현재 위협 환경에서 대부분의 공격 맬웨어, 악성 해킹과 이용, 디자인 및 관리 호스트 안전한 구현 하는 경우에 물리적 보안을 생략 하지 마십시오.  
  
### <a name="account-configuration"></a>계정 구성  
조직에서 스마트 카드를 사용 하지 않을 경우에 권한이 있는 계정 및 안전 하 게 관리 호스트 구현 하는 것이 좋습니다. 관리 호스트 관리 호스트가 포함 된 Ou에 연결 되어 있는 gpo 다음 설정을 수정 하 여 모든 계정에 대해 스마트 카드 로그온을 요구 하도록 구성 해야 다음과 같습니다.  
  
**컴퓨터 구성 \ 옵션 \ 대화형 로그온: 필요 스마트 카드**  
  
이 설정을 구성 개별 계정을 Active Directory에 관계 없이 스마트 카드를 사용 하 여 모든 대화형 로그온을 해야 합니다.  
  
안전 하 게 관리 호스트만에서 구성 될 수 있는 허가 된 계정으로 로그온을 허용 하도록 구성도 해야 합니다.  
  
**컴퓨터 구성 로컬 보안 설정 정책 \ 사용자 권한 할당**  
  
이 대화형 부여 (되며, 필요한 경우 원격 데스크톱 서비스) 안전 하 게 관리 호스트 권한이 있는 사용자만 로그온 권한이.  
  
### <a name="physical-security"></a>실제 보안  
신뢰할 수 있는 것으로 간주 됩니다 관리 호스트에 대 한 이러한 구성와 같은 각도로 자신이 관리 시스템을 보호 합니다. 대부분의 제공 된 권장 [공격 으로부터 도메인 컨트롤러 보안](../../../ad-ds/plan/security-best-practices/Securing-Domain-Controllers-Against-Attack.md) 도메인 컨트롤러 및 AD DS 데이터베이스를 관리 하는 데 사용 되는 호스트에 해당 됩니다. 대부분의 환경에 안전 하 게 관리 시스템 구현 하는 문제를 해결할 중 하나는 물리적 보안 더욱 어렵게 이러한 컴퓨터 자주 등 관리 사용자 데스크톱, 데이터 센터에 호스팅된 서버로 안전 하지 않은 영역에 위치를 구현할 수 있습니다.  
  
실제 보안 물리적 관리 호스트 액세스 제어 포함 되어 있습니다. 소규모 기업에서 전용된 관리 워크스테이션 유지 되는 office 스토어 또는 사용에 없는 경우 책상 서랍 잠긴 상태를 유지이 의미 될 수 있습니다. 또는 Active Directory 또는 도메인 컨트롤러의 관리를 수행 해야 할 경우 로그온 도메인 컨트롤러에 직접 것일 수 있습니다.  
  
조직에서 중소형 구현을 고려할 수 있습니다 안전 하 게 관리 "이동 서버" 사무실에 안전한 위치에 있는 있고 Active Directory 또는 도메인 컨트롤러의 관리 해야 하는 때 사용 하는 합니다. 사용 하거나 사용 하지 않고 점프 서버에 없는 경우 안전한 위치에 잠깁니다 관리 워크스테이션 구현할 수 있습니다.  
  
대기업, Active directory; 정확 하 게 제어 액세스를 제공 하는 데이터 센터 보관 점프 서버 배포할 수 있습니다. 도메인 컨트롤러; 하 고 파일, 인쇄, 응용 프로그램 서버 합니다. 점프 서버가 아키텍처 구현 가능성이 가장 큰 환경에서 보안 워크스테이션과 서버의 조합이 포함 합니다.  
  
조직의 크기와 관리 호스트 디자인에 관계 없이 무단된 액세스 또는 도난 로부터 물리적 컴퓨터를 보호 해야 및 암호화 하 고 관리 호스트의 드라이브 보호 BitLocker 드라이브 암호화를 사용 해야 합니다. BitLocker 관리 호스트에서를 구현 하 여 해당 디스크를 제거 하거나 호스트 도난당 한 경우에 확인할 수 있습니다 드라이브에 데이터 권한 없는 사용자나에 액세스할 수 없는.  
  
### <a name="operating-system-versions-and-configuration"></a>운영 체제 버전 및 구성  
모두 관리 호스트 실행할지 여부를 서버 또는 워크스테이션 최신 운영 체제가이 문서의 앞에 설명 된 이유로 조직에서 사용 됩니다. 현재 운영 체제를 실행 하면 새로운 보안 기능, 전체 공급 업체 지원 및 추가 기능 운영 체제에 새롭게 적용 된 관리 담당자 도움이 됩니다. 또한 관리 호스트를 먼저 배포 하는 새 운영 체제를 평가 하는 때와 새로운 기능, 설정 및 운영 체제의 광범위 한 배포를 계획에 이후 활용할 수 있는 관리 메커니즘 제공 하는 것을 이해 하기 위해 해야 합니다. 다음으로 조직에서 가장 고급 사용자는 사용자에 게는 새 운영 체제에 및 가장 위치를 지 원하는 될 예정입니다.  
  
### <a name="microsoft-security-configuration-wizard"></a>Microsoft Security 구성 마법사  
점프 서버 관리 호스트 전략의 일환으로, 구현 하는 경우 서비스, 레지스트리, 감사 및 서버의 공격을 줄이기 위해 방화벽 설정 기본 보안 구성 마법사를 사용 해야 합니다. 보안 구성 마법사 구성 설정을 수집 하 고 구성, 설정은 모든 점프 서버에서 일관 되 게 초기 구성 적용 하는 데 사용 되는 GPO로 변환 될 수 있습니다. 이동 서버에만 적용 됨 보안 설정 구현 GPO 더 편집할 수와 추출 Microsoft 보안 준수 관리자에서 추가 초기 설정으로 설정의 모든 결합할 수 있습니다.  
  
### <a name="microsoft-security-compliance-manager"></a>Microsoft 보안 준수 관리자  
[Microsoft 보안 준수 관리자](https://technet.microsoft.com/library/cc677002.aspx) 단일 도구 및 UI를 만들고 도메인 컨트롤러에 보안 설정을 초기 구성에서 사용할 수 있는에서 수집 하 고 운영 체제 버전 및 역할 구성에 따라 Microsoft가 권장 되는 보안 구성 통합 무료로 사용할 수 있는 도구입니다. Microsoft 보안 준수 관리자 템플릿 보안 구성 마법사 설정에 대 한 점프 서버에 배포 되 고 있는 점프에서에 서버가 Active Directory Ou에 배포 Gpo로 적용 포괄적인 구성 기준 생성 하기 위해와 함께 사용할 수 있습니다.  
  
> [!NOTE]  
> 현재, Microsoft 보안 준수 관리자 점프 서버 또는 다른 보안 관리 호스트 고유한 설정이 포함 되지 않습니다 하지만 관리 호스트에 대 한 초기 초기 만들 관리자 SCM (보안 준수) 계속 사용할 수 있습니다. 그러나 제대로 호스트를 보호 하기 위해 추가 보안 설정 매우 안전한 워크스테이션과 서버에 적합 한를 적용 해야 합니다.  
  
### <a name="applocker"></a>AppLocker  
관리 호스트 및 가상 machinesshould 스크립트, 도구 및 응용 프로그램 whitelists AppLocker 또는 제 3 자 응용 프로그램 제한 소프트웨어를 통해으로 구성할 수 있습니다. 관리 응용 프로그램 또는 보안 설정을 준수 하지 않는 유틸리티를 업그레이드 하거나 보안 개발 및 관리 방법을 준수 하는 도구를 교체 해야 합니다. 새로운 기능이 나 추가 도구 관리 호스트에 필요 하면 응용 프로그램 및 공공 설비 해야 철저 하 게 및 도구는 관리 호스트에 배포를 위해 적합 한 경우, 시스템은 whitelists에 추가할 수 있습니다.  
  
### <a name="rdp-restrictions"></a>RDP 제한  
제한 된 계정 및 컴퓨터 데 사용할 수 원격 데스크톱 게이트웨이의 (RD 게이트웨이)를 사용 하는 등의 관리 되는 시스템에 대 한 원격 데스크톱 프로토콜 RDP () 연결 이동 도메인 컨트롤러에 대 한 액세스를 제어 하는 서버 및 관리 하는 다른 설정 포함할지 특정 구성 아키텍처 관리 시스템에 따라 다르지만 권한이 있는 사용자와 시스템에서 시스템 합니다.  
  
대화형 로그온 권한이 있는 사용자가 허용 하 고 해야 제거 하거나 해야도 서버 액세스 필요 하지 않은 다른 로그온 형식을 차단 합니다.  
  
### <a name="patch-and-configuration-management"></a>패치 및 구성 관리  
중소기업 같은 Windows 업데이트 서비스를 사용할 수 또는 [Windows Server Update Services](https://technet.microsoft.com/windowsserver/bb332157) (WSUS) 대기업 엔터프라이즈 패치 및 구성 관리 소프트웨어 System Center Configuration Manager 같은 구현 하는 동안 Windows 시스템에서 업데이트 배포를 관리할 수 있습니다. 일반 서버 및 워크스테이션 인구에 업데이트를 배포 하는 사용 하 여 메커니즘을 관계 없이 호스트 관리, 인증 기관 도메인 컨트롤러 등 높은 보안 시스템에 대 한 별도 배포 고려해 야 합니다. 일반 관리 infrastructure에서 이러한 시스템을 분리 하 여 관리 소프트웨어 또는 서비스 계정이 손상 된 경우 손상 쉽게 확장할 수 없습니다 인프라에 가장 안전한 시스템에 있습니다.  
  
보안 시스템에 대 한 업데이트를 수동으로 프로세스를 구현 해서는 안 되지만 보안 시스템 업데이트에 대 한 별도 인프라를 구성 해야 합니다. 매우 큰 조직의 경우에이 인프라 일반적으로 구현할 수 있습니다 전용된 WSUS 서버 및 Gpo을 통해 보안된 시스템에 대 한 합니다.  
  
### <a name="blocking-internet-access"></a>인터넷 액세스 차단  
호스트 관리자는 인터넷 액세스를 허용 되지 않은 없으며 해야 하는 회사 인트라넷 탐색할 수 없습니다. 웹 브라우저와 유사한 응용 프로그램 관리 호스트에서 허용 되지 않은 합니다. 주변 방화벽 설정, WFAS 구성 및 안전 호스트에서 "블랙홀" 프록시 구성을 조합 하 여 안전 하 게 호스트 인터넷 액세스를 차단할 수 있습니다. 또한 웹 브라우저 관리 호스트에서 사용 되지 않도록 하려면 whitelisting 응용 프로그램을 사용할 수 있습니다.  
  
### <a name="virtualization"></a>Virtualization  
가능한 경우 가상 컴퓨터 관리 호스트 구현 하는 것이 좋습니다. Virtualization를 사용 하 여 시스템 생성할 수 있습니다 각 사용자 관리 하는 중앙 저장 되 고, 관리 하 고 있는 종료 될 수 쉽게 때 사용에는 자격 증명은에 남아 있지 활성 관리 시스템을 보장 합니다. 가상 관리 호스트 다시 설정 됩니다 초기 스냅숏의을 사용할 때마다 가상 컴퓨터는 그대로 유지 되도록 할 수 있습니다. 관리 호스트 virtualization 옵션에 대 한 자세한 내용은 다음 섹션에서 제공 됩니다.  
  
## <a name="sample-approaches-to-implementing-secure-administrative-hosts"></a>샘플 구현 하는 방법 관리 호스트 보안  
어떻게를 디자인 하 고 관리 호스트 인프라 배포 관계 없이 하면 유념 해야 앞부분이 항목에서에서 "원칙을 만드는 보안 관리 호스트"에 제공 된 지침입니다. 여기에 설명 된 방식의 각 "관리" 및 "생산성" 시스템 IT 부서에서 사용 하는 구분할 수 방법에 대 한 일반 정보를 제공 합니다. 생산성 시스템 확인 메일, 인터넷을 검색 하 고 Microsoft Office 등 일반 생산성 소프트웨어를 사용 하도록 IT 관리자를 사용 하는 컴퓨터가 있습니다. 시스템 관리자는 컴퓨터 전념는 IT 환경 일상적인 관리를 사용 하 고 강화입니다.  
  
안전 하 게 관리 호스트 구현 하는 가장 간단한 방법은 관리 작업을 수행할 수 있는 보안 워크스테이션 IT 부서 제공 하는 것입니다. 워크스테이션 전용 구현에서 각 관리 워크스테이션 관리 도구와 서버 및 기타 인프라 관리 RDP 연결을 시작 하는 데 사용 됩니다. 큰, 복잡 한 인프라 도움이 될 수 있습니다 분산된 디자인부터 워크스테이션과 전용된 관리 서버 사용 되는, 관리 호스트 "구현 보안을 유지 관리 워크스테이션 이동" 서버 및에 설명 된 대로이 항목 뒷부분에 있지만 워크스테이션 전용 구현 소규모 조직에서 적용 될 수 있습니다.  
  
### <a name="implementing-separate-physical-workstations"></a>별도 실제 워크스테이션 구현  
관리 호스트를 구현할 수 있습니다 한 방법은 두 가지 워크스테이션 각 IT 사용자 드릴 것입니다. 하나의 워크스테이션 메일을 확인 하 고 관리 기능을 두 번째 워크스테이션 엄격히 전용 생산성 응용 프로그램을 사용 하 여와 같은 작업을 수행 하려면 "일반" 사용자 계정으로 사용 됩니다.  
  
생산성 워크스테이션 IT 직원 권한이 있는 계정 비보안된 컴퓨터에 로그온 할 때 사용 하는 것이 아니라 일반 사용자 계정 지정할 수 있습니다. 관리 워크스테이션 엄격 제어 구성으로 설정 해야 및 IT 직원 관리 워크스테이션에 로그온 할 때 다른 계정을 사용 해야 합니다.  
  
스마트 카드를 구현 하는 경우 관리 워크스테이션 스마트 카드 로그온을 요구 하도록 구성 해야 및 IT 직원 제공 해야 별도 계정을 스마트 카드 로그온 요구 하도록 구성 된 관리 사용 합니다. 관리 호스트 강화 된 앞에서 설명한 대로, 고 지정된 IT 사용자만을 관리 워크스테이션 로컬로 로그온 할 수 있어야 합니다.  
  
#### <a name="pros"></a>전문가  
별도 실제 시스템을 구현 하 여 각 컴퓨터 역할 올바르게 구성 되어 있고 IT 사용자가 관리 시스템 위험에 노출 실수로 수 없는 확인할 수 있습니다.  
  
#### <a name="cons"></a>단점  
  
-   별도 실제 컴퓨터 구현 하드웨어 비용을 높여 줍니다.  
  
-   메모리에서 자격 증명을 캐시 자격 증명 원격 시스템 관리 하는 데 사용 하는 물리적 컴퓨터에 로그온 합니다.  
  
-   관리 워크스테이션 안전 하 게 저장 되어 있지는 메커니즘 물리적 하드웨어 키로 거 나 실제 기타 공격 으로부터 등을 통해 손상에 노출 될 수 있습니다.  
  
### <a name="implementing-a-secure-physical-workstation-with-a-virtualized-productivity-workstation"></a>가상화 생산성 워크스테이션 안전한 물리적 워크스테이션 구현  
이 방법을 IT 증명이 사용자 일상적인 관리 기능을 수행할 수 있는 보안된 관리 워크스테이션 책임 해당 범위 내에서 서버 관리 RSAT (원격 도구) 또는 RDP 서버에 연결을 사용 하 여 합니다. IT 사용자의 생산성 작업을 수행할 필요가 가상 컴퓨터 실행 원격 생산성 워크스테이션 RDP를 통해 연결할 수 있습니다. 각 워크스테이션에 대 한 별도 자격 증명을 사용 해야 하 고 스마트 카드와 같은 컨트롤을 구현 합니다.  
  
#### <a name="pros"></a>전문가  
  
-   관리 워크스테이션과 생산성 워크스테이션 분리 됩니다.  
  
-   별도 자격 증명 및 스마트 카드 IT 부서 보안 워크스테이션을 사용 하 여 생산성 워크스테이션에 연결할 사용할 수 있으며 권한 자격 증명 하지 덜 보안 컴퓨터에 보관해 둘 합니다.  
  
#### <a name="cons"></a>단점  
  
-   디자인 및 구현 작업 및 강력한 virtualization 옵션 필요 솔루션을 구현 합니다.  
  
-   실제 워크스테이션 안전 하 게 저장 되지 않은 경우 하드웨어 또는 운영 체제의 손상 하 고 통신 가로채기를 받을 수 있도록 하는 물리적 공격에 노출 될 수 있습니다.  
  
### <a name="implementing-a-single-secure-workstation-with-connections-to-separate-productivity-and-administrative-virtual-machines"></a>으로 구분 "생산성" 및 "관리" 가상 컴퓨터에 연결 된 단일 보안 워크스테이션 구현  
이 방법을 IT 사용자 단일 실제 워크스테이션 하는 잠겨 앞에서 설명한 대로, 하 고 있는 IT 사용자 없는 액세스 권한을된 실행할 수 있습니다. 메일 및 기타 생산성 응용 프로그램을 실행 하는 하나의 가상 컴퓨터와 사용자의 전용된 관리 호스트도 구성 된 가상 컴퓨터 두 번째 IT 부서 제공 가상 컴퓨터를 호스팅할 전용된 서버에 연결 원격 데스크톱 서비스를 제공할 수 있습니다.  
  
하면 필요 스마트 카드 또는 기타 다단계 로그온 가상 컴퓨터의 경우 컴퓨터에 로그온 할 때 사용 되는 계정 이외의 별도 계정을 사용 하 여 선택 합니다. IT 사용자의 컴퓨터에 로그온 한 후 생산성 원격 컴퓨터와 별도 계정 및 관리 원격 컴퓨터에 연결 하려면 스마트 카드에 연결 하는 생산성 스마트 카드를 사용할 수 있습니다.  
  
#### <a name="pros"></a>전문가  
  
-   IT 사용자 단일 실제 워크스테이션을 사용할 수 있습니다.  
  
-   가상 호스트 및 사용 하 여 가상 컴퓨터에 연결 원격 데스크톱 서비스에 대 한 별도 계정 하도록 요구 하 여 로컬 컴퓨터의 메모리에 IT 사용자의 자격 증명 캐시 되지 않습니다.  
  
-   실제 호스트 로컬 컴퓨터의 줄일 관리 호스트 동일한 수준의 보호 수 있습니다.  
  
-   경우에는 IT 사용자의 생산성 가상 컴퓨터 또는 관리 가상 컴퓨터의 손상 된 가상 컴퓨터 "으로 성공한" 상태로 초기화 쉽게 수 있습니다.  
  
-   컴퓨터 물리적 손상 되는 경우 메모리에서 권한 없이 자격 증명 캐시 됩니다 및 스마트 카드를 사용 하는 키 입력 거에서 자격 증명의 손상 되지 않을 수 있습니다.  
  
#### <a name="cons"></a>단점  
  
-   디자인 및 구현 작업 및 강력한 virtualization 옵션 필요 솔루션을 구현 합니다.  
  
-   실제 워크스테이션 안전 하 게 저장 되지 않은 경우 하드웨어 또는 운영 체제의 손상 하 고 통신 가로채기를 받을 수 있도록 하는 물리적 공격에 노출 될 수 있습니다.  
  
### <a name="implementing-secure-administrative-workstations-and-jump-servers"></a>안전 하 게 관리 워크스테이션과 점프 서버를 구현합니다.  
관리 워크스테이션 보안을 유지 하는 대신 또는와 함께에서 안전한 점프 서버를 구현할 수 있습니다 및 관리자가 사용자가 관리 작업을 수행할 RDP 및 스마트 카드를 사용 하 여 점프 서버에 연결할 수 있습니다.  
  
점프 서버 및에서 관리할 수 있는 대상 서버 점프 서버에 연결에 제한 구현할 수 있도록 원격 데스크톱 게이트웨이의 역할 실행 하도록 구성 합니다. 가능 하면도 Hyper-v 역할을 설치 하 고 해야 만들 [개인 가상 데스크톱](https://technet.microsoft.com/library/dd759174.aspx) 또는 다른 사용자 가상 컴퓨터 점프 서버에 작업에 사용 하 여 사용자가 관리 합니다.  
  
하 여 사용자 사용자별 가상 컴퓨터 점프 서버에, 관리 워크스테이션 물리적 보안을 제공 하 고 관리자가 사용자가 다시 설정 하거나 사용 하지에서 가상 컴퓨터를 종료 수 있습니다. Hyper-v 역할 및 원격 데스크톱 게이트웨이의 역할 같은 관리 호스트에 설치 하지 않으려는 경우 별도 컴퓨터에 설치할 수 있습니다.  
  
가능 하면 서버 관리 원격 관리 도구를 사용 해야 합니다. 원격 서버 관리 도구 RSAT () 기능에는 사용자의 가상 컴퓨터 (또는 점프 서버 관리에 대 한 사용자 가상 컴퓨터 구현 하지 않는 경우)을 설치 해야 관리 담당자 RDP 통해 관리 작업을 수행할 가상 컴퓨터에 연결 해야 합니다.  
  
경우에 관리자가 사용자를 직접 관리할 수 대상 서버에 RDP를 통해 연결 해야 RD 게이트웨이 구성 해야 적절 한 사용자 및 컴퓨터를 사용 하 여 목적지 서버에 연결 하는 경우에 연결할 수 있도록 합니다. 실행 서버 이동 하지 일반 사용 워크스테이션 및 있는 구성원 서버와 같은 도구 관리 시스템을 지정 되지 않은 시스템에서 금지 해야 RSAT (또는 유사한).  
  
#### <a name="pros"></a>전문가  
  
-   점프 서버 만들기 하면 특정 네트워크에 "영역" (비슷한 구성과 연결, 보안 요구 사항 만족 시스템 컬렉션) 서버 지도 하 고 각 영역의 관리 지정한 "시간대" 서버에 안전 하 게 관리 호스트에서 연결 하는 관리 담당자가 달성는 요구 하도록 합니다.  
  
-   영역에 웹 서버를 매핑하여 연결 속성 및 구성 요구 사항에 대 한 자세한 컨트롤을 구현할 수 있고 무단된 시스템에서 연결을 시도 쉽게 식별할 수 있습니다.  
  
-   점프 서버에 따라 관리자 가상 컴퓨터를 구현 하 여 종료 및 관리 작업이 완료 되는 알려진 정리 된 상태로 가상 컴퓨터의 재설정 적용 합니다. 관리 작업을 완료할 때 가상 컴퓨터의 종료 (또는 다시 시작)를 적용 하 여 가상 컴퓨터 공격자을 대상으로 지정할 수 받지 않으며 가능 자격 증명 도용의 공격 있으므로 다시 부팅 이상의 자격 증명 메모리 캐시 유지 되지 않습니다.  
  
#### <a name="cons"></a>단점  
  
-   실제 또는 가상 전용된 서버가 점프 서버에 대 한 필요 합니다.  
  
-   지정 된 구현 서버를 관리 워크스테이션 신중 하 게 계획 및 환경에서 구성 된 보안 영역에 매핑하 구성 필요 합니다.
