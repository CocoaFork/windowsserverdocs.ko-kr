# <a name="audit-policy-recommendations"></a>정책 추천 감사 합니다.

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

이 섹션 해결 Windows 기본 감사 정책 설정, 기본 권장 감사 정책 설정 및 보다 적극적인 권장 되는 Microsoft에서 워크스테이션과 서버 제품에 대 한 합니다.  

함께 손상 발견할 수 있도록 권장 설정, 여기에 표시 된 SCM 초기 추천 관리자에 게 시작 초기 안내에 사용 됩니다. 각 조직 마주 위협, 사용할 수 있는 위험 공차 및 사용 해야 감사 정책 범주 나 하위 범주 기능에 대해 고유한 결정을 내릴 해야 합니다. 위협 요소에 대 한 자세한 내용은 참조는 [위협과 대책 가이드](https://technet.microsoft.com/library/hh125921(v=ws.10).aspx)합니다. 장소에 진심으 감사 감사 정책 없이 관리자는 시작, 권장 설정 하는 것이 좋습니다 하 고 수정 하 고 테스트, 생산 환경에을 구현 하기 전에 다음 합니다.  

권장 되는 Microsoft 정의 컴퓨터 평균 보안 요구 사항이 있는 작동 하는 기능의 개략적으로 엔터프라이즈급 컴퓨터에 적용 됩니다. 높은 보안 요구 사항을 보다 적극적인 고려해 야 않아도 엔터티 정책 감사 합니다.  

> [!NOTE]  
> Microsoft Windows에 기본 설정 및 초기 추천에서 찍은 [Microsoft 보안 준수 관리자 도구](https://technet.microsoft.com/library/cc677002.aspx)합니다.  

다음과 같은 기본 감사 정책 설정은 지 알 수 없는 활성, 성공 되었을 수 결정된 악성 또는 맬웨어에 하는 일반적인 보안 컴퓨터에 대 한 것이 좋습니다.

## <a name="set-audit-policy-on-workstations-and-servers"></a>워크스테이션과 서버에 감사 정책 설정  
모든 이벤트 로그 관리 계획 워크스테이션과 서버 모니터링 해야 합니다. 실수만 서버 또는 도메인 컨트롤러를 모니터링 하는 것입니다. 워크스테이션 악성 해킹과 처음 자주 발생 하기 때문에 워크스테이션 모니터링 하 고 있지를 정보의 최상의 및 빠른 소스를 무시 합니다.  

관리자 해야 면밀히 검토 하 고 생산 환경에 구현 하기 전에 모든 감사 정책을 테스트 합니다.  

## <a name="events-to-monitor"></a>모니터에 이벤트  
보안 경고를 얻기 위해 완벽 하 게 이벤트 ID 다음과 같은 특성 포함 됩니다.  

-   높은 가능성 해당 항목을 나타냅니다 무단된 활동  

-   낮은 가양성 수  

-   항목을 설명/조사 응답 유발  

두 가지 유형의 이벤트를 모니터링 하 고 알림을 해야 합니다.  

1.  한 번 수행도 무단된 활동을 나타냅니다 이벤트  

2.  예상 되 고 허용 초기 위의 이벤트 누적  

첫 번째 이벤트의 예는 다음과 같습니다.  

도메인 관리자 DAs 도메인 컨트롤러 없는 컴퓨터에 로그온 할 등록이 차단, DA 회원 최종 사용자 워크스테이션 로그온 한 번 수행 경고를 생성 하 고 조사 합니다. 이러한 유형의 경고는 4964 감사 특별 한 로그온 이벤트를 사용 하 여 생성할 쉽게 (특별 한 그룹에 할당 된 새로운 로그온). 다른 단일 인스턴스 경고의 예입니다.  

-   서버 A 서버 B 경고 서로에 연결할 때에 연결 하지 해야 합니다.  

-   정상 최종 사용자 계정이 예기치 않게 중요 한 보안 그룹에 추가 되 면 알립니다.  

-   경우 A 공장 위치에는 직원 밤 경고 로그온 할 때 사용자 자정 작동 하지 않습니다.  

-   무단된 서비스가 도메인 컨트롤러에 설치 된 경우 알립니다.  

-   일반 최종 사용자가 직접에 로그온 한 지우기 없는 이유가 있을 SQL Server 조사 합니다.  

-   DA 그룹에서 구성원이 있고 다른 사용자 추가 하는 자체 있는 경우 즉시 확인 합니다.  

두 번째 이벤트의 예는 다음과 같습니다.  

로그온 실패 어 베 번호 추측 공격 암호를 것일 수 있습니다. 로그온 실패 수가 대 한 경고를 제공 하기 위해 엔터프라이즈용 먼저 악성 보안 이벤트가 되기 전에 해당 환경 내에서 로그온 실패 정상 수준 이해 해야 합니다.  

침입 모니터링할 때 포함 해야 하는 이벤트의 전체 목록을 참조 하세요 [모니터 부록 l: 이벤트](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md)합니다.  

## <a name="active-directory-objects-and-attributes-to-monitor"></a>Active Directory 개체 및 모니터에 특성  
계정, 그룹 및 모니터링 해야 손상 Active Directory 도메인 서비스 설치를 시도 하는 데 도움이 되는 특성은 다음과 같습니다.  

-   시스템을 사용 하지 않도록 설정 했거나 바이러스 백신 및 맬웨어 방지 소프트웨어 (자동으로 다시 시작 보호 않으면 수동으로) 제거  

-   관리자 계정에 대 한 무단으로 변경  

-   권한이 있는 계정 (자동으로 완료 또는 할당 시간 의심 스러운 활동 때 계정 제거에 만료 됨)를 사용 하 여 수행 하는 활동  

-   권한 및 AD ds에서 VIP 계정 합니다. 모니터 특성 (예: cn, 이름, sAMAccountName, 파티션에서, 또는 userAccountControl) 계정 탭에서 변경 내용 특히 변경 합니다. 모니터링 외에도 계정, 작은으로 계정에 가능한 관리자가 사용자의 집합을 수정할 수 있는 사람을 제한 합니다.  

참조 [모니터 부록 l: 이벤트](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md) 모니터, 중요도 등급 및 이벤트 메시지 요약 권장된 이벤트 목록에 대 한 합니다.  

-   신속 하 게 가장 잘 모니터링와 엄격 가장 구성 된 해야 하는 서버를 식별할 수 있는 작업의 분류도 그룹화 서버  

-   다음 AD DS 그룹의 회원 속성을 변경 내용: Enterprise 관리자 (EA), 도메인 관리자 (DA), 관리자가 (모음) 및 스키마 관리자 (SA)  

-   계정 설정에 대 한 사용 안 함된 권한이 있는 계정 (예: 기본 제공 된 관리자 계정 회원 시스템 및 Active Directory)  

-   모든 기록 계정에 로그인 계정 관리  

-   기본 제공 보안 구성 마법사 서비스, 레지스트리, 감사, 서버의 공격을 줄이기 위해 방화벽 설정을 구성할 수 있습니다. 점프 서버 관리 호스트 전략의 일환으로 구현 하는 경우이 마법사를 사용 합니다.

## <a name="general-list-of-security-event-id-recommendation-criticalities"></a>이벤트 ID 권장 Criticalities 보안 일반 목록  
이벤트 ID 추천 모든 다음과 같은 rating 중요도 함께 제공 됩니다.  

**높음:** 높은 중요도 등급이 이벤트 Id 항상 및 즉시 알림을 고 조사 합니다.  

**보통:** 중간 중요도 등급이 이벤트 ID 악성 활동을 나타낼 수 있지만 일부 다른 비정상적인 종료 (예를 들어,는 비정상적인 번호 특정 시간, 예기치 않은으로 발생 하는 일반적으로 하지 예상 되는 이벤트를 기록 하는 컴퓨터에 발생에서 발생 합니다.)가 포함 되어 있어야 합니다. 보통 중요도 이벤트 있을 수 있습니다 r를 기준으로 수집 하 고 시간이 지남에 따라 비교할 수 있습니다.  

**낮은:** 이벤트 ID 낮은 중요도 이벤트와 함께 주의 차지 하거나 안 중간 또는 중요도 이벤트와 상관 하지 않는 한 경고를 유발 하 고 있습니다.  

이러한 권장은 관리자에 대 한 초기 가이드 제공 하기 위한 것입니다. 모든 권장 생산 환경에서 실행 하기 전에 철저 하 게 검토 해야 합니다.  

참조 [모니터 부록 l: 이벤트](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md) 모니터 중요도 등급을 이벤트 메시지 요약 권장된 이벤트의 목록을 보려면 합니다.
