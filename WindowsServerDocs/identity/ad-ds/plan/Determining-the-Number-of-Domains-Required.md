# <a name="determining-the-number-of-domains-required"></a>필요한 도메인의 수를 확인

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

모든 숲 단일 도메인으로 시작합니다. 단일 도메인 숲 포함 될 수 있는 사용자가 최대 도메인 컨트롤러 및 대역폭이 Active Directory Domain Services (AD DS)를 할당 하려는 간의 복제 수용 해야 하는 느린 링크를 기반으로 합니다. 다음 표에서 최대 느린 링크의 속도 복제 예약 하려는 대역폭의 비율 단일 도메인 숲에 따라 도메인 포함 될 수 있는 사용자 수를 권장 합니다. 이 정보는 최대 100, 000 사용자를 포함 하는 및 28.8 k 이상의 초당 56kbps의 연결 된 숲에 적용 됩니다. 숲 100, 000 개 이상의 사용자 또는 28.8 미만의 연결을 포함 하는에 적용 되는 추천 숙련 된 Active Directory 디자이너를 참조 하십시오. 다음 표에 있는 값 다음과 같은 특징이 하는 환경에서 발생 한 복제 소통을 기반으로 합니다.  
  
-   새 사용자 연간 20% 비율로 숲에 참여 합니다.  
  
-   사용자가 출발해 1 년 15%의 속도로 숲입니다.  
  
-   각 사용자는 5 글로벌 그룹과 5 범용 그룹의 회원 합니다.  
  
-   컴퓨터에 사용자의 1:1입니다.  
  
-   Active Directory 통합 시스템 DNS (도메인 이름) 사용 됩니다.  
  
-   DNS 청소 사용 됩니다.

> 다음 표에 그림은 대략적인 합니다. 복제 교통 수량에서 지정된 된 시간 동안 디렉터리에 변경 번호 크게에 따라 다릅니다. 네트워크 도메인 배포 하기 전에 랩에서 예상된 크기와 디자인에 변경 속도 테스트를 통해 복제 사용량을 사용할 수 있는지 확인 합니다.

2.  연결 속도 느린에 해당 하는 행에서 AD DS를 할당 하려는 백분율 대역폭 나타내는 열을 찾습니다. 해당 위치에 값은 최대 단일 도메인 숲 속의 도메인 포함 될 수 있는 사용자입니다.  
  
사용자 숲 속의 사용자의 총 도메인 포함 될 수 있는 사용자가 최대 보다 작으면를 확인 하는 경우 단일 도메인을 사용할 수 있습니다. 이 확인 때 계획된 증대 고려 하 여 해야 합니다. 사용자 숲 속의 사용자의 총은 도메인 포함 될 수 있는 사용자가 최대 보다 더 깁니다를 확인 하는 경우는 훨씬 더 비율의 복제, 대역폭을 예약 하 연결 속도 향상 하거나 조직 도메인 지역으로 구분 합니다.
