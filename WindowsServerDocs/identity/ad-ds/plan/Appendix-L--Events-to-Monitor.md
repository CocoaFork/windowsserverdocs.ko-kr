# <a name="appendix-l-events-to-monitor"></a>부록 l: 모니터링할 이벤트

>적용 대상: Windows Server

다음 표에 나오는 권장 사항에 따라 사용자 환경에서 모니터링 해야 하는 이벤트 [손상의 기호에 대 한 Active Directory 모니터링](../../ad-ds/plan/security-best-practices/Monitoring-Active-Directory-for-Signs-of-Compromise.md)합니다. 다음 표에서 "현재 Windows 이벤트 ID" 열 현재 일반 지원에는 Windows 및 Windows Server 버전에서 구현 되는 이벤트 ID를 나열 합니다.  
  
Windows XP를 실행 하는 클라이언트 컴퓨터와 같은 레거시 버전의 Windows에서 해당 이벤트 ID를 나열 하는 "레거시 Windows 이벤트 ID" 열 이전 및 Windows Server 2003 이전 버전을 실행 하는 서버입니다. "잠재적 중요도" 낮은의 이벤트를 고려할지 여부를 식별 하는 열, 보통 또는 높은 중요도 공격과 "이벤트 요약" 열을 검색 이벤트의 간략 한 설명을 제공 합니다.  
  
높음 잠재적 중요도 의미는 이벤트 발생을 조사 해야 합니다. 보통 또는 낮음 잠재적 중요도 이러한 이벤트에만 조사 해야 크게 시간 측정 기간 동안에서 예상된 기준을 초과 하는 숫자 또는 예기치 않게 발생 하는 경우를 의미 합니다. 모든 조직은 필수 조사 응답이 필요한 경고를 만들기 전에 해당 환경에서 이러한 권장 사항을 테스트해야 합니다. 모든 환경은 다르며, 및 높음 잠재적 중요도 사용 하 여 순위를 지정 하는 이벤트의 일부 다른 무해 한 이벤트가 발생할 수 있습니다.
