# <a name="appendix-l-events-to-monitor"></a>모니터에 부록 l: 이벤트

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

  
## <a name="appendix-l-events-to-monitor"></a>모니터에 부록 l: 이벤트  
다음 표에서 이벤트에 제공 된 권장 따르면 귀하의 환경에 모니터링 해야 하는 [손상 기호에 대 한 Active Directory 모니터링](../../ad-ds/plan/security-best-practices/Monitoring-Active-Directory-for-Signs-of-Compromise.md)합니다. 다음 표에서 "최신 Windows 이벤트 ID" 열 일반 지원에서 현재 사용 중인 버전의 Windows 및 Windows Server에서 구현 이벤트 ID 나열 됩니다.  
  
"레거시 Windows 이벤트 ID" 열 나열 레거시 Windows 버전의 Windows XP를 실행 하는 클라이언트 컴퓨터와 같은 이벤트 ID 이전 및 Windows Server 2003 이전 버전을 실행 하는 서버 또는 합니다. "잠재적인 중요도" 열 식별 낮은의 이벤트 간주할 수 있는지 여부, 중간 또는 공격 하 고 "이벤트 요약" 열 검색 높은 중요도 이벤트에 대 한 간략 한 설명을 제공 합니다.  
  
높은의 잠재적인 중요도 의미는 하나 이벤트가 발생 조사 해야 합니다. 중간 또는 낮음으로 잠재적 중요도 의미 예기치 않게 또는 크게 예상된 초기 측정 기간 동안 시간 초과 하는 숫자 발생할 경우 이러한 이벤트 조사만 해야 합니다. 모든 조직이 필수 조사 응답 필요한 경고를 만들기 전에 해당 환경에서 이러한 권장 테스트 해야 합니다. 모든 환경 다른, 이며 순위가 높은의 잠재적 중요도 된 이벤트의 일부를 다른 해 이벤트 인해 발생할 수 있습니다.
