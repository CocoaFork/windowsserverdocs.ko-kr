# <a name="ad-ds-design-and-planning"></a>AD DS 디자인 및 계획

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

귀하의 환경에 Windows Server Active Directory Domain Services (AD DS)을 배포 하 여 위임된 하는 중앙 집중식된 관리 모델 및 단일 로그온 (SSO) 기능이 있는 AD DS 제공 하는 기능을 사용할 수 있습니다. 조직에 대 한 배포 작업과 현재 환경을 확인 한 후 조직의 요구를 충족 하 고 AD DS 배포 전략을 만들 수 있습니다.  
  
## <a name="about-this-guide"></a>이 가이드 정보  
이 가이드 조직 및 만들려는 디자인에서는 특정의 요청에 따라 AD DS 배포 전략 개발 하는 데 도움이 되는 추천을 제공 합니다. 이 가이드는 infrastructure 전문가 또는 시스템 설계자 사용에 대 한 것입니다. 이 가이드를 읽으려면 먼저 AD DS 기능 수준에서 작동 하는 방법을는 잘 알고가 있어야 합니다. 또한 AD DS 배포 전략에서 구성 요구 사항이 반영 될 수 있는는 잘 알고가 있어야 합니다.  
  
이 가이드 Windows Server 2008 AD DS 배포의 몇 가지 가능한 시작 지점을 작업 집합에 설명 합니다. 이 가이드에서는 귀하의 환경에 대 한 배포 전략 가장 적합 한 결정 합니다.  
  
이 가이드에 전략 거의 모든 서버 운영 체제 배포에 대 한 적절 한 옵션은 된 테스트 하 고 했는지 환경 100, 000 미만의 사용자와 미만인 1, 000 사이트에 대해 특별히의 유효성을 검사 최소 28.8 초당 56kbps의 네트워크 연결을 사용 합니다. 귀하의 환경 조건 이러한, AD DS 복잡 한 환경에서 배포 경험이 컨설팅 회사를 사용 하 여 하는 것이 좋습니다.  
  
테스트 AD DS 배포 프로세스에 대 한 자세한 내용은 참조 테스트 하 고 확인 하는 배포 프로세스 ([https://go.microsoft.com/fwlink/?LinkId=100206](https://go.microsoft.com/fwlink/?LinkId=100206)).  
  
> [!NOTE]  
> 이 가이드의 정보는 Windows Server 2008 R2 AD DS에도 적용 됩니다.
