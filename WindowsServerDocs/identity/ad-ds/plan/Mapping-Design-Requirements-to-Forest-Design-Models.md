# <a name="mapping-design-requirements-to-forest-design-models"></a>숲 디자인 모델을 매핑 디자인 요구 사항

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

조직에서 대부분의 그룹 단일 조직 숲 단일 정보 it 그룹에서 관리 되는 사용자 계정 및 모든 숲 공유 하는 그룹에 대 한 리소스를 포함 하 고 공유할 수 있습니다. 초기 구성 숲 라는이 공유 숲 속 조직에 대 한 숲 디자인 모델의 기반입니다.  
  
초기 조직 숲 조직에서 여러 그룹 호스트할 수을 하기 때문에 모든 당사자 예상 되는 결과 이해할 수 있도록 숲 소유자 각 그룹으로 수준 계약 서비스를 설정 해야 합니다. 이 설정에 동의한 서비스 기대 하 여 개별 그룹과 숲 소유자 보호 됩니다.  
  
그렇지 않은 경우 단일 조직 숲 공유할 수 조직에서 그룹의 모든, 다양 한 그룹의 요구에 맞게 숲 디자인 확장 해야 합니다. 이 디자인 요구 사항이 적용 자율성 격리와 제한 된 연결 네트워크 있는지 여부에 필요한 사항에 따라 그룹을 식별 한 다음 해당 요구 사항에 맞게 사용할 수 있는 숲 모델 포함 됩니다. 다음 표에서 숲 디자인 모델 시나리오 자율성, 격리와 연결 요인에 따라 보여 줍니다. 요구에 가장 잘 맞는 숲 디자인 시나리오를 확인 한 후 디자인 사양에 맞게 추가 결정 해야 하는지 확인 합니다.

## <a name="BKMK_1"></a>시나리오 1: 가입 자율성 데이터에 대 한 기존 숲  
조직에서 기존 조직 숲 (Ou)에 있는 그룹 호스트 하는 것 만으로 자율성 데이터에 대 한 요구 사항을 충족 수 있습니다. 데이터 자율성 해당 그룹에서 데이터 관리자에 게 Ou 제어할을 위임 합니다. 제어 Ou을 사용 하 여 위임에 대 한 자세한 내용은 참조 [조직 단위 디자인](../../ad-ds/plan/Creating-an-Organizational-Unit-Design.md)합니다.  
  
## <a name="BKMK_2"></a>시나리오 2: 조직 숲 또는 도메인 자율성 서비스에 대 한 사용  
그룹 조직에서 요구 사항으로 자율성 서비스를 파악 하는 경우이 요구 먼저 재고 하는 것이 좋습니다. 서비스 자율성 달성 관리 오버와 조직에 대해 추가 비용을 만듭니다. 고 것을 요구 사항에 대 한 서비스 자율성의 편의 위해 간단 하 게이 요구 사항을 충족 하는 데 필요한 비용 양쪽 맞춤 수 있는지 확인 합니다.  
  
다음 중 하나를 수행 하 여 자율성 서비스에 대 한 요구 사항을 충족 수 있습니다.  
  
-   조직 숲을 만드는 중입니다. 사용자가 그룹과 서비스 자율성 별도 조직 숲에 필요한 그룹에 대 한 컴퓨터를 놓습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 그룹 조직에서 다른 숲와 리소스를 액세스 또는 공유 하는 경우 자신의 조직 숲 및 기타 숲 간에 신뢰를 설정할 수 있습니다.  
  
-   조직 도메인을 사용 합니다. 사용자가 그룹과 컴퓨터 별도 기존 조직 숲 도메인에 놓습니다. 이 모델 도메인 수준 서비스 자율성만을 위한 제공 하 고, 격리 또는 격리 데이터 서비스 전체 서비스 자율성에 대 한 되지 않습니다.  
  
조직 도메인의 사용에 대 한 자세한 내용은 참조 [조직 도메인 숲 모델을 사용 하 여](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)합니다.  
  
## <a name="BKMK_3"></a>격리 서비스에 대 한 조직 숲 또는 숲 리소스를 사용 하는 시나리오 3:  
다음 중 하나를 수행 하 여 서비스 격리 요구 사항을 충족 수 있습니다.  
  
-   조직 숲을 사용 합니다. 사용자가 그룹과 서비스 격리 별도 조직 숲에 필요한 그룹에 대 한 컴퓨터를 놓습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 그룹 조직에서 다른 숲와 리소스를 액세스 또는 공유 하는 경우 자신의 조직 숲 및 기타 숲 간에 신뢰를 설정할 수 있습니다. 그러나 했습니다 권장 하지 않습니다이 이렇게 하므로 유니버설 그룹을 통해 리소스 액세스 많이 숲 신뢰 시나리오에서 제한 됩니다.  
  
-   리소스 숲을 사용 합니다. 리소스와 서비스 계정을 사용자 계정에서 기존 조직 숲 유지 별도 리소스 숲을 넣습니다. 필요한 경우 암호 확인용 계정에 리소스 숲 리소스에 액세스 리소스 숲 속의 조직 숲 사용할 수 없는 경우 만들 수 있습니다. 암호 확인용 계정 리소스 숲에에 로그인 하 고 리소스 제어 조직 숲 다시 온라인 상태가 될 때까지 유지 하는 데 필요한 기관이 있어야 합니다.  
  
    일반 사용자 계정을 사용 하는 동안 사용자의 숲 속의 리소스에 액세스할 수 있는 보안 리소스와 조직 숲 사이 설정 합니다. 이 구성 조직 숲 사용할 수 없는 경우 리소스 숲 속의 암호 확인용 계정에 대체 되도록 사용자가 허용 하면서 사용자 계정의 중앙 집중식된 관리할 수 있습니다.  
  
고려 격리 서비스에는 다음과 같습니다.  
  
-   숲 서비스 격리 용으로 만들어진 도메인 다른 숲에서 신뢰할 수 있는 하지만 해당 서비스 관리자가 그룹에 다른 숲에서 사용자를 포함 해야 합니다. 사용자가 다른 숲 격리 숲 속의 관리 그룹에 포함 되어, 격리 숲 속의 보안 잠재적으로 손상 될 수 있습니다 숲에서 서비스 관리자에 단독 제어 없기 때문에 합니다.  
  
-   네트워크에 액세스할 수 있는 도메인 컨트롤러를으로 서비스 거부 공격이) (등의 공격을 받기 해당 네트워크에서 악성 소프트웨어 로부터 합니다. 가능성 공격을 방지 하려면 다음을 수행할 수 있습니다.  
  
    -   호스트 도메인 컨트롤러 안전한 것으로 간주 되는 네트워크에 대해서만 합니다.  
  
    -   네트워크 또는 도메인 컨트롤러 호스트 하는 네트워크에 대 한 액세스를 제한 합니다.  
  
-   서비스 격리 추가 숲 생성을 해야합니다. 유지 관리 추가 숲 지원 하기 위해 인프라 비용이 보다 리소스를 사용할 수 없는 조직 숲 때문에 대 한 액세스 손실와 관련 된 비용 여부 평가 합니다.  
  
## <a name="BKMK_4"></a>격리 데이터에 대 한 조직 숲 또는 제한 된 권한 숲을 사용 하는 시나리오 4:  
다음 중 하나를 수행 하 여 격리 데이터를 얻을 수 있습니다.  
  
-   조직 숲을 사용 합니다. 사용자가 그룹과 격리 별도 조직 숲에 데이터를 필요로 하 고 그룹에 대 한 컴퓨터를 놓습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 그룹 조직에서 다른 숲와 리소스를 액세스 또는 공유 하는 경우 조직의 숲 및 기타 숲 간에 보안을 설정 합니다. 분류 정보에 액세스 해야 하는 사용자가 새로운 조직 숲 속의 존재 합니다. 사용자가 신뢰 관계를 사용 하 여 두 분류 데이터에에서 액세스 자체 숲 및 기타 숲에서 분류 데이터를 사용 하는 하나의 계정이 합니다.  
  
-   제한 된 권한 숲을 사용 합니다. 제한 된 데이터 및 해당 데이터에 액세스 하는 데 사용 된 사용자 계정을 포함 하는 별도 숲입니다. 별도 사용자 계정 네트워크에 제한 되지 않은 리소스에 액세스 하는 데 사용 하는 기존 조직 숲에 유지 됩니다. 제한 된 권한 숲와 기업에서 다른 숲 간에 없음 신뢰 만들어집니다. 또한 다른 숲에 연결할 수 없는에 별도 실제 네트워크 숲 배포 하 여 숲을 제한할 수 있습니다. 사용자가 두 워크스테이션 있어야 별도 네트워크에 숲 배포 하는 경우: 하나는 제한 된 숲 및에 액세스 하는 네트워크의 nonrestricted 영역에 대 한 액세스 합니다.  
  
숲 격리 데이터에 대 한를 만들기 위한 고려는 다음과 같습니다.  
  
-   조직 숲 만든 격리 데이터에 대 한 다른 숲에서 도메인 신뢰할 수 있는 되지만 사용자가 다른 숲에서 다음 중 하나를 포함 하지 않아야 합니다.  
  
    -   그룹 관리 서비스 나 서비스 관리자가 그룹의 회원 관리할 수 있는 그룹에 대해 책임이  
  
    -   보호 된 데이터를 저장 하는 컴퓨터 관리 제어 하는 그룹  
  
    -   보호 된 데이터에 대 한 액세스 권한이 있는 그룹 또는 사용자 개체 또는 액세스할 수 있는 그룹 개체를 관리 하는 그룹 보호 되는 데이터  
  
    사용자가 다른 숲 이러한 그룹에 포함 된, 격리 숲 속의 한 손상 하 고 보호 데이터의 공개 다른 숲 속의 한 손상 될 수 있습니다.  
  
-   다른 숲 격리 숲 속의 사용자의 다른 숲 리소스에 액세스할 수 있도록 격리 데이터에 대 한 만든 조직 숲 신뢰를 구성할 수 있습니다. 그러나 격리 숲에서 사용자가 절대 대화형에 로그온 해야 워크스테이션 신뢰 숲 속의 합니다. 신뢰 숲 속의 컴퓨터 악성 소프트웨어에 의해 손상 될 수 있습니다 및 캡처하는 사용자의 로그온 자격 증명을 사용할 수 있습니다.  
  
    > [!NOTE]  
    > 숲 소유자 신뢰 숲 속의 서버 격리 숲에서 사용자가 가장 하 고 격리 숲 속의 리소스에 액세스 하지 못하도록, 위임된 인증을 사용 하지 않도록 설정 하거나 제한 위임 기능을 사용할 수 있습니다. 제한 된 위임 위임된 인증에 대 한 자세한 내용은 참조 인증 위임 ([https://go.microsoft.com/fwlink/?LinkId=106614](https://go.microsoft.com/fwlink/?LinkId=106614)).  
  
-   사용자가 숲 외부에 있는 정보에 대 한 액세스를 제한 하는 조직의 조직 숲와 다른 숲 사이 방화벽을 설정 해야 할 수 있습니다.  
  
-   별도 숲 만드는 호스트 보호 된 정보를 네트워크에 액세스할 수 있는 격리 된 숲 및 컴퓨터에서 도메인 컨트롤러도 격리 데이터를 사용 하면, 있지만 공격 하는 해당 네트워크에서 컴퓨터 시작 적용 됩니다. 공격 가능성이 높습니다 너무 또는 공격이 나 보안 위반 결과 너무 크다는 결정 하는 조직의 네트워크 또는 도메인 컨트롤러를 호스트 하는 네트워크에 대 한 액세스를 제한 하 고 컴퓨터를 호스트 하는 데이터를 보호 합니다. 액세스를 제한 방화벽 및 인터넷 프로토콜 보안 같은 기술을 사용 하 여 편집할 수 있습니다. 최악의 경우 조직 조직에서 다른 네트워크에 연결할 수 없는 물리적 있는 개별 네트워크에서 보호 된 데이터를 유지 선택할 수 있습니다.  
  
    > [!NOTE]  
    > 모든 네트워크 연결이 제한 된 권한 숲 및 다른 네트워크 사이 있으면 가능성 데이터에 다른 네트워크로 전송 되지 제한 된 지역에 있습니다.  
  
## <a name="BKMK_5"></a>조직 숲, 사용 또는 제한 된 연결 방화벽을 다시 구성 시나리오 5:  
제한 된 연결 요구 사항에 맞게를 다음 중 하나를 수행할 수 있습니다.  
  
-   사용자는 기존 조직 숲 들을 한 다음 통과할 수의 Active Directory 교통 수 있도록 충분히 방화벽을 엽니다.  
  
-   조직 숲을 사용 합니다. 사용자가 그룹과 컴퓨터 연결이 제한 되어 그룹에 대 한 별도 조직 숲 넣습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 조직 숲 방화벽의 다른 측면에 별도 환경을 제공합니다. 숲은 사용자 계정 및 리소스는 숲 내에서 관리 되는 포함 하 여 일상적인 작업을 수행 하는 방화벽을 통해 이동 필요는 없습니다. 특정 사용자 또는 응용 프로그램 연락 숲 다른 방화벽을 통해 전달 하는 기능을 필요로 하는 특별 한 요구 사항이 있을 수 있습니다. 이러한 요구 사항을 비롯 하 여 작동 하는 데 어떤 신뢰 하는 데 필요한 방화벽에서 적절 한 인터페이스를 열어 개별적으로 해결할 수 있습니다.  
  
방화벽 사용할 수 있도록 Active Directory Domain Services (AD DS)와 구성에 대 한 자세한 내용은 네트워크에 대 한 Active Directory를 참조 하세요. 방화벽으로 분리 ([https://go.microsoft.com/fwlink/?LinkId=37928](https://go.microsoft.com/fwlink/?LinkId=37928)).  
  
## <a name="BKMK_6"></a>시나리오 6: 조직 숲 또는 도메인을 사용 하 고 다시 구성 자율성 서비스에 대 한 연결이 제한 된  
그룹 조직에서 요구 사항으로 자율성 서비스를 파악 하는 경우이 요구 먼저 재고 하는 것이 좋습니다. 서비스 자율성 달성 관리 오버와 조직에 대해 추가 비용을 만듭니다. 고 것을 요구 사항에 대 한 서비스 자율성의 편의 위해 간단 하 게이 요구 사항을 충족 하는 데 필요한 비용 양쪽 맞춤 수 있는지 확인 합니다.  
  
제한 된 연결 되는 문제에 대 한 서비스 자율성 요구 사항이 있는 경우 다음 중 하나를 수행할 수 있습니다.  
  
-   조직 숲을 사용 합니다. 사용자가 그룹과 서비스 자율성 별도 조직 숲에 필요한 그룹에 대 한 컴퓨터를 놓습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 조직 숲 방화벽의 다른 측면에 별도 환경을 제공합니다. 숲은 사용자 계정 및 리소스는 숲 내에서 관리 되는 포함 하 여 일상적인 작업을 수행 하는 방화벽을 통해 이동 필요는 없습니다. 특정 사용자 또는 응용 프로그램 연락 숲 다른 방화벽을 통해 전달 하는 기능을 필요로 하는 특별 한 요구 사항이 있을 수 있습니다. 이러한 요구 사항을 비롯 하 여 작동 하는 데 어떤 신뢰 하는 데 필요한 방화벽에서 적절 한 인터페이스를 열어 개별적으로 해결할 수 있습니다.  
  
-   사용자가 그룹과 컴퓨터 별도 기존 조직 숲 도메인에 놓습니다. 이 모델 도메인 수준 서비스 자율성만을 위한 제공 하 고, 격리 또는 격리 데이터 서비스 전체 서비스 자율성에 대 한 되지 않습니다. 숲에서 다른 그룹 숲 소유자 신뢰 하는 동일한도 새는 도메인의 서비스 관리자 신뢰 해야 합니다. 이런 이유이 때문에 대 한이 방법을 사용을 권장 하지 않습니다. 조직 도메인의 사용에 대 한 자세한 내용은 참조 [조직 도메인 숲 모델을 사용 하 여](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)합니다.  
  
통과할 수의 Active Directory 교통 수 있도록 충분히 방화벽을 엽니다 해야 할 수도 있습니다. 사용 하기 위해 방화벽 AD DS와 구성에 대 한 자세한 내용은 참조 방화벽으로 분할 된 네트워크에 대 한 Active Directory ([https://go.microsoft.com/fwlink/?LinkId=37928](https://go.microsoft.com/fwlink/?LinkId=37928)).  
  
## <a name="BKMK_7"></a>7 시나리오: 리소스 숲을 사용 하 고 다시 구성 격리 서비스에 대 한 연결이 제한 된  
제한 된 연결 되는 문제에 대 한 서비스 격리 요구 사항이 있는 경우 다음 중 하나를 수행할 수 있습니다.  
  
-   조직 숲을 사용 합니다. 사용자가 그룹과 서비스 격리 별도 조직 숲에 필요한 그룹에 대 한 컴퓨터를 놓습니다. 개별 숲 소유자 해당 그룹에서 할당 합니다. 조직 숲 방화벽의 다른 측면에 별도 환경을 제공합니다. 숲은 사용자 계정 및 리소스는 숲 내에서 관리 되는 포함 하 여 일상적인 작업을 수행 하는 방화벽을 통해 이동 필요는 없습니다. 특정 사용자 또는 응용 프로그램 연락 숲 다른 방화벽을 통해 전달 하는 기능을 필요로 하는 특별 한 요구 사항이 있을 수 있습니다. 이러한 요구 사항을 비롯 하 여 작동 하는 데 어떤 신뢰 하는 데 필요한 방화벽에서 적절 한 인터페이스를 열어 개별적으로 해결할 수 있습니다.  
  
-   리소스 숲을 사용 합니다. 리소스와 서비스 계정을 사용자 계정에서 기존 조직 숲 유지 별도 리소스 숲을 넣습니다. 조직 숲 사용할 수 없는 경우 리소스 숲에 대 한 액세스를 유지할 리소스 숲 속의 몇 가지 다른 사용자 계정을 만들 수 있습니다. 암호 확인용 계정 리소스 숲에에 로그인 하 고 리소스 제어 조직 숲 다시 온라인 상태가 될 때까지 유지 하는 데 필요한 기관이 있어야 합니다.  
  
    일반 사용자 계정을 사용 하는 동안 사용자의 숲 속의 리소스에 액세스할 수 있는 보안 리소스와 조직 숲 사이 설정 합니다. 이 구성 조직 숲 사용할 수 없는 경우 리소스 숲 속의 암호 확인용 계정에 대체 되도록 사용자가 허용 하면서 사용자 계정의 중앙 집중식된 관리할 수 있습니다.  
  
고려 격리 서비스에는 다음과 같습니다.  
  
-   숲 서비스 격리 용으로 만들어진 도메인 다른 숲에서 신뢰할 수 있는 하지만 해당 서비스 관리자가 그룹에 다른 숲에서 사용자를 포함 해야 합니다. 사용자가 다른 숲 격리 숲 속의 관리 그룹에 포함 되어, 격리 숲 속의 보안 잠재적으로 손상 될 수 있습니다 숲에서 서비스 관리자에 단독 제어 없기 때문에 합니다.  
  
-   네트워크에 액세스할 수 있는 도메인 컨트롤러를으로 해당 네트워크에서 컴퓨터의 (예: 서비스 거부 공격이) 공격 적용 됩니다. 가능성 공격을 방지 하려면 다음을 수행할 수 있습니다.  
  
    -   호스트 도메인 컨트롤러 안전한 것으로 간주 되는 네트워크에 대해서만 합니다.  
  
    -   네트워크 또는 도메인 컨트롤러 호스트 하는 네트워크에 대 한 액세스를 제한 합니다.  
  
-   서비스 격리 추가 숲 생성을 해야합니다. 유지 관리 추가 숲 지원 하기 위해 인프라 비용이 보다 리소스를 사용할 수 없는 조직 숲 때문에 대 한 액세스 손실와 관련 된 비용 여부 평가 합니다.  
  
    특정 사용자 또는 응용 프로그램 연락 숲 다른 방화벽을 통해 전달 하는 기능을 필요로 하는 특별 한 요구 사항이 있을 수 있습니다. 이러한 요구 사항을 비롯 하 여 작동 하는 데 어떤 신뢰 하는 데 필요한 방화벽에서 적절 한 인터페이스를 열어 개별적으로 해결할 수 있습니다.  
  
사용 하기 위해 방화벽 AD DS와 구성에 대 한 자세한 내용은 참조 방화벽으로 분할 된 네트워크에 대 한 Active Directory ([https://go.microsoft.com/fwlink/?LinkId=37928](https://go.microsoft.com/fwlink/?LinkId=37928)).
