## <a name="BKMK_1"></a>연결 개체  
연결 개체 대상 도메인 컨트롤러에 원본 도메인 컨트롤러에서 복제 연결 나타내는 Active Directory 개체를입니다. 도메인 컨트롤러 단일 사이트의 회원 이며 Active Directory Domain Services (AD DS)에서 서버 개체로 사이트에 표시 됩니다. 각 서버 개체 NTDS 설정 사이트의 복제 도메인 컨트롤러를 나타내는 개체 자녀를 있습니다.  
  
연결 개체 대상 서버의 NTDS 설정을 개체의 자녀가입니다. 하나의 서버 개체 복제가 두 도메인 컨트롤러 사이 발생 하는 경우에 대 한 다른의 인바인드 복제가 나타내는 연결 있어야 합니다. 도메인 컨트롤러에 대 한 모든 복제 연결 NTDS 설정을 개체 아래 연결 개체도 저장 됩니다. 연결 개체 원본 서버 복제가 식별 복제 일정 있어서 복제 전송 지정 합니다.  
  
기술 일관성 검사 (KCC) 연결 개체 자동으로 만들지만 수동으로 만들 수도 수 있습니다. 연결 개체 KCC 만든 Active Directory 사이트에 표시 되며 서비스 스냅인로 **<automatically generated>** 정상 작동 조건 적절 한 것으로 간주 됩니다. 관리자가 만든 연결 개체 수동으로 연결 개체 만들어집니다 합니다. 만든된 수동으로 연결 개체를 만들 때 관리자가 할당 이름으로 식별 합니다. 수정 하는 경우는 **<automatically generated>** 연결 개체, 관리 수정된 연결 개체로 변환 및 개체 GUID 형태의에 나타납니다. KCC 수동 또는 수정 연결 개체를 변경할 하지 않습니다.  
  
## <a name="BKMK_2"></a>KCC  
KCC 모든 도메인 컨트롤러에서 실행 되 고 복제 토폴로지 Active Directory 숲을 생성 하는 기본 제공 된 프로세스입니다. KCC는 복제가 또는 사이트 (사이트 간) 간에 (내) 사이트 내에서 발생 있는지 여부에 따라 별도 복제 토폴로지에서 만듭니다. KCC 토폴로지 새로운 도메인 컨트롤러, 기존의 도메인 컨트롤러의 제거, 또는 사이트 변경 비용 및 일정을 도메인 컨트롤러 및 일시적으로 사용할 수 없는 도메인 컨트롤러의 움직임 오류 상태에서 추가 지원를 동적으로 조정 합니다.  
  
사이트를 내 쓰기 도메인 컨트롤러 간의 연결이 지연을 큰 사이트의 줄이기 위해 추가 바로 가기 연결 양방향 링에서 항상 정렬 됩니다. 반면에 간 토폴로지 간 연결에 대 한 각 파티션 두 사이트 간의 있으며 일반적으로 바로 가기 연결 하나 즉 트리를 확장 한 계층입니다. 밤나무와 Active Directory 복제 토폴로지 확장에 대 한 자세한 내용은 참조 Active Directory 복제 토폴로지 기술 참조 ([https://go.microsoft.com/fwlink/?LinkID=93578](https://go.microsoft.com/fwlink/?LinkID=93578)).  
  
각 도메인 컨트롤러에서 KCC 정의 다른 도메인 컨트롤러에서 연결 하는 단방향 인바인드 연결 개체 만들어 복제 경로 만듭니다. 도메인 컨트롤러의 같은 사이트, KCC 관리 작업을 수행 하지 않고도 자동으로 연결 개체 만듭니다. 둘 이상의 사이트를 사용 하는 사이트를 간 사이트 링크를 구성 하 고 각 사이트에 있는 단일 KCC 자동으로 만들어집니다 사이트 간의 연결이 합니다.  
  
### <a name="kcc-improvements-for-windows-server-2008-rodcs"></a>Windows Server 2008 Rodc KCC 개선  

Windows Server 2008에서 (RODC) 새로 제공 읽기 전용 도메인 컨트롤러에 맞게 KCC 개선 여러 가지가 있습니다. RODC에 대 한 일반적인 배포 시나리오 지사입니다. 가장 일반적으로이 시나리오에 배포 된 Active Directory 복제 토폴로지-hub-and-spoke 디자인을 분기 도메인 컨트롤러 여러 사이트에서 적은 수의 브리지 서버 허브 사이트에서 복제 하는 위치를 기반으로 합니다.  
  
이 시나리오 RODC 배포할 때의 이점 단방향 복제 것입니다. 브리지 서버 관리 및 네트워크 사용량을 줄일 수 있는에서 복제 필요 하지 않습니다.  
  
그러나 이전 버전의 Windows Server 운영 체제에 허브 스포크 토폴로지별 강조 표시 한 가지 관리 문제 허브에서 새 브리지 도메인 컨트롤러를 추가한 후 없는 자동 메커니즘 재배포 분기 도메인 컨트롤러 허브 도메인 컨트롤러 새 허브 도메인 컨트롤러 활용 하기 위해 간에 복제 연결 하는입니다.  
  
Windows Server 2003 도메인 컨트롤러에 대 한 사용자 수 균형을 다시 조정 작업은 Windows Server 2003 지점 배포 가이드에서 Adlb.exe 등 도구를 사용 하 여 ([https://go.microsoft.com/fwlink/?LinkID=28523](https://go.microsoft.com/fwlink/?LinkID=28523)).  
  
Windows Server 2008 Rodc kcc 일반 기능을 제공 일부 재조정 Adlb.exe 같은 추가적인 도구를 사용 하 여 않아도 합니다. 새로운 기능은 기본적으로 활성화 됩니다. 다음 레지스트리 키 RODC에 대 한 설정 추가 하 여 비활성화할 수 있습니다.  
  
**찾아**  
  
**"임의 BH Loadbalancing 허용."**  
**1 = 0 (기본) 사용 = 사용 안 함**  
  
이러한 KCC 개선 작동 방식에 대 한 자세한 내용은 참조 계획 및 배포 Active Directory Domain Services 지점의 ([https://go.microsoft.com/fwlink/?LinkId=107114 ](https://go.microsoft.com/fwlink/?LinkId=107114)).  
  
## <a name="BKMK_3"></a>(Failover)  
사이트는 복제가 네트워크 오류 및 오프 라인 도메인 컨트롤러 관련 전달가 있는지 확인 합니다. 지정 된 간격으로 새로운 도메인 컨트롤러를 추가할 때 등 AD DS 발생 하는 변경에 대 한 복제 토폴로지 조정 하 고 새 사이트에 KCC 실행 생성 됩니다. KCC는 연결이 작동 하지 않는 경우 확인을 기존 연결 복제 상태를 검토 합니다. 연결이 실패 도메인 컨트롤러 인해 작동 하지 않는 경우 자동으로 kcc 다른 복제 파트너에 게 임시 연결 (사용 가능한 경우) 복제가 발생 수 있도록 합니다. KCC 모든 도메인 컨트롤러에 사이트를 사용할 수 없는 다른 웹 사이트에서 도메인 컨트롤러 사이 복제 연결 자동으로 만들어집니다.  
  
## <a name="BKMK_4"></a>서브넷  
서브넷 논리 IP 주소 집합이 할당 된 TCP/IP 네트워크의 된 부분이입니다. 서브넷 그룹화 컴퓨터가 네트워크에는 물리적 근접을 식별 하는 방식에서입니다. AD DS에서 서브넷 개체 사이트로 컴퓨터 지도 하는 데 사용 되는 네트워크 주소를 확인 합니다.  
  
## <a name="BKMK_5"></a>사이트  
사이트는 매우 안정적이 고 빨리 네트워크 연결을 사용 하나 또는 여러 개의 TCP/IP 서브넷 나타내는 Active Directory 개체 합니다. 사이트 정보 관리자를 Active Directory 액세스 및 복제 실제 네트워크의 사용을 최적화를 구성할 수 있습니다. 사이트 개체 서브넷, 집합와 관련 된 하 고 각 도메인 컨트롤러에 숲 속의 IP 주소 따라 Active Directory 사이트와 연결 됩니다. 사이트에서 둘 이상의 도메인 도메인 컨트롤러 호스트할 수 있습니다 및 도메인 둘 이상의 사이트에 표시할 수 있습니다.  
  
## <a name="BKMK_6"></a>사이트 링크  
사이트 링크는 Active Directory 개체 KCC 복제 Active Directory에 대 한 연결을 사용 하는 논리 경로 표시 하는입니다. 사이트 링크 개체 uniform 비용 지정 된 간 전송 통해 통신할 수 있는 사이트의 집합을 나타냅니다.  
  
사이트 링크 내에 포함 된 모든 사이트 같은 네트워크 종류를 사용 하 여 연결 된 것으로 간주 됩니다. 한 사이트에 도메인 컨트롤러에서 다른 사이트 도메인 컨트롤러에서에서 디렉터리 변경 복제 수 있도록 사이트 링크를 사용 하 여 다른 사이트에 사이트를 수동으로 연결 되어 있어야 합니다. 실제 네트워크의 네트워크 패킷이 복제 하는 동안 촬영 실제 경로 사이트 링크와 일치 하지 않는 하기 때문에 복제 효율성 Active Directory를 개선 하기 위해 중복 사이트 링크를 만들 필요가 없습니다.  
  
두 개의 사이트 사이트 링크를 통해 연결, 복제 시스템 브리지 서버 라고 하는 각 사이트에서 특정 도메인 컨트롤러 간의 연결이 자동으로 만들어집니다. Windows Server 2008에서 브리지 서버로 선택 되 고은 대상 동일한 파티션을 호스트 하는 사이트의 모든 도메인 컨트롤러입니다. 모든 후보 브리지 서버 복제가 작업을 공유 하는 사이트의 KCC 만든 복제 연결 배포 임의로 됩니다. 기본적으로 임의 선택 과정에서 수행 하는 장소 연결 개체 처음에 추가할 때 해당 사이트 한 번만 합니다.  
  
## <a name="BKMK_7"></a>사이트 링크 브리지  
사이트 링크 다리는 일련의 사이트 링크를 나타내는 Active Directory 개체를 볼 사이트가 모두 일반적인 전송을 사용 하 여 통신할 수 있습니다. 사이트 링크 브리지 바로 복제 서로 통신 링크를 사용 하 여 연결 되지 않은 도메인 컨트롤러를 사용 하도록 설정 합니다. 일반적으로 사이트 링크 다리 IP 네트워크 라우터 (또는 라우터 집합)에 해당합니다.  
  
기본적으로 KCC는 일부 사이트 공통 된 모든 사이트 링크를 통해 전이적 경로 구성할 수 있습니다. 이 문제를 사용 하지 않도록 설정 하는 경우 각 사이트 링크는 자체 고유한 및 격리 된 네트워크를 나타냅니다. 단일 경로로 처리 될 수 있는 사이트 링크 집합이 사이트 링크 다리 통해 표시 됩니다. 각 다리는 네트워크 교통량 격리 통신을 나타냅니다.  
  
사이트 링크 다리는 논리적 사이트 간의 전이적 실제 연결을 대표 하는 메커니즘 합니다. 사이트 링크 다리 저렴 경로 해당 사이트를 소유 디렉터리 파티션 interconnect를 결정 포함 된 사이트 링크 조합을 사용 하 여 KCC 수 있게 합니다. 사이트 링크 다리 도메인 컨트롤러에 실제 연결을 제공 하지 않습니다. 사이트 링크 다리 제거 되 면 결합 된 사이트 링크를 통해 복제 KCC 링크를 제거할 때까지 계속 됩니다.  
  
사이트 링크 다리는 필요한 사이트 인접 한 사이트에 있는 도메인 컨트롤러에도 호스팅된 하지 파티션을 호스트 도메인 컨트롤러에 포함 된 경우만 있지만 해당 파티션을 호스트 도메인 컨트롤러는 하나 이상의의 다른 사이트 숲에에서 있습니다. 인접 한 사이트에 단일 사이트 링크를 포함 두 개 이상의 사이트도 정의 됩니다.  
  
사이트 링크 다리 두 사이트 링크, 중간 사이트를 사용 하 여 두 끊어져 사이트 간의 전이적 경로 제공 간의 논리 연결을 만듭니다. 에 다리는 간 토폴로지 generator (ISTG)의 목적을 위해 중간 사이트를 사용 하 여 실제 연결을 의미 합니다. 에 다리는 도메인 컨트롤러 중간 사이트에는 복제 경로 제공 하는 것을 의미 하지 않습니다. 그러나이 것 경우가 중간 사이트 복제할 파티션을 호스트 하는 도메인 컨트롤러 포함 되어 있는 경우,이 경우 사이트 링크 다리 필요 하지 않습니다.  
  
각 사이트 링크 비용 결과 경로 적은지 비용을 만드는 추가 됩니다. 중간 사이트 디렉터리 파티션을 호스트 도메인 컨트롤러 포함 되어 있지 않습니다 하 고 낮은 비용 링크 존재 하지 않는 경우 사이트 링크 다리 사용 됩니다. 중간 사이트 디렉터리 파티션을 호스트 하는 도메인 컨트롤러, 포함 두 끊어져 사이트 중간 도메인 컨트롤러 복제 연결을 설정 하 고에 다리 사용 하지 것입니다.  
  
## <a name="BKMK_8"></a>사이트 링크 전이성  
기본적으로 모든 사이트 링크는 전이적 또는 "브리지." 사이트 링크는 브리지 하 고 겹치는 일정을 KCC 복제 연결 사이트 사이트 직접 사이트 링크를 통해 연결 되지 않은 있지만 일반적인 사이트를 통해 전이적 연결 되어 있는 도메인 컨트롤러 복제 파트너 결정을 만듭니다. 즉, 모든 사이트를 조합해 사이트 링크를 통해 다른 사이트에 연결할 수 있습니다.  
  
일반적으로 완벽 하 게 라우팅된 네트워크에 대 한 사용자 하지 만들 필요가 사이트 링크 다리 복제 변경 흐름을 제어 하는 경우가 있습니다. 네트워크는 완전히 라우팅되지 사이트 링크 다리 임파서블 복제 시도 하지 만들어야 합니다. 특정 전송에 대 한 모든 사이트 링크 암시적 해당 전송에 대 한 단일 사이트 링크 다리에 속해 있습니다. 사이트 연결에 대 한 기본 브리지 자동으로 발생 하 고 없음 Active Directory 개체가 다리 나타냅니다. **모든 사이트 링크를 줄이는** (SMTP) 간 transport 컨테이너 IP 및 SMTP(Simple Mail Transfer Protocol) 속성 발견 설정을 자동 사이트 링크 브리지 구현 합니다.  
  
> [!NOTE]  
> SMTP 복제 AD DS;의 향후 버전에서 지원 될 것 따라서 사이트 링크 개체 SMTP 컨테이너에서 만드는 권장 되지 않습니다.  
  
## <a name="BKMK_9"></a>드 서버  
드 서버는 응용 프로그램 요청한 데이터를 저장 하는 특정 도메인 컨트롤러에 참조 없이 AD DS 검색할 수 있도록는 숲 속의 모든 개체에 대 한 정보를 저장 하는 도메인 컨트롤러입니다. 모든 도메인 컨트롤러 등 드 서버 스키마 및 구성 파티션의 완전 한 쓸 수 복제본 및 도메인 디렉터리 파티션 호스트 하는 도메인에 대 한 전체, 쓸 수 복제 저장 합니다. 또한 드 서버에서 숲 속의 다른 모든 도메인의 일부 읽기 전용 복제본을 저장합니다. 일부 읽기 전용 도메인 복제본 모든 개체 도메인 있지만 일부 특성 (개체 검색에 대해 가장 자주 사용 되는 특성)에 포함 됩니다.  
  
## <a name="BKMK_10"></a>범용 그룹 구성원 캐싱  
범용 그룹 구성원 캐싱 도메인 컨트롤러를 사용자에 대 한 캐시 범용 그룹 구성원 정보 수 있게합니다. Active Directory 사이트 및 서비스를 사용 하 여 Windows Server 2008 유니버설 그룹 구성원 캐시를 실행 하는 도메인 컨트롤러를 사용 하도록 설정할 수 있습니다.  
  
범용 그룹 구성원 캐싱 활성화 도메인 네트워크 대역폭 사용량이 최소화 하는 도메인 컨트롤러에서 모두 숲에 있는 개체 복제 필요가 없는 하기 때문에 모든 사이트에 드 서버에 대 한 필요가 없습니다. 또한 인증 도메인 컨트롤러 항상 필요가 없습니다 액세스 드 유니버설 그룹 구성원 정보를 얻을 수 있으므로 로그온 시간을 줄입니다. 범용 그룹 구성원 캐싱를 사용 하는 경우에 대 한 자세한 내용은 참조 [글로벌 카탈로그 서버 배치 계획](../../../ad-ds/plan/Planning-Global-Catalog-Server-Placement.md)합니다.
