# <a name="the-role-of-claim-rules"></a>클레임은 규칙의 역할
Active Directory Federation Services \(AD FS\)에 Federation 서비스의 전체 기능은 클레임의 집합을 포함 한 토큰 드릴 것입니다. 어떤 클레임 ADFS 수락 하 고 다음 문제에 대 한 결정 클레임 규칙 적용 됩니다.  
  
## <a name="what-are-claim-rules"></a>클레임은 규칙 이란 무엇 인가요?  
클레임은 규칙 비즈니스 논리를 하나 또는 여러 개의 들어오는 클레임, 조건이 적용 됩니다 인스턴스를 나타냅니다 \ (경우 다음 y\ x) 조건에 따라 보내는 클레임 하나 이상 생성 하 고 있습니다. 송수신 청구에 대 한 자세한 내용은 참조 [The 역할 클레임](The-Role-of-Claims.md)합니다.  
  
클레임 파이프라인 통해 흐름을 제어 하는 비즈니스 논리가 구현 해야 할 때 클레임 규칙을 사용 합니다. 클레임 파이프라인은의 더 많은 논리 개념 동안 end\ to\ 엔드 프로세스 흐르는 대 한 주장 클레임 규칙은 실제 관리 요소 클레임 발급 프로세스를 통해 클레임 흐름을 사용자 지정 하는 데 사용할 수 있는 합니다.  
  
클레임 파이프라인에 대 한 자세한 내용은 참조 [클레임 엔진 The 역할](The-Role-of-the-Claims-Engine.md)합니다.  
  
클레임 규칙은 다음 이점이 있습니다.  
  
-   청구 공급자 로부터 신뢰 청구에 대 한 논리가 run\ 시간 비즈니스 적용 하려면 관리자 제공  
  
-   관리자는 어떤 클레임 신뢰 자에 게 배포 정의 메커니즘을 제공  
  
-   허용 또는 특정 사용자에 대 한 액세스 거부 하려면 관리자 풍부 하 고 자세한 claims\ 기반 인증 기능을 제공 합니다.  
  
### <a name="how-claim-rules-are-processed"></a>클레임은 규칙은 처리 하는 방법  
사용 하 여 클레임 파이프라인 클레임 규칙 처리 되는 *클레임 엔진*합니다. 클레임 엔진 Federation 서비스는 사용자가 제공 들어오는 클레임 세트를 검사 하 논리 구성 요소 이며, 각 규칙에 대 한 논리를 따라 그러면 출력 일련의 요청 합니다.  
  
함께 클레임 규칙 엔진과 지정된 연합된 보안와 관련 된 규칙 또는 있는지 확인 들어오는 클레임 해야 수은 통과, 특정 조건 조건을 충족 하기 위해 필터링 Federation 서비스에서 보내는 클레임으로 발급 전 클레임은 완전히 새로운 집합으로 변환 클레임 설정 합니다.  
  
이 프로세스에 대 한 자세한 내용은 참조 [클레임 엔진 The 역할](The-Role-of-the-Claims-Engine.md)합니다.  
  
## <a name="what-are-claim-rule-templates"></a>클레임은 규칙 템플릿 이란 무엇 인가요?  
ADFS 미리 정의 된 집합이 클레임 규칙 수 있도록 쉽게 설계 된 템플릿 선택 하 고 특정 비즈니스 요구에 가장 적합 한 청구 규칙을 만들 포함 되어 있습니다. 주장 규칙 템플릿 클레임 규칙 작성 과정 중에 사용 됩니다.  
  
Snap\에서 광고 FS 관리에 규칙 만들 수만 클레임 규칙 템플릿을 사용 하 여 합니다. 를 사용한 후에 snap\ 클레임 규칙 템플릿을 선택 규칙 논리를 위해 필요한 데이터를 입력 하 고 구성 데이터베이스에 저장 하는 것이 \ (출처: 해당 시점 forward\)에서 참조 되는 UI 클레임 일반적으로 합니다.  
  
### <a name="how-claim-rule-templates-work"></a>어떻게 규칙 템플릿 작업 주장  
언뜻 보기에 청구 규칙 템플릿 기능으로 제공 되는 snap\ 들어오는 청구에 대 한 데이터와 프로세스 특정 논리 수집을 바로 입력된 양식 것 같습니다. 그러나 훨씬 더 자세한 수준 필요한 주장 언어 깊숙히 알고를 않고도 규칙을 신속 하 게 만드는 데 필요한 기본 논리를 구성 하는 규칙 언어 프레임 워크 규칙 템플릿 스토어를 청구 합니다.  
  
사용자 인터페이스에서 제공 되는 각 템플릿을 \(UI\) 가장 일반적으로 필수 관리 작업을 기반 미리 입력 된 청구 규칙 언어 구문을 나타냅니다. 그러나 하나의 규칙 템플릿을는 예외입니다. 이 서식 지정 규칙 템플릿으로 이라고 합니다. 이 템플릿으로 구문 없음 미리 입력 합니다. 대신 클레임 규칙 언어 구문을 사용 하 여 청구 규칙 템플릿 양식 본문에서 클레임 규칙 언어 구문을 작성 직접 해야 합니다.  
  
클레임 규칙 언어 구문을 사용 하는 방법에 대 한 자세한 내용은 참조 [주장 규칙 언어의 The 역할](The-Role-of-the-Claim-Rule-Language.md) AD FS 배포 가이드에 있습니다.  
  
> [!TIP]  
> 클릭 하 여 언제 든 지 규칙와 관련 된 청구 규칙 언어를 볼 수는 **보기 규칙 언어** 속성 클레임 규칙의 단추.  
  
### <a name="how-to-create-a-claim-rule"></a>클레임은 규칙 만드는 방법  
클레임은 규칙 Federation 서비스 내에서 각 연합된 신뢰 관계에 대 한 별도 생성 되 고 여러 신뢰에서 공유 되지 않습니다. 클레임 규칙 템플릿으로 규칙 만들, 클레임 규칙 언어를 사용 하 여 규칙 작성 처음부터 시작 하거나 Windows PowerShell 사용 규칙을 사용자 지정할 수 있습니다.  
  
이러한 옵션을 모두 함께 특정된 시나리오에 대 한 적절 한 방법을 선택 유연성을 제공 하 사용 합니다. 클레임은 규칙 만드는 방법에 대 한 자세한 내용은 참조 [주장 규칙 구성](https://technet.microsoft.com/library/ee913571.aspx) 광고 FSDeployment 가이드에 있습니다.  
  
#### <a name="using-claim-rule-templates"></a>클레임은 규칙 템플릿을 사용  
주장 규칙 템플릿 클레임 규칙 작성 과정 중에 사용 됩니다. 청구 규칙을 만들려면 다음과 같은 템플릿을 중 하나를 사용할 수 있습니다.  
  
-   통과 또는 수신 클레임 필터링  
  
-   수신 클레임 변형  
  
-   LDAP 특성 클레임으로 보내기  
  
-   클레임으로 그룹 구성원 보내기  
  
-   클레임 사용자 지정 규칙을 사용 하 여 보내기  
  
-   허용 하거나 사용자가 수신 클레임에 따라 거부  
  
-   모든 사용자가 허용  
  
이러한 각를 설명 하는 자세한 정보에 대 한 주장 규칙 템플릿 참조 하십시오 [종류의 주장 규칙 템플릿을 사용 결정](Determine-the-Type-of-Claim-Rule-Template-to-Use.md)합니다.  
  
#### <a name="using-the-claim-rule-language"></a>클레임은 규칙 언어를 사용 하 여  
표준 클레임 규칙 템플릿도 범위를 벗어나는 있는 비즈니스 규칙에 대 한 일련의 복잡 한 논리 조건 클레임 규칙 언어를 사용 하 여 표현 하 사용자 지정 규칙 서식을 사용할 수 있습니다. 사용자 지정 규칙 사용에 대 한 자세한 내용은 참조 [사용자 지정 클레임 규칙을 사용 하는 경우](When-to-Use-a-Custom-Claim-Rule.md)합니다.  
  
#### <a name="using-windows-powershell"></a>Windows PowerShell를 사용 하 여  
만들거나 ADFS 대로 규칙 관리할를 Windows PowerShell cmdlet ADFSClaimRuleSet 개체를 사용할 수 있습니다. Windows PowerShell이 cmdlet이를 사용 하는 방법에 대 한 자세한 내용은 참조 [광고 FS 관리를 Windows PowerShell](https://go.microsoft.com/fwlink/?LinkID=179634)합니다.  
  
## <a name="what-is-a-claim-rule-set"></a>클레임은 규칙 집합 란?  
다음 그림와 같이 클레임 규칙 집합은 클레임 규칙 엔진에서 클레임을 처리 하는 방법을 정의 하는 특정된 연합된 보안에 대 한 하나 또는 여러 개의 규칙의 그룹입니다. 수신 클레임 Federation 서비스에서 수신 하는 경우 청구 규칙 엔진 적절 한 청구 규칙 집합으로 지정 된 논리가 적용 됩니다. 최종 총 전체에서 지정된 보안에 대 한 청구 발생 하는 방법을 결정 하는 설정에서 각 규칙에서 논리입니다.  
  
![광고 FS 역할](media/adfs2_claimruleset.gif)  
  
클레임은 규칙 해당된 규칙 설정 내에서 시간 순서로 클레임 엔진에서 처리 됩니다. 이 순서는 하나의 규칙 출력 집합에 다음 규칙을 입력으로 사용할 수 있으므로 중요 합니다.  
  
## <a name="what-are-claim-rule-set-types"></a>종류를 설정 클레임 규칙 이란 무엇 인가요?  
청구 규칙 종류를 설정의 해커나 신뢰와 관련 된 청구 규칙 집합 클레임 발급, 인증 또는 수용에 사용할 수 있는지 여부를 식별 하는 연결 된 보안 된 논리 부분이입니다. 각 연합된 신뢰 수 있는 하거나 더 클레임 규칙 하는 데 사용 되는 보안 유형에 따라 것과 관련 된 종류를 설정 합니다.  
  
다음 표에서 다양 한 유형의 클레임 규칙 집합 및 청구 공급자 신뢰 또는 신뢰 파티 신뢰와의 관계를 설명 합니다.  
  
|클레임은 규칙 종류를 설정|설명|사용|  
|-----------------------|---------------|-----------|  
|승인 변환 규칙 설정|일련의 클레임 규칙 특정에 사용 하는 공급자 신뢰 클레임 공급자 조직에서 허용 되는 들어오는 클레임 및 신뢰 파티 신뢰를 보낼 수 있는 보내는 클레임 지정를 요구 합니다.<br /><br />원본이 규칙 설정 하는 데 사용 될 들어오는 클레임 클레임 공급자 조직에 지정 된 대로 발급 변환 규칙에 의해 출력 청구 됩니다.<br /><br />기본적으로 클레임 공급자 신뢰 노드 포함 라는 클레임 공급자 신뢰 **Active Directory** 수용 변환 규칙 집합에 대 한 소스 특성 저장소를 나타내는 데 사용 되는 합니다. 이 보안 개체 Federation 서비스에서 연결 네트워크의 Active Directory 데이터베이스를 나타내는 데 사용 됩니다. 이 기본 신뢰 Active Directory로 인증 된 사용자에 대 한 주장 처리 무엇 이며 삭제할 수 없습니다.|클레임 제공자 신뢰|  
|발급 변환 규칙 설정|당사자에 발생 하는 클레임 지정를 신뢰 파티 신뢰를 사용 하는 클레임 규칙의 설정 합니다.<br /><br />원본이 규칙 설정 하는 데 사용 될 들어오는 클레임 처음 클레임 출력 수용 변환 규칙 됩니다 합니다.|파티 신뢰 사용|  
|발급 승인 규칙 설정|당사자에 대 한 토큰 받으려는 허용 사용자 지정 하려면 신뢰 파티 신뢰를 사용 하는 클레임 규칙의 설정 합니다.<br /><br />이 규칙 사용자 클레임 당사자에 대 한 당사자, 따라서 액세스를 받을 수 있는지 여부를 결정 합니다.<br /><br />발급 승인 규칙 지정 하지 않는 한 모든 사용자가 액세스할 수 기본적으로 합니다.|파티 신뢰 사용|  
|위임 인증 규칙 설정|당사자에 다른 사용자에 대 한 대리인으로 작동 하도록 허용 하는 사용자 지정 하려면 신뢰 파티 신뢰에서 사용 하는 클레임 규칙의 설정 합니다.<br /><br />이 규칙 요청 자가 여전히 당사자로 전송 된 토큰에서 요청 자가 식별 하는 동안 사용자 가장 수 있는지 여부를 결정 합니다.<br /><br />발급 승인 규칙 지정 하지 않은 경우 사용자가 기본적으로 대리자로 작동할 수 있습니다.|파티 신뢰 사용|  
|가장 인증 규칙 설정|Windows PowerShell를 사용 하 여 사용자 하는지 여부를 결정 구성 규칙 완벽 하 게 수 클레임 집합 가장 당사자에 다른 사용자 합니다.<br /><br />이 규칙 요청 자가 사용자를 파악 당사자로 전송 된 토큰에서 요청 자가 하지 않고 가장 수 있는지 여부를 결정 합니다.<br /><br />이러한 방식으로 사용자를 다른 가장 당사자는 사용자가 가장 알지 강력한 기능을입니다.|파티 신뢰 사용|  
  
조직에서 사용 하 여 적절 한 청구 규칙 선택에 대 한 자세한 내용은 참조 [종류의 주장 규칙 템플릿을 사용 결정](Determine-the-Type-of-Claim-Rule-Template-to-Use.md)합니다.
