# <a name="access-control-policies-in-windows-server-2016-ad-fs"></a>Windows Server 2016 광고 FS에서 액세스 제어 정책

>Windows Server 2016 적용 됩니다.

  
## <a name="access-control-policy-templates-in-ad-fs"></a>Adfs의 액세스를 제어 정책 템플릿  
이제 Active Directory Federation Services 액세스 제어 정책 템플릿 사용을 지원합니다.  관리자는 액세스를 제어 정책 템플릿을 사용 하 여 정책 템플릿을 신뢰 당사자 (RPs) 그룹으로 지정 하 여 정책 설정을 적용할 수 있습니다. 관리자 정책 템플릿을에 업데이트를 수행할 수 고 변경 내용을 적용 됩니다 신뢰 당사자에 자동으로 필요한 사용자 상호 작용 없는 경우입니다.  
  
## <a name="what-are-access-control-policy-templates"></a>액세스를 제어 정책 템플릿 이란 무엇 인가요?  
정책 처리 하기 위해 ADFS 코어 파이프라인에는 세 단계: 인증 하 고 클레임 발급 합니다. 현재 ADFS 관리자 별도로 정책을 각 이러한 단계를 구성 해야 합니다.  이 정책의 의미를 이해이 포함 됩니다 수 있고이 정책 간 종속성 합니다. 또한 클레임 규칙 언어와 작가 사용자 지정 규칙 (예: 일부 간단한/공통 정책을 사용 하는 파악 해야 할 관리자 차단 외부 액세스).  
  
관리자를 사용 하 여 발급 승인 규칙 구성 해야 할 위치이 이전 모델 템플릿 수행 어떤 액세스 제어 정책 바꾸기가 언어를 요구 합니다.  오래 된 PowerShell cmdlet 발급 승인 규칙의 적용 이지만 상호 새 모델과 제외 합니다. 관리자 새 모델 또는 이전 모델을 사용 하 여 중 하나를 선택할 수 있습니다.  새 모델 관리자를 다단계 인증 적용 포함 하 여 액세스 권한을 부여 하는 시기를 제어할 수 있습니다.  
  
액세스를 제어 정책 템플릿 허용 모델을 사용 합니다.  이 따라서 기본적으로 아무도 액세스 및 액세스 명시적으로 허가 해야 합니다.  그러나 모든 것이 아닙니다 또는 것을 허용 합니다.  관리자 허용 규칙에 대 한 예외를 추가할 수 있습니다.  예를 들어, 관리자가이 옵션을 선택 하 고 IP 주소 범위를 지정 하 여 특정 네트워크에 따라 액세스 권한을 부여 할 수도 있습니다.  그러나 관리자 추가할 수 예외 관리자 예를 들어 예외 특정 네트워크에서 추가 하 고 IP 주소 범위 해당 지정 수 있습니다.
