# <a name="administer-software-restriction-policies"></a>소프트웨어 제한 정책을 관리합니다

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

이 항목 IT 전문가 위한 소프트웨어 제한 정책 (소매가) 시작을 사용 하 여 Windows Server 2008 및 Windows Vista와 응용 프로그램 제어 정책을 관리 하는 방법 절차에 포함 되어 있습니다.

## <a name="introduction"></a>소개
소프트웨어 제한 정책 (소매가)은 도메인에 있는 컴퓨터에서 실행 중인 소프트웨어 프로그램으로 식별 하 고 이러한 프로그램을 실행 하는 기능을 제어 하는 그룹 정책 기반 기능입니다. 제한 정책이 소프트웨어를 사용 하 여 매우 제한적된 구성을 식별된 응용 프로그램을 실행 하면 컴퓨터에 대 한 만들 수 있습니다. 이러한 그룹 정책을 Microsoft Active Directory Domain Services와 통합만 독립 실행형 컴퓨터에를 구성할 수도 있습니다. 소매가 대 한 자세한 내용은 참조는 [소프트웨어 제한 정책](software-restriction-policies.md)합니다.

Windows Server 2008 R2 및 Windows 7부터, Windows AppLocker에 사용할 수 대신 또는 소매가 협력 제어 전략 응용 프로그램의 일부입니다.

이 항목을 포함 되어 있습니다.

-   [제한 정책 소프트웨어 열기](#BKMK_Open_SRP)

-   [새 소프트웨어 제한 정책 만들려면](#BKMK_Create_SRP)

-   [지정 된 파일 형식 추가 또는 삭제 하려면](#BKMK_Add_Del)

-   [소프트웨어 제한 정책이 로컬 관리자에 게 적용 하지 않도록 하려면](#BKMK_Prevent_Admin)

-   [제한 정책 소프트웨어의 기본 보안 수준을 변경 하려면](#BKMK_Sec_Lvl)

-   [Dll 소프트웨어 제한 정책이 적용 하려면](#BKMK_Apply_SRP_DLLs)

소매가 사용 하 여 특정 작업을 수행 하는 방법에 대 한 정보를 참조 하십시오.

-   [거부 허용 목록 및 응용 프로그램 재고 제한 정책 소프트웨어에 대 한 확인](determine-allow-deny-list-and-application-inventory-for-software-restriction-policies.md)

-   [메일 바이러스 로부터 컴퓨터를 보호 하기 위해 제한 정책이 소프트웨어를 사용 하 여](use-software-restriction-policies-to-help-protect-your-computer-against-an-email-virus.md)

-   [도메인에 대 한 사이트 또는 조직 구성원 서버의 또는 작업할 도메인에 가입 워크스테이션](#BKMK_2)

-   [조직 또는 도메인에 대 한 장치를 있고 또는 설치 원격 서버 관리 도구 된 워크스테이션 도메인 컨트롤러에](#BKMK_3)

### <a name="BKMK_1"></a>로컬 컴퓨터에 대 한

1.  로컬 보안 설정을 엽니다.

2.  콘솔 트리에서 클릭 **소프트웨어 제한 정책**합니다.

    **어디에 있나요?**

    -   보안 설정/소프트웨어 제한 정책

> [!NOTE]
> 이 절차를 수행 하려면 로컬 컴퓨터에서 관리자가 그룹의 회원 해야 하거나 받아야 적절 한 권한이 위임 합니다.

### <a name="BKMK_2"></a>도메인에 대 한 사이트 또는 조직 구성원 서버의 또는 작업할 도메인에 가입 워크스테이션

1.  Microsoft Management Console MMC ()을 엽니다.

2.  에 **파일** 메뉴를 클릭 **스냅인 추가/제거**을 차례로 클릭 하 고 **추가**합니다.

3.  클릭 **로컬 그룹 정책 개체 편집기**을 차례로 클릭 하 고 **추가**합니다.

4.  **그룹 정책 개체 선택**, 클릭 **찾아보기**합니다.

5.  **그룹 정책 개체 찾아보려면**에서 적절 한 도메인 사이트나 조직을 그룹 정책 개체 (GPO)을 선택-또는 새 앨범 만들기 마우스 클릭 한 다음 **완료**합니다.

6.  클릭 **닫기**을 차례로 클릭 하 고 **확인**합니다.

7.  콘솔 트리에서 클릭 **소프트웨어 제한 정책**합니다.

    **어디에 있나요?**

    -   *그룹 정책 개체* [*ComputerName*] 정책을/컴퓨터 구성 또는

        사용자 구성/Windows 설정/보안 소프트웨어 설정/제한 정책

> [!NOTE]
> 이 절차를 수행 하려면 관리자 도메인 그룹의 회원 이어야 합니다.

### <a name="BKMK_3"></a>조직 또는 도메인에 대 한 장치를 있고 또는 설치 원격 서버 관리 도구 된 워크스테이션 도메인 컨트롤러에

1.  그룹 정책 Management Console을 엽니다.

2.  콘솔 트리에서 소프트웨어에 대 한 제한 정책 열려는 하는 그룹 정책 개체 (GPO)을 마우스 오른쪽 단추로 클릭 합니다.

3.  클릭 **편집** 편집할 GPO를 열 수 있습니다. 클릭 해도 **새로** 만들기를 누른 다음 **편집**합니다.

4.  콘솔 트리에서 클릭 **소프트웨어 제한 정책**합니다.

    **어디에 있나요?**

    -   *그룹 정책 개체* [*ComputerName*] 정책을/컴퓨터 구성 또는

        사용자 구성/Windows 설정/보안 소프트웨어 설정/제한 정책

> [!NOTE]
> 이 절차를 수행 하려면 관리자 도메인 그룹의 회원 이어야 합니다.

### <a name="BKMK_4"></a>사이트를 있고 또는 설치 원격 서버 관리 도구 된 워크스테이션 도메인 컨트롤러에 대 한

1.  그룹 정책 Management Console을 엽니다.

2.  콘솔 트리에서 대 한 그룹 정책 설정 하려면 해당 사이트를 마우스 오른쪽 단추로 클릭 합니다.

    **어디에 있나요?**

    -   Active Directory 사이트 및 서비스 [*Domain_Controller_Name.Domain_Name*] / 사이트/사이트

3.  항목을 클릭 **그룹 정책 개체 연결** 기존 그룹 정책 개체 (GPO)을 선택한 다음에 **편집**합니다. 클릭 해도 **새로** 만들기를 누른 다음 **편집**합니다.

4.  콘솔 트리에서 클릭 **소프트웨어 제한 정책**합니다.

    **여기서**

    -   *그룹 정책 개체* [*ComputerName*] 정책을/컴퓨터 구성 또는

        사용자 구성/Windows 설정/보안 소프트웨어 설정/제한 정책

> [!NOTE]
> -   이 절차를 수행 하려면 로컬 컴퓨터에서 관리자가 그룹의 회원 해야 하거나 받아야 적절 한 권한이 위임 합니다. 컴퓨터가 도메인에 가입, 도메인 관리자 그룹의 회원이이 절차를 수행 수 있습니다.
> -   로그온 한 사용자 수를 관계 없이 컴퓨터에 적용할 수 정책 설정을 지정 하려면 클릭 **컴퓨터 구성**합니다.
> -   로그인 하는 컴퓨터에 관계 없이 사용자에 게 적용 되는 정책 설정을 지정 하려면 클릭 **사용자 구성**합니다.

## <a name="BKMK_Create_SRP"></a>새 소프트웨어 제한 정책 만들려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  에 **알림** 메뉴를 클릭 **새 소프트웨어 제한 정책**합니다.

> [!WARNING]
> -   관리자 자격 증명 다른 귀하의 환경에 따라이 절차를 수행 해야 하는 다음과 같습니다.
> 
>     -   로컬 컴퓨터에 대 한 새 소프트웨어 제한 정책 만들면: 로컬의 회원 **관리자** 는이 절차를 수행 하는 데 필요한 최소 그룹 또는 오른쪽 단추를 클릭 합니다.
>     -   도메인에 가입 하는 컴퓨터에 대 한 새로운 소프트웨어 제한 정책을 만들 관리자 도메인 그룹의 회원이이 절차를 수행할 수 있습니다.
> -   제한 정책이 소프트웨어에 대 한 그룹 정책 개체 (GPO), 만든 이미 있는 경우는 **새 소프트웨어 제한 정책** 명령이에 표시 되지 않는 **작업** 메뉴 합니다. 하,의 콘솔 트리에서 적용 되는 소프트웨어 제한 정책을 삭제를 마우스 오른쪽 단추로 클릭 **소프트웨어 제한 정책**을 차례로 클릭 하 고 **소프트웨어 제한 정책 삭제**합니다. GPO 제한 정책을 소프트웨어를 삭제 하면 해당 GPO에 대 한 모든 소프트웨어가 제한 정책 규칙 삭제 합니다. 제한 정책을 소프트웨어를 삭제 하면 해당 GPO에 대 한 새 소프트웨어 제한 정책을 만들 수 있습니다.

## <a name="BKMK_Add_Del"></a>지정 된 파일 형식 추가 또는 삭제 하려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  세부 정보 창에서 두 번 클릭 **지정 된 파일 형식**합니다.

3.  다음 중 하나를 수행 합니다.

    -   파일 형식에 추가 하려면 **파일 이름 확장명**, 파일 이름 확장명을 입력 한 다음 **추가**합니다.

    -   파일 형식에 삭제 **지정 된 파일 형식**파일 형식의 클릭 한 다음 **제거**합니다.

> [!NOTE]
> -   관리자 자격 증명 다른 환경에 추가 하거나 삭제 지정된 된 파일 형식에 따라이 절차를 수행 해야 하는 다음과 같습니다.
> 
>     -   추가 또는 로컬 컴퓨터에 대 한 지정된 된 파일 형식을 삭제 하는 경우: 로컬의 회원 **관리자** 는이 절차를 수행 하는 데 필요한 최소 그룹 또는 오른쪽 단추를 클릭 합니다.
>     -   도메인에 가입 하는 컴퓨터에 대 한 새로운 소프트웨어 제한 정책을 만들 관리자 도메인 그룹의 회원이이 절차를 수행할 수 있습니다.
> -   이미 수행 하지 않은 경우 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수 있습니다.
> -   지정 된 파일 형식 목록은 GPO에 대 한 모든 규칙 컴퓨터 구성 및 사용자 구성 공유 됩니다.

## <a name="BKMK_Prevent_Admin"></a>소프트웨어 제한 정책이 로컬 관리자에 게 적용 하지 않도록 하려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  세부 정보 창에서 두 번 클릭 **집행**합니다.

3.  아래에서 **다음 사용자에 게 소프트웨어가 제한 정책이 적용**, 클릭 **로컬 관리자를 제외한 모든 사용자에 게**합니다.

> [!WARNING]
> -   로컬의 회원 **관리자** 는이 절차를 수행 하는 데 필요한 최소 그룹 또는 오른쪽 단추를 클릭 합니다.
> -   이미 수행 하지 않은 경우 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수 있습니다.
> -   일반적으로 사용자가 자신의 조직에서 컴퓨터에 로컬 관리자가 그룹의 회원 수 있으면 하지 못하게 하려는이 옵션을 사용할 수 있습니다.
> -   로컬 컴퓨터에 대 한 제한 정책을 소프트웨어를 정의 하는 경우이 절차를 사용 하 여 로컬 관리자 소프트웨어 제한 정책이 적용 하는 데 하지 못하도록 합니다. 네트워크에 대 한 제한 정책을 소프트웨어를 정의 하는 경우 사용자 정책 설정을 구성원 그룹 정책을 통해 보안 그룹에 따라를 필터링 합니다.

## <a name="BKMK_Sec_Lvl"></a>제한 정책 소프트웨어의 기본 보안 수준을 변경 하려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  세부 정보 창에서 두 번 클릭 **보안 수준**합니다.

3.  마우스 오른쪽 단추로 클릭 한 다음 한를 기본 디바이스로 설정 보안 수준을 **기본값으로 설정**합니다.

> [!CAUTION]
> 특정 디렉터리의 설정 기본 보안 수준이 **허용 안 함** 운영 체제 저하 될 수 있습니다.

> [!NOTE]
> -   관리자 자격 증명 다른 기본 보안 수준이 소프트웨어 제한 정책의 변경한 환경에 따라이 절차를 수행 하 필요 합니다.
> -   이미 수행 하지 않은 경우이 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수 있습니다.
> -   세부 정보 창에서 현재 기본 보안 수준이에 확인 표시가 있는 원 검은색으로 표시 됩니다. 현재 기본 보안 수준이 마우스 오른쪽 단추로 클릭 하는 경우는 **기본값으로 설정** 명령이 메뉴에 표시 되지 않습니다.
> -   소프트웨어 제한 정책 규칙 기본 보안 수준이에 대 한 예외를 지정 하려면 만들어집니다. 기본 보안 수준이로 설정 된 경우 **제한 없음**, 규칙 실행 하도록 허용 되지 않은 소프트웨어를 지정할 수 있습니다. 기본 보안 수준이로 설정 된 경우 **허용 안 함**, 규칙 실행 하도록 허용 하는 소프트웨어를 지정할 수 있습니다.
> -   시스템에 모든 파일에 제한 정책 소프트웨어 기본 보안 수준이 즉시 켜기로 설치 시 **제한 없음**합니다.

## <a name="BKMK_Apply_SRP_DLLs"></a>Dll 소프트웨어 제한 정책이 적용 하려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  세부 정보 창에서 두 번 클릭 **집행**합니다.

3.  아래에서 **소프트웨어 제한 정책을 다음에 적용**, 클릭 **소프트웨어 파일을 모두**합니다.

> [!NOTE]
> -   이 절차를 수행 하려면 로컬 컴퓨터에서 관리자가 그룹의 회원 해야 하거나 받아야 적절 한 권한이 위임 합니다. 컴퓨터가 도메인에 가입, 도메인 관리자 그룹의 회원이이 절차를 수행 수 있습니다.
> -   기본적으로 소프트웨어 제한 정책 동적 연결 (Dll 라이브러리)를 확인 하지 않습니다. Dll 확인 DLL 로드 될 때마다 제한 정책이 소프트웨어 평가 해야 하기 때문에 시스템 성능을 저하 시킬 수 있습니다. Dll Dll를 대상으로 하는 바이러스를 수신 하는 데 문제가 있는 경우 확인을 결정할 수 있습니다. 기본 보안 수준이으로 설정 된 경우 **허용 안 함**, DLL 만들어야 소프트웨어 제한 각 DLL 실행 하도록 허용 하는 정책 규칙을 검사할 수 있도록 합니다.
