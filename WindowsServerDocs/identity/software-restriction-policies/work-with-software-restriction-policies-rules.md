# <a name="work-with-software-restriction-policies-rules"></a>소프트웨어 제한 정책 규칙 사용

>적용 대상: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

절차 제한 정책 소프트웨어를 사용 하 여 영역과 해시 규칙 된 인증서를 경로, 인터넷 작업에 대해 설명 합니다.

## <a name="introduction"></a>소개
제한 정책 소프트웨어를 식별 하 고 실행할 수 있는 소프트웨어를 지정 하 여 컴퓨팅 환경 신뢰할 수 없는 소프트웨어 로부터 보호할 수 있습니다. 기본 보안 수준이 정의 수 **제한 없음** 또는 **허용 안 함** 기본적으로 실행 하도록 허용 하거나 가능한 소프트웨어 또는 되지 않도록 그룹 정책 개체 (GPO)에 대 한 합니다. 특정 소프트웨어에 대 한 정책을 규칙 소프트웨어 제한 만들어 기본 보안 수준이이에 대 한 예외 만들 수 있습니다. 예를 들어, 기본 보안 수준이으로 설정 된 경우 **허용 안 함**, 특정 소프트웨어를 실행할 수 있도록 규칙 만들 수 있습니다. 유형의 규칙은 다음과 같습니다.

-   **인증서 규칙**

    절차 참조 하십시오 [인증서 규칙 작업](#BKMK_Cert_Rules)합니다.

-   **Hash 규칙**

    절차 참조 하십시오 [해시 규칙 작업](#BKMK_Hash_Rules)합니다.

-   **인터넷 시간대 규칙**

    절차 참조 하십시오 [인터넷 시간대 규칙 작업](#BKMK_Internet_Zone_Rules)합니다.

-   **경로 규칙**

    절차 참조 하십시오 [경로 규칙 작업](#BKMK_Path_Rules)합니다.

소프트웨어 제한 정책을 관리 다른 작업에 대 한 정보를 참조 하세요. [소프트웨어 제한 정책을 관리](administer-software-restriction-policies.md)합니다.

## <a name="BKMK_Cert_Rules"></a>인증서 규칙 작업
또한 제한 정책이 소프트웨어 서명 인증서가 소프트웨어를 식별할 수 있습니다. 소프트웨어를 식별 하 고 다음을 허용 하거나 소프트웨어의 보안 수준에 따라 실행 하도록 허용 하지 않습니다 하는 인증서 규칙을 만들 수 있습니다. 예를 들어, 사용자의 메시지를 표시 하지 않고 도메인 출처를 신뢰할 수 있는 소프트웨어를 자동으로 신뢰할 인증서 규칙을 사용할 수 있습니다. 또한 인증서 규칙 파일을 실행 하도록 허용 되지 않은 영역 운영 체제에 사용할 수 있습니다. 인증서 규칙은 기본적으로 사용할 수 없습니다.

그룹 정책을 사용 하는 도메인을 규칙을 만들 때 만들거나 그룹 정책 개체 수정할 수 있는 권한이 있어야 합니다. 로컬 컴퓨터에 대 한 규칙을 만들 관리자 자격 증명 해당 컴퓨터에 있어야 합니다.

#### <a name="to-create-a-certificate-rule"></a>인증서 규칙을 만들려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  콘솔 트리에서 또는 세부 정보 창에서 마우스 오른쪽 단추로 클릭 **추가 규칙**을 차례로 클릭 하 고 **새 인증서 규칙**합니다.

3.  클릭 **찾아보기**, 다음는 인증서 나 서명 된 파일을 선택 합니다.

4.  **보안 수준**를 클릭 하거나 **허용 안 함** 또는 **제한 없음**합니다.

5.  **설명**이 규칙에 대 한 설명을 입력 한 다음, **확인**합니다.

> [!NOTE]
> -   이미 수행 하지 않은 경우 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수도 있습니다.
> -   인증서 규칙은 기본적으로 사용할 수 없습니다.
> -   에 나열 된 인증서 규칙 영향을 받는 파일 형식만 **지정 된 파일 형식** 제한 정책 소프트웨어에 대 한 세부 정보 창에서 합니다. 모든 규칙에서 공유 하는 지정 된 파일 형식 목록은 한 방법이 있습니다.
> -   소프트웨어 제한 사항이 적용 정책에서 로그 오프 한 해당 컴퓨터에 로그온 하 여 정책 설정 업데이트 해야 합니다.
> -   둘 이상의 소프트웨어 제한 정책 규칙 정책 설정에 적용 되 면는 충돌 규칙의 우선 순위가 합니다.

### <a name="enabling-certificate-rules"></a>인증서 규칙 활성화
귀하의 환경에 따라 규칙 인증서 사용에 대 한 다른 절차 가지가 있습니다.

-   [로컬 컴퓨터에 대 한](#BKMK_1)

-   [그룹 정책 개체, 작업할 때가 도메인에 가입 하는 서버에 대 한](#BKMK_2)

-   [그룹 정책 개체, 있고 원격 서버 관리 도구 설치 된에 워크스테이션 또는 도메인 컨트롤러에 대 한](#BKMK_3)

-   [만 도메인에 대 한 컨트롤러 있고 원격 서버 관리 도구 팩이 설치 된 워크스테이션 또는 도메인 컨트롤러에](#BKMK_4)

#### <a name="BKMK_1"></a>로컬 컴퓨터에 대 한 규칙 인증서를 사용 하도록 설정 하려면

1.  로컬 보안 설정을 엽니다.

2.  콘솔 트리에서 클릭 **보안 옵션** 보안 정책 설정/로컬 아래에 있습니다.

3.  세부 정보 창에서 두 번 클릭 **시스템 설정: Windows 실행 소프트웨어 제한 정책에 대 한 파일에 사용 하 여 인증서 규칙**합니다.

4.  다음 중 하나를 수행 하 고 클릭 한 다음 **확인**:

    -   인증서 규칙을 사용 하도록 설정 하려면 클릭 **사용**합니다.

    -   인증서 규칙을 해제 하려면 클릭 **사용 안 함**합니다.

#### <a name="BKMK_2"></a>규칙 인증서를 사용 하도록 설정 하려면에 대 한는 그룹 정책 개체, 작업할 때가 도메인에 가입 하는 서버에

1.  Microsoft Management Console MMC ()을 엽니다.

2.  에 **파일** 메뉴를 클릭 **스냅인 추가/제거**을 차례로 클릭 하 고 **추가**합니다.

3.  클릭 **로컬 그룹 정책 개체 편집기**을 차례로 클릭 하 고 **추가**합니다.

4.  **그룹 정책 개체 선택**, 클릭 **찾아보기**합니다.

5.  **그룹 정책 개체 찾아보려면**에서 적절 한 도메인 사이트나 조직을 그룹 정책 개체 (GPO)을 선택-또는 새 앨범 만들기 마우스 클릭 한 다음 **완료**합니다.

6.  클릭 **닫기**을 차례로 클릭 하 고 **확인**합니다.

7.  콘솔 트리에서 클릭 **보안 옵션** 아래에 있는 *GroupPolicyObject* [*ComputerName*] 정책을/컴퓨터 구성/Windows 설정/보안 설정/로컬 정책을 / 합니다.

8.  세부 정보 창에서 두 번 클릭 **시스템 설정: Windows 실행 소프트웨어 제한 정책에 대 한 파일에 사용 하 여 인증서 규칙**합니다.

9. 이 정책 설정을 아직 정의 있지 않으면 선택는 **이 정책 설정 정의** 확인란을 선택 합니다.

10. 다음 중 하나를 수행 하 고 클릭 한 다음 **확인**:

    -   인증서 규칙을 사용 하도록 설정 하려면 클릭 **사용**합니다.

    -   인증서 규칙을 해제 하려면 클릭 **사용 안 함**합니다.

#### <a name="BKMK_3"></a>인증서를 사용 하도록 설정 하려면 그룹 정책 개체, 규칙 하 고 설치 원격 서버 관리 도구 된 워크스테이션 또는 도메인 컨트롤러에는

1.  Active Directory 사용자와 컴퓨터를 엽니다.

2.  콘솔 트리에서 그룹 정책 개체 (GPO) 규칙 인증서를 사용 하도록 설정 하려면 마우스 오른쪽 단추로 클릭 합니다.

3.  클릭 **속성**을 차례로 클릭 하 고 있는 **그룹 정책** 탭 합니다.

4.  클릭 **편집** 편집할 GPO를 열 수 있습니다. 클릭 해도 **새로** 만들기를 누른 다음 **편집**합니다.

5.  콘솔 트리에서 클릭 **보안 옵션** 아래에 있는 *GroupPolicyObject*[*ComputerName*] 정책을/컴퓨터 구성/Windows 설정/보안 설정/로컬 정책입니다.

6.  세부 정보 창에서 두 번 클릭 **시스템 설정: Windows 실행 소프트웨어 제한 정책에 대 한 파일에 사용 하 여 인증서 규칙**합니다.

7.  이 정책 설정을 아직 정의 있지 않으면 선택는 **이 정책 설정 정의** 확인란을 선택 합니다.

8.  다음 중 하나를 수행 하 고 클릭 한 다음 **확인**:

    -   인증서 규칙을 사용 하도록 설정 하려면 클릭 **사용**합니다.

    -   인증서 규칙을 해제 하려면 클릭 **사용 안 함**합니다.

#### <a name="BKMK_4"></a>사용 하도록 설정 하려면만 도메인 컨트롤러 고 인증서 규칙은 도메인 컨트롤러 또는 설치 원격 서버 관리 도구 된 워크스테이션

1.  도메인 컨트롤러 보안 설정을 엽니다.

2.  콘솔 트리에서 클릭 **보안 옵션** 아래에 있는 *GroupPolicyObject* [*ComputerName*] 정책을/컴퓨터 구성/Windows 설정/보안 설정/로컬 정책입니다.

3.  세부 정보 창에서 두 번 클릭 **시스템 설정: Windows 실행 소프트웨어 제한 정책에 대 한 파일에 사용 하 여 인증서 규칙**합니다.

4.  이 정책 설정을 아직 정의 있지 않으면 선택는 **이 정책 설정 정의** 확인란을 선택 합니다.

5.  다음 중 하나를 수행 하 고 클릭 한 다음 **확인**:

    -   인증서 규칙을 사용 하도록 설정 하려면 클릭 **사용**합니다.

    -   인증서 규칙을 해제 하려면 클릭 **사용 안 함**합니다.

> [!NOTE]
> 인증서 규칙이 적용 되기 전에이 절차를 수행 해야 합니다.

### <a name="set-trusted-publisher-options"></a>신뢰할 수 있는 게시자 옵션을 설정
로그인 하는 소프트웨어 사용량 소프트웨어 게시자 및 응용 프로그램 개발자 점점 더 많은 응용 프로그램과 출처를 신뢰할 수 있는지 확인 하기 위해 합니다. 하지만 많은 사용자가 이해 하지 못하거나는 설치한 응용 프로그램과 관련 된 서명 인증서에 작은 주의 해야 합니다.

정책 설정에는 **신뢰할 수 있는 게시자** 인증서 경로 유효성 검사 정책 탭 관리자가 신뢰할 수 있는 게시자에서 제공한 것임을 허용할 수 있는 인증서를 제어할 수 있습니다.

##### <a name="to-configure-the-trusted-publishers-policy-settings-for-a-local-computer"></a>로컬 컴퓨터에 대 한 정책 설정을 신뢰할 수 있는 게시자 구성 하려면

1.  에 **시작** 화면에서 입력**gpedit.msc** ENTER 키를 누릅니다.

2.  아래에서 콘솔 트리에서 **로컬 컴퓨터 정책 구성할 보안 설정**, 클릭 **공개 키 정책**합니다.

3.  두 번 클릭 **인증서 경로 유효성 검사 설정**을 차례로 클릭 하 고 있는 **신뢰할 수 있는 게시자** 탭 합니다.

4.  선택는 **이 정책 설정 정의** 확인란을 적용 하며 클릭 한 다음 하려는 정책 설정을 선택 **확인** 새 설정을 적용 합니다.

##### <a name="to-configure-the-trusted-publishers-policy-settings-for-a-domain"></a>도메인에 대 한 정책 설정을 신뢰할 수 있는 게시자 구성 하려면

1.  열린 **그룹 정책 관리**합니다.

2.  콘솔 트리에서 두 번 클릭 **그룹 정책 개체** 숲 및 도메인을 포함 하 여 **기본 도메인 정책** 그룹 정책 개체 GPO () 편집 하려는 합니다.

3.  마우스 오른쪽 단추로 클릭는 **기본 도메인 정책** GPO 및 클릭 **편집**합니다.

4.  아래에서 콘솔 트리에서 **컴퓨터 구성 보안 설정을**, 클릭 **공개 키 정책**합니다.

5.  두 번 클릭 **인증서 경로 유효성 검사 설정**을 차례로 클릭 하 고 있는 **신뢰할 수 있는 게시자** 탭 합니다.

6.  선택는 **이 정책 설정 정의** 확인란을 적용 하며 클릭 한 다음 하려는 정책 설정을 선택 **확인** 새 설정을 적용 합니다.

##### <a name="to-allow-only-administrators-to-manage-certificates-used-for-code-signing-for-a-local-computer"></a>코드 로컬 컴퓨터에 대 한 로그인에 사용 되는 인증서를 관리 하려면 관리자만 수 있도록 하려면

1.  에 **시작** 화면, 종류 **gpedit.msc** 에 **프로그램 및 파일 검색** 또는 Windows 8, 바탕 화면을 켜고 다음 ENTER 키를 누릅니다.

2.  콘솔 트리에서 **기본 도메인 정책** 또는 **로컬 컴퓨터 정책**를 두 번 클릭 **컴퓨터 구성**, **Windows 설정**, 및 **보안 설정**, 클릭 한 다음 **공개 키 정책**합니다.

3.  두 번 클릭 **인증서 경로 유효성 검사 설정**을 차례로 클릭 하 고 있는 **신뢰할 수 있는 게시자** 탭 합니다.

4.  선택는 **이 정책 설정 정의** 확인란을 선택 합니다.

5.  아래에서 **게시자 관리 신뢰할 수 있는**, 클릭 **모든 관리자만 신뢰할 수 있는 게시자 관리할 수 있도록**을 차례로 클릭 하 고 **확인** 새 설정을 적용 해야 합니다.

##### <a name="to-allow-only-administrators-to-manage-certificates-used-for-code-signing-for-a-domain"></a>코드 도메인에 대 한 로그인에 사용 되는 인증서를 관리 하려면 관리자만 수 있도록 하려면

1.  열린 **그룹 정책 관리**합니다.

2.  두 번 클릭 콘솔 트리에서 **그룹 정책 개체** 숲 및 도메인을 포함 하는 **기본 도메인 정책** 편집할 GPO 합니다.

3.  마우스 오른쪽 단추로 클릭는 **기본 도메인 정책** GPO 및 클릭 **편집**합니다.

4.  아래에서 콘솔 트리에서 **컴퓨터 구성 보안 설정을**, 클릭 **공개 키 정책**합니다.

5.  두 번 클릭 **인증서 경로 유효성 검사 설정**을 차례로 클릭 하 고 있는 **신뢰할 수 있는 게시자** 탭 합니다.

6.  선택는 **이 정책 설정 정의** 확인란을 선택한 다음 변경을 구현 **확인** 새 설정을 적용 합니다.

## <a name="BKMK_Hash_Rules"></a>해시 규칙 작업
해시는 일련의 고유 하 게 식별 소프트웨어 프로그램 또는 파일을 고정된 길이를 바이트 합니다. 해시는 hash algorithm 하 여 계산 됩니다. 소프트웨어 프로그램에 대 한 해시 규칙 만들면 제한 정책이 소프트웨어 프로그램 콘텐츠의 해시를 계산 합니다. 사용자는 소프트웨어 프로그램인 열려고 프로그램 콘텐츠의 해시 소프트웨어 제한 정책에 대 한 기존 해시 규칙 비교 됩니다. 소프트웨어 프로그램 콘텐츠의 해시는 항상 프로그램에는 컴퓨터 위치에 관계 없이 동일 합니다. 어떤 식으로든에서 소프트웨어 프로그램을 변경 하는 경우에 변경, 및 소프트웨어 제한 정책을 해시 규칙에 해시 일치 하지 않습니다.

해시 규칙을 만들고 보안을 설정 하는 예를 들어, **허용 안 함** 사용자가 특정 파일을 실행 하지 못하도록 합니다. 파일 이름을 바꾸거나을 다른 폴더로 이동 동일한 해시 해도 됩니다. 그러나 자체 파일을 변경도 해시 값을 변경 하 고 파일을 제한 알림을 무시할 수 있도록 합니다.

#### <a name="to-create-a-hash-rule"></a>Hash 규칙을 만들려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  콘솔 트리에서 또는 세부 정보 창에서 마우스 오른쪽 단추로 클릭 **추가 규칙**을 차례로 클릭 하 고 **새로운 해시 규칙**합니다.

3.  클릭 **찾아보기** 파일을 찾을 수 있습니다.

    > [!NOTE]
    > Windows XP에서의 사전 계산된 해시 붙여 수는 **파일 콘텐츠의 해시**합니다. Windows Server 2008 R2, Windows 7 및 이후 버전에서이 옵션을 사용할 수 없습니다.

4.  **보안 수준**를 클릭 하거나 **허용 안 함** 또는 **제한 없음**합니다.

5.  **설명**이 규칙에 대 한 설명을 입력 한 다음, **확인**합니다.

> [!NOTE]
> -   이미 수행 하지 않은 경우 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수 있습니다.
> -   바이러스 또는 실행에서 방지 트로이 말 해시 규칙 만들 수 있습니다.
> -   다른 사람이 해시 규칙을 실행할 수 없는 바이러스를 사용 하도록 하려는 경우 바이러스 해시 제한 정책 소프트웨어를 사용 하 여 계산 및 다음 다른 사람에 게 해시 값 전자 메일 합니다. 바이러스 자체 전자 메일 하지 않습니다.
> -   바이러스를 메일로 보낼 수도 실행 메일 첨부 파일을 방지 하려면 경로 규칙을 만들 수 있습니다.
> -   파일을 바뀌거나 동일한 해시 다른 폴더를 이동 합니다. 다른 해시 자체 파일을 변경 하면 됩니다.
> -   에 나열 된 해시 규칙 영향을 받는 파일 형식만 **지정 된 파일 형식** 제한 정책 소프트웨어에 대 한 세부 정보 창에서 합니다. 모든 규칙에서 공유 하는 지정 된 파일 형식 목록은 한 방법이 있습니다.
> -   소프트웨어 제한 사항이 적용 정책에서 로그 오프 한 해당 컴퓨터에 로그온 하 여 정책 설정 업데이트 해야 합니다.
> -   둘 이상의 소프트웨어 제한 정책 규칙 정책 설정에 적용 되 면는 충돌 규칙의 우선 순위가 합니다.

## <a name="BKMK_Internet_Zone_Rules"></a>인터넷 시간대 규칙에 사용
인터넷 시간대 규칙 Windows Installer 패키지에만 적용 됩니다. 시간대 규칙 소프트웨어 Internet Explorer 통해 지정 하는 영역에서 확인할 수 있습니다. 이러한 영역에 인터넷, 로컬 인트라넷, 제한 된 사이트를 신뢰할 수 있는 사이트 및 내 컴퓨터는 합니다. 인터넷 시간대 규칙은 사용자가 소프트웨어 다운로드 및 설치 하지 못하도록 설계 되었습니다.

#### <a name="to-create-an-internet-zone-rule"></a>인터넷 시간대 규칙을 만들려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  콘솔 트리에서 또는 세부 정보 창에서 마우스 오른쪽 단추로 클릭 **추가 규칙**을 차례로 클릭 하 고 **새 인터넷 시간대 규칙**합니다.

3.  **인터넷 영역**, 인터넷 영역을 클릭 합니다.

4.  **보안 수준**, 클릭 **허용 안 함** 또는 **제한 없음**을 차례로 클릭 하 고 **확인**합니다.

> [!NOTE]
> -   이미 수행 하지 않은 경우 그룹 정책 개체 (GPO)에 대 한 새로운 소프트웨어 제한 정책 설정을 만들 수 있습니다.
> -   시간대 규칙은 Windows Installer 패키지 하는.msi 파일 형식에 포함 된 파일에만 적용 됩니다.
> -   소프트웨어 제한 사항이 적용 정책에서 로그 오프 한 해당 컴퓨터에 로그온 하 여 정책 설정 업데이트 해야 합니다.
> -   둘 이상의 소프트웨어 제한 정책 규칙 정책 설정에 적용 되 면는 충돌 규칙의 우선 순위가 합니다.

## <a name="BKMK_Path_Rules"></a>경로 규칙 작업
경로 규칙 파일 경로로 소프트웨어를 식별합니다. 예를 들어 컴퓨터의 기본 보안 수준이 **허용 안 함**를 계속 각 사용자에 대 한 특정 폴더로 무제한 액세스를 허용할 수 있습니다. 보안 수준을 경로 규칙의 설정 및 파일 경로 사용 하 여 경로 규칙 만들 수 **제한 없음**합니다. 이러한 유형의 규칙에 대 한 몇 가지 일반적인 경로 %userprofile%, %windir%, 응용 프로그램 데이터 %, %programfiles 및 임시 % 됩니다. 또한 경로 규칙 소프트웨어의 레지스트리 키 경로를 사용 하는 레지스트리를 만들 수 있습니다.

이 규칙은 경로로 소프트웨어 프로그램으로 이동 하는 경우, 지정 하기 때문에 경로 규칙 적용 되지 않습니다.

#### <a name="to-create-a-path-rule"></a>경로 규칙을 만들려면

1.  제한 정책을 소프트웨어를 엽니다.

2.  콘솔 트리에서 또는 세부 정보 창에서 마우스 오른쪽 단추로 클릭 **추가 규칙**을 차례로 클릭 하 고 **새 경로 규칙**합니다.

3.  **경로**경로 입력 하거나 클릭 **찾아보기** 파일 또는 폴더를 찾을 수 있습니다.

4.  **보안 수준**를 클릭 하거나 **허용 안 함** 또는 **제한 없음**합니다.

5.  **설명**이 규칙에 대 한 설명을 입력 한 다음, **확인**합니다.

> [!CAUTION]
> -   특정 폴더에서의 Windows 폴더와 같은 고 보안 수준을 설정에 **허용 안 함** 운영 체제의 작동 저하 될 수 있습니다. 중요 한 종속 프로그램 그 중 하나 또는 운영 체제의 구성 허용 하지 않는 있는지 확인 합니다.

> [!NOTE]
> -   이미 수행 하지 않은 경우 새 소프트웨어 제한 정책을 그룹 정책 개체 (GPO)에 대 한 만들 수 있습니다.
> -   하는 경우 소프트웨어에 대 한 경로 규칙을 받아서 만든 한 보안 수준 **허용 안 함**, 사용자가 다른 위치로 복사 하 여 소프트웨어 실행할 수 있습니다.
> -   와일드 카드 문자 경로 규칙에서 지원 되는 * 및 합니다.
> -   % Programfiles % 또는 시스템 루트 % 등 환경 변수 경로 규칙에서 사용할 수 있습니다.
> -   알 수 없는 어디 컴퓨터에 저장 되어 있지만 해당 레지스트리 키가 있는 경우 소프트웨어가 경로 규칙을 만들려는 경우 레지스트리 경로 규칙을 만들 수 있습니다.
> -   사용자가 전자 메일 첨부 파일을 실행 하지 못하도록 하려면 사용자가 전자 메일 첨부 파일을 실행 하지 못하는 첨부 디렉터리 전자 메일 프로그램에 대 한 경로 규칙을 만들 수 있습니다.
> -   에 나열 된 경로 규칙 영향을 받는 파일 형식만 **지정 된 파일 형식** 제한 정책 소프트웨어에 대 한 세부 정보 창에서 합니다. 모든 규칙에서 공유 하는 지정 된 파일 형식 목록은 한 방법이 있습니다.
> -   소프트웨어 제한 사항이 적용 정책에서 로그 오프 한 해당 컴퓨터에 로그온 하 여 정책 설정 업데이트 해야 합니다.
> -   둘 이상의 소프트웨어 제한 정책 규칙 정책 설정에 적용 되 면는 충돌 규칙의 우선 순위가 합니다.

#### <a name="to-create-a-registry-path-rule"></a>레지스트리 경로 규칙을 만들려면

1.  에 **시작** 화면 regedit 입력 합니다.

2.  콘솔 트리에서 레지스트리 키를 규칙을 만들고 클릭 한 다음 원하는 마우스 오른쪽 단추로 클릭 **키 이름 복사**합니다. 세부 정보 창에서 값 이름을 note 합니다.

3.  제한 정책을 소프트웨어를 엽니다.

4.  콘솔 트리에서 또는 세부 정보 창에서 마우스 오른쪽 단추로 클릭 **추가 규칙**을 차례로 클릭 하 고 **새 경로 규칙**합니다.

5.  **경로**, 레지스트리 키 이름 값 이름을 뒤에 붙여 넣습니다.

6.  퍼센트 기호 (%) %HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\PlatformSDK\Directories\InstallDir% 등의 레지스트리 경로 포함 합니다.

7.  **보안 수준**를 클릭 하거나 **허용 안 함** 또는 **제한 없음**합니다.

8.  **설명**이 규칙에 대 한 설명을 입력 한 다음, **확인**합니다.
